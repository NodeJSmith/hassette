## Objectives & Success Criteria

- After WS connects, the hook automatically sends `{type: "subscribe", data: {logs: true, min_log_level: "INFO"}}` â€” no component-level subscribe needed
- After WS reconnects, log subscription is re-established automatically
- `LogTable` can update the server-side `min_log_level` when the user changes the filter
- `AppState` exposes `updateLogSubscription(level: string)` callback (not a generic `send()`)
- Log messages stream to the frontend without manual intervention

## Subtasks

1. In `frontend/src/state/create-app-state.ts`: add `updateLogSubscription: (level: string) => void` to the `AppState` type. Initialize as a no-op. Add a `setUpdateLogSubscription` setter so `useWebSocket` can wire it up.
2. In `frontend/src/hooks/use-websocket.ts`: after the `"connected"` message is received (line ~59), send `{type: "subscribe", data: {logs: true, min_log_level: "INFO"}}` via the socket. This handles both initial connect and reconnect (the hook creates a new socket on reconnect, which triggers a new `connected` message).
3. In `frontend/src/hooks/use-websocket.ts`: create a `updateLogSubscription(level: string)` function that sends `{type: "subscribe", data: {logs: true, min_log_level: level}}`. Wire it to `state.updateLogSubscription` via the setter.
4. In `frontend/src/components/shared/log-table.tsx`: when the user changes `minLevel` filter, call `updateLogSubscription(newLevel)` to update server-side filtering. Use `""` or `"DEBUG"` when the filter is "All Levels".
5. Remove the comment in `log-table.tsx` that admits WS subscription is unimplemented (if present)
6. Write frontend test: `frontend/src/hooks/use-websocket.test.ts::test_sends_subscribe_on_connect` â€” mock WebSocket, simulate `connected` message, assert subscribe was sent
7. Write frontend test: `frontend/src/hooks/use-websocket.test.ts::test_resubscribes_on_reconnect` â€” simulate disconnect + reconnect, assert subscribe sent again
8. Check existing E2E tests in `tests/e2e/test_websocket.py` â€” if any assert that log messages do NOT arrive via WS, update them now that streaming works
9. Run frontend tests: `cd frontend && npm test`
10. Run `uv run nox -s dev`
