## Objectives & Success Criteria

- `BusService._dispatch()` bypasses `CommandExecutor.execute()` when `listener.db_id is None`, invoking the handler directly with structured error logging.
- `SchedulerService.run_job()` bypasses `CommandExecutor.execute()` when `job.db_id is None`, running the job callable directly with structured error logging.
- Internal handler errors are logged with topic and handler/job context (not generic TaskBucket crash message).
- `once` listeners are removed in a `finally` block even when the handler raises.
- No `InvokeHandler` or `ExecuteJob` commands are created for internal handlers/jobs.

## Subtasks

1. In `src/hassette/core/bus_service.py`, rewrite `_dispatch()`:
   - When `listener.db_id is None`: wrap `await listener.invoke(event)` in try/except/finally. Log exceptions with `self.logger.exception("Internal handler error (topic=%s, handler=%r)", topic, listener)`. In `finally`, handle `listener.once` removal. Return early.
   - When `listener.db_id` is set: existing path (create `InvokeHandler`, delegate to executor). Remove the `listener.db_id or 0` fallback — use `listener.db_id` directly.
   - Remove the debug log about "registration pending" — it's no longer relevant.
2. In `src/hassette/core/scheduler_service.py`, rewrite `run_job()`:
   - When `job.db_id is None`: call the job callable directly with structured error logging. Skip `ExecuteJob` command creation.
   - When `job.db_id` is set: existing path. Remove the `job.db_id or 0` fallback — use `job.db_id` directly.
   - Remove the debug log about "registration pending".
3. Add test `tests/integration/test_bus.py::test_dispatch_internal_handler_bypasses_executor`:
   - Create a listener with no `app_key` (so `db_id` stays `None`)
   - Fire an event
   - Assert the handler was called
   - Assert `executor.execute` was NOT called
4. Add test `tests/integration/test_bus.py::test_dispatch_internal_handler_logs_error_on_exception`:
   - Create an internal listener whose handler raises
   - Fire an event
   - Assert the exception is logged (not propagated)
5. Add test `tests/integration/test_scheduler.py::test_run_job_internal_bypasses_executor`:
   - Create a job with no `app_key`
   - Run it
   - Assert the job callable was called
   - Assert `executor.execute` was NOT called

## Review Guidance

- **Critical constraint**: The `try/except/finally` in `_dispatch()` must catch `Exception` (not `BaseException`) to avoid swallowing `CancelledError` or `KeyboardInterrupt`.
- **`once` removal**: Must be in `finally` block for the internal path. Verify this handles the case where `listener.invoke()` raises.
- **Error log fidelity**: The log line must include topic and listener repr — check the format string matches the design doc.
- **No sentinel values**: After this WP, no code path should produce `listener_id=0` or `job_id=0`. Grep for `or 0` in bus_service.py and scheduler_service.py to confirm removal.
