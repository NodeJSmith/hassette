# WP03: Scheduler names, SPA path fix, and dead endpoint removal

**Fixes:** 3 (SPA path check), 4 (scheduler name collision), 5 (remove bus metrics stub), 6 (remove unused apps endpoint)
**Lane:** todo
**Depends on:** nothing (independent of WP01/WP02)
**Blocks:** nothing

## Summary

Four independent, small fixes grouped into one WP: add `__str__` to trigger classes and use in auto-generated job names, fix the SPA containment check, and remove two dead endpoints.

## Acceptance Criteria

### Fix 3: SPA path check

- [ ] `web/app.py`: containment check uses `candidate.resolve().is_relative_to(_SPA_DIR.resolve())`
- [ ] Path traversal test: `GET /../../../etc/passwd` returns 404

### Fix 4: Scheduler auto-generated names

- [ ] `IntervalTrigger.__str__` returns `f"interval:{self.interval.in_seconds()}s"`
- [ ] `CronTrigger.__str__` returns `f"cron:{self.cron_expression}"`
- [ ] `ScheduledJob.__post_init__`: auto-generated name uses `f"{callable_name}:{self.trigger}"` when `name` is empty
- [ ] Two `run_every` calls with same callable but different intervals both succeed
- [ ] `if_exists="skip"` still works for truly identical jobs (same callable + same trigger)

### Fix 5: Remove bus metrics stub

- [ ] `get_bus_metrics_summary()` removed from `routes/bus.py`
- [ ] `BusMetricsSummaryResponse` removed from `web/models.py`
- [ ] Any tests asserting the hardcoded zeros removed or updated
- [ ] `GET /bus/metrics` returns 404 (or not registered in router)

### Fix 6: Remove unused apps endpoint

- [ ] `get_app()` removed from `routes/apps.py`
- [ ] `AppInstanceResponse` model retained (used by `AppStatusResponse.apps`)
- [ ] Any tests for `GET /apps/{app_key}` removed or updated
- [ ] `GET /apps/{app_key}` returns 404 (or not registered in router)
