## Objectives & Success Criteria

- The current Preact SPA is running on port 8126 (the normal hassette container).
- A screenshotter agent navigates to every page and interactive state in the same 42-scenario inventory and saves full-page screenshots to `design/audits/visual-parity/new/`.
- Filenames match the old UI screenshot names exactly for easy side-by-side comparison.
- A verification agent reads every captured screenshot and confirms correct content.
- Scenarios that don't exist in the new UI yet (e.g., multi-instance switcher, error traceback display) are captured as-is to document the gap visually.

## Subtasks

1. Confirm the current hassette container is running and healthy.
2. Launch a screenshotter agent with the same 42-scenario inventory. The agent navigates the new UI at `http://127.0.0.1:8126/` and captures each scenario. For scenarios where the new UI has missing features (e.g., no instance switcher), capture what IS shown — the gap will be visible in the comparison.
3. Launch a verification agent that reads every screenshot and confirms completeness.

## Test Strategy

- Same verification checklist as WP01: 42 scenarios, each PASS/SKIP/FAIL.
- SKIP is allowed only for scenarios that literally cannot be reached in the new UI (e.g., if a route doesn't exist). The screenshot should still be taken of the closest equivalent.

## Review Guidance

- Every filename in `new/` must match a filename in `old/` — this enables side-by-side comparison.
- Missing features should be captured as blank/missing states, not skipped — the comparison agent needs to see what the new UI shows (or doesn't show) for each scenario.
