## Objectives & Success Criteria

A new `SessionManager` service exists at `src/hassette/core/session_manager.py` and owns all session lifecycle logic. `Hassette` delegates `session_id` to it and no longer contains any session CRUD methods. `core.py` is reduced by approximately 235 lines. All existing tests continue to pass.

Specifically:
- `SessionManager` constructs with `(hassette, *, database_service, parent)` and follows the `Resource` base class pattern
- `session_id` property on `SessionManager` raises `RuntimeError` if no session has been created
- `Hassette.session_id` is a one-line delegating property: `return self._session_manager.session_id`
- `Hassette.run_forever()` calls `self._session_manager.mark_orphaned_sessions()`, `self._session_manager.create_session()`, and registers `self._session_manager.on_service_crashed` as the crash handler
- `Hassette.before_shutdown()` calls `self._session_manager.finalize_session()`
- Methods `_create_session`, `_do_create_session`, `_mark_orphaned_sessions`, `_do_mark_orphaned_sessions`, `_on_service_crashed`, `_do_on_service_crashed`, `_finalize_session`, `_do_finalize_session` are removed from `core.py`
- Instance variables `_session_id`, `_session_error`, `_session_lock` are removed from `Hassette.__init__`

## Subtasks

1. Create `src/hassette/core/session_manager.py` with `SessionManager(Resource)`:
   - Constructor: `(hassette, *, database_service: DatabaseService, parent: Resource | None = None)`
   - State: `_database_service`, `_session_id: int | None = None`, `_session_error: bool = False`, `_session_lock: asyncio.Lock()`
   - `session_id` property (raises `RuntimeError` if `_session_id is None`)
   - Move all 4 method pairs from `core.py`: `create_session`/`_do_create_session`, `mark_orphaned_sessions`/`_do_mark_orphaned_sessions`, `on_service_crashed`/`_do_on_service_crashed`, `finalize_session`/`_do_finalize_session` — rename from private `_method` to public `method` (e.g., `_create_session` → `create_session`)
   - Preserve the `_session_lock` coordination in `on_service_crashed` and `finalize_session`
   - Preserve the `DatabaseService.db` try/except guard in `on_service_crashed` and `finalize_session`

2. Register `SessionManager` in `Hassette.__init__` immediately after `_database_service`:
   ```python
   self._database_service = self.add_child(DatabaseService)
   self._session_manager = self.add_child(SessionManager, database_service=self._database_service)
   ```

3. Replace `Hassette.session_id` property body with `return self._session_manager.session_id`

4. Update `Hassette.run_forever()` — replace the three session callsites:
   - `await self._mark_orphaned_sessions()` → `await self._session_manager.mark_orphaned_sessions()`
   - `await self._create_session()` → `await self._session_manager.create_session()`
   - `handler=self._on_service_crashed` → `handler=self._session_manager.on_service_crashed`

5. Update `Hassette.before_shutdown()` — replace `await self._finalize_session()` with `await self._session_manager.finalize_session()`

6. Remove from `core.py`:
   - Instance vars `_session_id`, `_session_error`, `_session_lock` from `__init__`
   - All 8 session methods

7. Export `SessionManager` from `src/hassette/core/__init__.py` if other core services are exported there

8. Run `uv run pyright` and fix any type errors before declaring done

## Review Guidance

**Must verify:**
- `SessionManager` uses `Resource` (not `Service`) — no `serve()` method present
- Constructor matches the `add_child()` pattern exactly: `(hassette, *, database_service, parent=None)` with `super().__init__(hassette, parent=parent)`
- `_session_lock` is an instance variable on `SessionManager`, not on `Hassette`
- `Hassette.session_id` contains only the delegation — no residual session logic
- None of the 8 session methods remain in `core.py`
- `DatabaseService` and `CommandExecutor` are untouched — both access `self.hassette.session_id` which still works via delegation

**Blocker:** Any session method remaining on `Hassette` instead of `SessionManager`
**Blocker:** `_session_lock` not moved — leaving it on `Hassette` would make `on_service_crashed` / `finalize_session` coordination broken
