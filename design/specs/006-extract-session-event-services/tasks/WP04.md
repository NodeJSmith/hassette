## Objectives & Success Criteria

`HassetteHarness._start_bus()` in `src/hassette/test_utils/harness.py` creates a real `EventStreamService` child on the mock instead of manually managing streams. `_HassetteMock` delegates `send_event()` and `event_streams_closed` through the service. Manual `exit_stack` stream teardown is removed. No double-close errors occur. All harness-based integration tests pass.

Specifically:
- `_HassetteMock._send_stream` and `_HassetteMock._receive_stream` instance variables are removed (lines 79–80)
- `_HassetteMock.send_event()` delegates to `self._event_stream_service.send_event()` when the service is present, otherwise raises `RuntimeError("Bus is not enabled on this harness")`
- `_HassetteMock.event_streams_closed` delegates to `self._event_stream_service.event_streams_closed` when the service is present, otherwise returns `True`
- `HassetteHarness._start_bus()` creates an `EventStreamService` child (`self.hassette.add_child(EventStreamService)`) and removes the manual `create_memory_object_stream` call and both `exit_stack.push_async_callback` stream teardown registrations
- `BusService` registration in `_start_bus()` uses `self.hassette._event_stream_service.receive_stream.clone()`
- `tests/integration/test_event_stream_service.py` exists (may be created in WP03; extend here if needed)
- `uv run pytest -n auto` passes in full

## Subtasks

1. Add `self._event_stream_service: EventStreamService | None = None` to `_HassetteMock.__init__` (after removing `_send_stream` and `_receive_stream` from lines 79–80)

2. Update `_HassetteMock.send_event()` (currently lines 106–111):
   ```python
   async def send_event(self, topic: str, event: Event[Any]) -> None:
       if not self._event_stream_service:
           raise RuntimeError("Bus is not enabled on this harness")
       await self._event_stream_service.send_event(topic, event)
   ```

3. Update `_HassetteMock.event_streams_closed` property (currently lines 137–141):
   ```python
   @property
   def event_streams_closed(self) -> bool:
       if not self._event_stream_service:
           return True
       return self._event_stream_service.event_streams_closed
   ```

4. Rewrite `HassetteHarness._start_bus()` (currently lines 353–373):
   - Remove `send_stream, receive_stream = create_memory_object_stream[...](1000)`
   - Remove `self.hassette._send_stream = send_stream` and `self.hassette._receive_stream = receive_stream`
   - Add: `self.hassette._event_stream_service = self.hassette.add_child(EventStreamService)`
   - Update `BusService` registration: `stream=self.hassette._event_stream_service.receive_stream.clone()`
   - Remove both `self._exit_stack.push_async_callback(send_stream.aclose)` lines — `EventStreamService.on_shutdown()` handles teardown via the Resource lifecycle

5. Add `EventStreamService` to the imports in `harness.py`

6. Run the full test suite: `uv run pytest -n auto` — pay particular attention to any test that uses `HassetteHarness` with the bus enabled, as these exercise the harness stream management

7. Run `uv run pyright` to catch any type errors from the interface changes

## Test Strategy

The primary test signal for this WP is the existing integration test suite — all tests that use `HassetteHarness` with `"bus"` in their component list exercise the updated `_start_bus()` path. Confirm they all pass.

Additionally, add to `tests/integration/test_event_stream_service.py` (create if not already present from WP03):
- `test_harness_send_event_reaches_bus` — using `HassetteHarness` with bus enabled, call `harness.hassette.send_event(topic, event)`, assert the bus receives it
- `test_harness_event_streams_closed_after_teardown` — after harness stops, assert `hassette.event_streams_closed` is `True`

The critical regression check is: no `ClosedResourceError` on teardown (which would indicate double-close from stale `exit_stack` callbacks).

## Review Guidance

**Must verify:**
- `create_memory_object_stream` is NOT called in `harness.py` — grep confirms
- `self._exit_stack.push_async_callback` is NOT used for stream teardown — grep for `aclose` in `_start_bus`
- `_HassetteMock._send_stream` and `_HassetteMock._receive_stream` do not exist — grep confirms
- `_HassetteMock._event_stream_service` is initialised to `None` in `__init__` and set in `_start_bus()`
- No test that relied on `hassette._send_stream` directly is broken — grep tests for this attribute

**Blocker:** Any remaining `exit_stack.push_async_callback(stream.aclose)` call in `_start_bus()` — causes double-close `ClosedResourceError` on harness teardown.
**Blocker:** `_HassetteMock.send_event()` checking `self._send_stream` instead of `self._event_stream_service` — would cause `AttributeError` in any harness-based test that sends events.
**Warning:** If `EventStreamService` is added to the harness but its `on_shutdown()` is never called during teardown, streams will leak. Verify the `Resource` child lifecycle is exercised on harness stop.
