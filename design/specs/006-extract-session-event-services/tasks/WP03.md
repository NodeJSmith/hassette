## Objectives & Success Criteria

A new `EventStreamService` service exists at `src/hassette/core/event_stream_service.py` and owns the anyio memory channel. The hardcoded `1000` buffer literal is gone from both `core.py` and `src/hassette/test_utils/harness.py`. `hassette_event_buffer_size: int = Field(default=1000)` exists in `src/hassette/config/config.py`. `Hassette.send_event()` and `Hassette.event_streams_closed` are delegating wrappers. All 5 producer services are untouched. All tests pass.

Specifically:
- `src/hassette/core/event_stream_service.py` contains `EventStreamService(Resource)` with `send_event()`, `event_streams_closed`, `receive_stream` property, and `on_shutdown()`
- `EventStreamService.__init__` reads `hassette.config.hassette_event_buffer_size` for buffer capacity
- `EventStreamService` is registered as the first `add_child()` call in `Hassette.__init__` (before `_database_service`)
- `BusService` registration uses `self._event_stream_service.receive_stream.clone()` instead of `self._receive_stream.clone()`
- `Hassette._send_stream` and `Hassette._receive_stream` instance variables are removed
- Stream teardown in `Hassette.on_shutdown()` is removed (handled by `EventStreamService.on_shutdown()`)
- `Hassette.event_streams_closed` property delegates to `self._event_stream_service.event_streams_closed`
- `Hassette.send_event()` delegates to `await self._event_stream_service.send_event(event_name, event)`
- None of the 5 producer services (`app_handler.py`, `websocket_service.py`, `file_watcher.py`, `app_lifecycle.py`, `service_watcher.py`) are modified

## Subtasks

1. Add `hassette_event_buffer_size: int = Field(default=1000)` to `HassetteConfig` in `src/hassette/config/config.py`, near `web_api_event_buffer_size` (line 218). Add a docstring: `"""Buffer capacity of the internal anyio memory channel used to route events to the bus."""`

2. Create `src/hassette/core/event_stream_service.py` with `EventStreamService(Resource)`:
   - Constructor: `(hassette, *, parent: Resource | None = None)` — reads `hassette.config.hassette_event_buffer_size` and calls `create_memory_object_stream(buffer_size)`
   - `receive_stream` property — returns `self._receive_stream`
   - `send_event(event_name: str, event: Event[Any]) -> None` — `await self._send_stream.send((event_name, event))`
   - `event_streams_closed` property — `return self._send_stream._closed and self._receive_stream._closed`
   - `on_shutdown()` — `await self._send_stream.aclose()` then `await self._receive_stream.aclose()`
   - Full type annotations: `_send_stream: MemoryObjectSendStream[tuple[str, Event[Any]]]` etc.

3. Register `EventStreamService` as the **first** `add_child()` call in `Hassette.__init__`, before `_database_service`:
   ```python
   self._event_stream_service = self.add_child(EventStreamService)
   self._database_service = self.add_child(DatabaseService)
   ```
   This ordering is required — `BusService` receives `receive_stream.clone()` at construction.

4. Update `BusService` registration in `Hassette.__init__`:
   - Change `stream=self._receive_stream.clone()` → `stream=self._event_stream_service.receive_stream.clone()`

5. Remove from `Hassette.__init__`: the `create_memory_object_stream(1000)` line and the `_send_stream`/`_receive_stream` assignments

6. Update `Hassette.send_event()`: replace body with `await self._event_stream_service.send_event(event_name, event)`

7. Update `Hassette.event_streams_closed` property: replace body with `return self._event_stream_service.event_streams_closed`

8. Remove stream teardown from `Hassette.on_shutdown()` (the `if self._send_stream is not None` / `if self._receive_stream is not None` block)

9. Export `EventStreamService` from `src/hassette/core/__init__.py` if appropriate

10. Run `uv run pyright` and fix type errors; run `uv run pytest -n auto` to confirm no regressions
