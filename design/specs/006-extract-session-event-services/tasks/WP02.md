## Objectives & Success Criteria

Session lifecycle tests are fully migrated to `tests/integration/test_session_manager.py` and test `SessionManager` directly ŌĆö no unbound-method tricks, no mock-self patterns. A `create_session_manager(db_service)` fixture exists in `tests/conftest.py`. The old `test_session_lifecycle.py` is updated or thinned to remove tests that now belong to `test_session_manager.py` (tests that tested `Hassette._create_session` etc. directly). All tests pass.

Specifically:
- `tests/integration/test_session_manager.py` contains integration tests for all 4 session lifecycle operations
- `create_session_manager` fixture in `tests/conftest.py` constructs a `SessionManager` with a real test `DatabaseService`
- Any remaining tests in `test_session_lifecycle.py` test Hassette-level behaviour (e.g., session ID delegation) rather than session CRUD internals
- `uv run pytest -n auto` passes

## Subtasks

1. Add `create_session_manager` fixture to `tests/conftest.py`:
   ```python
   @pytest.fixture
   async def session_manager(db_service: DatabaseService) -> AsyncIterator[SessionManager]:
       mock_hassette = Mock(spec=["config", "logger", "hassette"])
       mock_hassette.config = <minimal HassetteConfig>
       sm = SessionManager(mock_hassette, database_service=db_service, parent=None)
       yield sm
   ```
   Follow the same fixture scope and pattern as existing `db_service`-dependent fixtures in conftest.

2. Create `tests/integration/test_session_manager.py`:
   - Import and use the `session_manager` fixture
   - `test_create_session_inserts_row` ŌĆö call `create_session()`, assert `session_manager.session_id` is a positive int and a row exists in the DB
   - `test_mark_orphaned_sessions_updates_status` ŌĆö insert a "running" session row manually, call `mark_orphaned_sessions()`, assert status updated to "unknown"
   - `test_on_service_crashed_records_failure` ŌĆö call `create_session()`, fire a mock crash event, assert session row updated with failure status
   - `test_finalize_session_writes_stopped_at` ŌĆö call `create_session()` then `finalize_session()`, assert `stopped_at` is set and status is "completed" or "failure"
   - `test_session_id_raises_before_create` ŌĆö assert `RuntimeError` on `session_manager.session_id` before `create_session()` is called

3. Review `tests/integration/test_session_lifecycle.py`:
   - Identify tests that called `Hassette._create_session(mock_self)` style ŌĆö these are now covered by `test_session_manager.py`
   - Remove or rewrite those tests to use the `HassetteHarness` or a real `Hassette` for integration-level assertions
   - Keep any tests that validate Hassette-level session behaviour (e.g., that `hassette.session_id` returns the right value after startup)

4. Run `uv run pytest tests/integration/test_session_manager.py -v` and confirm all pass
5. Run `uv run pytest -n auto` to confirm no regressions
