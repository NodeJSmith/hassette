## Objectives & Success Criteria

A new `docs/pages/testing/index.md` page exists with the full progressive-disclosure structure defined in the design doc. The page covers: installation (no `hassette[test]` reference), quick start, harness walkthrough, state seeding (with ordering warning), simulating events (with cross-handler and timeout warnings), asserting API calls (with `call_service` delegation note), time control (with canonical sequence and `whenever` imports), event factories (exactly 6 `__all__` symbols), configuration errors, `make_test_config`, and limitations/troubleshooting. `uv run mkdocs build` succeeds with no errors.

## Subtasks

1. Create `docs/pages/testing/index.md` with the 11-section structure from the design doc's Architecture section.
2. Write the **Installation** section — emphasize test_utils ships with the main package. Install runners separately: `pip install pytest pytest-asyncio` / `uv add --dev pytest pytest-asyncio`.
3. Write the **Quick Start** section — minimal 5-line AppTestHarness example. The `assert_called` example must use `"call_service"` as the method name with an explanation of why.
4. Write the **Test Harness** section — constructor parameters (`app_cls`, `config`, `tmp_path`), properties (`app`, `bus`, `scheduler`, `api_recorder`, `states`).
5. Write the **State Seeding** section — `set_state()`, `set_states()` with dict and tuple examples. Include `!!! warning` admonition about calling `set_state` before `simulate_*`.
6. Write the **Simulating Events** section — `simulate_state_change()`, `simulate_attribute_change()`, `simulate_call_service()`. Include `!!! warning` about `simulate_attribute_change` firing state-change handlers too, with code example. Include "Timeouts and slow handlers" callout: default 2s timeout, `timeout=` parameter, secondary-spawn limitation.
7. Write the **Asserting API Calls** section — `assert_called()`, `assert_not_called()`, `assert_call_count()`, `get_calls()`, `reset()`. Explicit note about `turn_on`/`turn_off`/`toggle_service` delegation to `call_service`. Note about partial matching semantics.
8. Write the **Time Control** section — lead with canonical worked sequence (`freeze_time` → schedule → `advance_time` → `trigger_due_jobs` → assert). All examples include `from whenever import Instant` with construction example and link to whenever docs. Explicit note about `advance_time` having no effect on jobs without `trigger_due_jobs`.
9. Write the **Event Factories** section — document exactly: `create_state_change_event`, `create_call_service_event`, `make_state_dict`, `make_light_state_dict`, `make_sensor_state_dict`, `make_switch_state_dict`. No Tier 2 symbols.
10. Write the **Configuration Errors** section — `AppConfigurationError`, what triggers it, how to read the error message.
11. Write the **Advanced: make_test_config** section — for users who need `HassetteConfig` without the full harness. Include `data_dir` and `**overrides` usage.
12. Write the **Limitations and Troubleshooting** section — RecordingApi coverage boundary (enumerate unimplemented methods, `api.sync` Mock warning), concurrency/xdist constraints (`process-global lock`, `xdist_group('time_control')`), harness startup failures (5s timeout, swallowed exceptions at WARNING level).
13. Verify all code examples are syntactically valid Python by reading the source files for accurate signatures.
