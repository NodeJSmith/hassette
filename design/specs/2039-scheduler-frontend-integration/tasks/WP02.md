## Objectives & Success Criteria

- `app_jobs` route handler in `src/hassette/web/routes/telemetry.py` enriches `JobSummary` rows with live-heap data (`next_run`, `fire_at`, `jitter`, `cancelled`) by joining on `db_id`.
- Enrichment degrades gracefully: if `get_all_jobs()` raises, DB rows are returned without live fields (logged warning, no 500).
- `Scheduler.cancel_group()` and `ScheduledJob.cancel()` call `mark_job_cancelled(db_id)` to persist the cancellation signal.
- `/scheduler/jobs` route, `ScheduledJobResponse` model, and router registration are deleted.
- `frontend/openapi.json` and `frontend/src/api/generated-types.ts` are regenerated with the updated schema.
- `src/hassette/web/CLAUDE.md` is trimmed to ~30 lines of Python conventions (dependency aliases, `DB_ERRORS` pattern, route registration).
- Integration tests verify enrichment, graceful degradation, and that the deleted endpoint returns 404.
