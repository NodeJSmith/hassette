## Objectives & Success Criteria

- Migration `003_scheduler_frontend_columns.py` adds `group TEXT NULL` and `cancelled_at REAL NULL` columns to `scheduled_jobs` using the table-recreation pattern from migration 002.
- `JobSummary` model gains `group`, `next_run`, `fire_at`, `jitter`, `cancelled` fields with correct defaults. `repeat` field is removed.
- `ScheduledJobRegistration` gains `group: str | None` field.
- `SchedulerService._enqueue_then_register()` forwards `job.group` when constructing `ScheduledJobRegistration`.
- `telemetry_repository.register_job()` INSERTs the `group` value.
- `telemetry_repository` gains a `mark_job_cancelled(db_id: int)` method that sets `cancelled_at = time.time()`.
- All existing tests pass; new unit tests cover the new fields and methods.

## Subtasks

1. Write migration `src/hassette/migrations/versions/003_scheduler_frontend_columns.py` following the table-recreation pattern from `002_scheduler_api_redesign.py`. Add `group TEXT NULL` and `cancelled_at REAL NULL` columns. Downgrade recreates the table without these columns.
2. Update `src/hassette/core/telemetry_models.py`: add `group: str | None = None`, `next_run: float | None = None`, `fire_at: float | None = None`, `jitter: float | None = None`, `cancelled: bool = False` to `JobSummary`. Remove `repeat: int` field.
3. Add `group: str | None = None` field to `ScheduledJobRegistration` in `src/hassette/core/registration.py`.
4. Update `src/hassette/core/scheduler_service.py` `_enqueue_then_register()` to forward `job.group` when constructing `ScheduledJobRegistration`.
5. Update `src/hassette/core/telemetry_repository.py` `register_job()` to INSERT `group`. Remove `repeat` from the SQL SELECT. Add a `mark_job_cancelled(db_id: int)` method that sets `cancelled_at = time.time()` on the row.
6. Update `src/hassette/core/telemetry_query_service.py` `get_job_summary()` SQL to include `group` and `cancelled_at` columns, and remove `repeat` from the SELECT. Populate `JobSummary.cancelled` from `cancelled_at IS NOT NULL` when returning results (the live-heap enrichment in WP02 will override this for live jobs).
