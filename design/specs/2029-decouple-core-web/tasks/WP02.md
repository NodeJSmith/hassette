## Objectives & Success Criteria

- `track_execution()` in `src/hassette/utils/execution.py` accepts a `known_errors` parameter that suppresses `error_traceback` for matched exception types (via `isinstance`)
- `CommandExecutor` has a new `_execute()` method that wraps `track_execution()` with proper try/except for CancelledError re-raise and Exception suppression
- `_execute_handler()` and `_execute_job()` are thin wrappers (~10 lines each) that call `_execute()` and build their respective record types
- The `_run_error_hooks()` stub and its call sites are removed
- All 17 existing `test_command_executor.py` integration tests pass
- The `known_errors` `isinstance` semantics are documented in `track_execution()`'s docstring

## Subtasks

1. Add `known_errors: tuple[type[Exception], ...] = ()` parameter to `track_execution()` in `src/hassette/utils/execution.py`
2. In the `except Exception` branch of `track_execution()`, check `isinstance(exc, known_errors)` and set `result.error_traceback = None` if matched
3. Update the docstring to document the `known_errors` parameter and its `isinstance` semantics (subclasses are also suppressed)
4. Update existing unit tests in `tests/unit/test_execution.py` to cover `known_errors` behavior: known error suppresses traceback, unknown error preserves traceback, subclass of known error is also suppressed
5. Create `CommandExecutor._execute()` method following the design pseudocode: `result = ExecutionResult()` init, `async with track_execution(known_errors=...)`, outer try/except for CancelledError (queue record + re-raise) and Exception (pass), then queue record and run error hooks
6. Refactor `_execute_handler()` to call `_execute()` with `cmd.listener.invoke(cmd.event)` as the callable, then build `HandlerInvocationRecord` from the returned `ExecutionResult`
7. Refactor `_execute_job()` to call `_execute()` with `cmd.callable()` as the callable, then build `JobExecutionRecord` from the returned `ExecutionResult`
8. Remove the `_run_error_hooks()` stub method and all its call sites (the call in `_execute()` is also removed since the stub is deleted)
9. Verify all 17 existing `test_command_executor.py` integration tests pass

## Test Strategy

**Updated tests:** `tests/unit/test_execution.py`
- `test_known_errors_suppresses_traceback` — DependencyError with `known_errors=(DependencyError,)` produces `error_traceback=None`
- `test_unknown_error_preserves_traceback` — ValueError with `known_errors=(DependencyError,)` produces non-None `error_traceback`
- `test_known_errors_subclass_also_suppressed` — subclass of HassetteError is suppressed when `known_errors=(HassetteError,)`
- `test_known_errors_empty_tuple_preserves_all` — default `known_errors=()` preserves traceback for all exceptions

**Existing tests:** All 17 tests in `tests/integration/test_command_executor.py` validate the exception contract (traceback presence/absence, status values, CancelledError propagation). These must pass unchanged.

## Review Guidance

- **Exception contract preservation**: The 5-branch exception handling must produce identical outcomes: CancelledError → status='cancelled' + re-raise; DependencyError/HassetteError → status='error', no traceback; generic Exception → status='error', with traceback; success → status='success'.
- **`result` initialization**: `result = ExecutionResult()` MUST appear before the `async with` block to prevent `UnboundLocalError` if CancelledError fires before `track_execution()` yields.
- **`except Exception: pass` comment**: Must include the comment explaining that swallowing is intentional because `track_execution()` already re-raised.
- **Stub removal**: `_run_error_hooks()` and all 6 call sites must be fully removed — no dead code left behind. Note: the design doc pseudocode originally included `self._run_error_hooks()` in `_execute()`, but the open question was resolved during planning to remove the stub entirely (#268 never shipped).
