## Objectives & Success Criteria

- `src/hassette/core/domain_models.py` contains Pydantic BaseModel classes: `SystemStatus`, `StateChangedData`, `AppStatusChangedData`, `ConnectivityData`, `ServiceStatusData`
- Module docstring documents the separation: "Live runtime state and event payloads. See also telemetry_models.py for DB query result models."
- `src/hassette/web/mappers.py` contains 4 mapping functions that convert domain objects to web response models
- All mapper functions preserve every field from the source objects — no silent regressions
- `SystemStatus` includes all 6 fields: `status`, `websocket_connected`, `uptime_seconds`, `entity_count`, `app_count`, `services_running`
- Unit tests verify each mapper produces correct output

## Subtasks

1. Create `src/hassette/core/domain_models.py` with module docstring explaining the separation from `telemetry_models.py`
2. Define `SystemStatus(BaseModel)` with fields: `status: Literal["ok", "degraded", "starting"]`, `websocket_connected: bool`, `uptime_seconds: float | None`, `entity_count: int`, `app_count: int`, `services_running: list[str]`
3. Define `StateChangedData(BaseModel)` mirroring the fields of `StateChangedPayload` from `web/models.py`
4. Define `AppStatusChangedData(BaseModel)` mirroring `AppStatusChangedPayload` from `web/models.py`
5. Define `ConnectivityData(BaseModel)` mirroring `ConnectivityPayload` from `web/models.py`
6. Define `ServiceStatusData(BaseModel)` mirroring `WsServiceStatusPayload` from `web/models.py`
7. Create `src/hassette/web/mappers.py` with `app_status_response_from(snapshot: AppStatusSnapshot) -> AppStatusResponse` — merges `snapshot.running` and `snapshot.failed` into a single `apps` list
8. Add `app_manifest_list_response_from(full: AppFullSnapshot) -> AppManifestListResponse` — builds nested `AppInstanceResponse` from `AppManifestInfo.instances`; note `AppInstanceInfo.status` is `ResourceStatus` (use `.value`), `AppManifestInfo.status` is already `str`
9. Add `system_status_response_from(status: SystemStatus) -> SystemStatusResponse`
10. Add `connected_payload_from(status: SystemStatus, session_id: int | None) -> ConnectedPayload`
11. Write unit tests in `tests/unit/web/test_mappers.py` for each mapping function
12. Add docstring to `src/hassette/core/telemetry_models.py` clarifying: "Pydantic models for TelemetryQueryService DB query results. See also domain_models.py for live runtime state."

## Test Strategy

**New tests:** `tests/unit/web/test_mappers.py`
- `test_app_status_response_from_merges_running_and_failed` — snapshot with 2 running + 1 failed produces response with 3 apps
- `test_app_status_response_from_empty_snapshot` — no apps produces empty list
- `test_app_manifest_list_response_from_builds_nested_instances` — verify nested AppInstanceResponse objects are built correctly
- `test_app_manifest_list_response_from_coerces_resource_status_enum` — AppInstanceInfo.status (ResourceStatus enum) → string in response
- `test_system_status_response_from_preserves_all_fields` — all 6 fields including services_running
- `test_system_status_response_from_none_uptime` — uptime_seconds=None passes through
- `test_connected_payload_from_uses_system_status_fields` — entity_count and app_count from SystemStatus, session_id from parameter

## Review Guidance

- **Field completeness**: `SystemStatus` must have all 6 fields — especially `services_running: list[str]` which was a silent regression risk identified during challenge review.
- **Enum coercion in mappers**: `app_status_response_from()` and `app_manifest_list_response_from()` must call `.value` on `ResourceStatus` enum fields. `AppManifestInfo.status` is already a plain `str` — don't call `.value` on it.
- **No web.models imports in domain_models.py**: The new domain models must not import anything from `hassette.web.models`.
- **Module docstrings**: Both `domain_models.py` and `telemetry_models.py` must have docstrings explaining their distinct roles.
