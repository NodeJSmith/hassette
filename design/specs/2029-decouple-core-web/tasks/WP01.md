## Review Guidance

- **Sentinel filter location**: The `listener_id == 0` / `session_id == 0` drop logic MUST remain in `CommandExecutor._drain_and_persist()`, not inside the repository. Verify this.
- **Lazy db access**: Repository methods MUST access `self._db_service.db` inside the coroutine body, not accept `db` as a parameter. This preserves the safe lazy-access pattern.

- **No behavior change**: This is a pure code-movement refactor. No SQL changes, no schema changes, no new behavior.
