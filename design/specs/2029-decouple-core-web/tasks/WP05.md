## Objectives & Success Criteria

- Web routes (`apps.py`, `health.py`, `telemetry.py`, `ws.py`) call mapper functions from `web/mappers.py` to convert domain objects to response models
- WS payload types (`AppStatusChangedPayload`, `ConnectivityPayload`, `StateChangedPayload`, `WsServiceStatusPayload`) are moved from `web/models.py` to `core/domain_models.py` (renamed to `*Data` variants)
- `WsServerMessage` and wrapper message types stay in `web/models.py` and import from `core/domain_models`
- `web/telemetry_helpers.py` and `web/utils.py` type annotations are updated
- `frontend/ws-schema.json` is regenerated to reflect the class renames
- `check_schemas_fresh.py` pre-push guard passes
- All existing integration tests pass

## Review Guidance

- **WsServerMessage stays in web/models.py**: Only the payload data types move. The discriminated union and wrapper message types remain in `web/models.py` and import from `core/domain_models`.
- **ws-schema.json regenerated**: The `$defs` key names change (e.g., `AppStatusChangedPayload` → `AppStatusChangedData`). Verify the regenerated schema matches.
- **No response shape changes**: The JSON responses from all REST endpoints must be identical before and after. The OpenAPI schema (`frontend/openapi.json`) should also be regenerated and should not change (response models are unchanged, only the internal wiring changed).
- **ConnectedPayload stays in web/models.py**: It's a web-layer-only type. The mapper returns it but it doesn't move to domain_models.
- **telemetry_helpers.py and utils.py**: These consume `get_all_manifests_snapshot()` which now returns `AppFullSnapshot`. Verify type annotations are consistent — `AppFullSnapshot.manifests` has the same field names as `AppManifestListResponse.manifests` but different types.
