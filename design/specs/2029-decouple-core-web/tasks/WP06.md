## Objectives & Success Criteria

- `scripts/export_schemas.py` accepts a `--types` flag that also runs `npx openapi-typescript` after generating `openapi.json`
- `frontend/package.json` `types` script updated to output `src/api/generated-types.ts`
- `frontend/src/api/generated-types.ts` is committed to the repo and generated from the current `openapi.json`
- Hand-written interfaces in `frontend/src/api/endpoints.ts` are replaced with imports from `generated-types.ts`
- Old `frontend/src/api/types.ts` is deleted if it exists
- Frontend CI job validates `generated-types.ts` freshness via `npm run types && git diff --exit-code`
- All frontend builds pass with the generated types

## Subtasks

1. Add `--types` flag to `scripts/export_schemas.py` — after writing `openapi.json`, run `npx openapi-typescript openapi.json -o src/api/generated-types.ts` from the `frontend/` directory via `subprocess.run`
2. Update `frontend/package.json` `scripts.types` to: `openapi-typescript openapi.json -o src/api/generated-types.ts`
3. Run `python scripts/export_schemas.py --types` to generate the initial `frontend/src/api/generated-types.ts`
4. Delete `frontend/src/api/types.ts` if it exists
5. Identify which interfaces in `frontend/src/api/endpoints.ts` correspond to generated types — map each hand-written interface to its OpenAPI-generated equivalent
6. Replace hand-written interfaces with imports from `generated-types.ts` in `endpoints.ts`
7. Update any other frontend files that import types from `endpoints.ts` if the import paths change
8. Regenerate `frontend/openapi.json` to capture any model changes from WP03-WP05
9. Add `generated-types.ts` freshness check to CI — in the frontend job of `.github/workflows/tests.yml`, add: `npm run types && git diff --exit-code frontend/src/api/generated-types.ts`
10. Verify `npm run build` succeeds with the generated types

## Test Strategy

**CI validation:**
- The freshness check (`npm run types && git diff --exit-code`) in the frontend CI job validates that `generated-types.ts` is up to date
- `tests/integration/test_schema_freshness.py` validates `openapi.json` freshness (already exists)
- `npm run build` validates that all TypeScript imports resolve correctly

No new Python unit tests needed — this WP is infrastructure/tooling, validated by CI and the build.
