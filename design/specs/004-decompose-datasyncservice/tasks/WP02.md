## Objectives & Success Criteria

All 9 `TelemetryQueryService` methods are backed by real SQL queries against `DatabaseService.db`. Integration tests verify correct results against a seeded in-memory SQLite database. Routes that depend on `TelemetryDep` now return real data.

- All 9 methods in `TelemetryQueryService` execute real SQL (no more `return []`/`return None` stubs)
- Methods match the query patterns in `design/research/2026-02-16-sqlite-command-pattern/prereq-04-frontend-query-requirements.md` exactly (SQL, column names, ordering, LIMIT)
- `session_id: int | None = None` parameter on all applicable methods applies the session-scoping filter from prereq 4 (JOIN condition for LEFT JOIN + GROUP BY queries; WHERE clause for direct queries)
- `get_global_summary()` returns a `dict` with listener and job aggregate totals matching prereq 4 §3
- `get_current_session_summary()` returns `None` when no running session exists; returns the summary dict otherwise
- `tests/integration/test_telemetry_query_service.py` exists and covers all 9 methods (see Test Strategy)
- `uv run pytest -n auto` passes
- `uv run pyright` passes

## Subtasks

1. Read `prereq-04-frontend-query-requirements.md` in full before writing any SQL. The exact queries, column aliases, and JOIN patterns are specified there.
2. Implement `get_listener_summary(app_key, instance_index, session_id=None)`: LEFT JOIN `handler_invocations` with optional session filter on JOIN condition; GROUP BY `listeners.id`.
3. Implement `get_job_summary(app_key, instance_index, session_id=None)`: same pattern against `scheduled_jobs` + `job_executions`.
4. Implement `get_global_summary(session_id=None)`: two queries (listener summary + job summary); return combined dict with both result sets.
5. Implement `get_handler_invocations(listener_id, limit=50)`: direct query, ORDER BY `execution_start_ts DESC`, LIMIT.
6. Implement `get_job_executions(job_id, limit=50)`: direct query, ORDER BY `execution_start_ts DESC`, LIMIT.
7. Implement `get_recent_errors(since_ts, limit=50, session_id=None)`: JOIN both `handler_invocations` + `job_executions` against parent tables; filter `status = 'error'`; time filter `execution_start_ts > since_ts`; optional session filter.
8. Implement `get_slow_handlers(threshold_ms, limit=50)`: JOIN `handler_invocations` → `listeners`; filter `duration_ms > threshold_ms`; ORDER BY `duration_ms DESC`.
9. Implement `get_session_list(limit=20)`: query `sessions` table; ORDER BY `started_at DESC`; compute `duration_seconds` via `COALESCE(stopped_at, last_heartbeat_at) - started_at`.
10. Implement `get_current_session_summary()`: query `sessions WHERE status = 'running'`; subquery counts for invocations and executions; return `None` if no running session.
11. Write `tests/integration/test_telemetry_query_service.py`.
12. Run `uv run pytest -n auto` and `uv run pyright`; fix failures.

## Test Strategy

**Approach**: new integration test module — `tests/integration/test_telemetry_query_service.py`.

Use real in-memory SQLite via `DatabaseService` pointed at a `tmp_path`. The fixture pattern from `test_command_executor.py`:

```python
@pytest.fixture
def mock_hassette(tmp_path: Path) -> MagicMock:
    hassette = MagicMock()
    hassette.config.data_dir = tmp_path
    hassette.config.db_path = None
    hassette.config.db_retention_days = 7
    hassette.ready_event = asyncio.Event()
    return hassette

@pytest.fixture
async def db(mock_hassette: MagicMock) -> AsyncIterator[tuple[DatabaseService, int]]:
    db_service = DatabaseService(mock_hassette, parent=mock_hassette)
    await db_service.on_initialize()
    cursor = await db_service.db.execute(
        "INSERT INTO sessions (started_at, last_heartbeat_at, status) VALUES (?, ?, 'running')",
        (time.time(), time.time()),
    )
    session_id = cursor.lastrowid
    await db_service.db.commit()
    mock_hassette.session_id = session_id
    mock_hassette.database_service = db_service
    yield db_service, session_id
    await db_service.db.close()
```

Instantiate `TelemetryQueryService(mock_hassette, parent=mock_hassette)` and call `await svc.on_initialize()` before each test that needs it. Seed additional rows (listeners, scheduled_jobs, handler_invocations, job_executions) via raw SQL INSERT before calling the method under test. Assert on returned dicts — column names, values, ordering, LIMIT.

**Test cases** (minimum one per method, more for edge cases):

| Test | Seeds | Asserts |
|------|-------|---------|
| `test_get_listener_summary_aggregates` | 2 listeners, 3 invocations (2 success, 1 error) | `total_invocations=3`, `failed=1`, `successful=2`, `avg_duration_ms` correct |
| `test_get_listener_summary_empty` | 1 listener, no invocations | returns 1 row, all counts 0 |
| `test_get_listener_summary_session_scoped` | 2 invocations in session A, 1 in session B | with `session_id=A` returns counts for A only |
| `test_get_job_summary_aggregates` | 2 jobs, mixed results | correct aggregate totals |
| `test_get_global_summary` | mixed listener + job records | correct combined totals |
| `test_get_handler_invocations_ordered` | 5 invocations at different timestamps | returns most recent first, respects limit |
| `test_get_job_executions_ordered` | 3 executions | ordered DESC, respects limit |
| `test_get_recent_errors_filter` | mix of success + error records, some old | only errors after `since_ts` returned |
| `test_get_recent_errors_session_scoped` | errors in two sessions | session filter applied |
| `test_get_slow_handlers` | mix of fast + slow invocations | only above threshold returned, ordered by duration |
| `test_get_session_list` | 3 sessions with different statuses | ordered by `started_at DESC`, correct duration |
| `test_get_current_session_summary_running` | 1 running session with invocations | correct invocation/execution counts |
| `test_get_current_session_summary_no_session` | no sessions seeded | returns `None` |

## Review Guidance

Verify:
- No raw string concatenation in SQL (use parameterized `?` placeholders for all values)
- `aiosqlite` `fetchall()` / `fetchone()` results are converted to `list[dict]` / `dict | None` (use `conn.row_factory = aiosqlite.Row` or manual column mapping — whichever pattern `CommandExecutor` uses)
- Session filter is applied to the JOIN condition (not WHERE) in GROUP BY queries, so rows with no invocations in that session still appear with zero counts
- `get_current_session_summary()` correctly returns `None` when `fetchone()` returns `None`
- All SQL queries match the exact patterns in prereq 4 (column aliases matter — routes will reference them by name)
