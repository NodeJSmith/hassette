## Objectives & Success Criteria

- All CSS rules in `global.css` whose selectors are not referenced by any `.tsx` component are removed
- `ht-spinner` has a working CSS definition (keyframe animation + display styles) — spinner is visible
- `ht-alert-danger` renamed to `ht-alert--danger` in `alert-banner.tsx` with corresponding CSS rule
- `ht-text-mono` has a CSS definition (`font-family: monospace`)
- `ht-tag` / `ht-tag-handler` / `ht-tag-job` have CSS definitions for error feed badges
- `ht-alert-list` verified and fixed if unstyled
- No visual regressions on any page

## Subtasks

1. **Build class inventory**: Grep all `ht-` class names from `frontend/src/**/*.tsx` files. For template literals (e.g., `` `ht-badge--${variant}` ``), expand to all known variant values by reading the component context. Record every unique class name.
2. **Cross-reference with global.css**: For each CSS rule in `global.css`, check if its selector matches any class in the inventory. Mark unmatched rules as candidates for removal.
3. **Remove confirmed dead CSS sections**: Delete rules for `.ht-entity-*`, `.ht-timeline*`, `.ht-invocation*`, and any other unmatched legacy selectors from the htmx/Alpine era.
4. **Add missing `ht-spinner` CSS**: Add a `@keyframes ht-spin` rotation animation and `.ht-spinner` styles (centered, visible, animated) to `global.css`.
5. **Fix `ht-alert-danger`**: In `frontend/src/components/layout/alert-banner.tsx`, rename `ht-alert-danger` to `ht-alert--danger`. Ensure `.ht-alert--danger` rule exists in `global.css` (if not, add it with appropriate danger color from design tokens).
6. **Add `ht-text-mono`**: Add `.ht-text-mono { font-family: var(--ht-font-mono, monospace); }` to `global.css`.
7. **Add `ht-tag` styles**: Add `.ht-tag` base styles and `.ht-tag-handler`, `.ht-tag-job` variant styles to `global.css` for the error feed badges.
8. **Verify `ht-alert-list`**: Check if styled. If not, add appropriate list styles or remove the class from the component.

## Test Strategy

- No unit tests — CSS changes are verified visually.
- **Playwright screenshots**: Take before/after screenshots of all main pages (Dashboard, Apps, App Detail, Logs) and compare. No visual differences should appear except for the fixed missing styles (spinner now visible, alert banner properly styled).
- ESLint must pass.

## Review Guidance

- Verify no live CSS rules were deleted (every removed rule must have zero references in .tsx files)
- Verify dynamic class construction was accounted for (template literals expanded)
- Verify `ht-spinner` is now visible (was an invisible empty div)
- Verify `ht-alert--danger` follows BEM convention
- BLOCKER if: any page has visual regressions from deleted CSS rules
- WARNING if: dead CSS line count is significantly different from the confirmed sections

## Pitfalls to Avoid

- Do not delete CSS rules for dynamically constructed classes (template literals). Do not assume all `ht-*` classes are dead — verify each against the `.tsx` inventory.
