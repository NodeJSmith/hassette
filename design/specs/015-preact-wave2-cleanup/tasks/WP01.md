## Objectives & Success Criteria

- `useApi` accepts an optional `lazy: true` parameter that skips the initial mount fetch, returning `loading: false` and `data: null` until `refetch()` is called manually
- `getAppJobs`, `getHandlerInvocations`, `getJobExecutions` in `endpoints.ts` return typed interfaces instead of `unknown[]`
- `handler-row.tsx` and `job-row.tsx` use `useApi` with `lazy: true` instead of raw `await` + `loaded` signal
- All `as never` casts removed from `handler-row.tsx`, `job-row.tsx`, `job-list.tsx`, `handler-invocations.tsx`, `job-executions.tsx`
- Expanding a row calls `refetch()`, stale data stays visible during refetch (stale-while-revalidate)
- Rapid expand/collapse does not cause race conditions (requestIdRef guard)
- Reconnect-auto-refetch works for lazy useApi instances

## Subtasks

1. Add `lazy?: boolean` option to `useApi` in `frontend/src/hooks/use-api.ts`. When `lazy` is true: initialize `loading` signal to `false` (not `true`), and skip the initial `useEffect` fetch (guard with `if (!lazy)` or set initial `depsVersion` to -1 so the effect doesn't match). Ensure `refetch()` still works when called manually.
2. Add `useApi` lazy test in `frontend/src/hooks/use-api.test.ts`: mount with `lazy: true`, assert fetcher is NOT called, then call `result.current.refetch()`, assert fetcher is called once.
3. Define `JobData` interface in `frontend/src/api/endpoints.ts` matching the backend `JobSummary` Pydantic model fields: `job_id`, `app_key`, `instance_index`, `job_name`, `schedule_type`, `schedule_expression`, `total_executions`, `successful`, `failed`, `avg_duration_ms`, `last_executed_at`, `next_run_at`.
4. Define `HandlerInvocationData` interface in `frontend/src/api/endpoints.ts`: `invocation_id`, `listener_id`, `timestamp`, `duration_ms`, `success`, `error_message`, `error_type`, `error_traceback`, `trigger_event_type`.
5. Define `JobExecutionData` interface in `frontend/src/api/endpoints.ts`: `execution_id`, `job_id`, `timestamp`, `duration_ms`, `success`, `error_message`, `error_type`.
6. Update return types: `getAppJobs` → `JobData[]`, `getHandlerInvocations` → `HandlerInvocationData[]`, `getJobExecutions` → `JobExecutionData[]`.
7. Refactor `handler-row.tsx`: replace the raw `await` + `loaded` + `invocations` signals with `useApi(() => getHandlerInvocations(listener.listener_id), [], { lazy: true })`. On toggle expand, call `refetch()`. Render `data.value` when non-null (show stale data during refetch). Remove `loaded` signal and manual `invocations` signal.
8. Refactor `job-row.tsx`: same pattern as step 7 with `getJobExecutions(job.job_id)`.
9. Update `job-list.tsx`: remove `as never` cast, accept `JobData[] | null` prop type.
10. Update `handler-invocations.tsx` and `job-executions.tsx`: keep their local `Invocation`/`Execution` interfaces as component props. Remove `as never` casts at the call sites in handler-row/job-row — the typed endpoint return should be assignable or use a minimal boundary cast.

## Test Strategy

- **`frontend/src/hooks/use-api.test.ts`**: Add test `it("does not fetch on mount when lazy is true")` — mount with `lazy: true`, assert fetcher called 0 times, call refetch, assert called 1 time.
- **`frontend/src/hooks/use-api.test.ts`**: Add test `it("reconnect refetch works for lazy instances after first manual fetch")` — mount lazy, call refetch, increment reconnectVersion, assert fetcher called 2 times total.
- Existing tests must continue passing (lazy defaults to false, no behavior change for existing callers).

## Review Guidance

- Verify `lazy: true` does NOT fetch on mount — this is the core behavioral change
- Verify the typed interfaces match the backend Pydantic models (compare with `src/hassette/web/models.py`)
- Verify no `as never` or `unknown[]` remains in the affected components
- Verify stale-while-revalidate: `refetch()` sets `loading=true` but does NOT clear `data.value`
- BLOCKER if: lazy option is not implemented (useApi fires on mount), or types don't match backend

## Pitfalls to Avoid

- Do not clear `data.value` on refetch. Do not fetch on mount when `lazy` is true. Do not normalize `useApi` signature — `lazy` is an options object field, not a positional param.
