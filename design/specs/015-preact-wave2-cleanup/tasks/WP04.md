## Objectives & Success Criteria

- `createAppState()` returns a `logs` object with `push(entry)`, `toArray()`, and `version` (signal)
- `push()` atomically appends to the buffer and increments version, wrapped in `batch()`
- `use-websocket.ts` calls `state.logs.push(msg.data)` (single call instead of two separate operations)
- `log-table.tsx` reads `state.logs.version.value` and `state.logs.toArray()` through the encapsulated API
- Existing log streaming behavior unchanged

## Subtasks

1. Create a `createLogStore()` factory function in `frontend/src/state/create-app-state.ts` (or a new `utils/log-store.ts` if cleaner). The factory:
   - Creates a `RingBuffer<WsLogPayload>(1000)` internally
   - Creates a `version: signal(0)` internally
   - Returns `{ push(entry: WsLogPayload): void, toArray(): WsLogPayload[], version: Signal<number> }`
   - `push` calls `batch(() => { buffer.push(entry); version.value++ })`
   - `toArray` delegates to `buffer.toArray()`
2. Replace `logs: { buffer: new RingBuffer<WsLogPayload>(1000), version: signal(0) }` in `createAppState()` with `logs: createLogStore()`.
3. Update `frontend/src/hooks/use-websocket.ts`: replace `state.logs.buffer.push(msg.data); state.logs.version.value++;` (lines 66-67) with `state.logs.push(msg.data)`.
4. Update `frontend/src/components/shared/log-table.tsx`: replace `state.logs.buffer.toArray()` with `state.logs.toArray()`, keep `state.logs.version.value` read for signal subscription.
5. Update `frontend/src/state/create-app-state.test.ts`: update the "log buffer and version signal are independent" test to use the new `push`/`toArray` API.

## Test Strategy

- **`frontend/src/state/create-app-state.test.ts`**: Update existing test to use `a.logs.push(entry)` instead of `a.logs.buffer.push(entry); a.logs.version.value++`. Assert `a.logs.toArray()` returns the entry and `a.logs.version.value === 1`. Assert `b.logs.toArray()` is empty (independence).

## Review Guidance

- Verify `push()` uses `batch()` internally — this is critical for safe reuse outside existing batch blocks
- Verify the `AppState` type still exports correctly (check that `log-table.tsx` and `use-websocket.ts` compile)
- Verify `toArray()` returns entries in correct order
- BLOCKER if: `push()` does not batch, or buffer/version can be accessed directly bypassing the encapsulation

## Pitfalls to Avoid

- Do not expose `buffer` or `version` directly — all access through `push`/`toArray`/`version` API.
