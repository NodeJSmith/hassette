## Objectives & Success Criteria

Backend event payloads and domain models carry `ready: bool` and `ready_phase: str | None` fields with safe defaults. All lifecycle transition methods thread readiness state into their emitted events. A new `Resource._emit_readiness_event()` async helper exists for mid-operation readiness changes. ServiceWatcher passes explicit readiness fields. All existing tests continue to pass — new fields use defaults that preserve current behavior.

## Subtasks

1. Add `ready: bool = False` and `ready_phase: str | None = None` fields to `ServiceStatusPayload` in `src/hassette/events/hassette.py`
2. Add `ready: bool = False` and `ready_phase: str | None = None` parameters to `HassetteServiceEvent.from_data()` and pass them through to the `ServiceStatusPayload` constructor
3. Add `ready` and `ready_phase` parameters to `LifecycleMixin._create_service_status_event()` in `src/hassette/resources/mixins.py`, defaulting to `False` and `None`
4. Update every call site of `_create_service_status_event()` in `LifecycleMixin` to pass `ready=self.is_ready()` and `ready_phase=self._ready_reason`: `handle_running`, `handle_stop`, `handle_failed`, `handle_starting`, `handle_crash`
5. Update the `_LifecycleHostStubs` Protocol (TYPE_CHECKING block in `mixins.py`) to match the new signature
6. Add `async def _emit_readiness_event(self) -> None` to `Resource` in `src/hassette/resources/base.py` — constructs and sends a `service_status` event with the current readiness state using `_create_service_status_event(self._status, ready=self.is_ready(), ready_phase=self._ready_reason)`
7. Add `ready: bool = False` and `ready_phase: str | None = None` to `ServiceStatusData` in `src/hassette/core/domain_models.py`
8. Map `data.ready` and `data.ready_phase` in `RuntimeQueryService._on_service_status()` in `src/hassette/core/runtime_query_service.py`
9. Update `ServiceWatcher._emit_service_status_event()` in `src/hassette/core/service_watcher.py` to pass `ready=False, ready_phase=None` explicitly in all `ServiceStatusPayload(...)` constructor calls (both the method and the two synthetic event sites)
10. Add `ready: bool = False` and `ready_phase: str | None = None` parameters to `simulate_hassette_service_status()` in `src/hassette/test_utils/simulation.py` and pass them through to `HassetteServiceEvent.from_data()`. Add a `simulate_hassette_service_ready()` convenience that passes `status=RUNNING, ready=True`.
11. Regenerate WS schema: `uv run python scripts/export_schemas.py`

## Test Strategy

**New tests in `tests/unit/events/test_service_status_payload.py`:**
- `test_service_status_payload_defaults`: construct `ServiceStatusPayload` with no `ready`/`ready_phase` — assert `ready=False, ready_phase=None`
- `test_service_status_payload_explicit`: construct with `ready=True, ready_phase="connected"` — assert values pass through
- `test_from_data_includes_ready_fields`: call `HassetteServiceEvent.from_data(ready=True, ready_phase="test")` — assert payload carries both fields
- `test_from_data_ready_defaults`: call `from_data()` without ready params — assert `ready=False, ready_phase=None`

**New tests in `tests/unit/resources/test_emit_readiness_event.py`:**
- `test_emit_readiness_event_sends_service_status`: construct a stub Resource in RUNNING state with `is_ready()=True` and `_ready_reason="test reason"`, call `_emit_readiness_event()`, assert `hassette.send_event` called with a `ServiceStatusPayload` where `ready=True, ready_phase="test reason", status=RUNNING`
- `test_handle_running_includes_readiness`: construct a stub Resource with `is_ready()=False`, call `handle_running()`, assert the emitted event has `ready=False`

**New tests in `tests/unit/resources/test_lifecycle_side_effect_free.py`:**
- `test_mark_ready_does_not_emit`: call `mark_ready()` on a stub, assert `hassette.send_event` was NOT called
- `test_mark_not_ready_does_not_emit`: call `mark_not_ready()` on a stub, assert `hassette.send_event` was NOT called

**Update existing tests:**
- `tests/unit/test_ws_models.py`: add `ready` and `ready_phase` fields to `ServiceStatusPayload` constructions in `TestServiceStatusPayloadMatchesDataclass` and `TestServiceStatusDataRetryAt`
- `tests/integration/test_service_watcher.py`: add `ready=False, ready_phase=None` to direct `ServiceStatusPayload(...)` constructions at lines 388 and 442
- `tests/integration/test_session_manager.py`: add `ready=False, ready_phase=None` to the `ServiceStatusPayload(...)` construction at line 31

## Review Guidance

- Verify all `_create_service_status_event()` call sites pass `ready=self.is_ready()` and `ready_phase=self._ready_reason` — missing any call site means that status transition silently omits readiness
- Verify `_emit_readiness_event()` lives on `Resource` (not `LifecycleMixin`) — it needs the hassette context which only Resource has at runtime
- Verify `mark_ready()` and `mark_not_ready()` remain side-effect-free — no event emission, no async calls
- Verify the `_LifecycleHostStubs` Protocol stub in the TYPE_CHECKING block of `mixins.py` is updated to match the new `_create_service_status_event()` signature — this is type-checking-only but Pyright will flag mismatches
- Verify ServiceWatcher passes explicit `ready=False, ready_phase=None` — not relying on dataclass defaults
- Verify `ws-schema.json` was regenerated and includes `ready` and `ready_phase` in `ServiceStatusData`
