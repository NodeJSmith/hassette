## Objectives & Success Criteria

When both `immediate=True` and `duration=N` are specified, the framework consults `last_changed` at registration time to compute elapsed time. If elapsed >= duration, fires immediately. If elapsed < duration, starts a timer for the remaining time. Clock skew is handled with `[0, duration]` clamping. `on_attribute_change` always starts from zero. All combo tests pass.

## Subtasks

1. Implement elapsed-time computation in the immediate-fire task (`_spawn_immediate_fire_task` in bus_service.py):
   - Parse `last_changed` from state dict via `convert_datetime_str_to_system_tz()` (from `hassette.events.hass`)
   - Compute `now = date_utils.now()` (ZonedDateTime, same clock as `App.now`)
   - Compute `elapsed = (now - last_changed).total_seconds()` — clamp to `[0, duration]`
   - If `last_changed` is None → elapsed = 0 (full timer restart)
   - If elapsed >= duration → check `listener._fired` (once-guard), then dispatch immediately
   - If elapsed < duration → call `listener._duration_timer.start()` with remaining time `(duration - elapsed)`
2. For `on_attribute_change + immediate + duration`: always set elapsed = 0 (skip `last_changed` consultation). This is the documented known limitation.
3. Add DB migration in `src/hassette/core/telemetry_repository.py` for the new `immediate` and `duration` columns on the listener_registrations table (nullable columns with defaults)

## Test Strategy

Integration tests in `tests/integration/test_bus_immediate.py` (additions to existing file from WP03):
- `test_immediate_duration_fires_when_elapsed_exceeds` — entity held for 10s, duration=5 → fires immediately
- `test_immediate_duration_starts_timer_for_remaining` — entity held for 3s, duration=5 → timer fires after 2s
- `test_immediate_duration_last_changed_none` — last_changed missing → full timer (elapsed=0)
- `test_immediate_duration_negative_elapsed_clamped` — clock skew produces negative → treated as 0
- `test_immediate_duration_attribute_change_always_zero` — on_attribute_change always starts full timer
- `test_immediate_duration_once_fires_exactly_once` — immediate fire + once → consumed; no subsequent fire

## Review Guidance

- Verify clock source is `date_utils.now()` (ZonedDateTime), NOT `datetime.now(tz=UTC)` or `datetime.utcnow()`
- Verify `last_changed` is parsed via `convert_datetime_str_to_system_tz()`, NOT raw string comparison
- Verify elapsed is clamped to `[0, duration]` — negative values become 0, not negative timer
- Verify `on_attribute_change` explicitly skips `last_changed` computation (always elapsed=0)
- Verify DB migration is backward-compatible (nullable columns with defaults)
- Verify the once-guard (`listener._fired` check) is in the immediate-fire elapsed >= duration path
