## Objectives & Success Criteria

Handlers registered with `duration=N` fire only after the entity holds the matching state continuously for N seconds. The cancellation listener re-evaluates predicates before cancelling (does not cancel on attribute-only refreshes). The `_dispatch()` finally block skips removal for duration listeners. Once-guard is handled by routing through `listener.dispatch()`. All duration tests pass.

## Subtasks

1. Implement duration dispatch path in `BusService._dispatch()`:
   - After `listener.matches(event)` passes and `listener.duration is not None`, call `listener._duration_timer.start(event, on_fire_callback)` instead of `listener.dispatch(invoke_fn)`
   - The `on_fire_callback` coroutine: re-reads state from StateProxy via `self.hassette._state_proxy.states.get(entity_id)`, re-evaluates predicates, if still matching → calls `listener.dispatch(invoke_fn)` (delegates once-guard to existing code path), if not matching → silently drops
   - After `listener.dispatch()` returns (regardless of success/exception), if `listener.once` → remove listener unconditionally
2. Modify `_dispatch()` finally block: skip `once` listener removal when `listener.duration is not None` — removal is delegated to the DurationTimer callback
3. Wire the cancellation subscription creation:
   - `DurationTimer.start()` creates the cancellation listener using `BusService._create_cancel_listener()` (new private method)
   - Cancellation listener: same `owner_id` as main listener, `source_tier="framework"`, bypasses DB registration, registered directly on Router
   - Cancellation handler: receives every `state_changed` event for the entity, calls `DurationTimer._on_cancel_event(event)` which re-evaluates predicates before cancelling
4. Add logging at DEBUG level for DurationTimer state transitions: start, cancel, fire, dropped (matches the pattern from RateLimiter)

## Test Strategy

Integration tests in `tests/integration/test_bus_duration.py` (new file):
- `test_duration_fires_after_held` — state held for duration → handler fires with original triggering event
- `test_duration_cancelled_on_state_exit` — state changes away before duration → no fire
- `test_duration_resets_on_re_entry` — state leaves and returns → timer restarts from zero, fires after second hold
- `test_duration_double_check_before_fire` — state reverts between timer start and fire → no fire (state re-verification)
- `test_duration_with_once_fires_exactly_once` — once=True + duration: fires once, subsequent events ignored
- `test_duration_once_removal_on_exception` — handler raises → listener still removed (once contract upheld)
- `test_duration_subscription_cancel_stops_timer` — cancel subscription while timer pending → no fire, no leak
- `test_duration_not_cancelled_by_attribute_refresh` — attribute-only state_changed event (same state value) does not cancel timer for on_state_change
- `test_duration_attribute_change_cancel_only_on_predicate_fail` — for on_attribute_change, unrelated attribute changes don't cancel
- `test_duration_multiple_listeners_independent` — two listeners with different durations on same entity maintain independent timers
- `test_duration_cancel_listener_uses_framework_tier` — cancellation listener has source_tier="framework"
- `test_duration_cancel_listener_same_owner_id` — cleaned up by remove_listeners_by_owner()

## Review Guidance

- Verify cancellation listener re-evaluates predicates (NOT blind cancel on any state_changed)
- Verify `_dispatch()` finally block change is conditional on `listener.duration is not None` — non-duration path unchanged
- Verify DurationTimer callback routes through `listener.dispatch(invoke_fn)` for once-guard — NOT manual `_fired` check + `invoke_fn()`
- Verify once removal in callback is unconditional (regardless of success/exception)
- Verify cancellation listener uses same `owner_id` as main listener
- Verify cancellation listener bypasses DB registration
