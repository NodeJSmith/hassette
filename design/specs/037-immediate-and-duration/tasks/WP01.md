## Objectives & Success Criteria

Listener and ListenerRegistration dataclasses accept `immediate`, `duration`, and `entity_id` fields. Validation rejects invalid combinations (`duration + debounce`, `duration + throttle`). All existing tests continue to pass unchanged ‚Äî new fields have defaults so no construction sites break.

## Subtasks

1. Add `immediate: bool = False`, `duration: float | None = None`, and `entity_id: str | None = None` fields to the `Listener` dataclass in `src/hassette/bus/listeners.py`
2. Add `immediate: bool = False` and `duration: float | None = None` parameters to `Listener.create()` and thread them to the dataclass constructor
3. Extend `Listener._validate_options()` to reject `duration + debounce`, `duration + throttle` (ValueError), and require `duration > 0` when provided. `once + duration` is explicitly allowed ‚Äî do NOT add a prohibition.
4. Add `immediate: bool = False` and `duration: float | None = None` as optional fields with defaults to `ListenerRegistration` in `src/hassette/core/registration.py` (appended after existing fields to preserve all 15+ existing construction sites)
5. Add `immediate` and `duration` parameters to `on_state_change()` and `on_attribute_change()` in `src/hassette/bus/bus.py` ‚Äî these are explicit named parameters on these methods only, NOT added to the `Options` TypedDict. Extract them before calling `_subscribe` and pass directly to `Listener.create()`.
6. Add `entity_id` population: `on_state_change` and `on_attribute_change` pass the `entity_id` argument to `Listener.create()` which stores it on the Listener
7. Add glob + `immediate=True` validation in `on_state_change` and `on_attribute_change`: reject with ValueError if entity_id contains `*` or `?` and `immediate=True`
8. Update `BusService._register_then_add_route()` in `src/hassette/core/bus_service.py` to include `immediate` and `duration` in the `ListenerRegistration` construction

## Test Strategy

Unit tests in `tests/unit/bus/test_listeners.py`:
- `test_listener_create_with_immediate_true` ‚Äî Listener.create(immediate=True) sets field correctly
- `test_listener_create_with_duration` ‚Äî Listener.create(duration=5.0) sets field correctly
- `test_listener_create_with_entity_id` ‚Äî entity_id is stored on Listener
- `test_validate_duration_must_be_positive` ‚Äî duration=0 and duration=-1 raise ValueError
- `test_validate_duration_conflicts_with_debounce` ‚Äî ValueError
- `test_validate_duration_conflicts_with_throttle` ‚Äî ValueError
- `test_validate_once_plus_duration_allowed` ‚Äî no error raised

Integration tests in `tests/integration/test_bus.py` (or inline in existing test file):
- `test_on_state_change_accepts_immediate_param` ‚Äî registration succeeds
- `test_on_state_change_accepts_duration_param` ‚Äî registration succeeds
- `test_on_state_change_rejects_glob_with_immediate` ‚Äî ValueError raised
- `test_on_attribute_change_accepts_immediate_param` ‚Äî registration succeeds
