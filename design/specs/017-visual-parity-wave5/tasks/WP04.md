## Objectives & Success Criteria

- Level, App, and Message columns in the log table are sortable (in addition to existing Timestamp sort)
- Sort comparator is extracted to a named `sortEntries()` function at the top of the file
- `aria-sort` attribute appears only on the active sort column
- Each new sort button has a `data-testid` attribute (`sort-level`, `sort-app`, `sort-message`)
- When a non-timestamp sort is active, WebSocket log entries stop accumulating and a "Live updates paused" indicator appears
- Live streaming resumes when the user switches back to timestamp sort or clicks "Resume"
- Default sort remains Timestamp descending
- Existing `log-table.test.tsx` tests pass; new tests cover multi-column sort and pause behavior
