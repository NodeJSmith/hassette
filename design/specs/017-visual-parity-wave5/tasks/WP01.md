## Objectives & Success Criteria

- Dashboard app cards display error rate as a percentage (e.g., "2.1% errors") when `total_invocations + total_executions > 0`
- Error rate text is colored using `healthGradeToVariant(app.health_status)` as a proxy for severity
- No error rate text is shown when total is 0
- Existing `app-card.test.tsx` passes; new test cases cover the error rate display

## Subtasks

1. Read `frontend/src/components/dashboard/app-card.tsx` to confirm current state matches the design doc description
2. Write test in `frontend/src/components/dashboard/app-card.test.tsx`:
   - `test_error_rate_shown_when_invocations_exist` — render with `total_invocations=100, total_errors=5`, assert "5.0% errors" text present
   - `test_error_rate_hidden_when_no_invocations` — render with `total_invocations=0, total_executions=0`, assert no error rate text
   - `test_error_rate_uses_health_grade_color` — render with `health_status="critical"`, assert the text has `ht-text-danger` (or equivalent variant class)
3. In `frontend/src/components/dashboard/app-card.tsx`:
   - Import `healthGradeToVariant` from `../../utils/status`
   - Compute `const rate = total > 0 ? (errors / total) * 100 : 0;`
   - Add a `<span>` after the `ht-app-card__counts` div (inside the conditional `total > 0` block) displaying `{rate.toFixed(1)}% errors` with class `ht-text-xs ht-text-${healthGradeToVariant(app.health_status)}`
   - Add a code comment: `// health_status is a grade proxy, not error_rate_class — see design doc 017`
4. Run tests to verify all pass
