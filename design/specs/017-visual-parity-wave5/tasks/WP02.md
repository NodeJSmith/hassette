## Objectives & Success Criteria

- Error feed badge displays the exception type name (e.g., "ValueError") instead of "handler"/"job"
- Badge falls back to `err.kind` when `err.error_type` is empty
- Long module-qualified exception names are truncated to the last dotted component (e.g., "ServiceNotFound")
- CSS class interpolation for `kind` is guarded against unknown values (falls back to "neutral")
- `handler_method` (for handler errors) or `job_name` (for job errors) appears in the error entry subtitle
- ErrorEntry key uses `listener_id`/`job_id` instead of timestamp+index
- Existing tests pass; new test cases added

2. Write tests (create `frontend/src/components/dashboard/error-feed.test.tsx` if it doesn't exist):
   - `test_badge_shows_error_type_not_kind` — render with `error_type="ValueError"`, assert badge text is "ValueError"
   - `test_badge_falls_back_to_kind_on_empty_error_type` — render with `error_type=""`, assert badge text is "handler"
   - `test_long_error_type_truncated` — render with `error_type="homeassistant.exceptions.ServiceNotFound"`, assert badge text is "ServiceNotFound"

   - Add a helper `function shortErrorType(t: string): string` that returns the substring after the last `.`, or the full string if no dots

   - Change key from `${err.timestamp}-${err.app_key}-${i}` to `${err.listener_id ?? err.job_id ?? err.timestamp}-${err.app_key}`
