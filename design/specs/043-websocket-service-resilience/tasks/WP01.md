## Objectives & Success Criteria

Add 8 new configuration fields to `HassetteConfig` for the three-layer retry model, and add an optional `close_code` parameter to `RetryableConnectionClosedError`. After this WP:

- All 8 new fields are readable on a default `HassetteConfig` instance with correct types and defaults
- `RetryableConnectionClosedError` accepts an optional `close_code: int | None` keyword argument
- `RetryableConnectionClosedError.close_code` is accessible on instances
- Existing code that constructs `RetryableConnectionClosedError` with a message-only argument continues to work (close_code defaults to `None`)
- `pyright` and `ruff` pass on changed files

## Subtasks

1. Add 8 config fields to `HassetteConfig` in `src/hassette/config/config.py` after the existing `websocket_heartbeat_interval_seconds` field (line 191):
   - `websocket_connect_retry_max_attempts: int = Field(default=5)` with docstring
   - `websocket_connect_retry_initial_wait_seconds: float = Field(default=1.0)` with docstring
   - `websocket_connect_retry_max_wait_seconds: float = Field(default=32.0)` with docstring
   - `websocket_early_drop_stable_window_seconds: float = Field(default=30.0)` with docstring
   - `websocket_early_drop_max_retries: int = Field(default=5)` with docstring
   - `websocket_early_drop_backoff_initial_seconds: float = Field(default=2.0)` with docstring
   - `websocket_early_drop_backoff_max_seconds: float = Field(default=60.0)` with docstring
   - `websocket_max_recovery_seconds: float = Field(default=300.0)` with docstring
   - Follow existing docstring style: single-line description per field (see lines 178-191 for pattern)

2. Add `close_code` parameter to `RetryableConnectionClosedError` in `src/hassette/exceptions.py` (line 48). Follow the `FailedMessageError` pattern (lines 73-82):
   ```python
   class RetryableConnectionClosedError(ConnectionClosedError):
       def __init__(self, msg: str, *, close_code: int | None = None) -> None:
           super().__init__(msg)
           self.close_code = close_code
   ```

3. Write unit tests for the new config fields. Check for existing config tests first: look in `tests/unit/config/`, `tests/integration/test_config.py`, or grep for `HassetteConfig` in the test directories. Add to the existing config test file if one exists; otherwise create `tests/unit/config/test_config.py`. Each field must be verified: correct default value, correct type.

4. Write unit tests for `RetryableConnectionClosedError.close_code`: message-only construction (close_code is None), construction with close_code, and that the exception is still a subclass of `ConnectionClosedError`.

## Test Strategy

**Config tests** in the existing config test file (locate via `find tests -name '*config*'`):
- `test_websocket_connect_retry_defaults` — verify all 3 connect_retry fields have expected defaults
- `test_websocket_early_drop_defaults` — verify all 4 early_drop fields have expected defaults
- `test_websocket_max_recovery_default` — verify the recovery cap default is 300.0

**Exception tests** in `tests/unit/` (new file `tests/unit/test_exceptions.py` if none exists, or add to existing):
- `test_retryable_connection_closed_error_default_close_code` — close_code is None when not provided
- `test_retryable_connection_closed_error_with_close_code` — close_code is set when provided
- `test_retryable_connection_closed_error_inheritance` — still isinstance of ConnectionClosedError

## Review Guidance

- Config field names must match the design doc exactly (design.md § New config fields)
- Defaults must match current hardcoded behavior for connect_retry_* fields (5 attempts, 1.0s initial, 32.0s max)
- The `close_code` parameter must be keyword-only to avoid breaking existing callers that pass message as positional
- No behavioral changes in this WP — only additive types and config
