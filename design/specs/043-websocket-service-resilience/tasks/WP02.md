## Objectives & Success Criteria

Replace the `connected` property's access to `self._ws._conn.closed` (private aiohttp internal) with the public `self._ws.closed` API. Add `WSMsgType.ERROR` handling to `_raw_recv` to prevent silent recv loop spin. After this WP:

- `connected` returns correct values for all states (ws=None, ws.closed=True, ws.closed=False) without accessing `_conn`
- `_raw_recv` raises `RetryableConnectionClosedError` when it receives a `WSMsgType.ERROR` frame
- The `_build_fake_ws()` test helper no longer stubs `_conn`
- All existing integration tests pass
- `pyright` and `ruff` pass

## Subtasks

1. Replace the `connected` property in `src/hassette/core/websocket_service.py` (lines 118-126):
   ```python
   @property
   def connected(self) -> bool:
       return self._ws is not None and not self._ws.closed
   ```

2. Add `WSMsgType.ERROR` handling to `_raw_recv` in `src/hassette/core/websocket_service.py` (after the CLOSING branch around line 405). When `msg.type == WSMsgType.ERROR`, raise `RetryableConnectionClosedError` with the error details, chaining the original exception if `msg.data` is a `BaseException`.

3. Update `_build_fake_ws()` in `tests/integration/test_websocket_service.py` (line 30): remove `fake_ws._conn = SimpleNamespace(closed=is_closed)`. The `closed` attribute is already set on the fake_ws itself.

4. Update `test_connected_reflects_websocket_state` and any other tests that set `fake_ws._conn.closed` — replace with setting `fake_ws.closed` directly.

5. Write a test for `WSMsgType.ERROR` handling: mock `_ws.receive()` to return a `WSMessage(WSMsgType.ERROR, RuntimeError("socket error"), None)` and verify `_raw_recv` raises `RetryableConnectionClosedError`.
