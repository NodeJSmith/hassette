## Objectives & Success Criteria

Extract `_connect_ws()`, `_start_recv_and_subscribe()`, and `_partial_cleanup()` from `_make_connection()`. Replace hardcoded tenacity retry parameters with config values from WP01. Add `_connected_at` instance attribute. After this WP:

- `_connect_ws(session)` opens the WebSocket and authenticates (sets `self._ws`)
- `_start_recv_and_subscribe()` spawns recv loop, sends connection event, subscribes, marks ready, and records `_connected_at`
- `_partial_cleanup()` cancels recv task, closes WebSocket, clears futures and subscriptions (preserves session), suppresses all errors, bounded by a 2s timeout on the cancel+gather
- `_make_connection` delegates to `_connect_ws` and `_start_recv_and_subscribe` within the tenacity retry decorator
- Tenacity `stop_after_attempt`, `wait_exponential_jitter` use config values instead of hardcoded 5/1/32
- `_connected_at: float | None` initialized to `None` in `__init__`, reset at start of each `_make_connection` call
- Observable behavior of `_make_connection()` is unchanged (same retry logic, same success/failure semantics)
- All existing integration tests pass
- `pyright` and `ruff` pass

## Subtasks

1. Add `self._connected_at: float | None = None` to `WebsocketService.__init__` in `src/hassette/core/websocket_service.py` (around line 91).

2. Extract `_connect_ws(self, session: aiohttp.ClientSession) -> None` from the first half of `_inner_connect` (lines 163-176):
   - Sets `self._session = session`
   - Calls `session.ws_connect()` with heartbeat and SSL config
   - Converts `ClientConnectorError` with `ConnectionRefusedError` to `CouldNotFindHomeAssistantError`
   - Calls `self.authenticate()`
   - Sets `self._ws`

3. Extract `_start_recv_and_subscribe(self) -> asyncio.Task` from the second half of `_inner_connect` (lines 178-185):
   - Spawns recv loop via `self.task_bucket.spawn(self._recv_loop(), name="ws:recv")`
   - Calls `self._send_connection_established_event()`
   - Subscribes via `self._subscribe_events()` and adds to `self._subscription_ids`
   - Calls `self.mark_ready(reason="WebSocket connected, authenticated, and subscribed")`
   - Records `self._connected_at = time.monotonic()`
   - Returns the recv task

4. Create `_partial_cleanup(self) -> None` as a new async method:
   - Cancel `_recv_task` if it exists, gather with `return_exceptions=True` bounded by `asyncio.wait_for(..., timeout=2.0)` (catching TimeoutError with a warning)
   - Close `_ws` if it exists and is not closed
   - Clear `_response_futures` (set `RetryableConnectionClosedError("WebSocket disconnected")` on pending futures, then clear — matches the exception type used in `cleanup()` at websocket_service.py:212)
   - Clear `_subscription_ids`
   - Set `_ws = None`, `_recv_task = None`
   - Wrap entire body in `try/except Exception` + log warning — cleanup must never prevent retry

5. Refactor `_make_connection` to delegate to the new methods. Reset `self._connected_at = None` at the top. Replace hardcoded tenacity params:
   ```python
   @retry(
       retry=retry_if_not_exception_type(NON_RETRYABLE) | retry_if_exception_type(RETRYABLE),
       wait=wait_exponential_jitter(
           initial=self.hassette.config.websocket_connect_retry_initial_wait_seconds,
           max=self.hassette.config.websocket_connect_retry_max_wait_seconds,
       ),
       stop=stop_after_attempt(self.hassette.config.websocket_connect_retry_max_attempts),
       reraise=True,
       before_sleep=before_sleep_log(self.logger, logging.WARNING),
   )
   ```

6. Add `import time` to the imports if not already present.

7. Write unit tests for each extracted method (see Test Strategy).

## Review Guidance

- `_partial_cleanup` MUST NOT close `self._session` — that's owned by `serve()`'s `async with` block
- `_partial_cleanup` MUST suppress all exceptions — a cleanup failure must never prevent retry
- The `_partial_cleanup` cancel+gather MUST have a 2s timeout to prevent hanging on non-cancellable tasks
- `_connected_at` MUST be reset to `None` at the start of `_make_connection`, not just in `__init__`
- Tenacity config values are captured at decoration time (inside `_make_connection`), which runs per-call — this is correct
- The `_inner_connect` function inside `_make_connection` MUST still exist as the tenacity-decorated wrapper
