## Objectives & Success Criteria

Implement the early-drop retry loop inside `serve()` that detects post-ready connection drops within the stable window and retries transparently. Add token revocation detection (distinctive log on `InvalidAuthError` during reconnection). Make `_send_connection_lost_event()` idempotent and self-suppressing. Add the resilience budget startup log. After this WP:

- A simulated early-drop within the stable window is retried without calling `handle_failed()` or incrementing ServiceWatcher restart counter
- After exhausting `websocket_early_drop_max_retries` or `websocket_max_recovery_seconds`, the failure propagates to `_serve_wrapper` normally
- A drop outside the stable window propagates immediately without retry
- Non-retryable exceptions (e.g., `RuntimeError`) within the stable window propagate immediately
- `InvalidAuthError` during reconnection (after at least one early-drop retry) logs "Authentication failed on reconnect — possible token revocation" before propagating
- `_send_connection_lost_event()` is idempotent (skips if already not-ready) and self-suppressing (wraps bus dispatch in `suppress(Exception)`)
- The close code from `self._ws.close_code` is passed to `RetryableConnectionClosedError` when raising in `_raw_recv`
- At startup, an INFO log shows the computed resilience budget
- All existing integration tests pass
- `pyright` and `ruff` pass

## Subtasks

1. Define `EARLY_DROP_RETRYABLE` tuple at module level in `src/hassette/core/websocket_service.py`:
   ```python
   EARLY_DROP_RETRYABLE = (RetryableConnectionClosedError, ServerDisconnectedError)
   ```

2. Implement `_early_drop_backoff(self, attempt: int) -> None` using the service_watcher pattern:
   ```python
   backoff = min(
       config.websocket_early_drop_backoff_initial_seconds * (2 ** (attempt - 1)),
       config.websocket_early_drop_backoff_max_seconds,
   ) + random.uniform(0, config.websocket_early_drop_backoff_initial_seconds)
   await asyncio.sleep(backoff)
   ```
   Add `import random` if not already present.

3. Rewrite `serve()` with the early-drop retry loop per the design doc pseudocode (design.md § Early-drop retry loop). Key elements:
   - Wrap both `_make_connection` and `recv_task` in a single try block
   - Explicit `except InvalidAuthError` branch that logs distinctive message when `early_drop_attempts > 0`, then re-raises
   - `except Exception` branch with the stable-window check: compute `elapsed` conditionally (`float('inf')` if `_connected_at` is None), check `recovery_elapsed < max_recovery`
   - If early drop: increment counter, set `recovery_started_at` on first drop, log with WARNING level (elapsed time, attempt count, max, close code if available), call `mark_not_ready`, `_send_connection_lost_event`, `_partial_cleanup`, `_early_drop_backoff`, continue
   - If not early drop: `mark_not_ready`, `_send_connection_lost_event`, raise

4. Make `_send_connection_lost_event()` idempotent and self-suppressing:
   - Add `if not self.is_ready(): return` guard at the top
   - Wrap the event bus dispatch in `with suppress(Exception):`
   - Remove any external `suppress()` wrappers from callers (the old `serve()` had one)

5. Update `_raw_recv` to pass `close_code=self._ws.close_code` when raising `RetryableConnectionClosedError` for CLOSE/CLOSED/CLOSING/ERROR frames.

6. Add resilience budget startup log in `on_initialize()` or at the top of `serve()`:
   ```python
   self.logger.info(
       "WebSocket resilience budget: max ~%.0f minutes to permanent shutdown "
       "(early-drop: %d retries capped at %ds, connection: %d retries, service: %d restarts)",
       max_recovery / 60, max_early_drops, max_recovery,
       config.websocket_connect_retry_max_attempts,
       config.service_restart_max_attempts,
   )
   ```

7. Write integration tests for all early-drop scenarios (see Test Strategy).

## Test Strategy

**New tests** in `tests/integration/test_websocket_service.py`:

- `test_early_drop_retries_and_succeeds` — Patch `_make_connection` to return a task that fails immediately on first 2 calls (with `RetryableConnectionClosedError`), succeeds on 3rd. Set `_connected_at` to recent timestamp. Verify: `handle_failed` never called, `_make_connection` called 3 times, `_partial_cleanup` called 2 times, connection-lost event emitted 2 times, connection-established event emitted on 3rd.

- `test_early_drop_exhausts_retry_budget` — Patch `_make_connection` so every recv task raises `RetryableConnectionClosedError` immediately. Set max_retries to 2. Verify: exception propagates out of `serve()` after 2 retries, `mark_not_ready` called on final failure.

- `test_early_drop_exhausts_recovery_timeout` — Patch `time.monotonic()` to simulate recovery_elapsed exceeding `max_recovery_seconds`. Verify: failure propagates without further retry even if attempt count is not exhausted.

- `test_stable_connection_failure_propagates_immediately` — Set `_connected_at` to 60 seconds ago (beyond stable window). Recv task raises `RetryableConnectionClosedError`. Verify: propagates immediately, no retry, no `_partial_cleanup` call.

- `test_non_retryable_exception_in_stable_window` — Recv task raises `RuntimeError` within stable window. Verify: propagates immediately, not treated as early drop.

- `test_auth_failure_on_reconnect_logs_distinctive_message` — First iteration: early drop triggers retry. Second iteration: `_make_connection` raises `InvalidAuthError`. Verify: `InvalidAuthError` propagates, logger.error was called with "possible token revocation" substring.

- `test_send_connection_lost_event_idempotent` — Call `_send_connection_lost_event` when service is already not-ready. Verify: no event emitted (bus.send_event not called).

- `test_send_connection_lost_event_self_suppressing` — Mock bus to raise on send_event. Verify: no exception propagates from `_send_connection_lost_event`.

- `test_raw_recv_passes_close_code` — Mock `_ws.close_code` to return 1001. Trigger CLOSE frame. Verify: raised `RetryableConnectionClosedError` has `close_code == 1001`.

- `test_service_status_stays_running_during_early_drop` — During early-drop retry, verify `websocket_service.status == ResourceStatus.RUNNING` and `websocket_service.is_ready() == False`.

## Review Guidance

- The `try` block in `serve()` MUST wrap both `_make_connection` and `recv_task` — without this, `InvalidAuthError` from `_make_connection` bypasses the early-drop handler and the distinctive token-revocation log message never appears
- `elapsed` computation MUST be conditional: `float('inf')` when `_connected_at` is None — without this, any exception before the first successful connection crashes with `TypeError` from `monotonic() - None` instead of propagating the real error
- `_send_connection_lost_event` MUST be self-suppressing — no external `suppress()` wrappers at call sites — otherwise a bus error during disconnect masks the original network exception
- The early-drop retry MUST check both `early_drop_attempts < max_early_drops` AND `recovery_elapsed < max_recovery` — the total recovery cap prevents three independent retry layers from multiplying to 21+ minutes before permanent shutdown
- `EARLY_DROP_RETRYABLE` is a SUBSET of `RETRYABLE` — it must NOT include `ClientConnectorError` or `CouldNotFindHomeAssistantError`
- Close code logging is for diagnostics only — no behavioral decisions based on close codes
