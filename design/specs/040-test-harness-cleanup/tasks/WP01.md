## Objectives & Success Criteria

- `Hassette.__init__` performs only bare setup: config assignment, `enable_logging()`, `super().__init__()`, and null service slot declarations. No context var registration, no env file loading, no `add_child()` calls.
- `Hassette.startup_tasks()` is a public method that performs one-time environment and manifest setup (formerly `_startup_tasks()`).
- `Hassette.wire_services()` is a public method that calls `startup_tasks()`, registers context vars, wires all 18 services via `add_child()`, and validates the dependency graph.
- `run_forever()` raises `RuntimeError("call wire_services() before run_forever()")` when `_init_waves` is empty.
- `event_streams_closed` returns `True` when `_event_stream_service is None`.
- All 3 production call sites (`__main__.py`, `tests/system/conftest.py`, `tests/integration/conftest.py:hassette_instance`) call `wire_services()` after construction with no behavioral change.
- `run_hassette_startup_tasks()` in `fixtures.py` uses `Hassette(config).startup_tasks()` — no SimpleNamespace hack.
- All existing tests pass.

## Subtasks

1. In `src/hassette/core/core.py`, extract the bare setup portion of `__init__`: keep `self.config`, `enable_logging()`, `super().__init__(self)`, loop vars. Add all ~18 service slot declarations as `None`. Add `self._init_order: list[type[Resource]] = []` and `self._init_waves: list[list[type[Resource]]] = []`. Remove `self.unique_id = ""` (dead — `Resource.__init__` sets a UUID). Declare `self.api`, `self.states`, `self.state_registry`, `self.type_registry` as `None`.

2. Rename `_startup_tasks()` to `startup_tasks()` (public). No behavioral change.

3. Create `wire_services(self) -> None` containing: `context.set_global_hassette(self)`, `context.set_global_hassette_config(self.config)`, `self.startup_tasks()`, all 18 `add_child()` calls (preserving current order), and the dependency graph validation block (`validate_dependency_graph`, `topological_sort`, `topological_levels`, logging).

4. Add guard to `run_forever()`: `if not self._init_waves: raise RuntimeError("call wire_services() before run_forever()")`.

5. Add `None` guard to `event_streams_closed` property: `if self._event_stream_service is None: return True`.

6. Update `src/hassette/__main__.py:53`: add `core.wire_services()` after `core = Hassette(config=config)`.

7. Update `tests/system/conftest.py:138`: add `hassette.wire_services()` after `hassette = Hassette(config)`.

8. Update `tests/integration/conftest.py:19`: add `instance.wire_services()` after `instance = Hassette(test_config)`.

9. Update `run_hassette_startup_tasks()` in `src/hassette/test_utils/fixtures.py`: replace `SimpleNamespace` hack with `Hassette(config).startup_tasks()`. Remove the `from hassette.core.core import Hassette as HassetteCore` lazy import.

## Test Strategy

- **Existing test suite**: Run `timeout 300 uv run nox -s dev -- -n 2` — all tests must pass with no behavioral change.
- **Guard test**: Add `test_run_forever_without_wire_services_raises` in `tests/unit/test_core.py` (or nearest unit test for Hassette lifecycle): construct bare `Hassette(config)`, call `run_forever()` in a task, assert `RuntimeError` with the expected message.
- **event_streams_closed guard**: Add `test_event_streams_closed_before_wiring` — bare `Hassette(config)`, assert `event_streams_closed` is `True`.

## Review Guidance

- Verify `enable_logging()` stays in `__init__` (before `super().__init__()`) — `Resource.__init__` calls `_setup_logger()` which needs the logger configured.
- Verify `context.set_global_hassette()` is in `wire_services()`, NOT in `__init__` — the harness captures its own context token and would get a stale one if `__init__` also registers.
- Verify no production behavioral change — `wire_services()` must be called at all 3 call sites.
- Verify `_startup_tasks` → `startup_tasks` rename is complete (no remaining private references outside of test comments).
