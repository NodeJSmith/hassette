## Objectives & Success Criteria

`validate_registries()` runs during `Hassette.wire_services()` and catches invalid or missing registry entries at startup. In strict mode, validation issues raise `RegistryValidationError`. In non-strict mode, they log warnings. The function returns a list of `RegistryValidationIssue` dataclasses for testability. Running against the unmodified real registries produces zero issues.

## Subtasks

1. Create `src/hassette/conversion/validation.py` with:
   - `RegistryValidationIssue` frozen dataclass: `registry: str` ("STATE_REGISTRY" or "TYPE_REGISTRY"), `severity: str` ("error" or "warning"), `message: str`
   - `validate_registries(state_registry: StateRegistry, type_registry: TypeRegistry, *, strict: bool = False) -> list[RegistryValidationIssue]` function
2. Implement STATE_REGISTRY validation:
   - Zero entries ‚Üí error issue ("STATE_REGISTRY is empty ‚Äî models package may not be imported")
   - For each entry: verify the class is a subclass of `BaseState` ‚Üí error if not
   - For each entry: verify `StateKey.domain` is not None ‚Üí error if None
   - Duplicate domain detection: track seen domains, report warning for duplicates (note which class wins)
3. Implement TYPE_REGISTRY validation:
   - Zero entries ‚Üí error issue ("TYPE_REGISTRY is empty ‚Äî conversion module may not have loaded")
   - For each entry: verify `TypeConverterEntry.func` is callable ‚Üí error if not
   - For each entry: verify `from_type` and `to_type` are types (not None, not strings) ‚Üí error if not
   - For each entry: verify `error_types` is a tuple of exception subclasses ‚Üí error if not
4. After collecting all issues, apply strict mode: if `strict=True` and any issues with severity "error" exist, raise `RegistryValidationError` with a formatted summary of all issues. Always return the full list regardless of mode.
5. In non-strict mode, log each issue as a WARNING, then log a summary line: "Registry validation: N issues found" or "Registry validation: OK".
6. Export `validate_registries` and `RegistryValidationIssue` from `src/hassette/conversion/__init__.py`.
7. Call `validate_registries()` from `Hassette.wire_services()` (`src/hassette/core/core.py`) after all service children are added. Pass `strict=self.config.strict_lifecycle`. Import the real `STATE_REGISTRY` and `TYPE_REGISTRY` singletons from `hassette.conversion`.

## Test Strategy

Tests in `tests/unit/conversion/test_registry_validation.py` (new file). Use `StateRegistry.snapshot()`/`restore()` and `TypeRegistry.snapshot()`/`restore()` in a fixture to isolate registry state.

- `test_real_registries_pass_validation` ‚Äî call `validate_registries()` against the real (unmodified) registries, assert empty issue list
- `test_empty_state_registry_error` ‚Äî clear STATE_REGISTRY, validate, assert error issue with "empty" message
- `test_empty_type_registry_error` ‚Äî clear TYPE_REGISTRY, validate, assert error issue with "empty" message
- `test_state_registry_none_domain_error` ‚Äî inject entry with `StateKey(domain=None)`, assert error issue
- `test_state_registry_non_subclass_error` ‚Äî inject entry where the value is not a BaseState subclass, assert error issue
- `test_state_registry_duplicate_domain_warning` ‚Äî inject two entries for same domain, assert warning issue
- `test_type_registry_non_callable_func_error` ‚Äî inject `TypeConverterEntry(func=None, ...)`, assert error issue
- `test_type_registry_none_from_type_error` ‚Äî inject entry with `from_type=None`, assert error issue
- `test_strict_mode_raises_on_errors` ‚Äî inject an error-level issue, call with `strict=True`, assert `RegistryValidationError` raised with summary
- `test_strict_mode_does_not_raise_on_warnings_only` ‚Äî inject only warning-level issues, call with `strict=True`, assert no error raised
- `test_nonstrict_mode_logs_warnings` ‚Äî inject issues, call with `strict=False`, assert warnings logged and no exception

## Review Guidance

- `validate_registries()` must collect ALL issues before raising ‚Äî not fail-fast on the first error. The user needs to see the full picture.
- The snapshot/restore fixture must be `autouse=True` or explicitly used in every test that mutates registries. Missing this causes test pollution.
- The `wire_services()` call must happen AFTER all `add_child()` calls ‚Äî registry imports happen during child construction (e.g., `StateProxy` imports state models). Verify the placement by reading `wire_services()` fully.
- `RegistryValidationError` should include the issue count and first few messages in its string representation ‚Äî a bare "validation failed" is useless for debugging.
