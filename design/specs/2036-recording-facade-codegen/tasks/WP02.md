## Objectives & Success Criteria

Extend `tools/generate_sync_facade.py` to produce `_RecordingSyncFacade` from `RecordingApi` using an AST body-copy strategy, iterating `ApiSyncFacade`'s method list as the source of truth. Add `--target`, `--recording-api-path`, `--recording-out`, and `--check` CLI flags. Harden `run_ruff()` against silent failures. Add comprehensive generator unit tests. Do **not** yet use the generator to replace `src/hassette/test_utils/sync_facade.py` — that happens in WP03.

Observable outcomes:

- `tools/generate_sync_facade.py` has new CLI flags: `--target {api,recording,both}` (default: `both`), `--recording-api-path` (default: `src/hassette/test_utils/recording_api.py`), `--recording-out` (default: `src/hassette/test_utils/sync_facade.py`), `--check` (drift detection mode, exit 1 on drift).
- Running `uv run tools/generate_sync_facade.py --target recording` produces valid Python to stdout (or a file — see subtasks) that is syntactically correct, passes `ruff format --check` and `ruff check`, and can be imported without errors.
- Running `uv run tools/generate_sync_facade.py --target recording --check` against the current (still hand-written) `sync_facade.py` exits with code 1 and prints the actionable remediation message (because the committed file differs from the generator output). This is **expected** — the generator and the hand-written file are supposed to differ at this point. WP03 replaces the hand-written file.
- Running `uv run tools/generate_sync_facade.py --target api` still produces the same output as the current generator (no regression on the existing path).
- Running `uv run tools/generate_sync_facade.py --target api --check` against the currently committed `src/hassette/api/sync.py` exits with code 0 (no drift).
- The AST body rewriter is scoped to `FunctionDef.body` only — default argument expressions are not rewritten. The generator unit tests pin this behavior.
- The generator-time static check raises `SystemExit` with a clear message if any body-copied method calls an `async def` peer method on `self` (via the rewritten `self._parent.X` form).
- The import block for the recording facade is dynamically derived from `recording_api.py` imports, filtered to symbols actually referenced in the generated bodies.
- `run_ruff()` has `check=True` on the format call, `timeout=30` on both subprocess calls, `FileNotFoundError` handling, and `SyntaxError` handling on the `ast.parse()` call.
- Stub message templates `_STUB_MSG_STATE_CONVERSION` and `_STUB_MSG_GENERIC` are defined as module-level constants.
- The class header template is defined as `_RECORDING_CLASS_HEADER` constant matching the text pinned in `design.md` under "Class header template (pinned)".
- New test file `tests/unit/test_generate_sync_facade.py` contains unit tests for the AST rewriter covering: await-stripping, `self` → `self._parent` at the outermost position only, body-scoped rewriting (default arguments untouched), lambda handling, and the generator-time async-peer-call static check.
- `uv run pytest tests/unit/test_generate_sync_facade.py -n 2` passes.
- `uv run pyright tools/generate_sync_facade.py tests/unit/test_generate_sync_facade.py` is clean.
- `uv run ruff check tools/generate_sync_facade.py tests/unit/test_generate_sync_facade.py` is clean.

## Subtasks

1. **Robustness fixes to `run_ruff()`** (Findings 4, 23, 24). Modify the existing `run_ruff()` function in `tools/generate_sync_facade.py`:
   - Add `check=True` to the `subprocess.run(["ruff", "format", str(path)])` call.
   - Add `timeout=30` to both `subprocess.run` calls.
   - Wrap both calls in `try/except FileNotFoundError` raising `SystemExit("ruff not found on PATH. Install with: uv tool install ruff")`.
   - Wrap both calls in `try/except subprocess.TimeoutExpired` raising `SystemExit("ruff timed out after 30s — check for filesystem stall")`.
2. **Wrap `ast.parse()` in the existing `generate_sync()` function** to raise `SystemExit(f"Syntax error in {path}: {e}")` on `SyntaxError` (Finding 22). This fix applies to both existing and new parse sites — make it a helper if that's cleaner.
3. **Add module-level stub message constants** at the top of `tools/generate_sync_facade.py`:
   ```python
   _STUB_MSG_STATE_CONVERSION = (
       "RecordingApi.sync.{name} is not implemented on the test facade. "
       "Call `harness.api_recorder.sync.get_state(entity_id)` and read the returned state directly."
   )
   _STUB_MSG_GENERIC = (
       "RecordingApi.sync.{name} is not implemented. "
       "Seed state via AppTestHarness.set_state() for read methods, "
       "or use a full integration test for methods requiring a live HA connection."
   )
   _STATE_CONVERSION_METHODS = frozenset({"get_state_value", "get_state_value_typed", "get_attribute"})
   ```
4. **Add `_RECORDING_CLASS_HEADER` constant** containing the exact class header template text from `design.md` → "Class header template (pinned)". This includes the `class _RecordingSyncFacade:` declaration, the pyright ignore comment, the docstring, `_parent: "RecordingApi"` annotation, `__init__`, and the `__getattr__` method that interpolates `_STUB_MSG_GENERIC`.
5. **Add `_RECORDING_MODULE_HEADER_TEMPLATE`** — a string template for the module docstring and `TYPE_CHECKING` import block. The concrete imports are derived dynamically in subtask 8.
6. **Implement the AST body rewriter** as an `ast.NodeTransformer` subclass `_RecordingBodyRewriter`:
   - Visit `ast.Attribute` nodes: if `node.value` is `ast.Name(id='self')`, replace `node.value` with `ast.Attribute(value=ast.Name(id='self'), attr='_parent', ctx=ast.Load())`. This rewrites only the outermost `self` reference.
   - Visit `ast.Await` nodes: return `node.value` (unwrap).
   - Do NOT override `visit_FunctionDef`. The transformer must be **dispatched explicitly on `node.body` subtrees**, not on the whole function node. Implement this by iterating `func.body` in the caller and calling `self.visit(stmt)` on each statement — never `self.visit(func)`.
7. **Implement the generator-time async-peer-call static check** (Findings 15, 17). After body-rewriting a method, walk the rewritten body's `ast.Call` nodes. For each call, check if the callee is an `ast.Attribute` whose `value` is `ast.Attribute(value=ast.Name(id='self'), attr='_parent')` (i.e., the `self._parent.X` form) and whose `.attr` name appears in the set of `async def` method names collected from `RecordingApi`'s class body. If any such call is found, raise `SystemExit` with:
   - The method name being body-copied
   - The line number of the offending call (from the original source)
   - A remediation hint naming `_get_raw_state` / `_convert_state` as the sanctioned sync helpers
8. **Implement `_derive_recording_imports(body_nodes, recording_api_source_module)`**. Walk the body nodes' `ast.Name` and `ast.Attribute` references, collect the set of top-level symbols used. Parse `recording_api.py` once to extract its `import` / `from X import Y` statements. For each referenced symbol, find the import statement that provides it and emit it into the generated file's import block. If a referenced symbol has no matching import, raise `SystemExit("Method `{method_name}` uses symbol `{symbol}` with no known import path; add `{symbol}` to `recording_api.py`'s imports")`.
9. **Implement `gen_recording_method(func: ast.AsyncFunctionDef, async_method_names: set[str]) -> str`**:
   - Clone the function body using `ast.parse(ast.unparse(...))` or `copy.deepcopy`.
   - Dispatch `_RecordingBodyRewriter().visit(stmt)` on each statement in `node.body` (explicitly — not on the whole node).
   - Run the invariant check: walk the rewritten body for any surviving `ast.Await` nodes. If found, raise `SystemExit` with the method name and a hint.
   - Run the async-peer-call static check (subtask 7).
   - Emit a sync `def` using the existing `format_signature_and_call` helper (for the signature) and `ast.unparse` (for the rewritten body). Preserve docstrings via `ast.get_docstring` so the generated methods document themselves.
10. **Implement `gen_recording_stub(name: str) -> str`** — emits a sync `def` raising `NotImplementedError` with the appropriate tiered message (`_STUB_MSG_STATE_CONVERSION` for names in `_STATE_CONVERSION_METHODS`, else `_STUB_MSG_GENERIC`). Uses `format_signature_and_call` on the corresponding `ApiSyncFacade` method node to get the signature (so stub signatures match the facade exactly).
11. **Implement `generate_sync_recording(api_path: Path, recording_api_path: Path) -> str`**:
    - Parse `api_path` to find the `Api` class (re-use logic from existing `generate_sync()`).
    - Build the list of public async method names on `Api` (the `ApiSyncFacade` source-of-truth list). Filter out `LIFECYCLE_METHODS` and `is_overload()`.
    - Parse `recording_api_path` to find `RecordingApi`. Build a map `name → ast.AsyncFunctionDef` of its async methods. Collect the set of all async-method names (for the peer-call static check).
    - For each `ApiSyncFacade` method name (in the order they appear on `Api`):
      - If present in `RecordingApi`'s async method map → call `gen_recording_method(recording_node, async_method_names)`.
      - Else → call `gen_recording_stub(api_facade_method_node)`.
    - Collect all generated bodies, walk them once to derive imports via `_derive_recording_imports`.
    - Emit: `_RECORDING_MODULE_HEADER_TEMPLATE` formatted with derived imports + `_RECORDING_CLASS_HEADER` + `"\n".join(generated_bodies)`.
    - **Optional default-mismatch warning** (Finding 3): walk paired method signatures from `ApiSyncFacade` and `RecordingApi`; for each method in both, compare defaults. Print a warning to stderr on mismatch — do NOT fail. This makes divergence visible without blocking generation.
12. **Implement `--check` mode normalization protocol** (Findings 5, 6). Factor out a helper `_format_via_ruff(content: str) -> str` that:
    - Writes `content` to `tempfile.NamedTemporaryFile(suffix=".py", delete=False)` **outside the repo** (use `tempfile.gettempdir()`).
    - Runs `subprocess.run(["ruff", "format", tmp.name], check=True, timeout=30)`.
    - Reads back and returns the formatted content.
    - Removes the temp file in a `finally` block.
    - In check mode: read committed file content, format both sides through the same helper, byte-compare. On mismatch, `sys.exit(1)` with the actionable remediation message. On match, return normally.
13. **Update `main()` CLI to support `--target {api,recording,both}`, `--recording-api-path`, `--recording-out`, `--check`**. Default target is `both`. In non-check mode, write to file (using existing behavior for api, using `_recording_out` for recording). In check mode, compare without writing. When `--target both`, run both paths; if either drifts in check mode, exit 1 with details for both.
14. **Add `tests/unit/test_generate_sync_facade.py`** with these test cases (use pytest parametrize where helpful):
    - `test_rewriter_rewrites_self_to_self_parent_at_outermost_only` — feeds a synthetic `async def foo(self): return self.hassette.registry.thing` and asserts the rewritten body contains `self._parent.hassette.registry.thing`, with exactly one `self._parent` rewrite at the leftmost position.
    - `test_rewriter_strips_await` — feeds `async def foo(self): return await self._get_raw_state("x")`; asserts the rewritten body contains no `ast.Await` nodes and the call is `self._parent._get_raw_state("x")`.
    - `test_rewriter_leaves_default_arguments_untouched` — feeds `async def foo(self, x=self.CONST): return x` and asserts the rewritten signature still references `self.CONST` (not `self._parent.CONST`) while the body's `x` reference is unchanged. **This is the Finding 9 regression test.**
    - `test_rewriter_rewrites_lambda_body_self_references` — a method body containing `filter(lambda item: self.predicate(item), items)` — asserts the lambda's `self.predicate` is rewritten to `self._parent.predicate`.
    - `test_static_check_raises_on_async_peer_call` — feeds a synthetic `RecordingApi` with two async methods where one calls the other without `await`: `async def foo(self): return self.bar()` and `async def bar(self): ...`. Asserts `gen_recording_method` raises `SystemExit` mentioning `foo` and `bar`.
    - `test_static_check_allows_sync_helper_call` — feeds a method calling `self._get_raw_state(x)` where `_get_raw_state` is a regular `def` (not async). Asserts no `SystemExit`.
    - `test_derive_imports_from_body_references` — feeds a body referencing `EntityNotFoundError` and `BaseState`, asserts the generated import block contains `from hassette.exceptions import EntityNotFoundError` and `from hassette.models.states.base import BaseState` (sourced from `recording_api.py`'s imports).
    - `test_derive_imports_raises_on_unknown_symbol` — feeds a body referencing a symbol not in `recording_api.py`'s imports, asserts `SystemExit` with an actionable message.
    - `test_check_mode_normalizes_through_ruff` — using a small synthetic fixture, verify that `_format_via_ruff` produces identical output for two semantically-equivalent formatted variants.
    - `test_generate_recording_produces_valid_python` — runs the full `generate_sync_recording()` against the real `recording_api.py` and `api.py`, writes output to a tempfile, verifies `py_compile.compile()` succeeds.
15. **Run the existing `tools/generate_sync_facade.py --target api`** to verify no regression on the existing path (output should be byte-equal to the current `src/hassette/api/sync.py`, modulo the new robustness fixes in `run_ruff`).
16. **Do NOT** commit any changes to `src/hassette/test_utils/sync_facade.py` in this WP. That file stays hand-written until WP03. The generator's output is verified only via the unit tests and an ad-hoc dry-run.

## Review Guidance

**Design constraints to verify:**

- `_RecordingBodyRewriter.visit(...)` is called on `func.body` statements individually, **never** on the function node itself. If the reviewer sees `_RecordingBodyRewriter().visit(func_node)` anywhere, that's a Finding 9 regression — default-argument `self` references will be rewritten incorrectly. Blocker.
- The generator-time async-peer-call static check must actually walk every body-copied method. Verify by checking `gen_recording_method`'s call sites and confirming the check runs before the final emit. Blocker.
- `_STUB_MSG_STATE_CONVERSION` and `_STUB_MSG_GENERIC` are defined as module-level constants, not string literals scattered through the emit path. Blocker for Finding 19.
- `_RECORDING_CLASS_HEADER` text matches the pinned template in `design.md` exactly (modulo trailing whitespace). Any wording change from the pinned template requires updating the design doc in the same commit. Blocker for Finding 12.
- `run_ruff()` has `check=True`, `timeout=30`, `FileNotFoundError` handling, and `TimeoutExpired` handling. Missing any of these is a Finding 4/23/24 regression. Blocker.
- `--check` mode uses `tempfile.NamedTemporaryFile` outside the repository (verify via `tempfile.gettempdir()` usage, not a path under the worktree). If the reviewer sees a relative temp path, that's a Finding 5 regression. Blocker.
- Both sides of the `--check` comparison are normalized through the **same** ruff binary (via `_format_via_ruff`). If the comparison reads the committed file without running it through ruff, that's a Finding 6 regression. Blocker.
- The AST body rewriter unit tests include the default-argument case (Finding 9) and the async-peer-call case (Findings 15/17). Missing either test is a blocker.

**Warning-level deviations (note but not block):**

- Internal helper naming (e.g., `_derive_recording_imports` vs `_build_recording_imports`) — style choice.
- Whether stub generation iterates `Api` async methods directly or goes through a parsed `ApiSyncFacade` AST — both work; the design allows either.
- Additional unit tests beyond the required set.
