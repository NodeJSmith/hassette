## Objectives & Success Criteria

Stream 3: Eliminate the ClassVar race in hermetic config construction, fix `get_states_raw` mock, make exception recorder support concurrent recorders, fix registry isolation deep copies, add deterministic event fixture seeding, and unblock module-scoped `hassette_with_app_handler`.

Success:
- `_make_hermetic_config()` uses a closure-based approach instead of the ClassVar `_hermetic_init_kwargs` channel — race-free by construction
- `_HERMETIC_CONFIG_CACHE` is retained (prevents Pydantic subclass accumulation)
- Default API mock's `get_states_raw` returns `[]` explicitly (not relying on `MagicMock.__iter__`)
- `TaskBucket._exception_recorder` is a list supporting multiple concurrent recorders with LIFO install/uninstall
- `_isolate_registries` in `tests/conftest.py` uses deep copies for mutable registry contents
- Event fixture shuffling uses `random.Random(seed)` with the seed logged for reproducibility
- `hassette_with_app_handler` fixture is `scope="module"` in `fixtures.py`

## Subtasks

1. Refactor `_get_hermetic_subclass()` in `app_harness.py` to capture `config_dict` in a closure passed to `settings_customise_sources` instead of writing to the ClassVar `_hermetic_init_kwargs`. The `_HERMETIC_CONFIG_CACHE` dict is retained — its purpose is preventing Pydantic `__subclasses__()` accumulation, not race amortization. Update the `_make_hermetic_config()` function accordingly
2. Apply the same closure-based pattern to `_get_hermetic_hassette_config_cls()` in `config.py` — consolidate the three config construction patterns (`TestConfig` in `tests/conftest.py`, `_HermeticHassetteConfig` in `config.py`, `_make_hermetic_config` in `app_harness.py`) into a consistent approach. `TestConfig` stays as-is (it's a user-facing session-scoped fixture, not a ClassVar race risk)
3. In `harness.py` `start()`, explicitly configure the default API mock: `self.hassette.api.get_states_raw = AsyncMock(return_value=[])` — do this in the existing block where `self.hassette.api = AsyncMock()` is set up
4. Change `TaskBucket._exception_recorder` from a single callable to a list in `src/hassette/task_bucket/task_bucket.py`. Update `install_exception_recorder` to append, `uninstall_exception_recorder` to remove (LIFO-safe), and the exception dispatch to call all recorders. Uninstalling an already-uninstalled recorder must not raise
5. In `tests/conftest.py`, verify `_isolate_registries` uses `StateRegistry.snapshot()`/`TypeRegistry.snapshot()` — confirm snapshot values are immutable types (class objects, named tuples) rather than mutable instances. The current snapshots return dicts of `type[BaseState]` class objects, which are immutable by nature — the existing shallow copy is likely correct. Only add `copy.deepcopy()` if mutable values are found
6. Update event fixture shuffling in `fixtures.py`: replace `random.shuffle(events)` with `rng = random.Random(seed); rng.shuffle(events)` where `seed` is logged via `logging.getLogger(__name__).info("Event shuffle seed: %d", seed)`. Use `int.from_bytes(os.urandom(8))` as the seed
7. Change `hassette_with_app_handler` fixture in `fixtures.py` from `scope="function"` (default) to `scope="module"` — remove the TODO comment
