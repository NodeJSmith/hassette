## Objectives & Success Criteria

Stream 4: Add a public accessor layer on `HassetteHarness`, remove `_test_seed_state` and `_test_mode` from production code, rename `_test_trigger_due_jobs` to public `trigger_due_jobs()`, make `ApiResource` injectable, consolidate config patterns, and migrate all test callers.

Success:
- `HassetteHarness` exposes public properties: `state_proxy`, `bus_service`, `scheduler_service`, `bus`, `scheduler`, `app_handler`
- `SimulationMixin`, `TimeControlMixin`, and `RecordingApi` access framework internals through harness properties, not through `_HassetteMock` private attributes
- Zero `harness.hassette._` accesses from `simulation.py`, `time_control.py`, and `recording_api.py`
- `_test_seed_state` is removed from `StateProxy` — replaced by a harness-level helper using `asyncio.wait_for(lock.acquire(), timeout=5.0)`
- `_test_mode` flag is removed from `_HassetteMock` and all `getattr(self.hassette, "_test_mode")` checks in production code are deleted
- `_test_trigger_due_jobs` is renamed to public `trigger_due_jobs()` on `SchedulerService`
- `ApiResource` accepts keyword-only `rest_url` and `headers_factory` constructor parameters (default `None`); harness passes explicit values instead of patching private properties
- `SimpleTestServer` has a `reset()` method; `reset_mock_api` delegates to it
- The no-op `cast("ApiProtocol", RecordingApi)` is removed
- All integration tests that previously called `_test_seed_state` directly are migrated to the new harness-level helper
- Zero `getattr(self.hassette, "_test_mode")` in production code
