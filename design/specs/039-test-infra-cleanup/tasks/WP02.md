## Objectives & Success Criteria

Stream 2: Make every assertion failure and cleanup error surface to the test framework. Fix `stop()` to not break on first failure, restore task factory and ContextVar token, remove unnecessary `asyncio.sleep(0)` after shutdown, and convert integration cleanup fixtures to yield-based patterns.

Success:
- `shutdown_resource()` logs failures instead of suppressing them
- The teardown loop in `stop()` attempts shutdown for all resources; exceptions are collected and re-raised as `ExceptionGroup` after all resources have been attempted
- `assert_clean()` errors surface to the test framework after all other cleanup completes
- Task factory is saved via `get_task_factory()` in `start()` and restored in `stop()`
- ContextVar token from `set_global_hassette()` is saved in `start()` and reset via `HASSETTE_INSTANCE.reset(token)` in `stop()`
- `asyncio.sleep(0)` after each `shutdown_resource()` call is removed
- All 4 autouse cleanup fixtures in `tests/integration/conftest.py` use yield-based patterns with error surfacing (no bare `except Exception: pass`)
- `live_server` fixture checks `thread.is_alive()` after `thread.join(timeout=5)`
- `_fastapi_app` patcher is properly stopped during teardown
- Zero `except Exception: pass` in test infrastructure files (verified by grep)

## Subtasks

1. Change `shutdown_resource()` in `harness.py` from `contextlib.suppress(Exception)` to logging the exception with `logger.warning()` and re-raising
2. Rewrite the teardown loop in `stop()`: wrap each `resource` shutdown in individual try/except, collect exceptions into a list, attempt shutdown for ALL resources regardless of failures, then re-raise collected exceptions as `ExceptionGroup` after the loop completes
3. Remove `asyncio.sleep(0)` after each `shutdown_resource()` call in the teardown loop — `shutdown()` is fully awaited, these are unnecessary cooperative yields
4. Move `assert_clean()` call to after `_exit_stack.aclose()` and raise the assertion after all cleanup is complete — the current position suppresses assertion errors when earlier cleanup fails
5. In `start()`, save the previous task factory: `self._previous_task_factory = self.hassette._loop.get_task_factory()`. In `stop()`, restore it: `self.hassette._loop.set_task_factory(self._previous_task_factory)` before setting `_loop = None`
6. In `start()`, save the ContextVar token: `self._hassette_ctx_token = context.set_global_hassette(...)`. In `stop()`, reset it: `context.HASSETTE_INSTANCE.reset(self._hassette_ctx_token)` before setting `_loop = None`. Follow the `AppTestHarness` pattern at `app_harness.py:316-317`
7. Convert all 4 autouse cleanup fixtures in `tests/integration/conftest.py` (`cleanup_state_proxy_fixture`, `cleanup_bus_fixture`, `cleanup_scheduler_fixture`, `cleanup_mock_api_fixture`) from try/except patterns to yield-based patterns that surface errors
8. Fix `live_server` in `tests/e2e/conftest.py`: replace the `time.sleep(0.1)` startup polling loop with a tight socket-connect retry that does not sleep (the existing `socket.create_connection(..., timeout=0.5)` already blocks for up to 0.5s per attempt, making the `time.sleep(0.1)` redundant). On teardown, after `thread.join(timeout=5)`, add `if thread.is_alive(): raise RuntimeError("Live server did not stop within 5s")`
9. Fix `_fastapi_app` in `tests/e2e/conftest.py`: convert to yield fixture and call `patcher.stop()` in teardown
10. Add `wait_for_ready` return value check in `start()`: verify the return value and raise `TimeoutError` on failure
