Success:
- `RecordingApi` has `assert_called_exact` for exact-match assertions and `assert_called_partial` as a non-deprecated alias for the existing `assert_called` (partial match)
- CRUD helper methods are reduced from 32 to a generic core dispatching via `_RECORD_TYPE_TO_DOMAIN` in `recording_api.py` (no production model changes) with thin typed delegations

- `tests/TESTING.md` documents both assertion methods and default partial-match behavior

## Subtasks

1. Add `assert_called_exact` method to `RecordingApi` in `recording_api.py`: performs exact kwargs match (no extra keys allowed). Add `assert_called_partial` as a non-deprecated alias for the existing `assert_called`. Update docstrings on both to clearly document partial vs exact behavior
2. Implement 4 generic CRUD methods on `RecordingApi` (`_list_helpers`, `_create_helper`, `_update_helper`, `_delete_helper`) dispatching via the existing `_RECORD_TYPE_TO_DOMAIN` dict. Encode `InputSelectRecord` deep-copy handling in the dispatch table. Convert the 32 existing per-domain methods to thin typed delegations that call the generic core. No production model changes
3. Regenerate `_RecordingSyncFacade` via `uv run python tools/generate_sync_facade.py` after CRUD changes. Verify the generated file compiles and passes Pyright
4. Add bus-level task drain warning to `SimulationMixin` in `simulation.py`: when `_drain_task_bucket` detects bus-level tasks that were not drained, emit a `logger.warning()`
5. Document `wait_for_ready` no-op behavior on `_HassetteMock`: add a docstring note explaining it always returns `True` and pointing to `CLAUDE.md` for the correct approach to testing startup races
6. Replace `_FREEZE_TIME_LOCK.locked()` assertion in `tests/integration/test_app_harness_simulation.py` with a behavioral test: attempt a second `freeze_time()` inside an active freeze context and assert it raises `RuntimeError`
7. Centralize timeout constants: create a `TIMEOUTS` namespace (dict or module-level constants) in `harness.py` with documented rationale for each value (e.g., `SHUTDOWN_TIMEOUT = 5.0`, `WAIT_FOR_READY_TIMEOUT = 1.0`, `STATE_SEED_LOCK_TIMEOUT = 5.0`). Update all hardcoded timeout values in test infrastructure to reference these constants
8. Update `tests/TESTING.md`: document `assert_called` (partial match, default), `assert_called_partial` (alias), and `assert_called_exact` (exact match) with examples showing when to use each. Document the default partial-match behavior prominently
9. Update `tests/unit/test_recording_api_helpers.py` if CRUD genericization changes `_RECORD_TYPE_TO_DOMAIN` structure. Verify the test still imports and asserts correctly after the refactor

- **assert_called_exact**: `tests/unit/test_recording_api.py::test_assert_called_exact_match`, `test_assert_called_exact_mismatch`, `test_assert_called_exact_no_calls` — verify exact match succeeds, extra-kwargs fails, and no-calls produces clear error
- **assert_called_partial alias**: `tests/unit/test_recording_api.py::test_assert_called_partial_is_alias` — verify it behaves identically to `assert_called`

- Existing test suite (2394 tests) must pass; Pyright clean
