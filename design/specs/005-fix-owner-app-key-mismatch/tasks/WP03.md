## Objectives & Success Criteria

Fix the 3 broken HTMX partial filters and 1 API route filter to use the correct identity fields. Rename `owner` to `owner_id` on API response models. Update template context serialization.

Success criteria:
- `scheduler_jobs_partial` filters jobs by `j.app_key == app_key` (was `j.owner == app_key`)
- `app_detail_jobs_partial` filters jobs by `j.app_key == app_key` (was `j.owner == app_key`)
- `instance_jobs_partial` filters jobs by `j.app_key == app_key` and `j.instance_index == index`
- `GET /api/scheduler/jobs?app_key=...` filters by `j.app_key == app_key` (was `j.owner == app_key`)
- `router.py` page filters updated from `j.owner` to `j.owner_id`
- `ScheduledJobResponse.owner` renamed to `.owner_id`
- `JobExecutionResponse.owner` renamed to `.owner_id`
- `job_to_dict()` in `context.py` uses `job.owner_id`
- `_job_to_dict()` in `routes/scheduler.py` uses `job.owner_id`
- All TODO comments about the owner_id mismatch removed from `partials.py`

## Subtasks

1. In `src/hassette/web/ui/partials.py`: update the 3 filter expressions from `j.owner == app_key` to `j.app_key == app_key`. For `instance_jobs_partial`, also add `j.instance_index == index` to the filter. Remove all 3 TODO comments about the mismatch.

2. In `src/hassette/web/routes/scheduler.py`: update the API filter from `j.owner == app_key` to `j.app_key == app_key`. Update `_job_to_dict()` to use `job.owner_id`.

3. In `src/hassette/web/ui/router.py`: update `j.owner` to `j.owner_id` on lines ~105 and ~147.

4. In `src/hassette/web/ui/context.py`: update `job_to_dict()` to use `job.owner_id` in the returned dict.

5. In `src/hassette/web/models.py`: rename `ScheduledJobResponse.owner` to `.owner_id`. Rename `JobExecutionResponse.owner` to `.owner_id` if it exists.

6. Run `uv run pyright` to verify no type errors.
