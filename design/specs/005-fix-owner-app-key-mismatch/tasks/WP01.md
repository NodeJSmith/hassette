## Objectives & Success Criteria

Add `app_key: str` and `instance_index: int` fields to `Listener` and `ScheduledJob` dataclasses. Rename `owner` to `owner_id` on both, plus `ListenerMetrics`. Update `Bus` and `Scheduler` to pass these values from their parent App at creation time.

Success criteria:
- `Listener` has fields `owner_id`, `app_key`, `instance_index` (and no `owner` field)
- `ScheduledJob` has fields `owner_id`, `app_key`, `instance_index` (and no `owner` field)
- `ListenerMetrics.owner` renamed to `ListenerMetrics.owner_id`
- `Bus._create_listener()` passes `app_key` and `instance_index` from parent App
- `Scheduler` job creation passes `app_key` and `instance_index` from parent App
- Non-App parents (e.g., RuntimeQueryService) get `app_key=""` and `instance_index=0`
- All references to `.owner` on these classes are updated throughout the codebase
- Type checking passes (`uv run pyright`)

## Subtasks

1. In `src/hassette/bus/listeners.py`: rename `owner: str` to `owner_id: str` on the `Listener` dataclass. Add `app_key: str = ""` and `instance_index: int = 0` fields. Update `Listener.create()` classmethod to accept and pass `app_key` and `instance_index`. Update `__repr__` if it references `owner`.

2. In `src/hassette/scheduler/classes.py`: rename `owner: str` to `owner_id: str` on `ScheduledJob`. Add `app_key: str = field(default="", compare=False)` and `instance_index: int = field(default=0, compare=False)`. Update `__repr__` to use `owner_id`.

3. In `src/hassette/bus/metrics.py`: rename `ListenerMetrics.owner` to `ListenerMetrics.owner_id`. Update `to_dict()` to use the new key name.

4. In `src/hassette/bus/bus.py`: update `Bus._create_listener()` (or wherever `Listener.create()` is called) to pass `app_key` and `instance_index`. Resolve these from `self.parent`: if `hasattr(self.parent, 'app_key')`, use `self.parent.app_key` and `self.parent.index`; otherwise use `""` and `0`.

5. In `src/hassette/scheduler/scheduler.py`: update `ScheduledJob(...)` construction to pass `app_key` and `instance_index` from parent App, with same non-App fallback.

6. Grep the entire `src/` directory for remaining `.owner` references on Listener or ScheduledJob and update them to `.owner_id`. Key files: `src/hassette/core/bus_service.py` (Router.owners dict, remove_listeners_by_owner), `src/hassette/core/scheduler_service.py` (_ScheduledJobQueue.remove_owner), `src/hassette/core/commands.py`, `src/hassette/core/command_executor.py`.

7. Run `uv run pyright` to verify no type errors.

## Test Strategy

Write tests in `tests/integration/test_listeners.py` and `tests/integration/test_scheduler.py`:

- `test_listener_has_app_key_and_instance_index` — create a Listener via `Listener.create()` with explicit `app_key="my_app"` and `instance_index=1`, verify the fields are set.
- `test_scheduled_job_has_app_key_and_instance_index` — create a ScheduledJob with `app_key="my_app"` and `instance_index=2`, verify the fields are set.
- `test_listener_defaults_empty_app_key` — create a Listener without `app_key`, verify it defaults to `""`.

Existing tests that use `owner=` must be updated to `owner_id=` (covered in WP04 but may need temporary fixes here to keep tests running).

## Review Guidance

- Verify that `owner_id` is never confused with `Resource.owner_id` property — they serve different purposes (string field vs. computed property).
- Verify that the `app_key` resolution in Bus/Scheduler correctly handles non-App parents.
- Check that no `.owner` references remain in production code after this WP.
- The `compare=False` on new ScheduledJob fields is important — these must not affect job ordering.
