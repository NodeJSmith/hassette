## Objectives & Success Criteria

Fix `BusService.add_listener()` and `SchedulerService.add_job()` to use `listener.app_key`/`listener.instance_index` and `job.app_key`/`job.instance_index` when creating `ListenerRegistration` and `ScheduledJobRegistration`. Skip DB registration entirely when `app_key` is empty (non-App owners like RuntimeQueryService).

Success criteria:
- `ListenerRegistration.app_key` receives the real app_key (e.g., `"my_app"`) not the owner_id
- `ListenerRegistration.instance_index` receives the actual instance index, not hardcoded `0`
- Same for `ScheduledJobRegistration`
- Listeners from non-App Bus instances (empty `app_key`) are not registered in the DB
- Jobs from non-App Scheduler instances (empty `app_key`) are not registered in the DB
- All `.owner` references in bus_service.py and scheduler_service.py updated to `.owner_id`

## Subtasks

1. In `src/hassette/core/bus_service.py` `add_listener()` (~line 86): change `app_key=listener.owner` to `app_key=listener.app_key` and `instance_index=0` to `instance_index=listener.instance_index`. Add early return before DB registration if `listener.app_key` is empty.

2. In `src/hassette/core/scheduler_service.py` `add_job()` (~line 171): change `app_key=job.owner` to `app_key=job.app_key` and `instance_index=0` to `instance_index=job.instance_index`. Add early return before DB registration if `job.app_key` is empty.

3. Update all remaining `.owner` references in both files to `.owner_id` (Router.owners dict keys, `remove_listeners_by_owner()`, `_ScheduledJobQueue.remove_owner()`).

4. Run `uv run pyright` to verify no type errors.

## Test Strategy

Write tests in `tests/integration/test_command_executor.py` or a new `tests/integration/test_registration.py`:

- `test_listener_registration_uses_app_key` — create a Listener with `app_key="my_app"`, `instance_index=1`, call `add_listener()`, verify the `ListenerRegistration` passed to the executor has `app_key="my_app"` and `instance_index=1`.
- `test_job_registration_uses_app_key` — same for ScheduledJob and `add_job()`.
- `test_listener_with_empty_app_key_skips_registration` — create a Listener with `app_key=""`, call `add_listener()`, verify no DB registration occurs (executor's register method not called or called without the registration).
- `test_job_with_empty_app_key_skips_registration` — same for ScheduledJob.

## Review Guidance

- The critical check: verify that `listener.app_key` and `job.app_key` are used, NOT `listener.owner_id`. The whole bug was caused by using the wrong identity field.
- Verify the skip guard fires for empty `app_key` — ensure RuntimeQueryService's internal listeners don't get registered.
- Verify that the Router/Queue operations still use `owner_id` (not `app_key`) for in-memory ownership tracking.

## Avoid

- Do NOT change `Router.owners` keys or `_ScheduledJobQueue` ownership tracking from `owner_id` to `app_key`. In-memory ownership tracking must continue using `owner_id` (unique_name). Only DB registration uses `app_key`.
- Do NOT register listeners/jobs to the DB when `app_key` is empty — these are internal resources with no telemetry value.
