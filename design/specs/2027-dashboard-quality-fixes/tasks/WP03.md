## Objectives & Success Criteria

The `/api/healthz` endpoint is removed and returns 404. The `/api/health` endpoint returns 503 with `SystemStatusResponse` JSON when system status is non-ok (degraded or starting), and 200 when healthy. All docker-compose files, documentation, and OpenAPI spec reference `/api/health` instead of `/api/healthz`.

## Subtasks

1. Read `src/hassette/web/routes/health.py` to understand both endpoints.

2. Write failing tests in `tests/integration/test_web_api.py`:
   - `test_health_returns_200_when_ok` ‚Äî `/api/health` returns 200 with `status: "ok"` when WebSocket is connected.
   - `test_health_returns_503_when_degraded` ‚Äî returns 503 with `status: "degraded"` when WebSocket is disconnected.
   - `test_health_returns_503_when_starting` ‚Äî returns 503 with `status: "starting"` during startup.
   - `test_healthz_returns_404` ‚Äî `/api/healthz` returns 404 after removal.

3. Delete the `healthz()` endpoint function from `src/hassette/web/routes/health.py`.

4. Modify `get_health()` to use FastAPI's `Response` parameter for conditional status codes:
   ```python
   async def get_health(runtime: RuntimeDep, response: Response) -> SystemStatusResponse:
       status_data = runtime.get_system_status()
       if status_data.status != "ok":
           response.status_code = 503
       return status_data
   ```
   Add `responses={503: {"model": SystemStatusResponse}}` to the decorator.

5. Update health check references in docker-compose files:
   - `examples/docker-compose.yml` ‚Äî change `healthz` to `health` in the healthcheck test command.
   - `docs/pages/getting-started/docker/snippets/docker-compose.yml` ‚Äî same.
   - `docs/pages/getting-started/docker/snippets/full-docker-compose.yml` ‚Äî same.

6. Update `docs/pages/getting-started/docker/troubleshooting.md` ‚Äî replace all 3 references to `/healthz` with `/health`.

7. Regenerate OpenAPI spec: run `uv run python scripts/export_schemas.py` to update `frontend/openapi.json`.

8. Remove or migrate existing `/healthz` tests (`test_healthz_ok`, `test_healthz_degraded`) ‚Äî these should be replaced by the new `/health` tests above.

9. Run all tests to confirm no regressions.

## Review Guidance

- **Design constraint**: The `Response` parameter pattern must be used (not `JSONResponse`). This preserves Pydantic serialization for both 200 and 503 paths.
- **Design constraint**: 503 must be returned for BOTH `"degraded"` AND `"starting"` states ‚Äî any non-`"ok"` status.
- **Blocker**: If any docker-compose file or doc page still references `/healthz` after this WP, that's a miss.
- **Blocker**: `frontend/openapi.json` must be regenerated ‚Äî stale OpenAPI spec is a contract violation.
- **Warning**: The `Response` import may need to be added from `starlette.responses` or `fastapi`.
