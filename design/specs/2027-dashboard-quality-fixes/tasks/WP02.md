## Objectives & Success Criteria

A `/api/telemetry/status` endpoint exists that returns `{"degraded": false}` when the telemetry database is healthy and `{"degraded": true}` when it's unavailable. All three dashboard endpoints (`dashboard_kpis`, `dashboard_errors`, `dashboard_app_grid`) catch `sqlite3.Error`, `OSError`, and `ValueError` via a shared `DB_ERRORS` constant. `ValueError` is logged at WARNING level.

## Subtasks

1. Read `src/hassette/web/routes/telemetry.py` to understand the current `except sqlite3.Error` pattern in the three dashboard endpoints.

2. Write failing tests in `tests/integration/test_web_api.py`:
   - `test_telemetry_status_healthy` — `/api/telemetry/status` returns 200 with `{"degraded": false}` when DB is healthy.
   - `test_telemetry_status_db_unavailable` — returns 200 with `{"degraded": true}` when DB query raises `sqlite3.Error`.
   - `test_dashboard_kpis_oserror_returns_fallback` — `OSError` triggers the same fallback as `sqlite3.Error`.
   - `test_dashboard_kpis_valueerror_returns_fallback` — `ValueError` triggers fallback and logs at WARNING.

3. Define `DB_ERRORS: tuple[type[Exception], ...] = (sqlite3.Error, OSError, ValueError)` as a module-level constant in `src/hassette/web/routes/telemetry.py`. Import `sqlite3` if not already imported.

4. Replace each `except sqlite3.Error` with `except DB_ERRORS` in the three dashboard endpoint handlers. For `ValueError`, log at WARNING level (existing `sqlite3.Error` catches log at WARNING already — match the pattern but ensure `ValueError` is explicitly called out).

5. Add `TelemetryStatusResponse` model to `src/hassette/web/models.py`:
   ```python
   class TelemetryStatusResponse(BaseModel):
       degraded: bool
   ```

6. Add the `/telemetry/status` endpoint to `src/hassette/web/routes/telemetry.py`:
   - Inject `TelemetryDep`
   - Execute a representative query that exercises the listeners → handler_invocations join path (e.g., `SELECT COUNT(*) FROM listeners l LEFT JOIN handler_invocations hi ON hi.listener_id = l.id LIMIT 1`)
   - Catch `DB_ERRORS`, return `TelemetryStatusResponse(degraded=True)`
   - On success, return `TelemetryStatusResponse(degraded=False)`

7. Add `getTelemetryStatus()` function and `TelemetryStatus` type to `frontend/src/api/endpoints.ts`.

8. Run tests to confirm all pass.

## Review Guidance

- **Design constraint**: `DB_ERRORS` must be a shared constant used by ALL dashboard endpoints AND the status endpoint. Grep for bare `sqlite3.Error` catches in telemetry.py — there should be none after this WP.
- **Design constraint**: `ValueError` must be logged at WARNING level, not INFO.
- **Design constraint**: The status endpoint query must exercise the listeners → handler_invocations join, not just `SELECT 1`.
- **Blocker**: If the status endpoint catches `asyncio.CancelledError`, that violates the spec.
- **Warning**: The `TelemetryStatusResponse` model should NOT be added to the three dashboard response models — per-endpoint degraded fields were explicitly dropped (YAGNI).
