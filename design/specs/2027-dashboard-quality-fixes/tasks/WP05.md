## Objectives & Success Criteria

Log message truncation detection in `log-table.tsx` re-evaluates on viewport resize, font loading, and data changes. Messages that become truncated gain the expand affordance; messages that are no longer truncated lose it. A single `ResizeObserver` observes individual message text elements. New elements from streaming data are observed for subsequent resizes. No unnecessary signal updates when the truncated set is unchanged.

## Subtasks

1. Read `frontend/src/components/shared/log-table.tsx` fully. Understand `checkTruncation` ref callback (lines 138-145), `truncatedRows` signal, `expandedRows` signal, and the `canExpand` logic (line 282: `truncatedRows.value.has(rowKey) || isExpanded`).

2. Add `data-row-key={rowKey}` attribute to each `.ht-log-message__text` div (around line 297) so `recheckTruncation()` can map DOM elements back to row keys.

3. Replace the `checkTruncation` ref callback with a `recheckTruncation()` function:
   - Query all `.ht-log-message__text` elements within the table via `querySelectorAll`.
   - Build a new `Set<string>` of row keys where `el.scrollWidth > el.clientWidth`.
   - Compare to current `truncatedRows.value` — only update the signal if the set changed (compare sizes first, then contents).
   - Add a code comment documenting that expanded rows have `text-overflow: ellipsis` removed by CSS, so they won't appear in `truncatedRows`. The `|| isExpanded` guard in the render path is load-bearing.

4. Create **Trigger path A — Viewport resize**: A `useEffect` with empty dependency array (mount-only) that:
   - Creates a single `ResizeObserver` whose callback calls `recheckTruncation()`.
   - Observes all current `.ht-log-message__text` elements.
   - Returns a cleanup function that calls `observer.disconnect()`.
   - Store the observer in a ref so Trigger path B can access it.

5. Create **Trigger path B — Data changes**: A `useEffect` with dependency on `sorted.length` (or equivalent visible entry count) that:
   - Wraps `recheckTruncation()` in `requestAnimationFrame` to ensure layout is complete after render.
   - Observes any new `.ht-log-message__text` elements that the ResizeObserver doesn't yet track (so they participate in subsequent viewport resizes).

6. Add **Font load detection**: In the mount-only `useEffect`, call `document.fonts.ready.then(() => recheckTruncation())`.

7. Remove the old `checkTruncation` ref callback and its usage in the render function. Replace the `ref={checkTruncation(rowKey)}` with just the `data-row-key` attribute (no ref needed).

8. Write E2E tests in `tests/e2e/test_logs.py` (or a new file if test_logs.py doesn't exist):
   - `test_truncation_affordance_appears_on_narrow_viewport` — resize to narrow width, verify expand icons appear on long messages.
   - `test_truncation_affordance_disappears_on_wide_viewport` — start narrow (truncated), widen viewport, verify expand icons disappear for messages that now fit.

9. Run `uv run nox -s dev` to verify no regressions. Run E2E tests via `uv run nox -s e2e`.

## Test Strategy

E2E tests via Playwright in `tests/e2e/`:
- `test_truncation_affordance_appears_on_narrow_viewport`
- `test_truncation_affordance_disappears_on_wide_viewport`

Unit testing of `recheckTruncation()` is difficult (requires DOM measurement). E2E tests with Playwright viewport resizing are the appropriate test strategy here.

## Review Guidance

- **Design constraint**: Must observe individual `.ht-log-message__text` elements, NOT the scroll container. The scroll container may not resize on viewport change.
- **Design constraint**: Two distinct trigger paths — ResizeObserver for viewport, useEffect for data changes. Both call the same `recheckTruncation()`.
- **Design constraint**: New elements from streaming data must be observed by the ResizeObserver (Trigger path B handles this).
- **Blocker**: If `recheckTruncation()` updates the signal on every call regardless of changes, that causes unnecessary re-renders — signal update must be suppressed when the set is unchanged.
- **Blocker**: The `|| isExpanded` guard in `canExpand` must NOT be removed. It's load-bearing for expanded rows whose CSS removes text-overflow.
- **Warning**: `requestAnimationFrame` in Trigger path B ensures layout is complete before measuring. Omitting it may cause stale measurements.
