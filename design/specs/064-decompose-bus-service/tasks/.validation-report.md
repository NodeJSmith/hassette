## Step 4: Gap and Problem Analysis

### Coverage Gaps

None. Every FR and AC is claimed by at least one task.

### Dependency Analysis

- T01 and T02 have no dependencies — can run in parallel.
- T03 depends on T01 (imports `build_tracked_invoke_fn` from the sibling module). Correct: T03 needs the module to exist before it can import from it.
- T04 depends on T01, T02, and T03. Correct: integration wiring requires all three extracted modules.

No orphaned tasks. No circular dependency chains.

### Contradictions

None detected. The task files are consistent with the design doc on:
- FR#6 scope: The design doc says `read_current_state` does NOT move — its logic is absorbed into the `state_reader` callback. T03 correctly handles this by calling `self.state_reader(entity_id)` rather than claiming `read_current_state` moves literally. T04 formalizes this by rewriting `_read_entity_state` to absorb the error handling. The design doc's statement "Module-level helper functions (`read_current_state`, `compute_elapsed`) move to the duration hold module" is partially inaccurate in the FR text (the design doc itself contradicts it in the Architecture section), but the tasks correctly implement the Architecture section's intent (`compute_elapsed` moves, `read_current_state` is absorbed). No action needed — the tasks are correct.
- The `_should_log_event` exclusion from EventFilter is documented in both T02 and context.md.
- Dispatch tracking staying in BusService is documented in both T03 and context.md.
- The config_resolver lambda requirement is correctly specified in T04.

### Warnings

**W1 — AC#3 partially deferred:** AC#3 (`pyright` passes) is verified per-task in T01, T02, and T03, but there is no explicit AC#3 verify item in T04. However, T04's step 8 runs the full test suite (`timeout 300 uv run pytest -n 2 -x`), which would surface import-time type errors. A `uv run pyright` call after T04 would provide stronger assurance. This is a process gap, not a coverage gap — the criterion is likely met implicitly by the full-suite run.

**W2 — FR#6 wording ambiguity in design.md:** The FR text says `read_current_state` moves to duration_hold, but the Architecture section clarifies it is absorbed into the callback, not literally moved. The tasks implement the Architecture section's intent correctly. No fix needed in the tasks — but the design.md FR text is slightly misleading.

**W3 — docstring update in commands.py:** T04 includes a note to update the docstring reference in `core/commands.py:48` (which mentions `BusService._make_tracked_invoke_fn()`). This is a good-to-have but is not tied to any FR or AC. It is captured in the Focus section of T04. No coverage gap.
