## Objectives & Success Criteria

The `CommandExecutor._execute()` method enforces per-execution timeouts via `asyncio.timeout()`, recording `'timed_out'` status in telemetry. `ExecutionResult` gains an `is_timed_out` property. `track_execution` gains an `except TimeoutError` branch. Command dataclasses gain `effective_timeout: float | None` (required, no default). Timeout WARNING logs are rate-limited per entity with lazy eviction. `make_async_adapter._sync_fn` filters `TimeoutError` from its exception logging.

## Subtasks

1. Add `effective_timeout: float | None` field (required, no default) to `ExecuteJob` and `InvokeHandler` in `src/hassette/core/commands.py`. Update all existing construction sites to pass `effective_timeout=None` explicitly
2. Add `except TimeoutError` branch to `track_execution` in `src/hassette/utils/execution.py` — placed before `except Exception`, sets `result.status = "timed_out"` and re-raises. Add `is_timed_out` property to `ExecutionResult`
3. Add `asyncio.timeout(cmd.effective_timeout)` scope inside `_execute()` in `src/hassette/core/command_executor.py`, nested inside `track_execution`. Use unconditional `asyncio.timeout(cmd.effective_timeout)` — `None` is a valid no-op
4. Add per-entity timeout WARNING rate limiting to `CommandExecutor`: `dict[int, float]` keyed by in-memory `listener_id`/`job_id`, 60-second suppression window, lazy eviction of entries where `now - ts > 60s`
5. Keep the `log_error` gate as `result.is_error` only — timeouts are logged via the rate-limited WARNING path
6. Add `except TimeoutError: raise` branch to `make_async_adapter._sync_fn` in `src/hassette/task_bucket/task_bucket.py` before `except Exception`

## Review Guidance

- Verify `asyncio.timeout()` is nested INSIDE `track_execution` — wrong nesting produces `status='cancelled'` instead of `'timed_out'`
- Verify `except TimeoutError` is before `except Exception` in `track_execution` (not before `except CancelledError`)
- Verify `effective_timeout` has NO default on command dataclasses — this is intentional to catch missed construction sites
- Verify rate-limit dict uses in-memory ID (`listener_id`/`job_id`), not `db_id`
- Verify lazy eviction removes entries where `now - ts > 60s`
- FAIL if `log_error` gate includes `is_timed_out`
