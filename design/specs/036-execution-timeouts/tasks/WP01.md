## Objectives & Success Criteria

Extract the duplicated `_pending_registration_tasks` logic from `BusService` and `SchedulerService` into a standalone `RegistrationTracker` class. The `BusService` warning-count bug (always reports 0 incomplete tasks) is fixed by adopting the `asyncio.wait()` strategy uniformly. All existing tests pass without modification. A new regression test verifies the incomplete-count is non-zero at timeout (AC #10).

## Subtasks

1. Create `src/hassette/core/registration_tracker.py` with the `RegistrationTracker` class containing:
   - `_tasks: dict[str, list[asyncio.Task[None]]]` initialized as `defaultdict(list)`
   - `prune_and_track(app_key: str, task: asyncio.Task[None]) -> None`
   - `async def await_complete(self, app_key: str, timeout: float, logger: Logger) -> None` using `asyncio.wait(..., timeout=)`, cancelling stragglers, logging warning with correct incomplete count
   - `drain_framework_keys(await_fn: Callable[[str], Awaitable[None]]) -> None`
2. Wire `RegistrationTracker` into `SchedulerService` — replace `_pending_registration_tasks` dict, inline prune-and-append, and `await_registrations_complete()` body with `self._reg_tracker` calls
3. Wire `RegistrationTracker` into `BusService` — same replacement, fixing the `asyncio.wait_for(asyncio.gather(...))` bug
4. Wire `RegistrationTracker` into `BusService.drain_framework_registrations()` using `drain_framework_keys()`

## Test Strategy

- **`tests/unit/core/test_registration_tracker.py`** (new):
  - `test_track_adds_task` — verifies `prune_and_track` stores the task
  - `test_prune_removes_done_tasks` — verifies completed tasks are pruned on next `prune_and_track`
  - `test_await_complete_returns_immediately_for_unknown_key` — no-op for missing key
  - `test_await_complete_awaits_pending_tasks` — tasks complete within timeout
  - `test_await_complete_logs_warning_on_timeout` — incomplete count is non-zero in warning message (AC #10)
  - `test_await_complete_cancels_stragglers` — timed-out tasks are cancelled
  - `test_drain_framework_keys` — iterates framework-prefixed keys
- **Existing tests**: `test_scheduler_service_barrier.py` and `test_bus_service_public_accessors.py` must pass unchanged

## Review Guidance

- Verify `RegistrationTracker` has NO dependency on `Resource` or `Service` — it's a plain utility class
- Verify `BusService` now uses `asyncio.wait()` (not `asyncio.wait_for(asyncio.gather(...))`)
- Verify the warning message format matches the existing format in both services
- Verify `await_complete` signature matches the design: `(self, app_key: str, timeout: float, logger: Logger) -> None`
- FAIL if any existing test is modified (except for import path changes)
