## Objectives & Success Criteria

`Listener` gains `timeout: float | None` and `timeout_disabled: bool` fields (participating in `__eq__`). `Options` TypedDict includes both fields. `Bus.on()` accepts both as explicit keyword arguments. `BusService` resolves the effective timeout and passes it through `InvokeHandler`. Config gains `event_handler_timeout_seconds: float | None = 600.0`.

## Subtasks

1. Add `timeout: float | None = None` and `timeout_disabled: bool = False` fields to `Listener` in `src/hassette/bus/listeners.py` (participating in equality — no `compare=False`). Validate in `_validate_options()`: `if isinstance(timeout, (int, float)) and timeout <= 0: raise ValueError("timeout must be a positive number")`
2. Update `Listener.create()` signature to accept `timeout` and `timeout_disabled`, passing through to the constructor
3. Add `timeout: float | None` and `timeout_disabled: bool` to the `Options` TypedDict in `src/hassette/bus/bus.py` — both fields must be added together
4. Add `timeout: float | None = None` and `timeout_disabled: bool = False` as explicit keyword arguments on `Bus.on()`. Thread to `Listener.create()` call
5. Add `event_handler_timeout_seconds: float | None = Field(default=600.0)` to `HassetteConfig` in `src/hassette/config/config.py`
6. In `BusService._make_tracked_invoke_fn()`, resolve effective timeout from `listener.timeout`, `listener.timeout_disabled`, and `config.event_handler_timeout_seconds`. Pass as `effective_timeout` on `InvokeHandler` construction

## Test Strategy

- **`tests/unit/bus/test_listener_timeout.py`** (new):
  - `test_listener_create_with_timeout` — `Listener.create(..., timeout=5.0)` stores `timeout=5.0`
  - `test_listener_create_default_timeout` — defaults to `None`
  - `test_listener_create_timeout_disabled` — `timeout_disabled=True` stores correctly
  - `test_listener_timeout_validation_rejects_zero` — `timeout=0` raises `ValueError`
  - `test_listener_timeout_in_equality` — two listeners with different timeout are not equal
  - `test_listener_timeout_disabled_in_equality` — different `timeout_disabled` are not equal
- **`tests/unit/bus/test_bus_timeout_threading.py`** (new):
  - `test_bus_on_passes_timeout_to_listener` — `bus.on(topic=..., handler=..., timeout=5.0)` creates listener with `timeout=5.0`
  - `test_on_state_change_passes_timeout_via_options` — convenience method forwards through `_subscribe` → `on()`
  - `test_bus_on_passes_timeout_disabled` — `timeout_disabled=True` threads through
- **`tests/unit/core/test_bus_service_timeout.py`** (new — uses real CommandExecutor):
  - `test_dispatch_resolves_effective_timeout_from_listener` — listener.timeout=5 → effective_timeout=5
  - `test_dispatch_resolves_effective_timeout_from_config` — listener.timeout=None → uses config default
  - `test_dispatch_resolves_timeout_disabled` — listener.timeout_disabled=True → effective_timeout=None
  - `test_once_listener_removed_after_timeout` — once=True handler that times out is still removed from the bus (FR8, AC3)

## Review Guidance

- Verify `timeout` participates in `Listener.__eq__` (no `compare=False` or `field(compare=False)`)
- Verify both `timeout` and `timeout_disabled` are in `Options` TypedDict
- Verify `Bus.on()` has `timeout` and `timeout_disabled` as explicit kwargs (not via `**opts`)
- Verify effective timeout resolution happens in `_make_tracked_invoke_fn`, not in `_dispatch`
- FAIL if `Options` has `timeout` but not `timeout_disabled` (or vice versa)
