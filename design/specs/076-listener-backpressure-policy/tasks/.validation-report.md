The design states the runner applies numbered SQL files in order, so this is sound — but confirm the runner auto-discovers `008.sql`, or AC#8 has no executing test.
- **Several Edge Cases are documentation-only, routed to T05.** Fan-out-order sensitivity, sustained-saturation starvation, global-vs-per-listener, and live-only counts are "must document" items mapped to T05 prose (under FR#1/FR#2) with no dedicated AC — consistent with the design treating them as documentation rather than testable behavior. Acceptable.
- **Dependency ordering is sound.** T01 (no deps) → T02 (T01) → T03 (T01, T02) → T04 (T02, T03) → T05 (all). Shared-file writes to `bus_service.py` (T02 gate branch, T03 `build_registration`, T04 `live_execution_counts`) and to `tests/integration/bus/test_execution_modes.py` (T03, T04) are serialized via `depends_on` and called out in each task's Focus — no concurrent-write hazard if executed in dependency order.
