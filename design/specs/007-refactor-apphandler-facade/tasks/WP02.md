## Objectives & Success Criteria

Refactor `src/hassette/core/app_handler.py` from 369 lines to ~120-150 lines by delegating implementation methods to `AppLifecycleService` (created in WP01). Delete `src/hassette/core/app_lifecycle.py` (AppLifecycleManager) since its methods are now folded into AppLifecycleService.

**Observable outcomes:**
- `app_handler.py` is ~120-150 lines (thin coordinator)
- `app_lifecycle.py` is deleted
- `uv run pyright` passes
- `uv run ruff check` passes
- All existing tests pass (`uv run nox -s dev`)

## Subtasks

1. Refactor `AppHandler.__init__` (`app_handler.py:57-73`):
   - Keep: `self.registry = AppRegistry()`
   - Replace `self.factory`, `self.lifecycle` (AppLifecycleManager), `self.change_detector` with a single `self.lifecycle = self.add_child(AppLifecycleService, registry=self.registry)`
   - Remove: `self.bus = self.add_child(Bus)` â€” Bus moves to AppLifecycleService
   - Keep: log level configuration, but apply to `self.lifecycle` instead of individual components
   - Keep: `self.set_apps_configs()` call â€” delegate to `self.lifecycle.set_apps_configs()`

2. Keep public API methods as thin delegation:
   - `apps` property â†’ `self.registry.apps` (unchanged)
   - `get_status_snapshot()` â†’ `self.registry.get_snapshot()` (unchanged)
   - `get(app_key, index)` â†’ `self.registry.get(app_key, index)` (unchanged)
   - `all()` â†’ `self.registry.all_apps()` (unchanged)
   - `config_log_level` property (unchanged)

3. Refactor lifecycle hooks:
   - `on_initialize()` â€” keep the `wait_for_ready` call for `_websocket_service` and the `mark_ready()`. Remove the Bus subscription for file watcher (now in AppLifecycleService). Keep the dev_mode/allow_reload_in_prod check â€” but move the actual `bus.on()` subscription to AppLifecycleService's `on_initialize`
   - `after_initialize()` â€” this method now owns the dependency-readiness gate that was previously inside `bootstrap_apps` (lines 145-155 of the old app_handler.py). The implementation must:
     1. Call `self.hassette.wait_for_ready([websocket, api, bus, scheduler, state_proxy])`
     2. If `wait_for_ready` returns False (timeout), log the "Dependencies never became ready; skipping app startup" warning and return early â€” do NOT call `bootstrap_apps`
     3. Only on success: `self.task_bucket.spawn(self.lifecycle.bootstrap_apps())`
     This preserves the existing skip-on-timeout behavior while cleanly separating coordination (AppHandler) from execution (AppLifecycleService)
   - `on_shutdown()` â€” delegate to `self.lifecycle.shutdown_all()`

4. Replace implementation methods with delegation:
   - `start_app(app_key, force_reload)` â†’ `await self.lifecycle.start_app(app_key, force_reload)`
   - `stop_app(app_key)` â†’ `await self.lifecycle.stop_app(app_key)`
   - `reload_app(app_key, force_reload)` â†’ `await self.lifecycle.reload_app(app_key, force_reload)`
   - Remove: `start_apps`, `apply_changes`, `handle_change_event`, `refresh_config`, `bootstrap_apps`, `_resolve_only_app`, `_update_only_app_filter`, `_reconcile_blocked_apps`, `set_apps_configs` (all moved to AppLifecycleService)

5. Update imports:
   - Remove: `AppLifecycleManager`, `AppFactory`, `AppChangeDetector`, `Bus`, `ChangeSet` imports
   - Add: `AppLifecycleService` import from `hassette.core.app_lifecycle_service`

6. Delete `src/hassette/core/app_lifecycle.py` â€” all its methods are now in AppLifecycleService

7. Update `set_apps_configs` â€” this is currently called from `__init__` and from `AppHandler.refresh_config`. Since `refresh_config` moved to AppLifecycleService, `set_apps_configs` should also move there. AppHandler's `__init__` calls `self.lifecycle.set_apps_configs(hassette.config.app_manifests)` after creating the lifecycle service.

8. Verify all existing tests still pass by running `uv run nox -s dev`
