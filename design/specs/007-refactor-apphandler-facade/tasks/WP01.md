## Subtasks

1. Create `src/hassette/core/app_lifecycle_service.py` with `AppLifecycleService(Resource)`:
   - Constructor accepts `hassette`, `parent`, and `registry: AppRegistry`
   - Creates `AppFactory(hassette, registry)` internally
   - Creates `AppChangeDetector()` internally
   - Adds `Bus` as child via `self.add_child(Bus)`
   - `on_initialize()` calls `self.mark_ready(reason="AppLifecycleService initialized")`

2. Fold in all methods from `AppLifecycleManager` (`src/hassette/core/app_lifecycle.py`):
   - `initialize_instances(app_key, instances, manifest)` — adapt from `self.hassette.send_event()` pattern (already works via Resource base)
   - `shutdown_instance(inst)` — move as-is
   - `shutdown_instances(instances)` — move as-is
   - `shutdown_all()` — move as-is
   - `_emit_app_state_change(app, status, prev_status, exception)` — move as-is
   - `startup_timeout` / `shutdown_timeout` properties — read from `self.hassette.config`

3. Add the implementation methods that will be moved from `AppHandler` (write them here now, they'll be called from WP02's refactored AppHandler):
   - `bootstrap_apps()` — move from `app_handler.py:138-178`, but WITHOUT the `wait_for_ready` calls (those stay in AppHandler per the coordination split). The method starts from `_resolve_only_app()` through to emitting `APP_LOAD_COMPLETED`. Add a docstring/comment explaining: "Service readiness is checked by the caller (AppHandler.after_initialize) before this method is invoked. This method assumes all dependencies are already ready."
   - `start_app(app_key, force_reload)` — move from `app_handler.py:301-327`, including the `NOT_STARTED` event emission and `task_bucket.spawn` for `initialize_instances`
   - `stop_app(app_key)` — move from `app_handler.py:329-339`
   - `reload_app(app_key, force_reload)` — move from `app_handler.py:341-348`
   - `start_apps(apps)` — move from `app_handler.py:291-299`
   - `apply_changes(changes: ChangeSet)` — move from `app_handler.py:350-369`
   - `handle_change_event(changed_file_paths)` — move from `app_handler.py:252-289`, preserving the `typing.Annotated` DI signature exactly
   - `refresh_config()` — move from `app_handler.py:235-250`
   - `set_apps_configs(apps_config)` — move from `app_handler.py:91-103`
   - `_resolve_only_app(changed_file_paths)` — move from `app_handler.py:180-212`
   - `_update_only_app_filter(app_key)` — move from `app_handler.py:214-217`
   - `_reconcile_blocked_apps()` — move from `app_handler.py:219-233`

4. Add `config_log_level` property that reads from `self.hassette.config.app_handler_log_level`

5. Verify identity model: confirm `app_key` flows through `factory.create_instances()` → `registry.register_app()` unchanged, and `instance_index` is assigned by AppFactory (no new hardcoded `index=0`)
