## Objectives & Success Criteria

All existing tests pass against the rewritten scheduler:
- `tests/integration/test_scheduler.py` updated: the one test that calls the old `schedule(func, run_at, ...)` form directly is rewritten to use `schedule(func, After(seconds=N))` or `schedule(func, Every(...))`
- Any integration test that passes old parameter forms (`start=` tuple, `interval=N`, `days=N`, keyword cron) is updated to use the new API
- E2E fixtures in `tests/e2e/` that assert `trigger_type IN ("interval", "cron")` are updated to also accept `"once"`, `"after"`, `"custom"`
- New integration tests exercise the full `Once`/`After` exhaustion path via `HassetteHarness.trigger_due_jobs()`
- `uv run nox -s dev` (or `uv run pytest tests/integration/test_scheduler.py`) exits green

## Test Strategy

Use `HassetteHarness` for integration tests (it wires real bus, scheduler, state proxy components). Use `harness.trigger_due_jobs()` to fire due jobs without waiting for wall time.

All new tests go in `tests/integration/test_scheduler.py` (append to existing file rather than creating a new one unless it exceeds 800 lines).
