## Objectives & Success Criteria

> **Prerequisite**: Before starting WP01, pull latest `origin/main` into the implementation branch (`git pull --rebase origin main`) to ensure the scheduler rewrite begins on a clean base.

A new `src/hassette/scheduler/triggers.py` module exists containing:
- `TriggerProtocol` with all six methods: `first_run_time()`, `next_run_time() -> ZonedDateTime | None`, `trigger_label()`, `trigger_detail()`, `trigger_db_type()`, `trigger_id()`
- `After` trigger: fires once after a delay, `next_run_time()` returns `None`
- `Once` trigger: fires once at a wall-clock time, `if_past` param controls past-time behaviour, emits `WARNING` log on tomorrow-deferral, `next_run_time()` returns `None`
- `Every` trigger: fixed-interval, drift-resistant via preserved `_advance_past()` arithmetic, `start=ZonedDateTime | None` only
- `Daily` trigger: wraps `CronTrigger` internally, `at="HH:MM"` form only, no `every=N`
- `Cron` trigger: accepts 5-field (standard) and 6-field (seconds appended) expressions via `croniter`

`src/hassette/scheduler/__init__.py` re-exports: `TriggerProtocol`, `After`, `Once`, `Every`, `Daily`, `Cron` (alongside existing `ScheduledJob` and `Scheduler`; `IntervalTrigger` and `CronTrigger` are not removed here тАФ that happens in WP03).

Unit tests in `tests/unit/test_triggers.py` all pass.

## Subtasks

1. Create `src/hassette/scheduler/triggers.py` with the `TriggerProtocol` runtime-checkable `Protocol` class (six methods as defined in the design doc)
2. Implement `After(seconds=N | minutes=N | timedelta=TimeDelta(...))` тАФ `first_run_time()` returns `current_time.add(...)`, `next_run_time()` returns `None`
3. Implement `Once(at: str | ZonedDateTime, if_past: Literal["tomorrow", "error"] = "tomorrow")` тАФ parses `"HH:MM"` string into today's wall-clock time; if past and `if_past="tomorrow"` defer by one day and emit `logger.warning("Once(at=%r) constructed after the target time тАФ deferring to tomorrow.", at)`; `next_run_time()` returns `None`
4. Implement `Every(seconds=N, minutes=N, hours=N, start: ZonedDateTime | None = None)` тАФ port `_advance_past()` logic verbatim from `src/hassette/scheduler/classes.py:IntervalTrigger`; `trigger_id()` returns `f"every:{self.interval_seconds}"` where interval_seconds is total seconds
5. Implement `Daily(at: str)` тАФ parse `"HH:MM"` into a cron expression `f"0 {mm} {hh} * * *"`, delegate `first_run_time()` / `next_run_time()` to an internal `CronTrigger`; `trigger_id()` returns `f"cron:{expr}"`
6. Implement `Cron(expression: str)` тАФ validate at construction time; accept both 5-field and 6-field expressions (5-field: canonically `{mm} {hh} {dom} {mon} {dow}`; 6-field: `{mm} {hh} {dom} {mon} {dow} {sec}` тАФ appended seconds field per `croniter` convention); delegate `first_run_time()` / `next_run_time()` to an internal `CronTrigger`; `trigger_id()` returns `f"cron:{self.expression}"`
7. Update `src/hassette/scheduler/__init__.py` to re-export `TriggerProtocol`, `After`, `Once`, `Every`, `Daily`, `Cron` from `triggers.py`
8. Write `tests/unit/test_triggers.py` with unit tests for all trigger types (see Test Strategy)

## Review Guidance

**BLOCKERS (spec violations):**
- `next_run_time()` on `After` or `Once` must return `None` тАФ any non-None return is wrong
- `Once(if_past="tomorrow")` must emit a `WARNING` log тАФ silent deferral is wrong
- `Every.start=` must accept only `ZonedDateTime | None` тАФ accepting a string `start="07:00"` is wrong (DST trap)
- `Daily` must use cron internally (not `IntervalTrigger`) тАФ otherwise the run_daily bug is not fixed
- `trigger_id()` must return a stable string (no random components, no timestamps)

**WARNINGS (deviation from design):**
- If `_advance_past()` is not ported from `IntervalTrigger` verbatim, verify drift-resistance still holds
- `trigger_db_type()` return values must match the `Literal` exactly тАФ Pyright should enforce this
