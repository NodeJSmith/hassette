## Objectives & Success Criteria

> **Prerequisite**: Before starting WP01, pull latest `origin/main` into the implementation branch (`git pull --rebase origin main`) to ensure the scheduler rewrite begins on a clean base.

A new `src/hassette/scheduler/triggers.py` module exists containing:
- `TriggerProtocol` with all six methods: `first_run_time()`, `next_run_time() -> ZonedDateTime | None`, `trigger_label()`, `trigger_detail()`, `trigger_db_type()`, `trigger_id()`
- `After` trigger: fires once after a delay, `next_run_time()` returns `None`
- `Once` trigger: fires once at a wall-clock time, `if_past` param controls past-time behaviour, emits `WARNING` log on tomorrow-deferral, `next_run_time()` returns `None`
- `Every` trigger: fixed-interval, drift-resistant via preserved `_advance_past()` arithmetic, `start=ZonedDateTime | None` only
- `Daily` trigger: wraps `CronTrigger` internally, `at="HH:MM"` form only, no `every=N`
- `Cron` trigger: accepts 5-field (standard) and 6-field (seconds appended) expressions via `croniter`

`src/hassette/scheduler/__init__.py` re-exports: `TriggerProtocol`, `After`, `Once`, `Every`, `Daily`, `Cron` (alongside existing `ScheduledJob` and `Scheduler`; `IntervalTrigger` and `CronTrigger` are not removed here ‚Äî that happens in WP03).

Unit tests in `tests/unit/test_triggers.py` all pass.

## Subtasks

1. Create `src/hassette/scheduler/triggers.py` with the `TriggerProtocol` runtime-checkable `Protocol` class (six methods as defined in the design doc)
2. Implement `After(seconds=N | minutes=N | timedelta=TimeDelta(...))` ‚Äî `first_run_time()` returns `current_time.add(...)`, `next_run_time()` returns `None`
3. Implement `Once(at: str | ZonedDateTime, if_past: Literal["tomorrow", "error"] = "tomorrow")` ‚Äî parses `"HH:MM"` string into today's wall-clock time; if past and `if_past="tomorrow"` defer by one day and emit `logger.warning("Once(at=%r) constructed after the target time ‚Äî deferring to tomorrow.", at)`; `next_run_time()` returns `None`
4. Implement `Every(seconds=N, minutes=N, hours=N, start: ZonedDateTime | None = None)` ‚Äî port `_advance_past()` logic verbatim from `src/hassette/scheduler/classes.py:IntervalTrigger`; `trigger_id()` returns `f"every:{self.interval_seconds}"` where interval_seconds is total seconds
5. Implement `Daily(at: str)` ‚Äî parse `"HH:MM"` string (where HH=hour, MM=minute) into a **5-field** cron expression `f"{minute} {hour} * * *"` (e.g. `"07:00"` ‚Üí `"0 7 * * *"`); delegate `first_run_time()` / `next_run_time()` to an internal `CronTrigger` using this 5-field expression; `trigger_id()` returns `f"cron:{expr}"` where `expr` is the 5-field form. Do NOT use the 6-field `from_arguments()` helper from the existing `CronTrigger` ‚Äî that appends a seconds field and would produce a different format than the design's trigger table.
6. Implement `Cron(expression: str)` ‚Äî validate at construction time; accept both 5-field and 6-field expressions (5-field: canonically `{minute} {hour} {dom} {month} {dow}`; 6-field: seconds appended as a 6th field per `croniter` convention, e.g. `"0 9 * * 1-5 0"`); delegate `first_run_time()` / `next_run_time()` to an internal `CronTrigger`; `trigger_id()` returns `f"cron:{self.expression}"`
7. Update `src/hassette/scheduler/__init__.py` to re-export `TriggerProtocol`, `After`, `Once`, `Every`, `Daily`, `Cron` from `triggers.py`
8. Write `tests/unit/test_triggers.py` with unit tests for all trigger types (see Test Strategy)

## Test Strategy

File: `tests/unit/test_triggers.py`

Tests to write (use a frozen `ZonedDateTime` for determinism ‚Äî mock `hassette.utils.date_utils.now()` if needed via `monkeypatch`):

- `test_after_first_run_time` ‚Äî `After(seconds=30).first_run_time(t)` returns `t + 30s`
- `test_after_next_run_time_returns_none` ‚Äî `next_run_time()` returns `None`
- `test_after_trigger_id` ‚Äî `"after:30"`
- `test_once_today` ‚Äî `Once(at="07:00")` when current time is 06:00 ‚Üí fires today at 07:00
- `test_once_tomorrow_default` ‚Äî `Once(at="07:00")` when current time is 08:00 ‚Üí fires tomorrow at 07:00
- `test_once_tomorrow_warning` ‚Äî primary assertion is the fire time (tomorrow at 07:00); secondary assertion is that a `WARNING` log was emitted. Note: project rule `feedback_no_caplog_tests.md` discourages caplog assertions ‚Äî here it is acceptable only as a secondary check because the WARNING emission is a named design contract, not an implementation detail. The test passes or fails on the behavioral assertion (fire time); the caplog check is advisory.
- `test_once_past_raises` ‚Äî `Once(at="07:00", if_past="error")` when current time is 08:00 ‚Üí raises `ValueError`
- `test_once_next_run_time_returns_none` ‚Äî `next_run_time()` returns `None`
- `test_once_trigger_id` ‚Äî `"once:07:00"`
- `test_every_first_run_no_start` ‚Äî `Every(seconds=60).first_run_time(t)` returns `t + 60s`
- `test_every_first_run_future_start` ‚Äî `Every(hours=1, start=t+10m).first_run_time(t)` returns `t+10m`
- `test_every_first_run_past_start` ‚Äî `_advance_past()` skips missed intervals correctly
- `test_every_next_run_time` ‚Äî returns next interval aligned to previous_run anchor
- `test_every_drift_resistant` ‚Äî multiple calls with late `current_time` do not compound drift
- `test_every_trigger_id` ‚Äî `"every:3600"`
- `test_daily_fires_at_wall_clock_time` ‚Äî `Daily(at="07:00").next_run_time(prev, now)` uses cron grid, not 24h elapsed
- `test_daily_trigger_db_type` ‚Äî returns `"cron"`
- `test_daily_trigger_id` ‚Äî `"cron:0 7 * * *"`
- `test_cron_5field` ‚Äî `Cron("0 9 * * 1-5")` valid construction, correct `trigger_id()`
- `test_cron_6field` ‚Äî `Cron("0 9 * * 1-5 0")` valid construction, correct `trigger_id()`
- `test_cron_invalid_raises` ‚Äî malformed expression raises `ValueError` at construction
- `test_trigger_protocol_conformance` ‚Äî all five built-in trigger classes satisfy `isinstance(t, TriggerProtocol)` when `TriggerProtocol` is decorated `@runtime_checkable`

## Review Guidance

**BLOCKERS (spec violations):**
- `next_run_time()` on `After` or `Once` must return `None` ‚Äî any non-None return is wrong
- `Once(if_past="tomorrow")` must emit a `WARNING` log ‚Äî silent deferral is wrong
- `Every.start=` must accept only `ZonedDateTime | None` ‚Äî accepting a string `start="07:00"` is wrong (DST trap)
- `Daily` must use cron internally (not `IntervalTrigger`) ‚Äî otherwise the run_daily bug is not fixed
- `trigger_id()` must return a stable string (no random components, no timestamps)

**WARNINGS (deviation from design):**
- If `_advance_past()` is not ported from `IntervalTrigger` verbatim, verify drift-resistance still holds
- `trigger_db_type()` return values must match the `Literal` exactly ‚Äî Pyright should enforce this
