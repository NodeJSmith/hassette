## Objectives & Success Criteria

`src/hassette/web/utils.py:resolve_trigger()` no longer imports or checks `isinstance(trigger, IntervalTrigger)` / `isinstance(trigger, CronTrigger)` ‚Äî it calls `trigger.trigger_label()`, `trigger.trigger_detail()`, `trigger.trigger_db_type()` instead.

`src/hassette/web/models.py` reflects the expanded `trigger_type` value set (`"interval"`, `"cron"`, `"once"`, `"after"`, `"custom"`, `None`).

`src/hassette/web/routes/scheduler.py` (if it has any hardcoded trigger-type logic) is updated consistently.

No import of `IntervalTrigger` or `CronTrigger` remains in any file under `src/hassette/web/`.

Unit tests for `resolve_trigger()` in `tests/unit/test_web_utils.py` all pass. Pyright clean on web layer files.

## Subtasks

1. Rewrite `src/hassette/web/utils.py:resolve_trigger(job: ScheduledJob) -> tuple[str, str | None]` to: `return (job.trigger.trigger_db_type(), job.trigger.trigger_detail()) if job.trigger else ("one-shot", None)`; remove `isinstance` dispatch. 5. Write `tests/unit/test_web_utils.py` (see Test Strategy)
