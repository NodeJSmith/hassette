## Objectives & Success Criteria

`src/hassette/core/scheduler_service.py` is updated so that:
- `reschedule_job()` calls `trigger.next_run_time()` and removes the job immediately when `None` is returned; `repeat` flag is not consulted; exceptions from `next_run_time()` are caught, logged with full job context (job id, callable name, trigger repr), and treated as exhaustion (job removed)
- Jitter is applied to both first-run heap enqueue and every re-fire: `sort_index` offset = `random.uniform(0, job.jitter)` seconds when `job.jitter is not None`; `job.next_run` (unjittered) is never mutated by jitter
- `SchedulerService.register_removal_callback(owner_id: str, callback: Callable[[ScheduledJob], None])` stores per-owner callbacks called whenever a job is removed via `_remove_job()`
- `_enqueue_then_register()` replaces `isinstance(trigger, IntervalTrigger)` / `isinstance(trigger, CronTrigger)` dispatch with `trigger.trigger_db_type()`, `trigger.trigger_label()`, `trigger.trigger_detail()` protocol method calls; `ScheduledJobRegistration` constructed without `repeat=`; imports of `IntervalTrigger` and `CronTrigger` from `scheduler.classes` removed from this file

Unit tests pass. Pyright clean. No import of `IntervalTrigger` or `CronTrigger` remains in `scheduler_service.py`.

## Subtasks

1. Rewrite `reschedule_job(job)`:
   - If `job.cancelled`: call `_remove_job(job)` and return
   - Call `next_run = job.trigger.next_run_time(job.next_run, date_utils.now())` inside a `try/except Exception`
   - On exception: `logger.exception("reschedule_job: trigger raised for job_id=%s callable=%s trigger=%r", job.job_id, getattr(job.job, "__qualname__", str(job.job)), job.trigger)`; call `_remove_job(job)` and return
   - If `next_run is None`: call `_remove_job(job)` and return
   - If `next_run` is not in the future (delta ≤ 0): log WARNING as today, advance by 1s
   - Call `job.set_next_run(next_run)` (unjittered); then enqueue with jitter applied to `sort_index` only (see subtask 3)
2. Add `register_removal_callback(owner_id: str, callback: Callable[[ScheduledJob], None]) -> None` to `SchedulerService`; store in `_removal_callbacks: dict[str, Callable[[ScheduledJob], None]]`
3. Add `_apply_jitter_to_heap(job: ScheduledJob) -> None` helper: if `job.jitter is not None`, compute `offset = random.uniform(0, job.jitter)`; create a jittered `sort_index` tuple `(job.next_run.add(seconds=offset).timestamp_nanos(), job.job_id)` and set it on the job before enqueue; `job.next_run` must not be modified
4. Apply jitter in `_enqueue_job()` (before heap push) for re-fires, and in `add_job()` / `_enqueue_then_register()` for first-run enqueue
5. Update `_remove_job()`: after removing from heap, look up and call `_removal_callbacks.get(job.owner_id)` if present, passing the job
6. Rewrite `_enqueue_then_register()`: replace isinstance dispatch with `trigger.trigger_db_type()`, `trigger.trigger_label()`, `trigger.trigger_detail()` calls; construct `ScheduledJobRegistration` with `trigger_type=trigger.trigger_db_type()`, `trigger_label=trigger.trigger_label()`, `trigger_detail=trigger.trigger_detail()`; remove `repeat=` field from construction; remove `IntervalTrigger`/`CronTrigger` imports from this file
7. Write `tests/unit/core/test_scheduler_service_reschedule.py` (see Test Strategy)

## Review Guidance

**BLOCKERS:**
- `job.next_run` must never be mutated by jitter — if `job.next_run` contains the jittered time, `previous_run` passed to `next_run_time()` will drift on every cycle
- `reschedule_job` must not check `job.repeat` — the field no longer exists; any reference to it is a runtime `AttributeError`
- Exception from `next_run_time()` must NOT propagate — the scheduler must never crash due to a misbehaving custom trigger

**WARNINGS:**
- The jitter `sort_index` approach mutates the `sort_index` tuple on `ScheduledJob` directly; ensure heap ordering is consistent after this (re-push does a fresh `heapq.heappush`, so ordering is correct)
- `_removal_callbacks` should be keyed by `owner_id` (not `job_id`) — one callback per `Scheduler` instance per `owner_id`
