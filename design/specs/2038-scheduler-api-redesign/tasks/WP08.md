## Objectives & Success Criteria

The public documentation under `docs/` accurately describes the new scheduler API — no references to removed methods or parameter forms remain. `CLAUDE.md`'s Architecture section reflects the new trigger model. `uv run mkdocs build` exits without errors.

## Subtasks

1. Find all scheduler-related docs pages: `Glob docs/**/*.md` — read each page that mentions `run_daily`, `run_every`, `run_cron`, `IntervalTrigger`, `CronTrigger`, `ScheduleStartType`
2. Update each affected docs page:
   - Replace old method signatures and examples with new ones using trigger objects
   - Remove `start=` tuple examples; replace with `Once(at="HH:MM")` or `Daily(at="HH:MM")` examples
   - Remove `run_daily(days=N)` examples; replace with `Cron("0 HH */N * *")` guidance
   - Remove `run_cron(hour=N, minute=N, ...)` keyword form; replace with positional string form
   - Add a new "Custom Triggers" section with the `SolarPollTrigger` example from the design doc
   - Add a "Job Groups" section covering `schedule(..., group="morning")`, `cancel_group()`, `list_jobs(group=)`
   - Add a "Jitter" section documenting the `jitter=` parameter and its heap-only semantics
3. Update `CLAUDE.md`'s `## Architecture` section, `Scheduler` bullet: replace the current method list with: `schedule(func, trigger)`, `run_in()`, `run_daily()`, `run_every()`, `run_cron()` — mention trigger objects (`After`, `Once`, `Every`, `Daily`, `Cron`) and the `TriggerProtocol`
4. Remove any `docs/` references to `ScheduleStartType` (7-type union) — this type is removed
5. Run `uv run mkdocs build` and fix any broken links or undefined references
