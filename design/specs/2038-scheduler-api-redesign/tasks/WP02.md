## Objectives & Success Criteria

`src/hassette/scheduler/classes.py` is updated so that:
- `ScheduledJob` has a `group: str | None = None` field
- `ScheduledJob` has a `jitter: float | None = None` field
- `ScheduledJob.repeat` field is removed from the dataclass — it no longer exists at runtime
- `ScheduledJob.matches()` uses `trigger.trigger_id()` (not `==`) and includes `group` in the comparison; `repeat` is removed from the comparison
- `job.next_run` docstring updated: "Unjittered logical fire time — used as `previous_run` in subsequent trigger calls"
- `ScheduledJobRegistration` in `src/hassette/core/registration.py` has `repeat` field removed and `trigger_label: str` / `trigger_detail: str | None` fields added

Unit tests in `tests/unit/test_scheduler_job_names.py` and new tests in `tests/unit/test_scheduled_job.py` all pass. Pyright reports no errors on the modified files.

## Subtasks

1. Remove `repeat: bool` field from `ScheduledJob` dataclass in `src/hassette/scheduler/classes.py`
2. Add `group: str | None = field(default=None, compare=False)` to `ScheduledJob`
3. Add `jitter: float | None = field(default=None, compare=False)` to `ScheduledJob`
4. Rewrite `ScheduledJob.matches()` to compare `trigger_id()` instead of `==`: `self.trigger.trigger_id() == other.trigger.trigger_id() if (self.trigger and other.trigger) else self.trigger is other.trigger`, plus include `group` comparison; remove `repeat` from comparison
5. Update `ScheduledJob.__post_init__` — remove the `repeat`-related name formatting if present; update name to not reference `trigger` via `str()` (now uses `trigger.trigger_label()`)
6. In `src/hassette/core/registration.py`: remove `repeat: bool` from `ScheduledJobRegistration`; add `trigger_label: str` and `trigger_detail: str | None` fields
7. Grep for all construction sites of `ScheduledJobRegistration` in `src/` and update them to pass `trigger_label` / `trigger_detail` instead of `repeat` (primarily `src/hassette/core/scheduler_service.py:_enqueue_then_register` — but do not rewrite that method here, just flag it for WP04)
8. Write `tests/unit/test_scheduled_job.py` (see Test Strategy)

## Test Strategy

File: `tests/unit/test_scheduled_job.py`

Tests to write:
- `test_matches_same_trigger_id` — two jobs with `Every(hours=1)` triggers match regardless of object identity
- `test_matches_different_trigger_id` — `Every(hours=1)` vs `Every(hours=2)` do not match
- `test_matches_different_group` — same callable, same trigger, but `group="morning"` vs `group="evening"` do not match
- `test_matches_same_group` — same callable, same trigger, same group → match
- `test_matches_no_trigger` — two jobs with `trigger=None` compare by trigger identity (both None → match)
- `test_group_field_defaults_none` — freshly constructed job has `group=None`
- `test_jitter_field_defaults_none` — freshly constructed job has `jitter=None`
- `test_no_repeat_field` — `hasattr(job, "repeat")` is `False`
- `test_set_next_run_sort_index` — `set_next_run()` updates `sort_index` correctly
- `test_mark_registered_once` — second call to `mark_registered()` with different db_id logs warning and ignores new value

Existing `tests/unit/test_scheduler_job_names.py` must continue to pass after the `repeat` field removal — update any constructions in that file that pass `repeat=`.

## Review Guidance

**BLOCKERS:**
- `repeat` field must not exist on `ScheduledJob` at runtime — any runtime check of `job.repeat` anywhere in `src/` is wrong (DB column is fine in the migration, but the Python field is gone)
- `matches()` must use `trigger_id()` not `==` — falling back to `==` defeats the deduplication contract
- `group` must be included in `matches()` — two jobs with identical triggers but different groups must not match
