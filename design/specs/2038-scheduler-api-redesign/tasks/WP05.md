## Objectives & Success Criteria

A new Alembic migration `src/hassette/migrations/versions/002_scheduler_api_redesign.py` exists and runs cleanly. After running, the `scheduled_jobs` table has:
- `trigger_label TEXT NOT NULL DEFAULT ''` — populated from `trigger.trigger_label()` at registration
- `trigger_detail TEXT NULL` — populated from `trigger.trigger_detail()` at registration
- The `trigger_type` CHECK constraint accepts `"interval"`, `"cron"`, `"once"`, `"after"`, `"custom"`, and `NULL`
- `repeat` column is retained (it is a DB-only cached hint for reconciliation; the Python field was removed in WP02, but the DB column must stay for existing row compatibility)

`src/hassette/core/registration.py:ScheduledJobRegistration` has `trigger_label: str` and `trigger_detail: str | None` fields; `repeat: bool` field is removed (it was removed in WP02 but the migration must not drop the DB column).

`src/hassette/core/command_executor.py` (the `register_job()` path) passes `trigger_label` and `trigger_detail` through to the INSERT.

Migration runs forward without error. `spec-helper wp-validate 2038-scheduler-api-redesign` passes.

## Review Guidance

**BLOCKERS:**
- `trigger_type` CHECK constraint must accept `"once"` and `"after"` — otherwise `Once`/`After` job registration will fail with a CHECK constraint violation
- `trigger_label NOT NULL DEFAULT ''` — cannot be `NULL` for new rows; existing rows get empty string via `DEFAULT ''`
- SQLite does not support `ALTER TABLE ... DROP CONSTRAINT` or `ADD CONSTRAINT` — must use Alembic `batch_alter_table` for the constraint change
- `repeat` DB column must NOT be dropped — it is used by restart reconciliation queries

**WARNINGS:**
- `telemetry_repository.py` SELECT queries that do `SELECT *` from `scheduled_jobs` will automatically include new columns — verify the telemetry model can handle extra fields gracefully (ignore unknown columns or add them explicitly)
- The `DEFAULT ''` for `trigger_label` on existing rows means old jobs will have an empty label until they are re-registered; this is acceptable
