## Objectives & Success Criteria

`src/hassette/scheduler/scheduler.py` is rewritten so that:
- `Scheduler.schedule(func, trigger: TriggerProtocol, name, group, jitter, *, if_exists, args, kwargs)` is the new primary entry point; the old `schedule(func, run_at, trigger, repeat, ...)` signature is removed
- `_jobs_by_group: dict[str, set[ScheduledJob]]` tracks jobs by group on each `Scheduler` instance
- `cancel_group(group: str)` cancels all jobs in the group (no-op if group is nonexistent)
- `list_jobs(group: str | None = None)` returns all or group-filtered jobs
- `add_job()`, `remove_job()`, `remove_all_jobs()` all maintain `_jobs_by_group` in sync
- A removal callback is registered with `SchedulerService` so exhausted one-shot jobs are removed from `_jobs_by_group` automatically
- Convenience methods (`run_in`, `run_once`, `run_every`, `run_hourly`, `run_minutely`, `run_daily`, `run_cron`) are thin wrappers over `schedule()` using the new trigger objects; old parameter forms (`start=` tuple, `interval=N`, `days=N`, keyword-per-field cron) are removed
- `src/hassette/core/state_proxy.py` caller updated: `run_every(interval=TimeDelta(...))` → `run_every(seconds=config.state_proxy_poll_interval_seconds)`
- `get_start_dtme()` function removed from `scheduler.py`

Unit tests in `tests/unit/test_scheduler_resource.py` all pass. Pyright clean.

## Subtasks

1. Rewrite `Scheduler.schedule()` signature: `(self, func, trigger: TriggerProtocol, name="", group=None, jitter=None, *, if_exists="error", args=None, kwargs=None) -> ScheduledJob`; call `trigger.first_run_time(date_utils.now())` to get `run_at`; construct `ScheduledJob` with `group=group`, `jitter=jitter`; delegate to `add_job()`
2. Add `_jobs_by_group: dict[str, set[ScheduledJob]] = {}` in `Scheduler.__init__`
3. Update `add_job()` to add the job to `_jobs_by_group[job.group]` when `job.group is not None`
4. Update `remove_job()` to remove the job from `_jobs_by_group[job.group]` and drop the key if the set becomes empty
5. Update `remove_all_jobs()` to clear `_jobs_by_group` entirely before delegating to `SchedulerService`
6. Add `cancel_group(group: str) -> None`: iterate `_jobs_by_group.get(group, set()).copy()`, call `job.cancel()` and `scheduler_service.remove_job(job)` for each; del the group key; no-op if group not in dict
7. Add `list_jobs(group: str | None = None) -> list[ScheduledJob]`: return all jobs from `_jobs_by_name.values()` if `group=None`; else return `list(_jobs_by_group.get(group, set()))`
8. Register a removal callback with `SchedulerService.register_removal_callback(owner_id, callback)` in `Scheduler.__init__` so `reschedule_job` can notify this `Scheduler` when a one-shot job is auto-exhausted (the callback removes the job from `_jobs_by_group` and `_jobs_by_name`)
9. Rewrite convenience methods as thin wrappers:
   - `run_in(func, delay: TimeDelta | float, ...)` → `schedule(func, After(seconds=delay_secs), ...)`
   - `run_once(func, start: str | ZonedDateTime, ...)` → `schedule(func, Once(at=start), ...)`
   - `run_every(func, hours=0, minutes=0, seconds=0, ...)` → `schedule(func, Every(hours=h, minutes=m, seconds=s), ...)`; remove `interval=N` form
   - `run_minutely(func, minutes=1, ...)` → `schedule(func, Every(minutes=minutes), ...)`
   - `run_hourly(func, hours=1, ...)` → `schedule(func, Every(hours=hours), ...)`
   - `run_daily(func, at="00:00", ...)` → `schedule(func, Daily(at=at), ...)`; remove `days=N` form
   - `run_cron(func, expression: str, ...)` → `schedule(func, Cron(expression), ...)`; remove keyword-per-field form
10. Remove `get_start_dtme()` function entirely
11. Update `src/hassette/core/state_proxy.py` line 93: replace `run_every(self._load_cache, interval=TimeDelta(seconds=...))` with `run_every(self._load_cache, seconds=config.state_proxy_poll_interval_seconds)`
12. Write `tests/unit/test_scheduler_resource.py` (see Test Strategy)

## Review Guidance

**BLOCKERS:**
- Old `schedule(func, run_at, trigger, repeat, ...)` signature must not exist — if it does, the "wholesome replacement" intent is not met
- `_jobs_by_group` must be maintained by ALL three removal paths (WP design contract): `remove_job()`, `remove_all_jobs()`, and the removal callback. Missing any one is a bug.
- Convenience methods must not accept removed params: `start=` tuple, `interval=N`, `days=N`, keyword-per-field cron
- `run_daily` must delegate to `Daily` (not `IntervalTrigger`) — otherwise the 24h drift bug persists

**WARNINGS:**
- If `SchedulerService.register_removal_callback()` doesn't exist yet (it's added in WP04), use a forward-compatible stub; WP04 wires it up
- `run_once` param type changes: old form accepted `ScheduleStartType` (7-type union); new form accepts `str | ZonedDateTime` — ensure no existing test passes a tuple
