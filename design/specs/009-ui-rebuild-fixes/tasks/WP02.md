## Objectives & Success Criteria

After this WP:
1. The 5s polling partial (`app-handler-stats`) is consumed by JS that updates handler row counts, status dots, and duration text in the DOM
2. `data-live-on-app` is removed from `#app-handlers` and `#app-jobs` on the app detail page — these sections no longer morph on WS events
3. Expanding a handler row and waiting >5s for a poll does NOT collapse the row — Alpine state is preserved
4. Invocation counts, failed counts, and status dot colors update every 5s without page reload
5. The `app_handler_stats.html` partial includes `avg_duration_ms` in its data attributes

## Subtasks

1. Update `src/hassette/web/templates/partials/app_handler_stats.html` to include `data-avg-duration-ms="{{ listener.avg_duration_ms }}"` attribute on each span.

2. Add a stats-to-DOM updater function in `src/hassette/web/static/js/live-updates.js`:
   - Listen for `htmx:afterSwap` events where the target is `#app-handler-stats`
   - For each `<span>` with `data-listener-id` inside the swapped content:
     - Find the matching handler row `#handler-{id}`
     - Update the invocation count text (find `.ht-meta-item` with "calls" text)
     - Update the failed count text (find `.ht-meta-item--strong.ht-text-danger`)
     - Update the avg duration text
     - Compute dot class from counts: `danger` if failed > 0, `success` if total > 0, else `neutral`
     - Set `dot.className = 'ht-item-row__dot ht-item-row__dot--' + dotClass`

3. Remove `data-live-on-app` attribute from the `#app-handlers` div in `src/hassette/web/templates/pages/app_detail.html` (line ~112).

4. Remove `data-live-on-app` attribute from the `#app-jobs` div in `src/hassette/web/templates/pages/app_detail.html` (line ~132).

5. Verify the health strip `data-live-on-app` is retained (line ~101) — health strip morphing is safe.
