## Objectives & Success Criteria

After this WP:
1. `classify_error_rate(rate)` and `classify_health_bar(success_rate)` exist in `context.py` and return CSS class names (`good/warn/bad` and `excellent/good/warning/critical`)
2. Both functions are registered as Jinja2 template globals so templates can call them directly
3. All inline threshold logic is removed from `dashboard.html`, `app_health_strip.html`, and `dashboard_app_grid.html`
4. Thresholds are consistent: <5% = good, 5-10% = warn, >=10% = bad everywhere

## Subtasks

1. Add `classify_error_rate(rate: float) -> str` and `classify_health_bar(success_rate: float) -> str` to `src/hassette/web/ui/context.py`.

2. Register both functions as Jinja2 template globals. The template environment is initialized in `src/hassette/web/ui/__init__.py` — add them there via `templates.env.globals["classify_error_rate"] = classify_error_rate`.

3. Update `src/hassette/web/templates/pages/dashboard.html:22` — replace inline ternary `{{ 'good' if err_pct < 5 else ('warn' if err_pct < 10 else 'bad') }}` with `{{ classify_error_rate(err_pct) }}`.

4. Update `src/hassette/web/templates/partials/app_health_strip.html` — replace inline threshold logic with `classify_error_rate()` call.

5. Update `src/hassette/web/templates/partials/dashboard_app_grid.html` — replace inline threshold logic for both error rate classes and health bar gradient classes with `classify_error_rate()` and `classify_health_bar()`.
