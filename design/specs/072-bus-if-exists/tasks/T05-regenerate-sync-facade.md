## Summary
Regenerate the auto-generated synchronous bus facade so it exposes `if_exists` on the same
methods as the async `Bus`, and verify the codegen drift check passes. Runs after all `Bus`
signature changes (T03 + T04) are complete.

## Prompt
1. Regenerate the facade:
   ```bash
   uv run hassette-codegen sync-facade
   ```
   This rewrites `src/hassette/bus/sync.py` from the async `Bus` class. Do NOT hand-edit
   `sync.py` — it carries a "Do not edit this file directly" header.

2. Verify the regenerated facade exposes `if_exists` on `on()`, `add_listener`, and the
   event-specific `**opts` methods (`on_state_change`, `on_attribute_change`, `on_call_service`,
   etc.), and that the new `Subscription` return type on `add_listener` is reflected.

3. Run the drift check and the facade generation tests:
   ```bash
   uv run hassette-codegen sync-facade --check
   uv run pytest tests/integration/test_sync_facades.py tests/unit/tools/test_generate_sync_facade.py tests/unit/test_recording_sync_facade.py
   ```
   All must pass with no drift.

4. Confirm the reshaped collision method from T03 (`_resolve_collision`) does NOT appear on the
   facade — `sync_facade/ast_utils.py` skips `_`-prefixed methods. If it appears, the method was
   not underscore-prefixed; fix the name in T03's code (it must stay private).

## Focus
- The facade generator lives at `codegen/src/hassette_codegen/sync_facade/`. The CI lint/test
  pipeline runs the `--check` drift gate; a stale `sync.py` fails CI.
- `Options` is imported by both `bus.py` and the generated `sync.py` (options.py module docstring
  explains the import-cycle reason) — adding `if_exists` to the TypedDict flows to the facade
  signatures automatically on regeneration.
- This task produces a generated artifact only — no hand-written logic. If the generated output
  looks wrong, the fix belongs in the async `Bus` (T03/T04) or the generator, not in `sync.py`.
