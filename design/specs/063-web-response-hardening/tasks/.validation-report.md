## Warnings

**W1 — T01 unit test list omits `ErrorRateClass` and `ListenerKind`.**
The design's Test Strategy (FR#1 new coverage) explicitly lists `ErrorRateClass` and `ListenerKind` as types requiring reject-out-of-range unit tests. T01 Prompt step 7 lists five types to test (`InvocationStatus`, `ManifestStatus`, `ResourceStatus`, `HealthStatus`, `LOG_LEVEL_TYPE`) but omits `ErrorRateClass` and `ListenerKind`. The Verify item for AC#1 is broad ("all constrained types") and doesn't reproduce the gap. Implementer should add tests for these two types to `test_model_types.py`.

**W2 — T06 CLI spec update not reflected in a Verify item.**
T06 Prompt step 7 instructs updating the CLI design spec in the `worktree-cli` branch (changing `GET /api/logs/by-execution/{id}` → `GET /api/executions/{execution_id}`). This is required by the design (Architecture > Execution endpoint, last paragraph) and AC#8's scope, but no corresponding Verify checkbox exists in T06. Low risk since it's a one-line doc change, but it could be skipped without triggering a failing check.

**W3 — T06 references `web/CLAUDE.md` for `try/except DB_ERRORS` pattern.**
T06 Prompt step 3 says "Wrap in `try/except DB_ERRORS` → 503 pattern per `web/CLAUDE.md`". No `web/CLAUDE.md` is mentioned in the design doc or project CLAUDE.md. This is likely a stale internal reference. The pattern itself is correct (it matches `routes/logs.py:102` convention); the citation is just inaccurate. Implementer should look at `routes/logs.py` directly, not a non-existent file.
