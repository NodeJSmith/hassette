## Objectives & Success Criteria

Extend `AppTestHarness` with the three core testing primitives: event simulation, state seeding, and time control. After this WP, end users can write complete app tests:

```python
async with AppTestHarness(MotionLights, config={...}) as harness:
    harness.set_state("binary_sensor.motion", "off")
    await harness.simulate_state_change("binary_sensor.motion", old_value="off", new_value="on")
    harness.api_recorder.assert_called("turn_on", entity_id="light.kitchen")
```

1. `simulate_state_change(entity_id, old_value, new_value, old_attrs, new_attrs)` creates an event via the existing factory, sends through the bus, and drains handlers reliably.
2. `simulate_attribute_change(entity_id, attribute, old_value, new_value)` does the same for attribute changes.
3. `simulate_call_service(domain, service, **data)` does the same for service call events.
4. `set_state(entity_id, state, **attributes)` seeds the StateProxy cache via `_test_seed_state()` with a sentinel timestamp. Does NOT fire bus events.
5. `set_states(states_dict)` seeds multiple entities at once.
6. `freeze_time(instant)` patches `hassette.utils.date_utils.now` via `_TestClock`. Idempotent.
7. `advance_time(seconds, minutes, hours)` advances the frozen clock. Raises `RuntimeError` if not frozen. Does NOT auto-trigger jobs.
8. `trigger_due_jobs() -> int` snapshots due jobs, dispatches inline, returns count of completed jobs.

## Test Strategy

**File**: `tests/integration/test_app_harness_simulation.py`

Define a test app:
```python
class SimTestApp(App[AppConfig]):
    async def on_initialize(self):
        self.bus.on_state_change("sensor.temp", handler=self._on_temp)
        self.scheduler.run_daily(self._daily_task, start=(7, 0), name="daily")
    async def _on_temp(self, event):
        await self.api.turn_on("light.alert")
    async def _daily_task(self):
        await self.api.call_service("cover", "open_cover", target={"entity_id": "cover.blinds"})
```

- `test_simulate_state_change_triggers_handler` — simulate change on `sensor.temp`, verify `api_recorder.assert_called("turn_on", entity_id="light.alert")`.
- `test_simulate_state_change_drains_handlers` — verify the method returns only after handler completes (no sleep-based timing).
- `test_set_state_seeds_proxy` — call `set_state("light.kitchen", "on", brightness=255)`, verify `harness.states` returns the value.
- `test_set_state_does_not_fire_events` — spy on bus, call `set_state`, verify no events emitted.
- `test_set_states_multiple` — seed 3 entities, verify all appear in StateProxy.
- `test_freeze_time_patches_now` — freeze at a known instant, call `now()`, verify it returns the frozen time.
- `test_freeze_time_idempotent` — freeze twice, verify second call replaces (not stacks).
- `test_advance_time_without_freeze_raises` — call `advance_time` without `freeze_time`, verify `RuntimeError`.
- `test_advance_time_moves_clock` — freeze, advance 60s, verify `now()` is 60s later.
- `test_trigger_due_jobs_fires_scheduled` — freeze at 6:00, advance to 7:00, trigger, verify `call_service("cover", "open_cover")` was called.
- `test_trigger_due_jobs_returns_count` — verify return value matches jobs fired.
- `test_trigger_due_jobs_snapshot_prevents_infinite_loop` — freeze time, trigger a repeating job, verify it fires exactly once per trigger call (not infinitely).
- `test_freeze_time_cleanup_on_exit` — freeze time inside harness, exit, verify `now()` returns real time.

## Review Guidance

- **Drain target**: The implementation must trace the actual task ownership path. If `app.bus.task_bucket` is not where handler tasks land, the drain will be unreliable. Verify empirically.
- **Sentinel timestamp**: Must be `1970-01-01T00:00:00+00:00` (with timezone). Verify `make_state_dict` accepts a `last_updated` override.
- **trigger_due_jobs snapshot**: Must call `pop_due_and_peek_next` exactly ONCE. Do not loop — repeating jobs re-enqueue during `_dispatch_and_log` and would fire infinitely with a frozen clock.
- **Patcher cleanup**: `freeze_time` must stop the old patcher before creating a new one. The patcher's `stop` must be registered with the exit stack.
- **advance_time does NOT auto-trigger**: This is a deliberate design decision. The user must call `trigger_due_jobs()` explicitly.
