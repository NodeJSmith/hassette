## Objectives & Success Criteria

1. `hassette.test_utils._internal` subpackage contains all existing internal symbols (HassetteHarness, SimpleTestServer, etc.).
2. `hassette.test_utils.__init__.py` exports Tier 1 symbols in `__all__` and re-exports Tier 2 from `_internal` for backward compatibility.
3. `make_test_config(**overrides)` in `src/hassette/test_utils/config.py` creates a hermetic `HassetteConfig` with sensible defaults.
4. `pyproject.toml` has `[project.optional-dependencies] test = ["pytest>=8.0", "pytest-asyncio>=0.23"]`.
5. All existing internal tests (`tests/`) continue to work unchanged — imports like `from hassette.test_utils import HassetteHarness` resolve via re-export.
6. Module docstring updated to reflect Tier 1 APIs are supported for end users.

## Subtasks

1. Create `src/hassette/test_utils/_internal/__init__.py`. Move all current exports from `test_utils/__init__.py` into this file (all 48 symbols). The `_internal` package re-exports everything that was previously in `test_utils`.

2. Rewrite `src/hassette/test_utils/__init__.py`:
   - New docstring: "Test utilities for hassette apps. Tier 1 APIs (AppTestHarness, RecordingApi, make_test_config, event factories) are stable and documented for end users."
   - Tier 1 imports: `AppTestHarness`, `RecordingApi`, `ApiCall`, `make_test_config`, `create_state_change_event`, `create_call_service_event`, `make_state_dict`, `make_light_state_dict`, `make_sensor_state_dict`, `make_switch_state_dict`.
   - Tier 2 re-exports: `from ._internal import *` (or explicit names) for backward compatibility.
   - `__all__` contains only Tier 1 symbols.

3. Create `src/hassette/test_utils/config.py` with `make_test_config(**overrides) -> HassetteConfig`:
   - Uses the same hermetic settings pattern as `AppTestHarness` (override `settings_customise_sources` to `InitSettingsSource` only).
   - Defaults: `ha_host="test.invalid"`, `ha_port=8123`, `ha_token="test-token"`, `data_dir=tempfile.mkdtemp()`.
   - Overrides applied via the init kwargs.
   - Docstring lists the defaults.

4. Add to `pyproject.toml`:
   ```toml
   [project.optional-dependencies]
   test = ["pytest>=8.0", "pytest-asyncio>=0.23"]
   ```

5. Verify all existing test imports work by running the full test suite.

## Test Strategy

**File**: `tests/unit/test_public_api_surface.py`

- `test_tier1_in_all` — verify `hassette.test_utils.__all__` contains exactly the Tier 1 symbols.
- `test_tier2_importable` — verify `from hassette.test_utils import HassetteHarness` works (backward compat).
- `test_tier2_not_in_all` — verify `HassetteHarness` not in `__all__`.
- `test_star_import_only_tier1` — verify `from hassette.test_utils import *` only brings Tier 1 symbols into namespace.

**File**: `tests/unit/test_make_test_config.py`

- `test_defaults` — call `make_test_config()`, verify `ha_host="test.invalid"`, `ha_port=8123`, `ha_token="test-token"`.
- `test_override` — call `make_test_config(ha_host="192.168.1.1")`, verify override applied.
- `test_hermetic_no_env` — set env var, call `make_test_config()`, verify env var not picked up.
- `test_data_dir_is_tmpdir` — verify `data_dir` points to a temp directory.

## Review Guidance

- **Backward compatibility**: Every import that works today (`from hassette.test_utils import HassetteHarness`, `from hassette.test_utils import create_state_change_event`) must continue to work. Run the full existing test suite to verify.
- **`__all__` contents**: Must be exactly the Tier 1 symbols. No Tier 2 symbols.
- **`_internal` package**: Must contain all 48 previously exported symbols. Missing any will break internal tests.
- **make_test_config hermiticity**: Must suppress env vars and .env files. The default `ha_host="test.invalid"` must never resolve to a real HA instance.
- **pyproject.toml**: Version floor `pytest-asyncio>=0.23` must match the project's own pinned version. Check `uv.lock` or `[dependency-groups]` for the actual version.
