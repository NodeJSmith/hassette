## Objectives & Success Criteria

`AppTestHarness` at `src/hassette/test_utils/app_harness.py` — an async context manager that wires a user's App class into test infrastructure. After `async with AppTestHarness(MyApp, config={...}) as harness:`, the user has a fully initialized app with `harness.app`, `harness.bus`, `harness.scheduler`, `harness.api_recorder`, and `harness.states`.

1. `__init__` stores args only — no resource allocation.
2. `__aenter__` follows the 12-step sequence from the design doc, using `AsyncExitStack` with LIFO teardown.
3. Config validation uses the hermetic `_HermeticSettings` subclass pattern (only `InitSettingsSource`, no env/file).
4. Bad config raises `AppConfigurationError` with app class name and field details.
5. `__aexit__` tears down in correct order: app shutdown (while harness is live) → harness stop → restore `_api_factory` → restore manifest → reset ContextVar → cleanup tmpdir.
6. Auto-creates a tmpdir when `tmp_path` is not provided, cleaned up on exit.
7. `set_global_hassette()` called in `__aenter__` (not `HassetteHarness.__init__`), token stored and reset on exit.
8. `state_proxy.mark_ready()` called once during setup.
9. `hassette.api.get_states_raw = AsyncMock(return_value=[])` configured before StateProxy starts.
10. State proxy polling disabled in the minimal config.

## Subtasks

1. Create `src/hassette/test_utils/app_harness.py` with `AppTestHarness` class.

2. Implement `__init__(self, app_cls, config: dict, *, tmp_path: Path | None = None)` — stores `_app_cls`, `_config_dict`, `_tmp_path`. No other allocation.

3. Implement `AppConfigurationError` exception class — accepts `app_cls` and `original_error`, produces a message like `"AppConfigurationError for MotionLights: 1 validation error — field 'motion_entity' is required"`.

4. Implement the hermetic config factory: create `_make_hermetic_config(app_config_cls, config_dict)` that creates a transient subclass overriding `settings_customise_sources` to return only `InitSettingsSource(settings_cls, init_kwargs=config_dict)`. Catches `ValidationError` and re-raises as `AppConfigurationError`.

5. Implement `__aenter__`:
   - Create `AsyncExitStack` (stored as `self._exit_stack`).
   - Create tmpdir if `tmp_path` is None (via `tempfile.mkdtemp()`), register cleanup.
   - Create minimal `HassetteConfig` with `ha_host="test.invalid"`, polling disabled, `data_dir` from tmpdir.
   - Validate user config via hermetic factory.
   - Create `HassetteHarness` with `with_bus()`, `with_scheduler()`, `with_state_proxy()`, `with_state_registry()`.
   - Configure `hassette.api.get_states_raw = AsyncMock(return_value=[])`.
   - **Important**: Remove `set_global_hassette` from `HassetteHarness.__init__` — move it here. Start harness, register `harness.stop()` callback (registered early = unwinds late).
   - Call `set_global_hassette()`, store token, register `ContextVar.reset(token)`.
   - Call `state_proxy.mark_ready()`.
   - Synthesize `AppManifest`, set on class with save/restore.
   - Set `_api_factory = RecordingApi` on class with save/restore.
   - Instantiate app, add as child, start, wait for RUNNING.
   - Register app shutdown callback (registered late = unwinds early).

6. Implement `__aexit__` — delegates to `self._exit_stack.__aexit__()`.

7. Implement convenience properties: `app`, `bus`, `scheduler`, `api_recorder`, `states`.

8. Handle the `set_global_hassette` move: in `HassetteHarness.__init__` (line 212 of `harness.py`), add a `skip_global_set: bool = False` constructor parameter. When `True`, skip the `context.set_global_hassette()` call — `AppTestHarness` passes `skip_global_set=True` and handles the ContextVar itself in `__aenter__`. Internal tests continue to use `HassetteHarness` without the flag (defaults to `False`, preserving current behavior).

## Test Strategy

**File**: `tests/integration/test_app_test_harness.py`

Define a minimal test app class inline:
```python
class TestConfig(AppConfig):
    test_entity: str = "sensor.test"

class TestApp(App[TestConfig]):
    async def on_initialize(self):
        self.bus.on_state_change("sensor.test", handler=self._on_change)
    async def _on_change(self, event): pass
```

- `test_basic_lifecycle` — `async with AppTestHarness(TestApp, config={})` succeeds, `harness.app` is a `TestApp`, reaches RUNNING, exits cleanly.
- `test_api_recorder_is_recording_api` — verify `harness.api_recorder` is a `RecordingApi` instance.
- `test_bus_scheduler_states_exposed` — verify `harness.bus`, `harness.scheduler`, `harness.states` are the app's actual resources.
- `test_bad_config_raises_app_configuration_error` — pass `config={"nonexistent_required": "x"}` to an app with a required field, verify `AppConfigurationError`.
- `test_config_hermetic_ignores_env` — set env var matching the config prefix, verify it's not picked up.
- `test_cleanup_on_aenter_failure` — mock a failure partway through `__aenter__`, verify exit stack unwinds (no leaked tasks, ContextVar reset).
- `test_sequential_tests_dont_collide` — run two sequential `async with` blocks, verify no "already set" error.
- `test_manifest_restored_after_exit` — verify `TestApp.app_manifest` is restored to its original value.
- `test_api_factory_restored_after_exit` — verify `TestApp._api_factory` is `None` after exit.
- `test_auto_tmpdir_created_and_cleaned` — run without `tmp_path`, verify a tmpdir was used, verify it's removed after exit.

## Review Guidance

- **Exit stack order**: The LIFO order must result in app shutdown happening before harness stop. Verify by checking the registration order in `__aenter__` — harness stop must be registered BEFORE app shutdown.
- **ContextVar**: `set_global_hassette` must be called in `__aenter__`, not during `HassetteHarness` construction. Verify that sequential tests work.
- **Hermetic config**: The `_HermeticSettings` pattern must override `settings_customise_sources` to return only `InitSettingsSource`. Verify env vars are suppressed.
- **State proxy polling**: The minimal config must disable polling. Check that `_load_cache()` is not called periodically during tests.
- **HassetteHarness compatibility**: Internal tests that use `HassetteHarness` directly must still work. The `set_global_hassette` change must not break them.
