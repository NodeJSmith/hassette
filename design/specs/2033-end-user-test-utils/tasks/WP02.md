## Objectives & Success Criteria

A `RecordingApi` Resource subclass that records write calls and delegates read calls to `StateProxy`. Verified against an `ApiProtocol` at import time. Located at `src/hassette/test_utils/recording_api.py`.

1. All write methods (turn_on, turn_off, call_service, set_state, fire_event, toggle_service) record calls to `self.calls` and are no-ops.
2. All read methods (get_state, get_states, get_entity, get_entity_or_none, entity_exists, get_state_value, get_state_or_none) delegate to `StateProxy` and convert via state registry.
3. `get_state()` raises `EntityNotFoundError` for unseeded entities (matching real Api behavior).
4. Method signatures match real `Api` exactly — `call_service(domain, service, target=None, return_response=None, **data)`, `set_state(entity_id: str | StrEnum, state: Any, attributes: dict | None = None) -> dict`, etc.
5. `on_initialize()` calls `self.mark_ready()`.
6. `sync` attribute is a `Mock()` instance.
7. Unstubbed methods raise `NotImplementedError` with a message directing users to `use_api_server=True`.
8. Static `ApiProtocol` conformance assertion passes at import time.

## Subtasks

1. Define `ApiCall` dataclass in `src/hassette/test_utils/recording_api.py` — fields: `method: str`, `args: tuple`, `kwargs: dict`.

2. Define `ApiProtocol(Protocol)` covering all public async methods from `src/hassette/api/api.py`. Include: `turn_on`, `turn_off`, `call_service`, `set_state`, `fire_event`, `toggle_service`, `get_state`, `get_states`, `get_state_raw`, `get_states_raw`, `get_entity`, `get_entity_or_none`, `entity_exists`, `get_state_value`, `get_state_value_typed`, `get_state_or_none`, `get_attribute`, `get_history`, `render_template`.

3. Implement `RecordingApi(Resource)`:
   - Constructor accepts `hassette` and `state_proxy: StateProxy` (stored as `_state_proxy`).
   - `calls: list[ApiCall]` initialized to `[]`.
   - `sync = Mock()`.
   - `on_initialize` calls `self.mark_ready(reason="RecordingApi initialized")`.
   - Write methods: record `ApiCall(method_name, args, kwargs)` to `self.calls`, return appropriate stubs (`None`, `{}`, etc.). `call_service` with `return_response=True` returns a stub `ServiceResponse`.
   - Read methods: delegate to `_state_proxy._states` lookup, convert via state registry. `get_state` raises `EntityNotFoundError` if entity not in `_state_proxy._states`. `entity_exists` returns `bool`. `get_entity` calls through `get_state` logic. `get_state_or_none` / `get_entity_or_none` catch `EntityNotFoundError` and return `None`.
   - Remaining methods: raise `NotImplementedError(f"RecordingApi.{name}() is not implemented. Use use_api_server=True for full Api coverage.")`.

4. Implement assertion helpers: `assert_called(method, **kwargs)`, `assert_not_called(method)`, `assert_call_count(method, count)`, `get_calls(method=None)`, `reset()`.

5. Add static conformance assertion: `_: ApiProtocol = cast(ApiProtocol, RecordingApi)` or equivalent Pyright-checkable pattern at module level.

## Test Strategy

**File**: `tests/unit/test_recording_api.py`

- `test_turn_on_records_call` — call `turn_on("light.test")`, verify `calls` contains the right `ApiCall`.
- `test_call_service_records_target` — call `call_service("light", "turn_on", target={"entity_id": "light.x"})`, verify `target` is captured explicitly.
- `test_call_service_return_response` — call with `return_response=True`, verify a stub `ServiceResponse` is returned.
- `test_set_state_signature_matches_api` — call with `(entity_id, state, attributes={...})`, verify args recorded correctly with `dict` return.
- `test_fire_event_signature` — call with `(event_type, event_data={...})`, verify recorded.
- `test_get_state_delegates_to_state_proxy` — seed `_state_proxy._states` via `_test_seed_state`, call `get_state`, verify correct `BaseState` returned.
- `test_get_state_raises_entity_not_found` — call `get_state("nonexistent")`, verify `EntityNotFoundError`.
- `test_entity_exists_true_and_false` — seed one entity, verify `True` for seeded and `False` for unseeded.
- `test_get_state_or_none_returns_none` — verify `None` for unseeded entity (no exception).
- `test_unstubbed_method_raises` — call `render_template(...)`, verify `NotImplementedError` with helpful message.
- `test_assert_called_matching` — record a call, verify `assert_called` matches.
- `test_assert_called_fails` — verify `AssertionError` when no matching call.
- `test_assert_not_called` — verify passes when method not called, fails when called.
- `test_assert_call_count` — verify correct count.
- `test_reset_clears_calls` — record calls, reset, verify empty.
- `test_mark_ready_on_initialize` — start the resource, verify it reaches RUNNING.
- `test_protocol_conformance` — import the module, verify no Pyright/type errors at import time (this is a static check, but a runtime smoke test confirms the cast works).

## Review Guidance

- **Signature fidelity**: Every method signature must exactly match `src/hassette/api/api.py`. Diff the signatures. Pay special attention to `call_service` (`target`, `return_response`), `set_state` (`attributes: dict | None`), `turn_on`/`turn_off` (`domain` parameter).
- **EntityNotFoundError**: `get_state` must raise, not return `None`. This matches the real Api's 404 behavior.
- **State registry conversion**: Read methods must convert raw `HassStateDict` to typed `BaseState` via the state registry, not return raw dicts.
- **mark_ready**: Must be called in `on_initialize`. Without it, the harness hangs waiting for RUNNING status.
