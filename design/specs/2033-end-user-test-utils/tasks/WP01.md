## Objectives & Success Criteria

Three targeted changes to core framework classes that enable `AppTestHarness` (WP03) without changing any existing behavior:

1. `context.set_global_hassette()` returns the `Token` from `ContextVar.set()` — callers that ignore the return value are unaffected.
2. `App._api_factory: ClassVar[type[Resource] | None] = None` controls which class `add_child()` uses for the Api resource — defaults to `Api` when `None`.
3. `StateProxy._test_seed_state(entity_id, state_dict)` acquires the write lock and inserts state directly — for test harnesses only.

All existing tests pass unchanged.

## Subtasks

1. In `src/hassette/context.py`, modify `set_global_hassette()` to return the token from `HASSETTE_INSTANCE.set(hassette)` (line 50). Change return type from `None` to `Token[Hassette]`. The early-return path (line 30, `curr_inst is hassette`) should return `HASSETTE_INSTANCE.get(None)` — but since there's no stored token for that case, return `None` and change the return type to `Token[Hassette] | None`. Update docstring.

2. In `src/hassette/app/app.py`, add `_api_factory: ClassVar[type[Resource] | None] = None` to the `App` class body (after `app_config_cls` ClassVar). In `__init__` (line 94), change `self.api = self.add_child(Api)` to `factory = type(self)._api_factory or Api; self.api = self.add_child(factory)`.

3. In `src/hassette/core/state_proxy.py`, add `_test_seed_state(self, entity_id: str, state_dict: HassStateDict) -> None` method. It must: acquire `self._lock`, write to `self._states[entity_id] = state_dict`, and return. It does NOT call `mark_ready()`. Add a docstring noting this is for test harnesses only.

## Test Strategy

**File**: `tests/unit/test_framework_injection_points.py`

- `test_set_global_hassette_returns_token` — call `set_global_hassette`, verify return is a `Token`, reset with it, verify ContextVar is cleared.
- `test_set_global_hassette_same_instance_returns_none` — set once, call again with same instance, verify `None` return.
- `test_app_api_factory_default_uses_api` — instantiate App with `_api_factory=None`, verify `app.api` is an `Api` instance.
- `test_app_api_factory_custom_uses_factory` — set `_api_factory` to a mock Resource subclass, verify `app.api` is that type.
- `test_seed_state_writes_to_cache` — call `_test_seed_state`, verify entity appears in `_states`.
- `test_seed_state_acquires_lock` — verify the method acquires the write lock (use a lock spy or timeout test).

## Review Guidance

- **context.py**: The return type change must be `Token[Hassette] | None`, not just `Token`. The `None` case covers the early-return when the same instance is already set.
- **app.py**: `type(self)._api_factory` is correct (not `self._api_factory`) — ensures the ClassVar lookup follows MRO.
- **state_proxy.py**: `_test_seed_state` must NOT call `mark_ready()` — the design explicitly separates lifecycle from seeding. Verify no bus events are fired.
- **Blast radius**: All three changes must be backward-compatible. Existing tests must pass without modification.
