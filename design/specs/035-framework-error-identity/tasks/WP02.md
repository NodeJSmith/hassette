## Objectives & Success Criteria

Framework services register with component-specific keys instead of the bare sentinel. The registration drain correctly awaits all framework component tasks. After this WP:

- `register_framework_listener(component="service_watcher", ...)` produces `app_key="__hassette__.service_watcher"`
- `register_framework_listener(component="", ...)` raises `ValueError`
- `register_framework_listener(component="bad/name", ...)` raises `ValueError`
- AppHandler registers with `__hassette__.app_handler`
- Hassette core registers with `__hassette__.core`
- ServiceWatcher registers with `__hassette__.service_watcher` (4 listeners)
- `drain_framework_registrations()` drains all component-specific task lists
- `core.py` uses `drain_framework_registrations()` instead of `await_registrations_complete(FRAMEWORK_APP_KEY)`
- The test harness `_register_framework_listener()` wrapper accepts and passes through `component`

## Subtasks

1. Add `component: str` parameter with snake_case validation to `register_framework_listener()` in `src/hassette/core/bus_service.py`; construct `app_key` from `FRAMEWORK_APP_KEY_PREFIX + component`
2. Add `drain_framework_registrations()` method to `BusService` that iterates `list(self._pending_registration_tasks)` and drains all keys matching `is_framework_key()`
3. Update `src/hassette/core/app_handler.py` — pass `component="app_handler"` to `register_framework_listener()`
4. Update `src/hassette/core/core.py` — pass `component="core"` to `register_framework_listener()`; replace `await_registrations_complete(FRAMEWORK_APP_KEY)` with `drain_framework_registrations()`
5. Update `src/hassette/core/service_watcher.py` — pass `component="service_watcher"` to all 4 `register_framework_listener()` calls
6. Update `_register_framework_listener()` in `src/hassette/test_utils/harness.py` — add `component` parameter and pass it through to `BusService.register_framework_listener()`
7. Write unit tests for component validation and drain in a new section of an appropriate test file

## Test Strategy

- Test component validation: valid snake_case accepted, empty string rejected, invalid characters rejected
- Test `drain_framework_registrations()`: mock `_pending_registration_tasks` with mixed app/framework keys, verify only framework keys are drained
- Existing integration tests that call `_register_framework_listener()` must still pass — specifically: `tests/integration/test_framework_telemetry.py`, `tests/integration/test_core.py`, and `tests/integration/test_dispatch_unification.py` (all need to pass `component`)

## Review Guidance

- Verify `_pending_registration_tasks` iteration uses `list()` to avoid dict-size-changed-during-iteration errors (it's a defaultdict)
- Verify all 6 callers of `register_framework_listener()` pass `component`
- Verify `core.py` no longer references `await_registrations_complete(FRAMEWORK_APP_KEY)` anywhere
- Verify the component validation regex matches the design: `^[a-z][a-z_]*[a-z]$`
