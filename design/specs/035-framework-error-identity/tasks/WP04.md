## Objectives & Success Criteria

The dashboard shows a single unified error feed with both app and framework errors, tracebacks are expandable, and FrameworkHealth is a summary-only badge. After this WP:

- `dashboard_errors` endpoint defaults to `source_tier='all'` (was `'app'`)
- `dashboard_kpis` endpoint defaults to `source_tier='all'` (was `'app'`)
- `FrameworkSummaryResponse` only contains `total_errors` and `total_job_errors` (no `errors` list)
- `FrameworkHealth` badge count uses 24h time window matching the feed
- `FrameworkHealth` component shows count badge only, no expandable error list
- `dashboard.tsx` has a single "Recent Errors" section (no split)
- `error-feed.tsx` renders framework errors with "Framework" badge and component name (no broken links)
- `error-feed.tsx` renders traceback toggle using `ErrorCell` when `error_traceback` is present
- Frontend helpers `isFrameworkKey()`, `frameworkDisplayName()`, `frameworkDisplayLabel()` exist in shared utils
- CSS supports traceback toggle in flex context (error-feed layout)

## Subtasks

3. Change `dashboard_kpis()` default tier to `"all"` in `src/hassette/web/routes/telemetry.py`
4. Slim `FrameworkSummaryResponse` in `src/hassette/web/models.py` — remove `errors` list field, keep counts only
5. Update `dashboard_framework_summary()` in `src/hassette/web/routes/telemetry.py` — remove `get_recent_errors()` call, return counts only; use 24h time window for counts
6. Update `frontend/src/components/dashboard/framework-health.tsx` — remove expandable error feed, keep summary badge with count
7. Update `frontend/src/pages/dashboard.tsx` — remove split error sections, single "Recent Errors" card
8. Update `frontend/src/components/dashboard/error-feed.tsx` — import `isFrameworkKey`/`frameworkDisplayLabel`; guard link rendering with `!isFramework`; add traceback expand/collapse using `ErrorCell` from `app-detail/error-cell.tsx`
9. Add CSS for traceback toggle in flex context to `frontend/src/global.css` — ensure `ht-traceback` and toggle button work inside `ht-error-entry__body`
10. Regenerate `frontend/src/api/generated-types.ts` — run `npm run types` from the `frontend/` directory (which runs `openapi-typescript openapi.json -o src/api/generated-types.ts`). The `openapi.json` is exported by starting the dev server or running the schema export command.
