## Objectives & Success Criteria

The error feed query returns tracebacks, response models include them, traceback storage is capped, and framework-tier errors always preserve tracebacks. After this WP:

- `get_recent_errors()` SQL selects `error_traceback` from both `handler_invocations` and `job_executions`
- `HandlerErrorRecord` and `JobErrorRecord` include `error_traceback`
- `HandlerErrorEntry` and `JobErrorEntry` response models include `error_traceback`
- `dashboard_errors` and `dashboard_framework_summary` endpoints pass `error_traceback` through
- `track_execution()` caps tracebacks at 8 KB with a `"... [truncated]"` suffix
- `CommandExecutor._execute()` uses match/case on `cmd.source_tier`: app tier suppresses tracebacks for `DependencyError`/`HassetteError`, framework tier preserves all tracebacks
- Unexpected `source_tier` values raise `AssertionError`

## Subtasks

1. Add `error_traceback: str | None = None` to `HandlerErrorRecord` and `JobErrorRecord` in `src/hassette/core/telemetry_models.py`
2. Add `error_traceback` to both SELECT branches and the result construction in `get_recent_errors()` in `src/hassette/core/telemetry_query_service.py`
3. Add `error_traceback: str | None = None` to `HandlerErrorEntry` and `JobErrorEntry` in `src/hassette/web/models.py`
4. Pass `error_traceback` through in `dashboard_errors` and `dashboard_framework_summary` response construction in `src/hassette/web/routes/telemetry.py`
5. Add 8 KB traceback cap in `track_execution()` in `src/hassette/utils/execution.py`
6. Replace the `known_errors` assignment in `CommandExecutor._execute()` with match/case on `cmd.source_tier` in `src/hassette/core/command_executor.py`
7. Write tests for traceback cap in `tests/unit/test_execution.py` (existing file)
8. Write tests for source_tier branching (app suppresses, framework preserves, unknown asserts) in `tests/unit/test_command_executor.py` (new file)

## Test Strategy

- `tests/unit/test_execution.py`: `test_traceback_cap_under_limit` (no truncation), `test_traceback_cap_over_limit` (truncated with suffix), `test_traceback_cap_exactly_at_limit`
- Command executor tests: `test_app_tier_suppresses_known_error_traceback`, `test_framework_tier_preserves_known_error_traceback`, `test_unexpected_source_tier_raises`
- Integration: `tests/integration/test_web_api.py` — verify `error_traceback` appears in dashboard error response for errors that have one

## Review Guidance

- Verify the match/case has an explicit `case _: raise AssertionError(...)` arm — not a bare `else`
- Verify the 8 KB cap is applied before storing, not after retrieval
- Verify both UNION ALL branches in `get_recent_errors()` select `error_traceback`
- Verify `error_traceback` is passed through in both the `dashboard_errors` and `dashboard_framework_summary` endpoint handlers
