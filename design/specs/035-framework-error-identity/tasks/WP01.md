## Objectives & Success Criteria

Establish the prefix-based framework key convention with helpers, update all validation guards, and create the migration that updates CHECK constraints. After this WP:

- `is_framework_key("__hassette__.service_watcher")` returns `True`
- `is_framework_key("__hassette__")` returns `True` (bare key backward compat)
- `is_framework_key("my_app")` returns `False`
- `framework_display_name("__hassette__.service_watcher")` returns `"service_watcher"`
- `framework_display_name("__hassette__")` returns `"framework"`
- `AppConfig(app_key="__hassette__.my_app")` raises `ValueError`
- `config.py` rejects `__hassette__.anything` as an app key
- `_validate_source_tier()` accepts prefixed framework keys
- `reconcile_registrations()` skips all framework keys (not just bare)
- `get_all_app_summaries()` excludes all framework keys
- Migration 004 rebuilds `listeners` and `scheduled_jobs` with GLOB-based CHECK constraint
- Stale SQL comments in `reconcile_registrations()` are updated

## Subtasks

1. Add `FRAMEWORK_APP_KEY_PREFIX`, `is_framework_key()`, and `framework_display_name()` to `src/hassette/types/types.py`
2. Update `_validate_source_tier()` in `src/hassette/core/telemetry_repository.py` to use `is_framework_key()` instead of exact match
3. Update `reconcile_registrations()` guard in `src/hassette/core/telemetry_repository.py` to use `is_framework_key()`; update inline SQL comments referencing the old constant
4. Update `get_all_app_summaries()` in `src/hassette/core/telemetry_query_service.py` to filter with `is_framework_key()` comprehension instead of `discard(FRAMEWORK_APP_KEY)`
5. Update `_reject_hassette_sentinel` validator in `src/hassette/app/app_config.py` to use `is_framework_key()`
6. Update app key guard in `src/hassette/config/config.py` (`set_validated_app_manifests`) to use `is_framework_key()`
7. Create `src/hassette/migrations/versions/004_framework_key_prefix.py` — table-rebuild for `listeners` and `scheduled_jobs` with `CHECK (app_key NOT GLOB '__hassette__*' OR source_tier = 'framework')`
8. Write unit tests for `is_framework_key()` and `framework_display_name()` in `tests/unit/test_types.py` (new file)
9. Update existing tests in `tests/integration/test_framework_telemetry.py`, `tests/integration/test_dispatch_unification.py`, and `tests/system/test_telemetry_lifecycle.py` that hardcode `"__hassette__"` — update to accept component-specific keys where assertions reference the DB
