## Objectives & Success Criteria

Make the 35 new `Api` methods first-class citizens of the test harness. `RecordingApi` gains a `helper_definitions` seed dict, 32 CRUD method implementations that mutate the seed dict, and 3 counter action methods that record `ApiCall` directly. `AppTestHarness` gains a public `seed_helper(record)` method. `ApiProtocol` is extended with all 35 method signatures. `_KNOWN_READ_METHODS` in the parity test gains the 8 `list_*` names. The recording sync facade is regenerated.

This WP is where the test-harness parity gates are enforced. **Co-commit rule**: the `_KNOWN_READ_METHODS` update and the `list_*` additions to `RecordingApi` MUST land in the same commit, otherwise the parity test fails with a confusing error.

Concrete outcomes:

- `src/hassette/test_utils/recording_api.py` has:
  - Module-level `_SUPPORTED_HELPER_DOMAINS: frozenset[str]` — single source of truth for the 8 domains.
  - Module-level `_RECORD_TYPE_TO_DOMAIN: dict[type, str]` — hand-maintained lookup from Record class → domain name (with comment explaining the hand-maintained nature per round-3 finding F6).
  - Module-level `_slugify_helper_name(name: str) -> str` helper — HARNESS-ONLY; NOT in `api.py`. Uses either a local implementation or an existing hassette slugify utility if one exists (check via `grep -r slugify src/hassette/`).
  - `RecordingApi.helper_definitions: dict[str, dict[str, Any]]` attribute, initialized in `__init__` to `{d: {} for d in _SUPPORTED_HELPER_DOMAINS}`.
  - 8 `list_*` methods that return `cast(list[{Domain}Record], list(self.helper_definitions["{domain}"].values()))`.
  - 8 `create_*` methods that record an `ApiCall`, construct a `{Domain}Record` with `id=_slugify_helper_name(params.name)` and other fields from `params.model_dump(exclude_unset=True)`, insert into `helper_definitions`, return the new record.
  - 8 `update_*` methods that record an `ApiCall`, raise `KeyError` if `helper_id` not in seed dict, otherwise use `existing.model_copy(update=params.model_dump(exclude_unset=True))` to build the new record, replace in-place, return it.
  - 8 `delete_*` methods that record an `ApiCall`, raise `KeyError` on missing id, otherwise pop from the seed dict. (Per round-2 finding F7: strict, not silent.)
  - 3 counter action methods (`increment_counter`, `decrement_counter`, `reset_counter`) that record an `ApiCall` directly — NOT via `self.call_service(...)`. Following the existing `turn_on` pattern at `recording_api.py:167-176`.
  - `reset()` extended to reinitialize `self.helper_definitions` to empty-per-domain state (round-2 finding F8).
- `ApiProtocol` in `recording_api.py` gains 35 new method signatures (8 list + 8 create + 8 update + 8 delete + 3 counter actions).
- `src/hassette/test_utils/app_harness.py` gains `seed_helper(record: BaseModel) -> None` — derives domain from record type via `_RECORD_TYPE_TO_DOMAIN`, raises `ValueError` with the valid record class names on unknown type (round-2 finding F5).
- `tests/unit/test_recording_api_write_parity.py::_KNOWN_READ_METHODS` frozenset is extended with the 8 `list_*` names. **Co-commit required.**
- `src/hassette/test_utils/sync_facade.py` regenerated via `uv run tools/generate_sync_facade.py --target recording`.
- Drift gate `uv run tools/generate_sync_facade.py --target recording --check` exits 0.
- Parity gate `uv run pytest tests/unit/test_recording_api_write_parity.py -n 2` passes.
- `tests/unit/test_recording_api_helpers.py` (new file) exercises the full seed/list/create/update/delete lifecycle.
- `uv run pytest tests/unit/test_recording_api_helpers.py tests/unit/test_recording_api.py tests/unit/test_recording_sync_facade.py -n 2` passes.
- `uv run pyright` passes on all changed files.

Otherwise implement locally: `return name.lower().replace(" ", "_")` as a minimal version, with a docstring noting that WP01 documented HA's exact rule (if more complex transformation is needed, update this implementation).

7. Add the first full CRUD quartet for `input_boolean` to `RecordingApi`:
   - `list_input_booleans(self) -> list[InputBooleanRecord]` — `return cast(list[InputBooleanRecord], list(self.helper_definitions["input_boolean"].values()))`. No recording (this is a read).
   - `create_input_boolean(self, params: CreateInputBooleanParams) -> InputBooleanRecord` — record `ApiCall(method="create_input_boolean", args=(), kwargs=params.model_dump(exclude_unset=True))`, construct `record = InputBooleanRecord(id=_slugify_helper_name(params.name), **params.model_dump(exclude_unset=True))`, insert `self.helper_definitions["input_boolean"][record.id] = record`, return `record`.
   - `update_input_boolean(self, helper_id: str, params: UpdateInputBooleanParams) -> InputBooleanRecord` — record `ApiCall`, raise `KeyError(f"No input_boolean helper with id {helper_id!r} seeded. Use harness.seed_helper() or create_input_boolean() first.")` if `helper_id not in self.helper_definitions["input_boolean"]`, otherwise `existing = self.helper_definitions["input_boolean"][helper_id]; updated = existing.model_copy(update=params.model_dump(exclude_unset=True)); self.helper_definitions["input_boolean"][helper_id] = updated; return updated`.

9. Add 3 counter action methods to `RecordingApi`:

    ```python

    ```
11. Locate `src/hassette/test_utils/app_harness.py` (the `AppTestHarness` class) and add a new public method `seed_helper`:
    ```python
    def seed_helper(self, record: BaseModel) -> None:
        """Seed a stored helper config for tests that read helper CRUD.

        Domain is derived from the record class. Passing a record of a type
        not registered in _RECORD_TYPE_TO_DOMAIN raises ValueError immediately.
        """
        try:
            domain = _RECORD_TYPE_TO_DOMAIN[type(record)]
        except KeyError as e:
            raise ValueError(
                f"Unknown helper record type: {type(record).__name__}. "
                f"Expected one of: {sorted(t.__name__ for t in _RECORD_TYPE_TO_DOMAIN)}"
            ) from e
        self.api_recorder.helper_definitions[domain][record.id] = record
    ```
    Import `_RECORD_TYPE_TO_DOMAIN` from `recording_api`, or — if the import graph would be cleaner — move `_RECORD_TYPE_TO_DOMAIN` to `test_utils/_helpers.py` and import from both `recording_api.py` and `app_harness.py`. Either works; pick the lower-drama option.

## Review Guidance

**Code reviewer must verify:**

- Counter action methods on `RecordingApi` record an `ApiCall` directly — NO `await self.call_service(...)` call. The generator's `_check_no_async_peer_calls` guard at `generate_sync_facade.py:509` would abort regeneration if they did (round-2 finding F12).
- `_slugify_helper_name` lives in `recording_api.py` (or a shared test_utils helpers module), NOT in `api.py` (round-3 finding F5). Production `Api.create_*` methods must NOT call it — they let HA slugify server-side.
- `seed_helper(record)` has no `domain` parameter. It derives domain from `type(record)`. A call with `seed_helper("input_boolean", record)` (old-style) should fail at type-check time (Pyright) or raise at runtime. (Round-2 finding F5.)
- `update_*` and `delete_*` both raise `KeyError` on missing id — strictly consistent. Neither silently ignores. (Round-2 finding F7.)
- `reset()` clears `helper_definitions`, not just `calls`. (Round-2 finding F8.)
- `_KNOWN_READ_METHODS` update in `test_recording_api_write_parity.py` is IN THE SAME COMMIT as the `list_*` additions. Use `git log --oneline -3` and verify.
- The regenerated `test_utils/sync_facade.py` contains sync wrappers for all 35 new methods, and none of the module-level `_SUPPORTED_HELPER_DOMAINS`/`_RECORD_TYPE_TO_DOMAIN`/`_slugify_helper_name` helpers.
- `ApiProtocol` entries match the real `Api` method signatures exactly (including type annotations on `params` and return types).

**Integration reviewer must verify:**

- Drift gate passes for the recording target: `uv run pytest tests/unit/test_recording_sync_facade_generation.py::test_generator_check_mode_exits_zero -n 2`.
- Parity gate passes: `uv run pytest tests/unit/test_recording_api_write_parity.py -n 2`.
- No circular imports introduced between `recording_api.py` and `app_harness.py`.
- `harness.seed_helper(InputBooleanRecord(id="x", name="y"))` works at the public API level (end-to-end smoke test via the test harness fixture).

**Blocker-level deviations:**

- `self.call_service(...)` in any counter action method on `RecordingApi` → BLOCKER. Generator will abort.
- `_slugify_helper_name` in `api.py` → BLOCKER (F5 reversal). Harness-only logic leaking into production.
- `_KNOWN_READ_METHODS` NOT co-committed → BLOCKER. Parity test fails on commit.
- `delete_*` silently succeeds on missing id → BLOCKER (F7). Divergence from `update_*` and from real HA.
- `reset()` not clearing `helper_definitions` → BLOCKER (F8). Silent test-isolation bug.
