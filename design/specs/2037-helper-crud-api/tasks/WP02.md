## Objectives & Success Criteria

Introduce every Pydantic model the feature needs — one new state model (`CounterState`) and a new `models/helpers/` package containing Record / Create / Update models for all 8 helper domains. This WP produces only data classes + their tests; no `Api` methods yet.

Concrete outcomes:

- `python-slugify>=8.0` is added as a direct runtime dependency in `pyproject.toml` (WP05 uses this for `_slugify_helper_name` to match HA's exact slug rule — WP01 verified HA uses `python-slugify` with `separator="_"`). The package is already present in `uv.lock` as a transitive dep, but must be declared as direct so WP05's import is supported by the dependency graph.
- `uv.lock` is regenerated to reflect the direct dependency.
- `src/hassette/models/states/counter.py` exists with `CounterState` and `CounterAttributes`, following the pattern of `input.py` and `timer.py`.
- `CounterState` is registered in `src/hassette/models/states/__init__.py` imports and `__all__`.
- `STATE_REGISTRY` resolves `domain="counter"` to `CounterState` (no code change needed if the registry auto-walks `__all__`; otherwise add the explicit registration per existing pattern).
- `src/hassette/models/helpers/` package exists with `__init__.py` and 8 domain files: `input_boolean.py`, `input_number.py`, `input_text.py`, `input_select.py`, `input_datetime.py`, `input_button.py`, `counter.py`, `timer.py`.
- Each domain file exports three Pydantic models: `{Domain}Record`, `Create{Domain}Params`, `Update{Domain}Params`.
- `{Domain}Record` uses `model_config = ConfigDict(extra="allow")`; `Update{Domain}Params` uses `ConfigDict(extra="ignore")`; `Create{Domain}Params` uses default (`extra="forbid"`).
- `Create*Params` optional fields DO NOT default to `= None` — they rely on Pydantic's implicit unset semantics so `model_dump(exclude_unset=True)` can distinguish "not set" from "set to None".
- `CreateInputDatetimeParams` carries a `model_validator(mode="after")` enforcing `has_date or has_time`.
- `tests/unit/test_api_helper_models.py` exists with unit tests covering: each domain's Record extra="allow", each domain's Update empty-dict construction, the `CreateInputDatetimeParams` validator, and `exclude_unset=True` semantics on a representative Create.
- `uv run pyright` passes on all new files.
- `uv run pytest tests/unit/test_api_helper_models.py` passes.

## Review Guidance

**Code reviewer must verify:**

- `Create*Params` optional fields do NOT have `= None` defaults (this is the single most common mistake; `exclude_unset=True` fails silently if fields default to `None`).
- `Update*Params` classes use `ConfigDict(extra="ignore")` — a default `extra="forbid"` would break round-tripping after HA upgrades (round-1 finding F14).
- `{Domain}Record` classes use `ConfigDict(extra="allow")` — this is the defensive read-side posture for HA version drift.
- `CreateInputDatetimeParams.model_validator` is `mode="after"` (not `mode="before"` which operates on raw dict data before field validation).
- `CounterState` extends `NumericBaseState`, not `StringBaseState` — the state value is numeric. Compare to `TimerState` which correctly uses `StringBaseState` because timer values are strings like "active" / "idle".
- Every field name and type matches WP01's verified HA source table. Fields not in that table should not appear here.

**Integration reviewer must verify:**

- `CounterState` registration follows the same pattern as other numeric state models (e.g., `NumberState` in `number.py`).
- No drift between `models/helpers/__init__.py` exports and the per-domain files (check that every `Record`/`Create*Params`/`Update*Params` class defined in a domain file appears in `__init__.py`'s `__all__`).
- No circular imports introduced by the new package.

**Blocker-level deviations:**

- If `Create*Params` fields default to `= None`, the ExceptionsWP (WP04) integration tests will silently pass while production payloads drop fields. This is a BLOCKER even if all unit tests pass.
- If the state registry does not resolve `domain="counter"`, `api.get_state("counter.xxx")` will fall back to `BaseState` — an observable framework regression, not a feature gap. BLOCKER.
