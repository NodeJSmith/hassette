## Objectives & Success Criteria

After this WP:

- A new file `src/hassette/test_utils/sync_facade.py` exists containing the `_RecordingSyncFacade` plain class.
- `_RecordingSyncFacade` provides **sync** versions of all 6 write methods (`turn_on`, `turn_off`, `toggle_service`, `call_service`, `set_state`, `fire_event`) that append to the parent `RecordingApi`'s `calls` list, producing identical `ApiCall.kwargs` shapes as the async side.
- `_RecordingSyncFacade` provides **sync** versions of the 6 read methods (`get_state`, `get_states`, `get_entity`, `get_entity_or_none`, `entity_exists`, `get_state_or_none`) that delegate to the parent's state-proxy helpers (`_get_raw_state`, `_convert_state`, `_state_proxy.states`).
- `_RecordingSyncFacade.__getattr__` raises `NotImplementedError` with tailored messages (same scheme as `RecordingApi.__getattr__` from WP01) — `get_state_value`/`get_state_value_typed`/`get_attribute` redirect to `get_state`, everything else gets the "Seed state" message.
- `RecordingApi.__init__` changes: `self.sync: _RecordingSyncFacade = _RecordingSyncFacade(self)` (module-level import at top of `recording_api.py`; no lazy imports).
- The existing `test_sync_attribute_is_mock` test is replaced with tests that verify the new facade behavior.
- A new test file `tests/unit/test_recording_sync_facade.py` exercises each write and read method individually, verifying the sync path produces the same `ApiCall` shapes as the async path.
- A new test file `tests/unit/test_recording_sync_facade_drift.py` asserts that the public write-method name set of `_RecordingSyncFacade` covers every public method of `ApiSyncFacade`, catching future drift.
- `uv run pytest tests/unit/test_recording_api.py tests/unit/test_recording_sync_facade.py tests/unit/test_recording_sync_facade_drift.py -v -n 2` passes.

## Test Strategy

### New file: `tests/unit/test_recording_sync_facade.py`

Tests exercise each `_RecordingSyncFacade` method and verify it produces the same `ApiCall` shape as the corresponding async method:

- `test_sync_turn_on_records_with_correct_shape` — call `api.sync.turn_on("light.kitchen")`, assert `ApiCall(method="turn_on", args=("light.kitchen",), kwargs={"entity_id": "light.kitchen", "domain": "homeassistant"})` in `api.calls`.
- `test_sync_turn_on_coerces_strenum` — pass a `StrEnum` and assert the recorded kwargs value is a plain `str`.
- `test_sync_turn_on_passes_extra_data` — `api.sync.turn_on("light.kitchen", brightness=200)`, assert `brightness=200` is in kwargs.
- `test_sync_turn_off_records_with_correct_shape` — same pattern.
- `test_sync_toggle_service_records_with_correct_shape` — same pattern.
- `test_sync_call_service_records_with_correct_shape` — call `api.sync.call_service("light", "turn_on", target={"entity_id": "light.kitchen"})`, assert `method="call_service"` with the full kwargs dict.
- `test_sync_call_service_returns_service_response_when_requested` — `return_response=True` returns a `ServiceResponse` with null context.
- `test_sync_set_state_records_and_returns_empty_dict` — assert return value is `{}` and call is recorded.
- `test_sync_fire_event_records_and_returns_empty_dict` — same pattern.
- `test_sync_and_async_share_calls_list` — call `api.turn_on(...)` (async) and `api.sync.turn_on(...)` (sync); assert `len(api.calls) == 2` and both entries have `method="turn_on"`.
- `test_sync_get_state_delegates_to_state_proxy` — seed state via the test harness, call `api.sync.get_state(...)`, assert the typed state is returned correctly.
- `test_sync_get_states_returns_all_seeded_entities` — seed multiple entities, call `api.sync.get_states()`, assert all are present.
- `test_sync_get_entity_with_base_state_model` — seed state, call `api.sync.get_entity(entity_id)`, assert typed state.
- `test_sync_get_entity_with_base_entity_model` — seed state, call `api.sync.get_entity(entity_id, SomeBaseEntityModel)`, assert `model.model_validate` was used.
- `test_sync_get_entity_or_none_returns_none_for_unseeded` — assert `None` is returned, not an exception.
- `test_sync_entity_exists_returns_bool` — seed one entity, assert True for it and False for an unseeded one.
- `test_sync_get_state_or_none_returns_none_for_unseeded` — same pattern.
- `test_sync_getattr_raises_notimplementederror_with_tailored_message_for_get_state_value` — assert the error message contains "Call `harness.api_recorder.sync.get_state(entity_id)`".
- `test_sync_getattr_raises_notimplementederror_with_default_message_for_other_methods` — assert message contains "Seed state via AppTestHarness.set_state()".
- `test_sync_getattr_private_attributes_raise_attribute_error` — accessing `api.sync._something` raises `AttributeError`, not `NotImplementedError`.
- `test_recording_api_sync_is_recording_sync_facade` — `isinstance(api.sync, _RecordingSyncFacade)` is True.

### New file: `tests/unit/test_recording_sync_facade_drift.py`

One test:

- `test_recording_sync_facade_covers_api_sync_facade` — introspect `ApiSyncFacade` and `_RecordingSyncFacade`, assert that every public method (non-underscore, defined in `cls.__dict__`) on `ApiSyncFacade` is also defined on `_RecordingSyncFacade`. Use `inspect.getmembers(cls, predicate=inspect.isfunction)` filtered to non-underscore names and `cls.__dict__`. Error message on failure lists the missing methods and directs the reader to update `sync_facade.py`.

**Note**: the drift test only verifies name-set coverage, not signature compatibility. Per-method signature correctness is verified by the per-method tests above.

## Review Guidance

- **Blocker**: Any method in `_RecordingSyncFacade` producing a different `ApiCall.kwargs` shape than the async counterpart on `RecordingApi`. Cross-reference against the canonical shape table in the design.
- **Blocker**: `RecordingApi.sync` remaining a `Mock()` — the whole point of this WP is to replace it.
- **Blocker**: Lazy import of `_RecordingSyncFacade` inside `RecordingApi.__init__`. The import must be at module level.
- **Blocker**: `_RecordingSyncFacade` extending `Resource` — the design explicitly specifies a plain class.
- **Blocker**: Any write method not storing `entity_id` as a plain `str` in `kwargs` for the three convenience methods.
- **Warning**: Duplicated state-proxy traversal logic in the facade instead of delegating via `self._parent._get_raw_state` etc.
- **Warning**: Missing drift test — the CI guard is load-bearing for preventing future regressions.
- Confirm that `get_entity_or_none` and `get_state_or_none` call `self.get_entity`/`self.get_state` (not `self._parent.get_entity`/`self._parent.get_state`) — they should use the sync facade's own methods so the call recording path is correct.
- Verify that `test_recording_sync_facade_drift.py` actually fails when you manually rename a method in `_RecordingSyncFacade` (add a known-failing mutation test locally during review if uncertain).
