## Objectives & Success Criteria

After this WP:

- `TaskBucket` has a new public method `pending_tasks() -> list[asyncio.Task[Any]]` that returns a snapshot list of non-completed tasks in the bucket.
- `BusService` has a new public property `is_dispatch_idle -> bool` that returns `True` when no dispatch tasks are in flight (delegates to `self._dispatch_idle_event.is_set()`).
- `BusService` has a new public property `dispatch_pending_count -> int` that returns the number of currently-in-flight dispatch tasks (delegates to `self._dispatch_pending`).
- These additions unblock WP04's iterative drain, which requires public accessors to avoid private-attribute coupling.
- Existing tests for `TaskBucket` and `BusService` continue to pass; new tests cover the three additions.
- `uv run pytest tests/ -k "task_bucket or bus_service" -v -n 2` passes.

## Subtasks

1. In `src/hassette/task_bucket/task_bucket.py`, add a public method `pending_tasks` to the `TaskBucket` class:
   ```python
   def pending_tasks(self) -> list[asyncio.Task[Any]]:
       """Return a snapshot list of non-completed tasks in this bucket."""
       return [t for t in list(self._tasks) if not t.done()]
   ```
   Place it near `cancel_all_sync` (around line 216) so related task-set operations are grouped.
2. In `src/hassette/core/bus_service.py`, add a public property `is_dispatch_idle` to the `BusService` class:
   ```python
   @property
   def is_dispatch_idle(self) -> bool:
       """Return True when no dispatch tasks are in flight."""
       return self._dispatch_idle_event.is_set()
   ```
   Place it near `await_dispatch_idle` (around line 413) so dispatch-state accessors are grouped.
3. Add a public property `dispatch_pending_count` to `BusService` alongside `is_dispatch_idle`:
   ```python
   @property
   def dispatch_pending_count(self) -> int:
       """Return the number of currently-in-flight dispatch tasks."""
       return self._dispatch_pending
   ```
4. Add docstrings explaining that these are the recommended public accessors for drain helpers and test infrastructure.

## Test Strategy

### Updates to `tests/integration/test_bus.py` (or appropriate existing BusService test file)

### Updates to `tests/` for TaskBucket

- `test_pending_tasks_returns_empty_for_fresh_bucket` — create a `TaskBucket`, assert `pending_tasks() == []`.
- `test_pending_tasks_returns_active_tasks` — spawn 2 tasks that never complete (e.g., `asyncio.sleep(1)` inside a gated future), assert `len(pending_tasks()) == 2`.
- `test_pending_tasks_excludes_completed_tasks` — spawn a task that completes immediately, await its completion via `asyncio.sleep(0)`, assert it's not in `pending_tasks()`.
- `test_pending_tasks_excludes_cancelled_tasks` — spawn a task, cancel it, wait for the cancellation to propagate, assert it's not in `pending_tasks()`.

Place these in `tests/unit/test_task_bucket.py` (create the file if it doesn't exist; otherwise add to the existing location found via `grep "class TaskBucket" tests/`).

## Review Guidance

- **Blocker**: `pending_tasks()` returning a reference to the internal `_tasks` `WeakSet` rather than a fresh list. The returned object must be iterable after mutations to `_tasks` without raising `RuntimeError`.
- **Blocker**: `is_dispatch_idle` or `dispatch_pending_count` reading from a field other than the one documented (`_dispatch_idle_event` and `_dispatch_pending` respectively). These properties are trivial wrappers; any additional logic is out of scope.
- **Warning**: Missing docstrings — the new accessors are the recommended public surface and should be documented clearly so future drain maintainers know to use them instead of private fields.
- **Warning**: Tests missing the snapshot-semantics check for `pending_tasks()` — this is the only non-trivial behavior (list copy to avoid iteration-during-mutation issues).
- Verify that adding these methods does NOT change existing task_bucket or bus_service behavior in any way — this WP is purely additive.
