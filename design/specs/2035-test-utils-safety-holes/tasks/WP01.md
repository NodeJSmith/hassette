## Objectives & Success Criteria

After this WP:

- `ApiCall` is defined in a new file `src/hassette/test_utils/api_call.py` (extracted from `recording_api.py`) to break the circular import that would otherwise exist when `sync_facade.py` is added in WP02. The existing `hassette.test_utils.ApiCall` re-export in `__init__.py` continues to work.
- `RecordingApi.turn_on`, `turn_off`, and `toggle_service` record directly under their own method names via `ApiCall(method="turn_on"|"turn_off"|"toggle_service", ...)` — not via delegation to `call_service`.
- All three convenience methods accept `entity_id: str | StrEnum` and coerce via `str(entity_id)` before recording, producing uniform plain-string values in `ApiCall.kwargs["entity_id"]`.
- `RecordingApi.__getattr__` emits a tailored `NotImplementedError` message for `get_state_value`, `get_state_value_typed`, and `get_attribute` pointing users at `await self.api.get_state(entity_id)`; other missing methods get the existing "seed state" guidance.
- Existing `test_recording_api.py` assertions that matched the old `call_service` shape for `turn_on`/`turn_off`/`toggle_service` calls are updated to match the new `turn_on`/`turn_off`/`toggle_service` shape.
- Existing `test_app_test_harness.py` assertion at line 219 (`assert_called("call_service", service="turn_on", ...)`) is updated to match the new shape.
- `uv run pytest tests/unit/test_recording_api.py tests/unit/test_public_api_surface.py -v -n 2` passes.

## Subtasks

1. Create `src/hassette/test_utils/api_call.py` containing the `ApiCall` dataclass. Copy the class verbatim from `recording_api.py` along with its existing docstring.
2. Update `src/hassette/test_utils/recording_api.py` to import `ApiCall` from `.api_call` at the top of the file; remove the `ApiCall` class definition from `recording_api.py`.
3. Verify `src/hassette/test_utils/__init__.py` still re-exports `ApiCall` correctly (it currently imports from `.recording_api`; change to `.api_call` or add a compatibility re-export from `.recording_api` that forwards to `.api_call`).
4. In `src/hassette/test_utils/recording_api.py`, change `RecordingApi.turn_on` from delegating to `call_service` to recording directly:
   ```python
   async def turn_on(self, entity_id: str | StrEnum, domain: str = "homeassistant", **data) -> None:
       entity_id = str(entity_id)
       self.calls.append(
           ApiCall(
               method="turn_on",
               args=(entity_id,),
               kwargs={"entity_id": entity_id, "domain": domain, **data},
           )
       )
   ```
5. Change `RecordingApi.turn_off` similarly — broaden the type annotation to `str | StrEnum`, add `str(entity_id)` coercion, record as `method="turn_off"` with `kwargs={"entity_id": entity_id, "domain": domain}`.
6. Change `RecordingApi.toggle_service` similarly — `str | StrEnum`, coercion, record as `method="toggle_service"`.
7. Add `_STATE_CONVERSION_METHODS: ClassVar[frozenset[str]] = frozenset({"get_state_value", "get_state_value_typed", "get_attribute"})` as a class attribute on `RecordingApi`.
8. Update `RecordingApi.__getattr__` to check if `name in self._STATE_CONVERSION_METHODS` and raise `NotImplementedError` with the tailored message: `f"RecordingApi.{name} is not implemented. Call `await self.api.get_state(entity_id)` and read the returned state directly."`. For other names, keep the existing message text.
9. In `tests/unit/test_recording_api.py`, update assertions on lines 272 and 279 from `assert_called("call_service", service="turn_on", ...)` to `assert_called("turn_on", ...)` with the new kwargs shape (no `service=` key, flat `entity_id` and `domain` and `**data`).
10. In `tests/integration/test_app_test_harness.py` line 219, update the assertion to the new shape.
11. Update the `recording_api.py` module docstring example (line 13) and `RecordingApi` class docstring example if they show the old `call_service` assertion form.
12. Update the `app_harness.py` module docstring example (around line 13) similarly.

## Test Strategy

All tests live in `tests/unit/test_recording_api.py` (existing file, updated):

- **Updated**: `test_turn_on_records_call` (or equivalent — existing tests that call `turn_on` and assert on the old `call_service` shape) must assert on the new `method="turn_on"` shape.
- **New**: `test_turn_on_accepts_strenum` — pass a `StrEnum` to `turn_on` and assert that `ApiCall.kwargs["entity_id"]` is a plain `str`.
- **New**: `test_turn_off_records_call` — call `turn_off("light.kitchen")`, assert `ApiCall(method="turn_off", args=("light.kitchen",), kwargs={"entity_id": "light.kitchen", "domain": "homeassistant"})`.
- **New**: `test_turn_off_accepts_strenum` — same StrEnum coercion check.
- **New**: `test_toggle_service_records_call` — same pattern.
- **New**: `test_toggle_service_accepts_strenum` — same StrEnum coercion check.
- **New**: `test_call_service_still_records_as_call_service` — call `call_service("light", "turn_on", target={...})` directly; assert it still records as `method="call_service"` unchanged.
- **New**: `test_getattr_tailored_message_for_state_conversion` — access `api.get_state_value` (or call-and-expect) and assert the `NotImplementedError` message contains "Call `await self.api.get_state(entity_id)`".
- **New**: `test_getattr_default_message_for_other_methods` — access a non-state-conversion unimplemented method and assert the message contains "Seed state via AppTestHarness.set_state()".
- **New**: `test_apicall_import_from_api_call_module` — import `ApiCall` from `hassette.test_utils.api_call` directly and confirm it's the same class as `hassette.test_utils.ApiCall`.

## Review Guidance

- **Blocker**: Any of the three convenience methods still delegating to `call_service` internally. The design explicitly requires direct recording.
- **Blocker**: `ApiCall` remaining in `recording_api.py` — the extraction is load-bearing for WP02's circular import fix.
- **Blocker**: `entity_id` stored in `ApiCall.kwargs` as a `StrEnum` instance instead of a plain `str` for any of the three convenience methods.
- **Warning**: `hassette.test_utils.ApiCall` re-export breaking — verify `tests/unit/test_public_api_surface.py` still passes and that external callers who `from hassette.test_utils import ApiCall` get the same class.
- **Warning**: The `__getattr__` tailored message missing any of the three state-conversion method names.
- Ensure `entity_id = str(entity_id)` is the first line of each of the three convenience method bodies (before the `self.calls.append(...)` call).
- Verify the kwargs dict keys match the design's canonical table exactly — no extra keys, no renamed keys.
