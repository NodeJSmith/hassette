## Objectives & Success Criteria

Users can register error handlers on the Scheduler: app-level via `scheduler.on_error(handler)` and per-job via `on_error=` on any scheduling method. After this WP, `error_handler` is on `ScheduledJob` with `field(default=None, compare=False)`, added to `matches()` and `diff_fields()`, and `on_error=` is available on `schedule()` and all convenience methods. Unit tests verify field propagation, deduplication behavior, and precedence logic.

## Subtasks

1. Add `error_handler: SchedulerErrorHandlerType | None = field(default=None, compare=False)` to `ScheduledJob` in `src/hassette/scheduler/classes.py`. Use `compare=False` to exclude from `@dataclass(order=True)` heap ordering.
2. Add `error_handler` to `ScheduledJob.matches()` using identity comparison (`self.error_handler is other.error_handler`) on the raw callable.
3. Add `error_handler` to `ScheduledJob.diff_fields()`.
4. Add `on_error: SchedulerErrorHandlerType | None = None` parameter to `Scheduler.schedule()` in `src/hassette/scheduler/scheduler.py`. Pass through to `ScheduledJob` construction.
5. Add `on_error: SchedulerErrorHandlerType | None = None` to all convenience methods: `run_in`, `run_once`, `run_every`, `run_daily`, `run_cron`, `run_hourly`, `run_minutely`. Pass through to `schedule()`.
6. Add `on_error(handler: SchedulerErrorHandlerType)` method to `Scheduler` — stores `_error_handler` on the instance. Reset in `Scheduler.on_initialize()`.

## Test Strategy

- `tests/unit/scheduler/test_scheduler_error_handler.py`: `test_on_error_stores_handler`, `test_on_error_reset_on_initialize`, `test_per_job_on_error_stored`, `test_job_error_handler_default_none`, `test_on_error_replaces_previous`, `test_convenience_methods_pass_on_error`.
- `tests/unit/scheduler/test_scheduled_job.py` (extend existing): `test_matches_with_same_error_handler`, `test_matches_with_different_error_handler`, `test_diff_fields_includes_error_handler`, `test_error_handler_compare_false_no_heap_corruption`.

## Review Guidance

- Verify `compare=False` on `error_handler` field — without it, `Callable | None` in heap ordering causes `TypeError`.
- Verify `matches()` uses `is` (identity), not `==` (value) for `error_handler`.
- Verify `diff_fields()` includes `error_handler` so `if_exists="update"` catches handler changes.
- Verify all 7 convenience methods pass `on_error` through to `schedule()`.
- Verify `_error_handler` reset in `on_initialize()`, not `__init__()`.
