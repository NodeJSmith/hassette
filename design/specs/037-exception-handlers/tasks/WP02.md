## Objectives & Success Criteria

Users can register error handlers on the Bus: app-level via `bus.on_error(handler)` and per-listener via `on_error=` on any registration method. After this WP, `on_error` is in the `Options` TypedDict, on `Bus.on()`'s parameter list, threaded through `Listener.create()`, and stored on the `Listener` dataclass. The raw callable is stored (not normalized) for reliable `matches()` identity comparison. Unit tests verify field propagation, Options flow, and precedence logic.

## Subtasks

1. Add `on_error: BusErrorHandlerType | None` to the `Options` TypedDict in `src/hassette/bus/bus.py` (line ~109).
2. Add `on_error: BusErrorHandlerType | None = None` to `Bus.on()`'s explicit keyword-only parameter list (line ~212). Pass it through to `Listener.create()`.
3. Add `error_handler: BusErrorHandlerType | None = None` field to the `Listener` dataclass body in `src/hassette/bus/listeners.py` (required for `slots=True`). Add the parameter to `Listener.create()`'s signature and thread it through the `cls(...)` constructor call at line ~267.
4. Add `on_error(handler: BusErrorHandlerType)` method to `Bus` — stores `_error_handler` on the instance. Reset `_error_handler = None` in `Bus.on_initialize()`.
5. Explicitly verify all 14 convenience wrappers pass `on_error` through `**opts` → `_subscribe()` → `on()` without individual changes. Write a parameterized test that calls each wrapper with `on_error=mock_handler` and asserts the resulting `Listener.error_handler` is set. Wrappers to verify: `on_state_change`, `on_attribute_change`, `on_call_service`, `on_homeassistant_restart`, `on_homeassistant_close`, `on_websocket_connected`, `on_websocket_disconnected`, `on_app_running`, `on_app_stopped`, `on_service_registered`, `on_component_loaded`, `on_automation_reloaded`, `on_event`, `on_any_event`.

## Test Strategy

- `tests/unit/bus/test_listeners.py` (extend existing): `test_listener_create_with_error_handler`, `test_listener_create_without_error_handler_defaults_none`.

## Review Guidance

- Verify `on_error` appears in three places: `Options` TypedDict, `Bus.on()` parameter list, `Listener.create()` parameter list. Missing any one causes TypeError or silent discard.
- Verify the raw callable is stored on `Listener.error_handler`, not a `make_async_adapter` wrapper — normalization happens at invocation time (WP04).
- Verify `_error_handler` is reset in `on_initialize()`, not `__init__()` — hot-reload safety.
- Verify existing tests still pass — the new parameter has a default so no call sites should break.
