## Objectives & Success Criteria

Error handlers are invoked when callbacks fail. After this WP: `_execute_handler` and `_execute_job` capture `ExecutionResult`, check for error handlers (per-registration first, then app-level from command field), spawn a separate task to invoke the handler with a context object, and guard against handler failures. `BusService._dispatch()` resolves `Bus._error_handler` at dispatch time and carries it on `InvokeHandler.app_level_error_handler`. Same for `SchedulerService`. The `_error_handler_failures` counter is incremented on handler failure. Unit tests verify the full invocation path.

## Subtasks

1. Add `app_level_error_handler: BusErrorHandlerType | None = None` field to `InvokeHandler` in `src/hassette/core/commands.py`. Add equivalent `app_level_error_handler: SchedulerErrorHandlerType | None = None` to `ExecuteJob`.
2. In `BusService._dispatch()` (`src/hassette/core/bus_service.py`), when building the `InvokeHandler` command, resolve `self._bus._error_handler` and set it on `cmd.app_level_error_handler`.
3. In `SchedulerService` (`src/hassette/core/scheduler_service.py`), when building the `ExecuteJob` command, resolve `self._scheduler._error_handler` and set it on `cmd.app_level_error_handler`.
4. In `CommandExecutor._execute_handler()` (`src/hassette/core/command_executor.py`), change to `result = await self._execute(...)`. After `_execute()` returns, if `result.is_error and result.exc is not None`, determine the handler: `cmd.listener.error_handler` (per-registration) or `cmd.app_level_error_handler` (fallback). If a handler exists, spawn a separate task via `self._task_bucket.spawn()` to invoke it.
5. In the spawned error handler task: build `BusErrorContext` from `cmd` and `result` (traceback via `"".join(traceback.format_exception(result.exc))`), normalize the handler via `make_async_adapter`, wrap in `asyncio.timeout(self._config.error_handler_timeout_seconds)` and try/except. On handler failure, log at ERROR and increment `_error_handler_failures` counter. On timeout, log WARNING.
6. Add `_error_handler_failures: int = 0` counter to `CommandExecutor`. Expose via the existing counters API pattern.
7. Implement same pattern for `_execute_job()` with `SchedulerErrorContext`.

## Review Guidance

- Verify error handler invocation is in `_execute_handler`/`_execute_job`, NOT inside `_execute()` — abstraction boundary preservation.
- Verify the error handler runs in a separate spawned task, not inline in the dispatch task — dispatch slot starvation prevention.
- Verify `traceback` on context object is built from `traceback.format_exception(result.exc)`, not from `result.error_traceback` (which may be `None` for DependencyError).
- Verify `CancelledError` re-raise contract in `_execute()` is untouched.
- Verify `make_async_adapter` normalization happens at invocation time in the spawned task, not at registration time.
- Verify `_error_handler_failures` counter exists and is incremented on both handler exceptions and timeouts.
