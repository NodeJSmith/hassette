## Objectives & Success Criteria

`tests/smoke/conftest.py` exists and provides:

1. `ha_container` — a `scope="session"` pytest fixture that starts the Docker Compose HA service, waits for it to be healthy, yields the URL `http://localhost:18123`, and tears it down after the session. If Docker or HA fails to start within 60 seconds, the fixture calls `pytest.fail()` with the error.
2. `startup_context(config, timeout=30)` — an async context manager that runs `Hassette.run_forever()` as a background `asyncio.Task`, polls `hassette._session_id` until it is a positive integer (indicating `_create_session()` has run), yields the running `Hassette` instance, then sets `shutdown_event` and awaits the task.
3. `make_smoke_config(ha_url, tmp_path)` — a helper that returns a `HassetteConfig` instance pointed at the smoke HA container, with `data_dir` and `app_dir` set to `tmp_path` subdirectories.

**Acceptance check**: `pytest tests/smoke/conftest.py --collect-only -q` completes without import errors. The `ha_container` fixture appears in `pytest --fixtures --co -q` output.

## Subtasks

1. Write `tests/smoke/conftest.py` with the following structure:

   ```python
   import asyncio
   import subprocess
   import time
   from collections.abc import AsyncIterator
   from contextlib import asynccontextmanager
   from pathlib import Path

   import httpx
   import pytest

   from hassette import Hassette
   from hassette.config.config import HassetteConfig

   COMPOSE_FILE = Path(__file__).parent / "docker-compose.yml"
   HA_URL = "http://localhost:18123"
   HA_TOKEN = "hassette-smoke-test-token"
   STARTUP_TIMEOUT = 60
   ```

2. Implement `ha_container` as a `scope="session"` sync fixture (sync, not async — subprocess calls are synchronous). Use `subprocess.run(["docker", "compose", "-f", str(COMPOSE_FILE), "up", "-d", "homeassistant"], check=True)` to start, and `subprocess.run(["docker", "compose", "-f", str(COMPOSE_FILE), "down"], check=False)` in the teardown (in a `finally` block via `yield`). Poll with `httpx.get()` in a loop with `time.sleep(2)` between attempts.

3. Implement `startup_context` as an `@asynccontextmanager` async generator. Background task via `asyncio.create_task(hassette.run_forever())`. Poll `hassette._session_id` in a `while` loop with `await asyncio.sleep(0.1)`. Raise `TimeoutError` if the deadline passes. In the `finally` block: `hassette.shutdown_event.set()`, then `await task` (catch `asyncio.CancelledError` if the task was already cancelled).

4. Implement `make_smoke_config(ha_url: str, tmp_path: Path) -> HassetteConfig` that instantiates `HassetteConfig` with:
   - `_env_file=None` (avoid loading `.env` from the test suite)
   - `base_url=ha_url`
   - `token=HA_TOKEN`
   - `data_dir=tmp_path / "data"`
   - `app_dir=tmp_path / "apps"` (create the directory: `(tmp_path / "apps").mkdir(exist_ok=True)`)
   - `run_web_api=False` (no web server needed during smoke tests)
   - `autodetect_apps=False`
   - `startup_timeout_seconds=30`

   Use `HassetteConfig.model_construct()` or direct instantiation with the keyword arguments. Check how the existing `TestConfig` in `tests/conftest.py` is constructed to avoid CLI arg parsing side effects — specifically set `cli_parse_args=False` or equivalent.

5. Add a `pytest.ini.options` `filterwarnings` override in the smoke `conftest.py` to avoid httpx/Docker deprecation warnings causing test failures (since `pyproject.toml` has `filterwarnings = ["error"]` globally):
   ```python
   # Suppress non-Hassette warnings in smoke tests — Docker and httpx emit deprecation notices
   import warnings
   warnings.filterwarnings("default", category=DeprecationWarning, module="httpx")
   warnings.filterwarnings("default", category=DeprecationWarning, module="docker")
   ```

   Alternatively, use a `pytestmark` in test files. Evaluate during implementation which is cleaner.

## Test Strategy

Write a minimal test in `tests/smoke/test_conftest_smoke.py` (to be deleted or merged into `test_startup.py` in WP03):

```python
import pytest

pytestmark = pytest.mark.smoke

def test_ha_container_url(ha_container):
    """ha_container fixture provides a reachable URL."""
    import httpx
    r = httpx.get(f"{ha_container}/api/", headers={"Authorization": f"Bearer hassette-smoke-test-token"})
    assert r.status_code == 200
```

This test verifies the fixture works before building the async startup tests in WP03.

## Review Guidance

- **`ha_container` scope**: Must be `scope="session"` — HA starts exactly once per `pytest -m smoke` run. If it's function-scoped, HA would restart per test, which is extremely slow and likely broken.
- **`startup_context` shutdown**: The `finally` block must call `hassette.shutdown_event.set()` and `await task`. If the task raises, the exception should propagate to the test (don't swallow it).
- **`make_smoke_config` CLI parsing**: `HassetteConfig` normally parses CLI args in `__init__`. The smoke config must disable this (same pattern as `TestConfig` in `tests/conftest.py` which sets `cli_parse_args=False`).
- **`data_dir` and `app_dir`**: Must point to `tmp_path` subdirectories so each test gets a clean SQLite DB. `tmp_path` is function-scoped in pytest, so each test calling `make_smoke_config(ha_container, tmp_path)` gets isolation.
- **No `src/` changes**: This WP touches only `tests/smoke/`. If any changes to `src/` seem necessary, flag as a deviation.
