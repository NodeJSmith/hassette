## Objectives & Success Criteria

The following files exist and are committed to the repo:

- `tests/fixtures/ha-config/configuration.yaml` — contains `demo:` integration
- `tests/fixtures/ha-config/.storage/auth` — valid HA v7 auth JSON with a pre-seeded admin user and a long-lived token whose bcrypt hash corresponds to `hassette-smoke-test-token`
- `tests/fixtures/ha-config/.storage/onboarding` — `done` list suppresses the HA setup wizard
- `tests/fixtures/ha-config/.storage/core.config` — minimal location/timezone entry
- `tests/fixtures/ha-config/.storage/core.config_entries` — empty entries list
- `tests/smoke/docker-compose.yml` — starts `homeassistant/home-assistant:2025.3` on port `18123`, mounts the fixture config volume, and has a healthcheck that validates the pre-seeded token
- `tests/smoke/generate_ha_fixtures.py` — runnable script that regenerates the `.storage/auth` file from `hassette-smoke-test-token`

**Acceptance check**: Run `docker compose -f tests/smoke/docker-compose.yml up -d` and `curl -sf http://localhost:18123/api/ -H "Authorization: Bearer hassette-smoke-test-token"` returns HTTP 200. This confirms HA booted with the pre-seeded config and auth is working.

## Test Strategy

This WP creates fixture files, not test code. Verification is manual:

1. `docker compose -f tests/smoke/docker-compose.yml up -d`
2. `curl -sf http://localhost:18123/api/ -H "Authorization: Bearer hassette-smoke-test-token"` → must return HTTP 200 with `{"message":"API running."}`
3. `docker compose -f tests/smoke/docker-compose.yml down`

Document the manual verification result in the Activity Log.

## Review Guidance

- **Auth fixture correctness**: The `.storage/auth` JSON must have `"version": 7` and include the `access_tokens` array with a bcrypt hash of `hassette-smoke-test-token`. If the healthcheck passes, the fixture is correct.
- **Deterministic UUIDs**: The hardcoded UUIDs in `generate_ha_fixtures.py` must match those in the committed `.storage/auth` file — otherwise `generate_ha_fixtures.py` would overwrite with a different user and the token would break.
- **Image version pinned**: `docker-compose.yml` must use `homeassistant/home-assistant:2025.3`, not `stable`. Pinning prevents uncontrolled fixture drift.
- **Port collision**: Port `18123` (not `8123`) must be used to avoid conflict with a local homelab HA.
- **Non-goal check**: No Hassette code should be modified in this WP.
