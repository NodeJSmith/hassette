## Objectives & Success Criteria

`tests/smoke/test_startup.py` exists with 5 passing tests, all marked `pytest.mark.smoke`:

1. `test_startup_completes` — passes when `hassette._session_id` is a positive integer after `startup_context` exits, meaning `run_forever()` completed startup without raising
2. `test_session_created_in_db` — a row exists in the `sessions` table with `status='running'` for the current session ID
3. `test_demo_entities_visible` — `light.kitchen_lights` and `binary_sensor.movement_backyard` are present in the states returned by `hassette.api.get_states()`
4. `test_bus_handler_fires_on_state_change` — a `bus.on_state_change` handler registered for `light.kitchen_lights` receives at least one event after calling `api.call_service("light", "toggle", {"entity_id": "light.kitchen_lights"})`
5. `test_no_sentinel_records_dropped` — no log records matching `"Dropping"` and `"invocation record"` appear in `caplog` during a 2-second observation window after startup

All 5 tests pass when run with `pytest -m smoke -v` against a running HA container. No existing tests break.

   ```python
   async def test_bus_handler_fires_on_state_change(ha_container, tmp_path):
       config = make_smoke_config(ha_container, tmp_path)
       received: list[object] = []

       async def capture_event(event: object) -> None:
           received.append(event)

       async with startup_context(config) as hassette:
           hassette.bus.on_state_change("light.kitchen_lights", handler=capture_event)
           await hassette.api.call_service("light", "toggle", {"entity_id": "light.kitchen_lights"})

   ```

7. Implement `test_no_sentinel_records_dropped(ha_container, tmp_path, caplog)`:
   ```python
   async def test_no_sentinel_records_dropped(ha_container, tmp_path, caplog):
       config = make_smoke_config(ha_container, tmp_path)
       with caplog.at_level(logging.WARNING, logger="hassette.core.command_executor"):
           async with startup_context(config):
               await asyncio.sleep(2.0)

       dropped = [
           r for r in caplog.records
           if "Dropping" in r.message and "invocation record" in r.message
       ]
       assert dropped == [], f"Sentinel records dropped during startup: {[r.message for r in dropped]}"
   ```

## Test Strategy

These ARE the tests for this WP. Run them as:

```bash
docker compose -f tests/smoke/docker-compose.yml up -d
pytest -m smoke -v --tb=short
docker compose -f tests/smoke/docker-compose.yml down
```

All 5 must pass. The sentinel test (`test_no_sentinel_records_dropped`) is the most important — a fail here means a startup race is still present.

## Review Guidance

- **`test_no_sentinel_records_dropped` is the regression gate**: If this test fails (sentinel records ARE being dropped), a startup race is still active. This is a blocker, not a warning.
- **Demo entity IDs**: If `light.kitchen_lights` or `binary_sensor.movement_backyard` don't exist in HA 2025.3's demo integration, the test must be updated with actual entity IDs. Verify by running `curl http://localhost:18123/api/states -H "Authorization: Bearer hassette-smoke-test-token"` and checking the output.
- **`asyncio_mode = "auto"`**: No `@pytest.mark.asyncio` decorator needed — all async tests run automatically under the global setting.
- **Each test uses its own `tmp_path`**: All 5 tests must be function-scoped (default) and use `tmp_path` for `data_dir`. Do NOT share a single Hassette instance across tests — each test creates its own via `startup_context`.
- **`ha_container` is session-scoped**: HA starts once. All 5 tests connect to the same container. Tests must not leave HA in a state that breaks subsequent tests (e.g., if `toggle` leaves the light off, the next test must not depend on it being on).
- **No `src/` changes**: Blocker if any.
