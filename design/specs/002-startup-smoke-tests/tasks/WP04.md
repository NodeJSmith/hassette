## Objectives & Success Criteria

1. `nox -s smoke` runs successfully locally (with HA container already up) and prints results for the 5 smoke tests
2. `pytest` (bare, no args) does NOT discover or run smoke tests â€” the `addopts` skip filter includes `not smoke`
3. `.github/workflows/smoke-tests.yml` exists and passes on a branch that touches `src/hassette/core/core.py` (manually triggered via `workflow_dispatch` is sufficient for initial verification)
4. `pytest --markers` lists `smoke: marks startup smoke tests...`

## Subtasks

1. Add `smoke` session to `noxfile.py` after the existing `e2e` session:

   ```python
   @nox.session(python=["3.13"])
   def smoke(session: "Session"):
       """Startup smoke tests against a real HA Docker container.

       Requires Docker and the HA container to be running:
           docker compose -f tests/smoke/docker-compose.yml up -d
       Or run via CI which starts the container automatically.
       """
       session.run(
           "uv",
           "run",
           "--active",
           "--reinstall-package",
           "hassette",
           "pytest",
           "-m",
           "smoke",
           "-v",
           "-n",
           "0",
           "--tb=short",
           external=True,
       )
   ```

   Note: `-n 0` (not `-n auto`) is required â€” the `ha_container` session fixture is not xdist-safe. No `-W error` flag â€” smoke tests may see deprecation warnings from external libraries.

2. Update `pyproject.toml` `[tool.pytest.ini_options]`:

   a. Add `smoke` to the `markers` list:
   ```toml
   "smoke: marks startup smoke tests requiring a running HA Docker container (run explicitly with 'nox -s smoke')",
   ```

   b. Update `addopts` to also exclude smoke â€” append only `and not smoke` to the existing value. The current value is `"-m 'not docker'"`, so the result should be:
   ```toml
   addopts = "-m 'not docker and not smoke'"
   ```

   Do NOT add `not e2e` â€” that is not part of this design's scope. The nox `tests` session passes its own `-m "not docker and not e2e"` separately.

3. Create `.github/workflows/smoke-tests.yml`:

   ```yaml
   name: Smoke Tests

   on:
     push:
       branches: [main]
     pull_request:
       branches: [main]
       paths:
         - "src/hassette/core/core.py"
         - "src/hassette/core/command_executor.py"
         - "src/hassette/core/websocket_service.py"
         - "src/hassette/core/database_service.py"
         - "src/hassette/core/bus_service.py"
         - "tests/smoke/**"
         - "tests/fixtures/ha-config/**"
         - ".github/workflows/smoke-tests.yml"
     workflow_dispatch:

   jobs:
     smoke:
       runs-on: ubuntu-latest

       steps:
         - name: Checkout code
           uses: actions/checkout@v4

         - name: Install uv
           uses: astral-sh/setup-uv@v5
           with:
             enable-cache: true
             python-version: "3.13"

         - run: uv sync

         - name: Start HA container
           run: docker compose -f tests/smoke/docker-compose.yml up -d homeassistant

         - name: Wait for HA to be ready
           run: |
             for i in $(seq 1 30); do
               curl -sf http://localhost:18123/api/ \
                 -H "Authorization: Bearer hassette-smoke-test-token" && break
               echo "Attempt $i/30 â€” waiting..."
               sleep 3
             done || (docker compose -f tests/smoke/docker-compose.yml logs homeassistant && exit 1)

         - name: Run smoke tests
           run: uv run nox -s smoke

         - name: Tear down HA container
           if: always()
           run: docker compose -f tests/smoke/docker-compose.yml down
   ```

4. Verify the change to `addopts` doesn't break the default `pytest` run: confirm `uv run pytest --collect-only -q` shows no tests from `tests/smoke/`.

5. Verify `nox -s tests` and `nox -s tests_with_coverage` still exclude smoke: the existing `-m 'not docker and not e2e'` filter won't match `smoke`, but since `addopts` now adds `not smoke`, bare `pytest` invocations (used by the nox sessions) will also skip smoke. Confirm by dry-run.

## Review Guidance

- **`addopts` update is critical**: Without `not smoke` in `addopts`, running bare `pytest` (or `nox -s tests`) would attempt to collect smoke tests and fail with a `docker not available` or fixture error. This is a blocker if omitted.
- **`-n 0` in nox session**: xdist with `ha_container` session fixture causes the fixture to be instantiated per worker, starting multiple HA containers. Must be `-n 0`.
- **No `-W error` in nox smoke session**: The `e2e` session also omits `-W error` â€” follow the same pattern. Smoke tests run against external containers where we can't control all warnings.
- **CI path filter**: The `paths:` filter on the PR trigger must include `src/hassette/core/bus_service.py` â€” BusService is where the startup race windows open. Missing it from the filter would mean BusService changes don't trigger smoke tests.
- **`uv sync` step**: The e2e workflow uses `uv sync`; the smoke workflow should match. Check that `uv sync` includes dev dependencies (it should by default).
