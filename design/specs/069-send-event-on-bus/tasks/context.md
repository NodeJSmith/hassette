# Context: Bus.emit — ergonomic inter-app broadcast

## Problem & Motivation

Broadcasting between apps currently requires manually constructing a `HassettePayload` + `Event` envelope and calling `App.send_event`. This puts the send API on `App` instead of `Bus`, breaking the on/emit symmetry users expect. The migration docs also incorrectly state inter-app communication is "not supported." This change adds `Bus.emit(topic, data)` as the ergonomic broadcast method, ships a `D.EventData[T]` DI accessor for typed receive, and removes `App.send_event` as a clean break.

## Visual Artifacts

None.

## Key Decisions

1. `emit` goes on `Bus` for on/emit symmetry (`self.bus.on(...)` / `self.bus.emit(...)`).
2. `emit(topic, data)` builds the envelope internally — user passes plain data, framework wraps in `HassettePayload` + `Event`.
3. No `emit_event(event)` raw escape hatch — YAGNI, no named caller.
4. Clean break: delete `App.send_event` and `AppSync.send_event_sync` with no deprecation.
5. TypeVar named `EmitDataT` (not `DataT`) to avoid collision with the covariant `DataT` in `events/base.py`.
6. `D.EventData[T]` ships in this PR to fulfill the type-safety promise end-to-end.
7. BusSyncFacade is auto-generated by codegen but requires `BUS_HEADER` update first.
8. Framework-level `exclude_self` deferred — handler-side guard pattern documented instead.

## Constraints & Anti-Patterns

- `Bus.emit` MUST delegate through `hassette.send_event(event)` — never write to the event stream directly. The guards (streams-closed, wire_services) live there.
- Do NOT add `emit_event(event)`.
- Do NOT implement any non-goals (targeted sends, schema enforcement, exclude_self parameter).
- Do NOT migrate internal `hassette.send_event` callers to Bus.
