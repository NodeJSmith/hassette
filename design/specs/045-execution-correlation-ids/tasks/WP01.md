## Objectives & Success Criteria

`HassettePayload.event_id` returns a UUID4 string (not an incrementing integer). `EventPayload` declares `origin` and `event_id` fields so `_build_record()` can access them on any payload type without Pyright errors. `HassettePayload.origin` returns `"HASSETTE"`. All existing tests pass with the type change. `service_watcher.py` logs event IDs correctly with `%s`.

## Subtasks

1. Add `origin: str = "UNKNOWN"` and `event_id: str = ""` fields to `EventPayload` in `src/hassette/events/base.py`. Convert `HassPayload.event_id` from a property to a regular field — remove the `@property` method and set `event_id` at construction from `context.id`. This may require a `__post_init__` or updating all `HassPayload(...)` construction sites to pass `event_id=context.id` explicitly. Both `HassPayload.__repr__` calls that reference `self.event_id` continue to work unchanged since it's still a `str` attribute.
2. Remove `HASSETTE_EVENT_ID_SEQ = itertools.count(1)` from `src/hassette/events/base.py`.
3. Change `HassettePayload.event_id` from `int` field with counter default to `str` field with `uuid.uuid4()` default. Add docstring noting the UUID is payload-instance-scoped (not event-scoped).
4. Add `origin: str = "HASSETTE"` to `HassettePayload`, overriding the base default.
5. Fix `src/hassette/core/service_watcher.py:451` — change `%d` to `%s` in the format string for `event.payload.event_id`.
6. Run `grep -rn '\.event_id'` across the full codebase to audit for any integer assumptions beyond `service_watcher.py`. Fix any found.
7. Run `uv run pyright` and fix any type errors from the changes.

## Test Strategy

**File: `tests/unit/events/test_event_payload_unification.py` (new)**

- `test_hassette_payload_event_id_is_uuid_string` — construct `HassettePayload`, assert `isinstance(payload.event_id, str)` and that it's a valid UUID4.
- `test_hassette_payload_event_id_is_unique` — construct two payloads, assert `event_id` values differ.
- `test_hassette_payload_origin_is_hassette` — assert `payload.origin == "HASSETTE"`.
- `test_hass_payload_event_id_matches_context_id` — construct `HassPayload` with a known `HassContext(id="abc", ...)`, assert `payload.event_id == "abc"`.
- `test_hass_payload_origin_is_literal` — assert `payload.origin` is `"LOCAL"` or `"REMOTE"` based on construction.
- `test_event_payload_base_has_origin_default` — assert `EventPayload("test", data=None).origin == "UNKNOWN"`.
