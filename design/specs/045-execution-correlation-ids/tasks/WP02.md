## Objectives & Success Criteria

`CURRENT_EXECUTION_ID` ContextVar is set for the duration of each handler/job execution and reset in `finally`. `HandlerInvocationRecord` carries `execution_id`, `trigger_context_id`, and `trigger_origin`. `JobExecutionRecord` carries `execution_id`. `_build_record()` populates all fields from the command. Unit tests verify ContextVar isolation, UUID uniqueness, and trigger field extraction.

## Subtasks

1. Add `CURRENT_EXECUTION_ID: ContextVar[str | None] = ContextVar("CURRENT_EXECUTION_ID", default=None)` to `src/hassette/context.py`.
2. Add `execution_id: str | None = None`, `trigger_context_id: str | None = None`, `trigger_origin: str | None = None` to `HandlerInvocationRecord` in `src/hassette/bus/invocation_record.py`.
3. Add `execution_id: str | None = None` to `JobExecutionRecord` in `src/hassette/scheduler/classes.py`.
4. Update `CommandExecutor._execute_handler()` in `src/hassette/core/command_executor.py`: generate `execution_id = str(uuid.uuid4())`, set `CURRENT_EXECUTION_ID` via token, wrap the entire method body (including `_execute()` call and error handler `task_bucket.spawn()`) in `try/finally` with `CURRENT_EXECUTION_ID.reset(token)`. Pass `execution_id` to `_execute()`.
5. Apply the same pattern to `CommandExecutor._execute_job()`.
6. Update `CommandExecutor._execute()` signature to accept `execution_id: str` parameter; pass it to `_build_record()`.
7. Update `CommandExecutor._build_record()` to accept `execution_id: str`. For `InvokeHandler`: extract `trigger_context_id = cmd.event.payload.event_id` and `trigger_origin = cmd.event.payload.origin`. For `ExecuteJob`: only `execution_id` (trigger fields stay `None`).
8. Run `uv run pyright` and fix any type errors.

## Test Strategy

**File: `tests/unit/core/test_command_executor_execution_id.py` (new)**

- `test_execution_id_set_during_handler_execution` — mock a handler that reads `CURRENT_EXECUTION_ID.get()` and stores the value. Execute via `_execute_handler()`. Assert the stored value is a valid UUID4 string matching `result.execution_id` (read from the enqueued record).
- `test_execution_id_none_after_handler_execution` — after `_execute_handler()` returns, assert `CURRENT_EXECUTION_ID.get()` is `None`.
- `test_execution_id_none_after_cancelled_execution` — mock a handler that raises `CancelledError`. Assert `CURRENT_EXECUTION_ID.get()` is `None` after the re-raise.
- `test_execution_id_unique_per_execution` — run two executions, assert their `execution_id` values differ.
- `test_handler_record_has_trigger_context_id` — execute a handler with a `HassPayload`-based event. Assert the enqueued record's `trigger_context_id` matches the payload's `event_id`.
- `test_handler_record_has_trigger_origin_local` — execute with `HassPayload(origin="LOCAL", ...)`. Assert record's `trigger_origin == "LOCAL"`.
- `test_handler_record_has_trigger_origin_hassette` — execute with a `HassettePayload`-based event. Assert record's `trigger_origin == "HASSETTE"`.
- `test_job_record_has_execution_id_no_trigger` — execute a job. Assert record has non-None `execution_id` and `None` for `trigger_context_id` and `trigger_origin`.
- `test_error_handler_inherits_execution_id` — mock a handler that raises, with an error handler that reads `CURRENT_EXECUTION_ID.get()`. Assert the error handler sees the same UUID as the main execution.

## Review Guidance

- The `try/finally` in `_execute_handler()` must wrap the entire body including the `task_bucket.spawn()` for error handlers — spawned error handler tasks inherit the ContextVar snapshot at spawn time, so the reset must happen after the spawn.
- Verify that `_execute()` does NOT set/reset the ContextVar — only the outer methods do.
- The `execution_id` parameter on `_execute()` and `_build_record()` is `str` (not `str | None`) because it's always generated before the call.
- `trigger_context_id` extraction uses `cmd.event.payload.event_id` which now exists on `EventPayload` (from WP01).
