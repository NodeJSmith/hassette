## Objectives & Success Criteria

OpenAPI schema and TypeScript types include the new fields. Handler invocations table shows "Trace ID", "Trigger", and "Origin" columns. Job executions table shows "Trace ID" column. Frontend compiles cleanly (`tsc --noEmit`). Factories updated. `database-telemetry.md` documents the new columns.

## Subtasks

1. Regenerate OpenAPI spec: `uv run python scripts/export_schemas.py`.
2. Regenerate TypeScript types: `cd frontend && npx openapi-typescript openapi.json -o src/api/generated-types.ts`.
3. Update `frontend/src/test/factories.ts`: add `execution_id: null`, `trigger_context_id: null`, `trigger_origin: null` to `createInvocation()` and `execution_id: null` to `createExecution()`. (Fields are `str | None` → nullable in OpenAPI → `string | null` in TypeScript.)
4. Update `frontend/src/components/app-detail/handler-invocations.tsx`:
   - Change `COL_COUNT` from `4` to `7`.
   - Add three `<th>` headers after Error: "Trace ID", "Trigger", "Origin".
   - Add three `<td>` cells per row: `execution_id` (monospace, truncated to 8 chars with full value in `title`), `trigger_context_id` (same styling), `trigger_origin` (plain text). Render `null`/`undefined` as "—".
5. Update `frontend/src/components/app-detail/job-executions.tsx`:
   - Change `COL_COUNT` from `4` to `5`.
   - Add one `<th>` header after Error: "Trace ID".
   - Add one `<td>` cell per row: `execution_id` (monospace, truncated). Render `null` as "—".
6. Install frontend deps if needed: `cd frontend && npm install`.
7. Run `cd frontend && npm run build` (or `tsc --noEmit`) to verify compilation.
8. Update `docs/pages/core-concepts/database-telemetry.md`: document `execution_id`, `trigger_context_id`, and `trigger_origin` columns with their semantics. Note that `trigger_context_id` for hassette events is payload-instance-scoped (not event-scoped).

## Review Guidance

- `COL_COUNT` must match the actual number of `<th>` headers — count them manually.
- Monospace styling for trace IDs and trigger context should use `ht-text-mono ht-text-xs` per existing conventions in the same files.
- Truncation: first 8 characters of UUID + "…" with full value in `title` attribute for hover copy. Consistent with the design's "Trace" domain concept from `design/context.md`.
- Verify the factory `satisfies` assertion compiles after adding the new fields.
- The documentation update should mention that `execution_id` is always populated for new rows and `null` for pre-migration rows.
