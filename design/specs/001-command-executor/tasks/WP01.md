## Objectives & Success Criteria

All new dataclasses and utility functions exist, are importable, fully type-annotated, and pass their unit tests. No other code is changed in this WP.

- `HandlerInvocationRecord` frozen dataclass exists at `src/hassette/bus/invocation_record.py`
- `JobExecutionRecord` in `src/hassette/scheduler/classes.py` is modernized: `frozen=True`, `execution_start_ts: float`, `job_id` is FK-ready (no `job_name`/`owner`/`started_at`), `session_id: int` added

- `capture_registration_source()` exists at `src/hassette/utils/source_capture.py`, returns `(source_location: str, registration_source: str | None)`, caches parsed ASTs per file, never raises
- All unit tests in `tests/unit/test_source_capture.py` pass
- All unit tests in `tests/unit/test_serialization.py` pass
- `uv run pytest tests/unit/test_source_capture.py tests/unit/test_serialization.py -v` passes

4. Add `db_id: int | None = None` field to `ScheduledJob` in `src/hassette/scheduler/classes.py`.

5. Create `src/hassette/core/registration.py` with `ListenerRegistration` and `ScheduledJobRegistration` frozen dataclasses matching the spec fields exactly.

7. Create `src/hassette/utils/serialization.py` with `safe_json_serialize(value: Any) -> str` ŌĆö `json.dumps(value, default=str, sort_keys=True)` with fallback to `'"<NON_SERIALIZABLE>"'` on `TypeError`/`ValueError`.

8. Create `src/hassette/utils/source_capture.py` with `capture_registration_source(*, frames_to_skip: int = 0) -> tuple[str, str | None]`:
   - Walk `inspect.stack()`, skip internal hassette frames (frames whose filename contains `hassette/bus/` or `hassette/scheduler/`), arrive at the app frame
   - `source_location = f"{filename}:{lineno}"`
   - Try `ast.parse()` the file (cache per filename in a module-level `dict[str, ast.Module]`), find the `Call` node at `lineno`, extract with `ast.get_source_segment()`
   - If anything fails (REPL, no source, parse error): `registration_source = None`, `source_location` still returned from frame
   - Return `(source_location, registration_source)`

9. Write unit tests for `safe_json_serialize` in `tests/unit/test_serialization.py`:

   - `test_serialize_with_default_str` ŌĆö Path, datetime, enum
   - `test_serialize_non_serializable` ŌĆö returns `'"<NON_SERIALIZABLE>"'` for unserializable object
   - `test_serialize_sort_keys` ŌĆö keys are sorted

10. Write unit tests for `capture_registration_source` in `tests/unit/test_source_capture.py`:
    - `test_capture_from_real_call` ŌĆö called from a real module file, returns non-None `registration_source`
    - `test_cache_reuse` ŌĆö calling twice from same file parses AST only once (monkeypatch `ast.parse` to assert call count)
    - `test_graceful_on_no_source` ŌĆö simulate REPL frame (no source file), returns `(source_location, None)` without raising

## Review Guidance

- `JobExecutionRecord` must be `frozen=True` ŌĆö mutable dataclass is a FAIL
- `Listener.db_id` and `ScheduledJob.db_id` must default to `None` ŌĆö never initialized in `__init__`, set later by executor
- `Listener` uses `slots=True` ŌĆö adding a field without listing it in the dataclass will silently fail or raise; verify the field is accessible
- `capture_registration_source()` must never raise ŌĆö any failure in AST parsing/stack walking must return `None` for `registration_source`, not propagate the exception
- `safe_json_serialize()` must never raise ŌĆö the `'"<NON_SERIALIZABLE>"'` fallback is a contract, not an option
- Registration dataclasses in `src/hassette/core/registration.py` must use float for timestamps (Instant epoch seconds), not `Instant` objects ŌĆö the DB layer works in floats
