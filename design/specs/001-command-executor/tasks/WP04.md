## Objectives & Success Criteria

`SchedulerService` is thinned: `run_job()` is ~5 lines, `_execution_log` is gone, and job registration upserts to the DB via the executor. The existing scheduler integration tests pass without modification.

- `SchedulerService.__init__` accepts `executor: CommandExecutor` as a keyword argument
- `SchedulerService.run_job()` body is ≤ 10 lines: guards cancelled/behind-schedule, creates `ExecuteJob`, calls `await self._executor.execute(cmd)`
- `SchedulerService._execution_log` deque is removed
- Any `get_execution_log()` accessor returns `[]` with a `# TODO(#267)` comment
- Job registration calls `await self._executor.register_job(reg)` and sets `job.db_id`
- `tests/integration/test_scheduler.py` passes without modification
- `uv run pyright src/hassette/core/scheduler_service.py` passes

   - Accept `executor: "CommandExecutor"` keyword argument
   - Set `self._executor = executor`
   - Remove `self._execution_log = deque(maxlen=...)` initialization

4. Rewrite `run_job()`:
   ```python
   async def run_job(self, job: "ScheduledJob") -> None:
       if job.cancelled:
           self.logger.debug("Job %s is cancelled, skipping", job)
           await self._remove_job(job)
           return

       run_at_delta = job.next_run - now()
       if run_at_delta.in_seconds() < -self.hassette.config.scheduler_behind_schedule_threshold_seconds:
           self.logger.warning("Job %s is behind schedule by %s seconds, running now.", job, abs(run_at_delta.in_seconds()))

       cmd = ExecuteJob(
           job=job,
           callable=self.task_bucket.make_async_adapter(job.job),
           job_db_id=job.db_id or 0,
       )
       await self._executor.execute(cmd)
   ```

5. Remove `track_execution` import from `scheduler_service.py` (it is no longer used after `run_job()` is rewritten). Remove `ExecutionResult` import too if unused.

## Test Strategy

Behavioral equivalence: `tests/integration/test_scheduler.py` must pass. This is the primary check.

New test to add (in `tests/integration/test_scheduler.py` or a new fixture):
- `test_run_job_calls_executor` — mock executor, schedule a job, trigger `run_job()`, assert `executor.execute()` called once with an `ExecuteJob`
- `test_job_registration_sets_db_id` — after adding a job, assert `job.db_id` is not `None` (i.e., `register_job()` was called and returned an id)

## Review Guidance

- `run_job()` must delegate ALL exception handling to the executor — no try/except in `run_job()` itself (FAIL)
- `track_execution()` context manager must NOT remain in `run_job()` — the executor owns timing (FAIL if still present)
- `job.db_id` must be set at registration time (when the job enters the queue), not at `run_job()` time — if `db_id` is still `None` when `run_job()` fires, the FK is 0 (a sentinel, not an error, but indicates the registration path was missed)
- The behind-schedule warning log must stay in `run_job()` — it is scheduler logic, not execution logic
- `job.cancelled` guard must stay in `run_job()` — it is scheduler logic
