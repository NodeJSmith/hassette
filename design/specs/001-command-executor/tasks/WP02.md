## Objectives & Success Criteria

`CommandExecutor` exists as a fully functional `Service` with all execution, persistence, and registration logic. It can be instantiated with a mock `Hassette` and a real in-memory SQLite DB (following the `test_database_service.py` pattern) and all behaviors are verified.

- `src/hassette/core/command_executor.py` exists and is importable
- `CommandExecutor(Service)` class with `execute(cmd)`, `serve()`, `register_listener()`, `register_job()`
- Exception contract verified by tests: `CancelledError` → queued + re-raised; `DependencyError` → `logger.error`, swallowed; `HassetteError` → `logger.error`, swallowed; `Exception` → `logger.exception`, swallowed; success → queued with `status="success"`
- Write queue drains in batches: records appear in `handler_invocations` and `job_executions` tables after `serve()` drains
- `_flush_queue()` on shutdown persists remaining records before returning
- Parent table upsert: `register_listener()` inserts on first call, updates `last_registered_at` on second call with same natural key, never duplicates `first_registered_at`
- `_run_error_hooks()` exists as a no-op stub (wired in #268)
- All tests in `tests/integration/test_command_executor.py` pass
- `uv run pyright src/hassette/core/command_executor.py` passes

3. Implement `_execute_handler(cmd: InvokeHandler) -> None`:
   - Capture `execution_start_ts = time.time()` and `_mono_start = time.monotonic()` before invocation
   - `try`: `await cmd.listener.invoke(cmd.event)`
   - `except asyncio.CancelledError`: build record with `status="cancelled"`, `self._write_queue.put_nowait(record)`, `raise`
   - `except DependencyError as e`: build record with `status="error"`, `error_type="DependencyError"`, `self.logger.error(...)`, `self._write_queue.put_nowait(record)`
   - `except HassetteError as e`: build record with `status="error"`, `self.logger.error(...)` (no traceback), queue record
   - `except Exception as e`: build record with `status="error"`, `self.logger.exception(...)` (with traceback), queue record
   - `else`: build record with `status="success"`, queue record
   - `finally`: compute `duration_ms = (time.monotonic() - _mono_start) * 1000`
   - Note: `record` must be created in each branch (not `finally`) because `CancelledError` re-raises before `finally` completes the record

4. Implement `_execute_job(cmd: ExecuteJob) -> None` with the same structure, using `JobExecutionRecord` instead of `HandlerInvocationRecord`.

   - Return `cursor.lastrowid`

7. Implement `register_job(registration: ScheduledJobRegistration) -> int` — same pattern for `scheduled_jobs` table.

8. Implement `serve()`:
   - `self.mark_ready(reason="CommandExecutor started")`
   - Loop: use `asyncio.wait` on `[queue.get(), shutdown_event.wait()]` with `FIRST_COMPLETED`

   - Commit once per batch

10. Implement `_flush_queue()`: drain all remaining items (no size limit), persist via `_drain_and_persist()` or inline write.

## Test Strategy

Follow `tests/integration/test_database_service.py` pattern: `mock_hassette` fixture with `tmp_path`, real DB setup via `DatabaseService.on_initialize()`, then instantiate `CommandExecutor` pointing at the same DB.

Key test functions:

- `test_cancelled_error_reraises_and_queues_record` — mock listener raises `CancelledError`, assert `pytest.raises(asyncio.CancelledError)`, then drain queue and verify record with `status="cancelled"`
- `test_dependency_error_swallowed_logged_error` — mock listener raises `DependencyError`, assert no re-raise, verify logger `error` called (not `exception`), record has `error_type="DependencyError"`
- `test_hassette_error_uses_logger_error` — mock listener raises `HassetteError`, assert logger `error` (not `exception`)
- `test_unexpected_exception_uses_logger_exception` — mock listener raises `RuntimeError`, assert logger `exception` called
- `test_serve_drains_to_db` — queue 5 records manually via `_write_queue.put_nowait()`, run `_drain_and_persist()`, query DB and assert 5 rows in `handler_invocations`
- `test_register_listener_upsert` — call `register_listener()` twice with same natural key, assert one row in DB, assert `first_registered_at` unchanged, `last_registered_at` updated

## Review Guidance

- `CancelledError` MUST be re-raised — swallowing it would break asyncio cancellation machinery (FAIL)
- `DependencyError` MUST use `logger.error` (not `logger.exception`) — no traceback for framework errors (FAIL if wrong)
- `HassetteError` MUST use `logger.error` (not `logger.exception`) (FAIL if wrong)
- Records must be queued BEFORE re-raising `CancelledError` — use `put_nowait()` not `put()` to avoid a second await point after the exception
- `first_registered_at` must NOT be in the `DO UPDATE SET` clause — only `last_registered_at` updates on conflict
- `_write_queue` must be unbounded `asyncio.Queue()` (no `maxsize`) — bounded queue risks `put_nowait` raising `QueueFull` under bursts
- `serve()` must call `_flush_queue()` before returning on shutdown — unflushed records are lost
