## Review Guidance

- `CommandExecutor` must be added AFTER `DatabaseService` in the `add_child()` order — it waits for DB ready in `on_initialize()`. Order matters for startup sequencing (FAIL if reversed)
- `CommandExecutor` must be added BEFORE `BusService` and `SchedulerService` — they need the executor instance at construction time (FAIL if reversed)
- The executor is passed by reference at construction time, not looked up at runtime — `add_child(BusService, ..., executor=self._command_executor)` not `executor=lambda: self._command_executor`
- DataSyncService stubs must return the correct empty type (e.g., `BusMetricsSummaryResponse(listeners=[])` not just `[]`) — callers expect the response object, not a raw list
- Every stubbed method must have `# TODO(#267)` so they're findable during #267
