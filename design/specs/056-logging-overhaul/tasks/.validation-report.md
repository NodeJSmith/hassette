## Warnings (non-blocking)

**W1 — T03 test for AC#13 scope is narrow:**
T03's Verify item for AC#13 reads "all emitted records appear in capture handler buffer and persistence handler batch." The design doc's AC#13 says "confirm all emitted records appear in **the database** and on console output." T03 verifies the pre-DB handoff (records in the persistence handler batch), which is a reasonable split since T04 owns DB wiring. However, a post-T04 integration test that verifies AC#13 end-to-end (records actually in DB after shutdown) would close the gap. T05's test suite does not currently list this. Consider adding an integration test covering the full shutdown flush → DB path once T04 is complete.

**W2 — T04 `seq` column population is unspecified:**
The `log_records` table schema includes a `seq INTEGER NOT NULL` column (for ordering records within a session). Neither T03's `LogPersistenceHandler._flush()` description nor T04's `insert_log_records()` description specifies how `seq` is generated. The design doc does not specify this either. If `seq` is intended to be a monotonic counter (rather than a DB-level sequence), the implementer needs to decide: per-record counter in `LogPersistenceHandler`, or an auto-increment alias using `rowid`. Leaving it undecified risks a `NOT NULL` constraint violation or incorrect ordering. Recommend clarifying before T03/T04 implementation.

**W3 — T05 `AC#7` verify scope is shared with T06:**
Both T05 and T06 claim `AC#7` in their `implements` lists. T05's Verify item covers `GET /api/logs/by-execution/{id}` returning only that execution's records (backend). T06's Verify item covers the frontend rendering at `/logs?execution_id=<uuid>`. This split is legitimate and correct, but the implementer should be aware that AC#7 is not fully verifiable until T06 completes. No action needed — just awareness.

**W4 — `log_queue_max` config field added in T03 item 7 with a conditional ("if not already added by T01"):**
T01 adds `log_format` and T03 item 7 says to add `log_queue_max` "if not already added by T01." T01's prompt does not add `log_queue_max` — only `log_format`. The "if not" qualifier is mildly confusing but not a contradiction; `log_queue_max` clearly belongs to T03 and will not be added by T01. Recommend removing the qualifier to avoid implementer uncertainty.
