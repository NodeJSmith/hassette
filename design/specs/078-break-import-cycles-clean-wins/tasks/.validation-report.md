## Coverage Gaps

**AC#3 — web/ planted-violation check has no Verify item in any task.**

AC#3 requires that `check_module_boundaries.py` exits non-zero when a `from hassette.core import ...` is planted in an `api/`, `web/`, OR `utils/` file. T01 covers the `api/` variant; T02 covers the `utils/` variant. T03 adds the `web → core` boundary RULE but does not claim AC#3 in its `implements` list and has no corresponding Verify item for the planted-violation check in a `web/` file. The `web` RULE is the highest-impact new rule (largest extraction surface) and its rejection behavior is unverified by the plan.

Resolution: add an AC#3 Verify item to T03 — e.g. "inserting a throwaway top-level `from hassette.core.core import Hassette` into any `web/` file makes `python tools/check_module_boundaries.py` exit non-zero; the clean tree exits zero."

## Warnings

1. **T04 AC#2 Verify is not fully binary.** The criterion reads "all pass (or any unrunnable suite is named as an explicit observability gap with the reason)". The conditional escape hatch makes pass/fail depend on a judgment call rather than a deterministic output. This is consistent with the design doc's own language ("or e2e is named as explicit observability gap"), so it is not a gap — but reviewers should be aware the criterion is not purely binary.

2. **FR#7 is split across three tasks (T01, T02, T03), each adding one rule.** This is structurally correct and intentional (per depends_on chain), but the full FR#7 satisfaction is only confirmed in aggregate across tasks. No single task's Verify confirms all three RULES simultaneously. Consider adding a T04 Verify item that runs `python tools/check_module_boundaries.py` on the fully-assembled tree and checks all three rules are present.

3. **T03 AC#6 Verify scope is narrower than T01's.** T01's AC#6 Verify includes `uv run pyright`; T03's AC#6 Verify also includes `uv run pyright`. Both are correct individually, but pyright is run mid-sequence (not just at T04). This is intentional per the design's incremental-verification strategy — not a gap, noted for awareness.
