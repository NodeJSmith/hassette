## Objectives & Success Criteria

- `TelemetryRepository.register_listener()` uses `INSERT ... ON CONFLICT(idx_listeners_natural) DO UPDATE SET <mutable fields>, retired_at = NULL RETURNING id`.
- `TelemetryRepository.register_job()` uses `INSERT ... ON CONFLICT(idx_scheduled_jobs_natural) DO UPDATE SET <mutable fields>, retired_at = NULL RETURNING id`.
- Upsert returns the existing row ID when the natural key matches (FK preservation), or a new ID when inserting.
- `once=True` listeners use plain `INSERT` (not ON CONFLICT) — partial index excludes them.
- `name` field included in INSERT column list from `ListenerRegistration.name`.
- Mutable fields list matches design: `debounce`, `throttle`, `priority`, `predicate_description`, `source_location`, `registration_source`, `retired_at` (listener); `handler_method`, `trigger_type`, `trigger_value`, `repeat`, `args_json`, `kwargs_json`, `source_location`, `registration_source`, `retired_at` (job). `once` is NOT mutable.
- Existing "always-new-ID" unit tests deleted and replaced with upsert contract tests.

## Subtasks

1. Modify `TelemetryRepository.register_listener()` in `src/hassette/core/telemetry_repository.py`: replace plain `INSERT` with `INSERT ... ON CONFLICT DO UPDATE` when `registration.once is False`. Keep plain `INSERT` for `once=True`.
2. Modify `TelemetryRepository.register_job()`: replace plain `INSERT` with `INSERT ... ON CONFLICT DO UPDATE`.
3. Add `name` to the INSERT column list and parameter tuple in `register_listener()`.
4. Delete `test_register_listener_two_calls_return_distinct_ids` and `test_register_job_two_calls_return_distinct_ids` from `tests/unit/core/test_telemetry_repository.py`.
5. Add upsert contract tests.

## Test Strategy

- **Test file**: `tests/unit/core/test_telemetry_repository.py`
- `test_upsert_same_natural_key_returns_same_id` — register listener, register again with same natural key but different `source_location`, assert same ID returned.
- `test_upsert_different_natural_key_returns_new_id` — register two listeners with different topics, assert different IDs.
- `test_upsert_resets_retired_at` — register listener, manually set `retired_at` on the row, upsert again, assert `retired_at IS NULL`.
- `test_upsert_updates_mutable_fields` — register, upsert with different `debounce`, read row, assert `debounce` updated.
- `test_once_true_always_inserts` — register two `once=True` listeners with same natural key, assert different IDs.
- `test_upsert_does_not_update_human_description` — register, upsert with different `human_description` in a different natural key column, verify identity columns unchanged.
- `test_upsert_with_name_overrides_human_description_in_key` — register with `name="a"`, register with same handler+topic but `name="b"`, assert different IDs.

## Review Guidance

- Verify the `ON CONFLICT` target matches the expression index name — SQLite requires the conflict target to match the index expression verbatim.
- Verify `once=True` registrations use plain `INSERT`, not the upsert path.
- Verify `once` is NOT in the `DO UPDATE SET` clause.
- Verify `retired_at = NULL` is included in the `DO UPDATE SET` clause.
