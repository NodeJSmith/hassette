## Objectives & Success Criteria

- `TelemetryRepository.reconcile_registrations(app_key, live_listener_ids, live_job_ids)` deletes stale rows without history, sets `retired_at` on stale rows with history, and deletes `once=True` rows from previous sessions (guarded by NOT EXISTS).
- `CommandExecutor.reconcile_registrations()` delegates to the repository via `DatabaseService.submit()`.
- `clear_registrations()` removed from `TelemetryRepository` and `CommandExecutor`.
- `app_lifecycle_service.py` updated: remove `clear_registrations()` call from `start_app()`; add post-ready reconciliation inside `initialize_instances()` after instance loop.
- `BusService.await_registrations_complete(app_key)` method added with timeout.
- `SchedulerService` exposes job ID collection mechanism.
- Live listener/job IDs collected by iterating instances via registry.
- Reconciliation failure logs a warning and allows the app to continue.
- Harness mock updated: `clear_registrations` mock replaced with `reconcile_registrations = AsyncMock()`; `register_listener` mock returns distinct IDs via `side_effect`; `spec=CommandExecutor` added.

Add post-ready reconciliation at the end of `initialize_instances()` — after the instance loop, await registration completion, collect live IDs from instances via registry iteration (each instance's `Bus.get_listeners()` and `Scheduler` jobs), **union with Router listener IDs** (`{listener.db_id for listener in router.get_listeners_by_owner(owner_id)}` per instance, wrapped in try/except with conservative fallback) as a safety guard for mid-session active handlers, then call `reconcile_registrations()`. Wrap in try/except for error tolerance.

## Test Strategy

- **Test file**: `tests/unit/core/test_telemetry_repository.py`
- `test_reconcile_deletes_stale_without_history` — register listener, don't create invocations, reconcile with empty live_ids, assert row deleted.
- `test_reconcile_retires_stale_with_history` — register listener, create invocation, reconcile with empty live_ids, assert `retired_at IS NOT NULL`.
- `test_reconcile_preserves_live_listeners` — register two listeners, reconcile with one ID, assert the live one preserved.
- `test_reconcile_deletes_once_true_previous_session` — register `once=True` listener in session 1, reconcile in session 2, assert deleted (no current-session invocations).
- `test_reconcile_preserves_once_true_with_current_invocations` — register `once=True`, create invocation in current session, reconcile, assert preserved.
- `test_reconcile_empty_ids_no_crash` — reconcile with empty `live_listener_ids`, assert no error.
- `test_reconcile_resets_retired_at_on_reupsert` — retire a row, then upsert same key, verify `retired_at` is NULL.

- **Test file**: `tests/integration/test_command_executor.py`
- `test_fk_preserved_across_restart` — register listener, create invocation, upsert with same key, assert invocation still references same listener_id.
- `test_reconciliation_ordering` — verify reconciliation runs after initialization completes (replaces deleted `clear_registrations` assertion).

## Review Guidance

- Verify reconciliation runs INSIDE `initialize_instances()`, not in `start_app()` — the current `clear_registrations` call in `start_app` runs BEFORE initialization, which is the wrong timing for post-ready reconciliation.
- Verify `await_registrations_complete()` has a timeout and handles task errors gracefully.
- Verify the empty ID list guard: `NOT IN ()` is a SQLite syntax error.
- Verify the `once=True` DELETE guard uses `NOT EXISTS` with `session_id = ?` to protect current-session invocations.
- Verify harness mock returns distinct IDs per call (not fixed `42`).
