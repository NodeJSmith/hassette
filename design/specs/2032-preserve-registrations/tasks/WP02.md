## Objectives & Success Criteria

- `Bus.on()`, `Bus.on_state_change()`, `Bus.on_attribute_change()`, `Bus.on_call_service()` accept `name: str | None = None` as an explicit named parameter.
- `Options` TypedDict includes `name` for type checking.
- `Listener` dataclass has a `name: str | None` field, set via `Listener.create()`.
- `ListenerRegistration` dataclass has `name: str | None = None` as the **last field** (preserving positional construction).
- `BusService._register_then_add_route()` passes `listener.name` into `ListenerRegistration`.
- `Bus` instances maintain `_registered_keys: set` for collision detection, cleared at initialization start.
- Duplicate natural keys raise `ValueError` synchronously in `Bus.on()` / `add_listener()`.
- All existing bus tests continue to pass.

## Subtasks

1. Add `name: str | None = None` as the last field on `ListenerRegistration` in `src/hassette/core/registration.py`.
2. Add `name: str | None = None` field to `Listener` dataclass in `src/hassette/bus/listeners.py`. Add `name` parameter to `Listener.create()`.
3. Add `name: str | None = None` as explicit named parameter on `Bus.on()` in `src/hassette/bus/bus.py`. Extract it alongside `once`, `debounce`, `throttle`. Pass to `Listener.create(name=name)`.
4. Update `Options` TypedDict in `src/hassette/bus/bus.py` to include `name: str | None`.
5. Propagate `name` through `Bus.on_state_change()`, `Bus.on_attribute_change()`, `Bus.on_call_service()` — extract from `**opts` and pass to `self.on()`.
6. Add `_registered_keys: set[tuple]` to `Bus.__init__()`. Clear it at the start of each initialization cycle (add a clear call in `on_initialize()` or equivalent).
7. Add collision detection in `Bus.on()` / `add_listener()`: compute natural key tuple `(app_key, instance_index, handler_method, topic, name or human_description or '')`, check against `_registered_keys`, raise `ValueError` if duplicate. Add key to set after validation.
8. Update `BusService._register_then_add_route()` in `src/hassette/core/bus_service.py` to pass `listener.name` into `ListenerRegistration(name=listener.name)`.

## Test Strategy

- **Test file**: `tests/unit/bus/test_bus.py` (create if needed; or `tests/integration/test_bus.py` if unit isolation isn't feasible)
- `test_name_parameter_propagates_to_listener` — register with `name="my_listener"`, assert `listener.name == "my_listener"`.
- `test_duplicate_natural_key_raises_value_error` — register two listeners with same handler+topic+predicate, assert `ValueError` on second.
- `test_duplicate_name_raises_value_error` — register two listeners with same handler+topic and `name="x"`, assert `ValueError`.
- `test_name_disambiguates_otherwise_identical_keys` — register two listeners with same handler+topic but different `name=`, assert both succeed.
- `test_registered_keys_cleared_on_reinit` — verify `_registered_keys` is empty at start of initialization.

## Review Guidance

- Verify `name` is extracted as an **explicit named parameter** in `Bus.on()`, not via `**opts` passthrough — `Bus.on()` uses keyword-only args, so `name` must appear in the signature.
- Verify `ListenerRegistration.name` is the **last field** — inserting it elsewhere breaks positional construction at `bus_service.py:102-113`.
- Collision detection must be synchronous (in `Bus.on()`, not in `BusService._register_then_add_route()`).
- The `_registered_keys` set must be cleared at initialization start, not only on shutdown.
