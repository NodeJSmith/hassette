## Objectives & Success Criteria

- `ValueIs.summarize()` returns `f"value is {self.condition} from {callable_name(self.source)}"` for non-callable conditions, and `f"custom condition from {callable_name(self.source)}"` for callable conditions with non-callable sources.
- `Guard.summarize()` behavior is investigated — if `Guard` is redundant with `ValueIs`, document the finding in the Open Questions resolution (not blocking).
- Golden/snapshot tests assert exact `summarize()` output for every predicate type, catching accidental format changes that would orphan historical rows.
- All existing predicate tests continue to pass.

## Subtasks

1. Add a `callable_name(fn)` utility function (e.g., in `src/hassette/event_handling/predicates.py` or a shared util) that returns `fn.__qualname__` for named functions, `fn.__name__` for builtins, and `"<callable>"` for objects with no stable name (lambdas, partials). Follow APScheduler's `callable_to_ref` pattern for stable names.
2. Update `ValueIs.summarize()` in `src/hassette/event_handling/predicates.py` to include condition value and source accessor name per design spec.
3. Investigate `Guard` — read its constructor and compare with `ValueIs`. Document whether consolidation is warranted (add a comment to the Open Questions section of design.md, not a code change).
4. Verify whether `ServiceDataWhere.summarize()` can receive callable conditions. If so, add a companion fix. If not, add a comment noting the verification.
5. Check for existing `tests/unit/test_predicate_summarize.py` — if it exists with overlapping coverage, consolidate into `tests/unit/bus/test_predicates.py` rather than maintaining two snapshot suites. Add golden/snapshot tests in `tests/unit/bus/test_predicates.py` asserting exact `summarize()` output for: `EntityMatches`, `DomainMatches`, `ServiceMatches`, `StateDidChange`, `AttrDidChange`, `StateTo`, `StateFrom`, `AttrTo`, `AttrFrom`, `StateComparison`, `AttrComparison`, `ValueIs` (literal condition), `ValueIs` (callable condition), `Guard`, `AllOf`, `AnyOf`, `Not`, `IsPresent`, `IsMissing`, `DidChange`, `ServiceDataWhere`.

## Test Strategy

- **Test file**: `tests/unit/bus/test_predicates.py`
- `test_value_is_summarize_literal_condition` — asserts `ValueIs(source=get_state_value_new, condition="on").summarize()` returns a string containing `"on"` and the source name.
- `test_value_is_summarize_callable_condition` — asserts `ValueIs(source=some_fn, condition=lambda v: v > 50).summarize()` returns `"custom condition from ..."`.
- `test_predicate_summarize_golden_<type>` — one test per predicate type asserting exact output. These are stability contract tests — they fail loudly when `summarize()` output changes.
- All existing tests in the file must continue to pass.

## Review Guidance

- Verify `callable_name()` handles lambdas, `functools.partial`, bound methods, and `get_path()` accessors correctly — these are the edge cases that motivated the fix.
- Verify the golden tests cover every predicate class in `predicates.py` — missing one means a future `summarize()` change could orphan rows silently.
- `Guard` investigation should document its finding but NOT consolidate `Guard`/`ValueIs` in this WP.
