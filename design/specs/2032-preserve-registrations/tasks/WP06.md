## Objectives & Success Criteria

- `get_global_summary()` uses `active_listeners` / `active_scheduled_jobs` views for registration counts (`total_listeners`, `total_jobs`).
- `get_all_app_summaries()` uses views for `handler_count` and `job_count`.
- `get_listener_summary()` and `get_job_summary()` query base tables (intentionally include retired rows for history).
- `get_recent_errors()` and `get_slow_handlers()` INNER JOIN comments updated — remove references to `clear_registrations()`.
- `DatabaseService._do_run_retention_cleanup()` extended to delete retired registration rows older than `retention_days`.
- "All Time" queries return data spanning multiple sessions (regression test).

## Subtasks

1. Update `get_global_summary()` in `src/hassette/core/telemetry_query_service.py` — change `(SELECT COUNT(*) FROM listeners)` to `(SELECT COUNT(*) FROM active_listeners)` and same for `scheduled_jobs`.
2. Update `get_all_app_summaries()` — change `listener_reg_query` and `job_reg_query` to query `active_listeners` / `active_scheduled_jobs` instead of base tables.
3. Update INNER JOIN comments in `get_recent_errors()` (lines 430, 449, 471, 489) and `get_slow_handlers()` (line 524) — remove references to `clear_registrations()`, note that rows persist across restarts.
4. Extend `_do_run_retention_cleanup()` in `src/hassette/core/database_service.py` — add `DELETE FROM listeners WHERE retired_at IS NOT NULL AND retired_at < ?` (same for `scheduled_jobs`), using `retention_days` threshold.

## Test Strategy

- **Test file**: `tests/integration/test_telemetry_query_service.py`
- `test_all_time_aggregates_across_sessions` — register listener in session 1, create invocations, simulate restart (upsert same key in session 2), create more invocations, query without `session_id`, assert total covers both sessions.
- `test_registration_counts_exclude_retired` — register two listeners, retire one via reconciliation, query `get_global_summary()`, assert `total_listeners = 1`.
- `test_listener_summary_includes_retired` — register, retire, query `get_listener_summary()`, assert retired row still appears with its invocation history.
- `test_retention_cleanup_deletes_old_retired_rows` — insert retired registration with old `retired_at`, run retention cleanup, assert deleted.

## Review Guidance

- Verify `get_global_summary` and `get_all_app_summaries` use view names (`active_listeners`, `active_scheduled_jobs`), not base table names with WHERE clauses — the view-based approach is the enforcement mechanism against forgotten filters.
- Verify `get_listener_summary` and `get_job_summary` still query base tables — these intentionally include retired rows.
- Verify retention cleanup uses the same `retention_days` config as invocation cleanup.
