## Objectives & Success Criteria

- Migration 006 drops and recreates `listeners` and `scheduled_jobs` tables with new schema (add `name TEXT`, `retired_at REAL` columns).
- Partial unique expression index `idx_listeners_natural` created on `listeners(app_key, instance_index, handler_method, topic, COALESCE(name, human_description, '')) WHERE once = 0`.
- Unique index `idx_scheduled_jobs_natural` created on `scheduled_jobs(app_key, instance_index, job_name)`.
- Database views `active_listeners` and `active_scheduled_jobs` created.
- Single-column covering index `idx_hi_listener_id` on `handler_invocations(listener_id)` created.
- Child tables (`handler_invocations`, `job_executions`) rebuilt to fix FK references after parent table rename (following migration 004 pattern).
- `_DDL` constant in `tests/unit/core/test_telemetry_repository.py` updated to match new schema.

## Subtasks

1. Create `src/hassette/migrations/versions/006_preserve_registrations.py` with `revision = "006"`, `down_revision = "005"`.
2. In `upgrade()`: rename → create → insert → drop pattern for `listeners` (adding `name TEXT` and `retired_at REAL` columns). Same for `scheduled_jobs` (adding `retired_at REAL`).
3. Rebuild `handler_invocations` and `job_executions` child tables after parent rebuilds (following migration 004 lines 108-175 pattern) to fix FK references.
4. Create the partial unique expression index on `listeners` and the unique index on `scheduled_jobs`.
5. Create `active_listeners` and `active_scheduled_jobs` views.
6. Create `idx_hi_listener_id` covering index on `handler_invocations(listener_id)`.
7. Recreate existing indexes that were dropped during table rebuilds (copy index definitions from migration 004).
8. Write `downgrade()` — reverse the schema changes (drop views, rebuild tables without new columns, restore original indexes).
9. Update `_DDL` constant in `tests/unit/core/test_telemetry_repository.py` to include `name TEXT`, `retired_at REAL`, the views, and the new indexes.

## Test Strategy

- **Test file**: `tests/unit/core/test_telemetry_repository.py`
- `test_schema_has_name_column` — verify `PRAGMA table_info(listeners)` includes `name`.
- `test_schema_has_retired_at_column` — verify both `listeners` and `scheduled_jobs` have `retired_at`.
- `test_unique_index_enforced` — insert two non-once listeners with same natural key, assert `IntegrityError`.
- `test_partial_index_allows_once_duplicates` — insert two `once=1` listeners with same natural key, assert both succeed.
- `test_active_views_exist` — `SELECT * FROM active_listeners` succeeds.
- All existing tests in the file must pass with the updated `_DDL`.

## Review Guidance

- Verify the child table rebuild follows migration 004's exact pattern — SQLite silently rewires FK references on parent table rename.
- Verify `COALESCE(name, human_description, '')` in the index expression matches the design's natural key specification.
- Verify `WHERE once = 0` on the partial index — `once=True` rows must not participate in uniqueness enforcement.
- Verify all existing indexes from migration 004 are recreated (they're dropped when tables are rebuilt).
