## Objectives & Success Criteria

- All telemetry endpoints accept `source_tier: Literal['app', 'framework', 'all'] | None = Query(default=None)` parameter. FastAPI returns 422 for invalid values.
- `dashboard_errors` defaults to `source_tier='app'` when parameter is None.
- `app_health`, `app_listeners`, `app_jobs`, `handler_invocations`, `job_executions` have `try: ... except DB_ERRORS:` guards returning empty/zero-value responses with 503 status.
- `/telemetry/status` returns `dropped_overflow` and `dropped_exhausted` counts from `CommandExecutor` via `Hassette.get_drop_counters()`.
- `dashboard_errors` renders orphaned records (null FK) as "deleted handler" / "deleted job" instead of failing.
- `__hassette__` is documented in OpenAPI description strings on `app_key` path parameters.
- `since_ts` parameter on `dashboard_errors` defaults to `time.time() - 86400` (last 24h) instead of `0`.

## Subtasks

1. Add `source_tier: Literal['app', 'framework', 'all'] | None = Query(default=None)` to all telemetry endpoints in `src/hassette/web/routes/telemetry.py`. Pass to query methods (map `None` → `'app'`).
2. Add `try: ... except DB_ERRORS:` guards to `app_health`, `app_listeners`, `app_jobs`, `handler_invocations`, `job_executions`. Return empty lists or zero-value responses with 503 status.
3. Update `/telemetry/status` endpoint to include `dropped_overflow` and `dropped_exhausted` from `Hassette.get_drop_counters()` via `HassetteDep` (or `RuntimeDep`).
4. Update `dashboard_errors` to handle null FK fields in error records — render "deleted handler" / "deleted job" labels.
5. Change `since_ts` default in `dashboard_errors` from `0` to `time.time() - 86400`.
6. Add OpenAPI description to `app_key` path parameter: "Use `__hassette__` to query framework-internal actor telemetry."
7. Update `TelemetryStatusResponse` model degraded-path: `dropped_overflow=0, dropped_exhausted=0` when `CommandExecutor` is not initialized.

## Test Strategy

- **`tests/integration/test_api.py`** (extend existing or new `test_telemetry_endpoints.py`):
  - `test_dashboard_errors_defaults_to_app_tier` — call without `source_tier`, verify only app-tier errors returned.
  - `test_dashboard_errors_framework_filter` — call with `?source_tier=framework`, verify only framework errors.
  - `test_dashboard_errors_all_filter` — call with `?source_tier=all`, verify both tiers.
  - `test_source_tier_invalid_returns_422` — call with `?source_tier=invalid`, verify 422.
  - `test_app_health_db_error_returns_503` — mock DB unavailability, call `app_health`, verify 503 with zero-value response.
  - `test_app_listeners_db_error_returns_503` — same for `app_listeners`.
  - `test_status_includes_drop_counters` — increment drop counters, call `/telemetry/status`, verify non-zero counts.
  - `test_dashboard_errors_orphan_rendering` — insert orphan error record (null FK), call endpoint, verify "deleted handler" label.
  - `test_hassette_app_key_returns_framework_data` — call `GET /telemetry/app/__hassette__/listeners`, verify framework actors returned.

## Review Guidance

- Verify FastAPI's `Literal` validation handles `source_tier` — no custom validation code needed.
- Verify DB_ERRORS guard on all 5 unguarded endpoints (compare against the already-guarded `dashboard_kpis`, `dashboard_errors`, `sessions`).
- Verify `dropped_*` counters reach the status endpoint via `Hassette.get_drop_counters()`, not via a direct `CommandExecutor` import.
- Verify orphan rendering produces a user-readable label, not `null` in JSON.
