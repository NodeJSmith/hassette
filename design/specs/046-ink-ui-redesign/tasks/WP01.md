## Objectives & Success Criteria

Replace the Graphite+Emerald design foundation with the Ink design system. After this WP, all CSS custom properties resolve to Ink values, self-hosted fonts serve Newsreader/Geist/Geist Mono, and base typography/reset/layout-grid CSS uses the new token vocabulary. Existing component CSS will be visually broken (expected — component styles are rebuilt in subsequent WPs). The build succeeds with no missing token references.

## Subtasks

1. Download and add self-hosted `.woff2` font files to `frontend/public/fonts/`: Newsreader (400), Geist (400, 500), Geist Mono (400, 500). Remove the old font files (dm-sans, space-grotesk, jetbrains-mono).
2. Rewrite `frontend/src/tokens.css` with the complete Ink token system from the design bundle's `colors_and_type.css`. Light theme as `:root` default, dark theme via `[data-theme="dark"]`. Drop the `--ht-*` prefix — use unprefixed tokens (`--bg-page`, `--ink-1`, `--line-1`, `--ok`, `--accent`, etc.). Include all surface, ink, line, status, accent, shadow, spacing, radius, motion, and typography tokens.
3. Rewrite the `@font-face` declarations at the top of `frontend/src/global.css` to reference the new font files and map to `--font-display` (Newsreader), `--font-body` (Geist), `--font-mono` (Geist Mono).
4. Rewrite the base CSS in `global.css`: reset (box-sizing, margin, img, button, table), body element styling (bg, color, font-family from tokens), heading styles (h1-h4 using `--font-display`), code/mono styles, link styles, selection styles, reduced-motion media query. Keep only the base — component-specific classes are rebuilt in later WPs.
5. Add the 2-column layout grid class (240px sidebar + 1fr main) and the main content area styles.
6. Update `frontend/src/utils/theme.ts` if the theme toggle logic needs changes (the type `"dark" | "light"` should still work; verify `[data-theme]` attribute is set correctly).
7. Update `design/context.md` to reflect the Ink design system: new token names, new font families, new color vocabulary, new aesthetic direction.
8. Verify `npm run build` succeeds (token references resolve, fonts load).

## Test Strategy

- `theme.ts` has an existing test (`utils/theme.test.ts`) — verify it still passes after any changes.
- No new component tests needed — this WP is CSS and font files only.
- Manual verification: run `npm run dev`, confirm both themes load without console errors, fonts render correctly.

## Review Guidance

- Every value in `tokens.css` must match the design bundle's `colors_and_type.css` exactly (hex values, font stacks, spacing scale, radii, shadows, motion).
- No `--ht-*` prefixed tokens should remain in `tokens.css`.
- No raw hex, pixel, or font-family values in global.css base styles — all must reference tokens.
- `design/context.md` must accurately reflect the Ink tokens (not the old Graphite+Emerald values).
- The Graphite+Emerald theme block must be completely removed (no `[data-theme="graphite-emerald"]`).
