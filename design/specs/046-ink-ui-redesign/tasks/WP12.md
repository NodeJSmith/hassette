## Objectives

Replace the current KPI strip + app card grid with the mockup's overview layout:
- Serif greeting headline with contextual status subtitle
- System metadata line (app count, run rate)

- Recent activity feed

## Subtasks

### 0. Framework error banner

**Mockup ref:** `overview.jsx:23–53, 187`

Render the `FrameworkErrorChip` above the hero card when boot-time issues exist. Full-width bar with error/warning count, top issue detail, and "view all →" link to config page. Background: `color-mix(in oklch, var(--err) 4%, var(--bg-surface))`, border: `var(--err)`.

Data source: `boot_issues` field on `SystemStatus` (added in WP16). Each issue has `severity` (err/warn), `label`, `detail`, and `route`. Render unconditionally in `OverviewPopulated` — only hidden when the array is empty.

### 1. Greeting + metadata header

**Mockup ref:** `overview.jsx:170–182`

Replace the current `HeroCard` at the top with:
- `<h1>` in `--font-display`: time-of-day greeting ("Good morning.", "Good afternoon.", "Good evening.")
- Right-aligned metadata in mono: `{appCount} apps · {runsPerHour} runs / hr`
- Subtitle line in `--ink-2`: contextual message that varies by system state (see mockup lines 175–181 for exact copy per state)

**HA version** is in the mockup (`home assistant 2026.4`) but requires a new backend field. Skip it for now — add when backend support exists. Leave the spot in the metadata line so it's easy to add later.

### 2. Three summary cards

**Mockup ref:** `overview.jsx:274–325`

Replace `KpiStrip` + `AppGrid` with a 3-column CSS grid (`repeat(3, 1fr)`, gap 16px):

**Card 1: "your apps"** (mockup lines 276–288)
- `<h3>` title "your apps"
- List of apps (first 6) with: StatusShape dot, app name in mono, invocation count right-aligned
- Multi-instance apps show `×{count}` suffix
- "see all →" link to `/apps`
- Data source: `appGrid` (already fetched)

**Card 2: "activity"** (mockup lines 290–312)
- `<h3>` title "activity"
- Big number in `--font-display` (total invocations + executions)
- "runs in last hour" subtitle
- "open inspector →" button at bottom (links to events route when it exists; disabled/hidden for now since events page is deferred)
- SVG sparkline: polyline rendering `activity_buckets` from the KPIs response (added in WP16). Each bucket is `{ok: int, err: int}` for one time slice. Normalize to 0–1 range for the polyline Y axis. Width ~260px, height ~36px.
- ok/err breakdown: `● {ok} ok  ■ {err} err`
- Quiet state: show empty state with "0 runs / hour" message (mockup lines 293–299)

**Card 3: "system"** (mockup lines 314–324)
- `<h3>` title "system"
- Service list: bus, scheduler, HA websocket, file watcher
- Each row: StatusShape dot, service name, right-aligned metric in mono dim
- Data source: `serviceStatus` signal from app state

**Run rate computation:** `(total_invocations + total_executions)` from KPIs divided by the time window in hours. For "since restart" with uptime < 1h, show the raw count instead.

### 3. Enrich hero card variants

**Mockup ref:** `overview.jsx:189–271` (hero cards), `overview.jsx:66–144` (first install)

Keep the existing hero card concept but enrich each variant to match the mockup:

**Healthy** (mockup lines 190–201): Green accent card with ✓ icon, "everything's running smoothly" title, subtitle mentioning last incident if available.

**Single failure** (mockup lines 249–271): Error-tinted card with ! icon, app name in title, error type + message + source location in subtitle, action buttons (open app →, view traceback, stop).

**Multiple failures** (mockup lines 202–248): Error card with ranked list of failing apps. Each row: status dot, app name in mono, crash count + time since, error type + message, "failing" pill. Bottom bar: "view all in events →" and "stop all failing" buttons.

**First install** (mockup lines 66–144): Two-column layout. Left: code snippet card showing a minimal hassette app + TOML config. Right: system status card + "coming from AppDaemon?" card. This is NOT just a text message.

**Quiet**: No separate quiet hero card — the greeting subtitle handles the quiet state ("all apps are running, but nothing has happened in a while."). The activity card shows the empty state ("0 runs / hour"). Do NOT add a quiet variant to the hero card component — remove it from the hero card variants list if it exists.

### 4. Recent errors as table

**Mockup ref:** `overview.jsx:327–371`

Replace the current `ErrorFeed` expandable card format with a table matching the mockup:
- Columns: TIME, APP, LOCATION, EXCEPTION, AGE
- TIME: timestamp in mono dim
- APP: app name linked to app detail
- LOCATION: `module.function()` + `apps/module.py:line` below it in dim
- EXCEPTION: error type in `--err` color + message
- AGE: relative time in mono dim (30s, 4m, 20m)

Keep the existing tier filter (All/Apps/Framework) toggle.

LOCATION uses `source_location` field on error entries (added in WP16). The error card uses `color-mix(in oklch, var(--err) 4%, var(--bg-surface))` background tint with `border-color: var(--err)` — NOT `border-left` (banned pattern).
