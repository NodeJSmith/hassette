## Objectives & Success Criteria

Remove `session_id` from all telemetry and bus route signatures and query methods. Replace with `since: float | None` (Unix epoch). Add `uptime_seconds` to the WS `connected` payload. After this WP, no route or query method accepts `session_id`, all accept `since`, and the WS connected message sends `uptime_seconds` instead of `session_id`. Existing tests pass. Schema regeneration reflects the changes.

## Subtasks

1. Add `_since_clause(since: float | None, timestamp_col: str) -> tuple[str, dict[str, float]]` helper to `src/hassette/core/telemetry_query_service.py`, mirroring the existing `_source_tier_clause()` pattern. Returns `("AND {col} >= :since", {"since": val})` when `since` is not None, else `("", {})`.
2. Refactor `get_listener_summary()` (line 234): replace `session_id: int | None` param with `since: float | None`. Replace the session_id SQL branch with `_since_clause()`. Update all callers.
3. Refactor `get_job_summary()` (line 316): same pattern.
4. Refactor `get_all_app_summaries()` (line 380): same pattern.
5. Refactor `get_global_summary()` (line 517): same pattern.
6. Refactor `get_handler_invocations()` (line 592): same pattern.
7. Refactor `get_job_executions()` (line 621): same pattern.
8. Update all telemetry route handlers in `src/hassette/web/routes/telemetry.py`: replace `session_id: int | None = Query(...)` with `since: float | None = Query(default=None)`. Update the calls to query methods.
9. Update `src/hassette/web/routes/bus.py`: remove `session_id` param from the listeners endpoint, replace with `since`.
10. Update `src/hassette/web/telemetry_helpers.py`: remove `safe_session_id()` if no longer referenced.
11. Update `ConnectedPayload` in `src/hassette/web/models.py`: remove `session_id`, add `uptime_seconds: float`.
12. Update `connected_payload_from()` in `src/hassette/web/mappers.py`: compute `uptime_seconds` from `RuntimeQueryService._start_time` (or accept it as a parameter).
13. Update `src/hassette/web/routes/ws.py`: stop sending `session_id` in the connected message; send `uptime_seconds`.
14. Run schema regeneration: `uv run python scripts/export_schemas.py --types` to update `frontend/openapi.json`, `frontend/ws-schema.json`, and `frontend/src/api/generated-types.ts`.
15. Update existing backend tests that pass `session_id` to routes or query methods — replace with `since` or remove the parameter.

## Test Strategy

- Existing integration tests in `tests/integration/` that call telemetry routes with `session_id` must be updated to use `since` or no filter.
- Test `_since_clause()`: verify it returns correct SQL for `None` (empty) and a float value. Test file: `tests/unit/test_telemetry_query_helpers.py` or inline in existing telemetry test file.
- Test that routes reject unknown query params gracefully (FastAPI handles this automatically).
- Run `uv run nox -s dev -- -n 2` to verify all existing tests pass.

## Review Guidance

- Every `session_id` reference in route files and query methods must be gone — grep for `session_id` in the web/ and telemetry_query_service paths.
