## Objectives & Success Criteria

Build the global command palette triggered by Cmd-K / Ctrl-K. After this WP: pressing the keyboard shortcut opens a modal search palette that indexes pages, apps, instances, handlers, and quick actions. Typing filters results in real-time. Arrow keys navigate, Enter selects, Escape closes. Selecting an item navigates to the correct page with correct focus.

## Subtasks

1. Create `frontend/src/components/layout/cmd-k-palette.tsx`: modal overlay with backdrop blur (`rgba(20,22,26,0.45)`, `backdrop-filter: blur(2px)`). Width: `min(620px, 90%)`, max-height 70vh. Border: `var(--line-1)`, radius: `var(--r-xl)`, shadow: `var(--shadow-3)`. Input at top (large size, placeholder "Search apps, handlers, pages, actions…"). Scrollable results area. Footer with keyboard hints (`↑↓ navigate · ↵ select · esc close`).
2. Build the search index: on palette open, build from current data:
   - **Pages**: Overview, Logs, Config (static items)
   - **Apps**: every app from manifests (label: display name, sub: app_key)
   - **Instances**: every instance child of multi-instance apps
   - **Handlers**: fetch from `GET /bus/listeners` (returns all listeners flat) on palette open. This endpoint already exists and returns `handler_method`, `app_key`, `instance_index`, and `topic` — sufficient for display and navigation. Cache the result for the session (refetch on palette re-open after WS reconnect).
   - **Actions**: "Reload all apps", "Stop all failing", "Open docs" (static items)
3. Implement filtering: case-insensitive substring match across label, sub, and kind fields. Group results by kind (pages, apps, instances, handlers, actions) with section headers.
4. Implement keyboard navigation: ↑/↓ to move selection, Enter to select, Escape to close. Selected row shows `background: var(--bg-active)` with left `borderLeft: 2px solid var(--accent)`.
5. Implement item glyphs: `▭` page, `□` app, `·` instance, `ƒ` handler, `⚡` action. Or use StatusShape for items that have a status.
6. Implement selection actions: pages → navigate to route. Apps → navigate to `/apps/{key}`. Instances → navigate to `/apps/{key}/{index}`. Handlers → navigate to `/apps/{key}?focus={handler_method}` (the app detail page handles the `focus` query param — see WP07). Actions → execute: "reload all" iterates running apps and calls `reloadApp(key)` per-app; "stop all failing" iterates failed apps and calls `stopApp(key)` per-app; "open docs" opens external link.
7. Wire into `app.tsx`: the Cmd-K/Ctrl-K listener from WP05 opens this palette. The palette manages its own open/close state.
8. Add component CSS for palette overlay, input, results list, section headers, result rows, keyboard hints footer.

## Review Guidance

- The palette must be a modal overlay (position fixed, z-index above everything) — not a sidebar panel or inline component.
- Backdrop must use `backdrop-filter: blur(2px)` as specified in the design prototype.
- Shadow must be `var(--shadow-3)` — this is one of the few places shadows are allowed (floating surface).
- The keyboard shortcut must handle both Cmd-K (Mac) and Ctrl-K (Windows/Linux) — check `e.metaKey || e.ctrlKey`.
- Handler index should come from `GET /bus/listeners`, not from per-app cache. If the fetch fails or is slow, degrade gracefully (show apps/pages immediately, add handlers when the response arrives).
