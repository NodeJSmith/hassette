## Objectives & Success Criteria

Build the structural shell of the new UI: the 240px sidebar with app tree, shared primitive components (StatusShape, confirm dialog, time-preset selector), the status bar, the off-canvas mobile drawer, and the top-level routing in `app.tsx`. After this WP, the application renders the new layout with sidebar navigation, the app list populates from the manifests API, and clicking nav items/apps routes correctly (page content is placeholder until subsequent WPs). Both themes work. Mobile drawer opens/closes.

## Subtasks

1. Create `frontend/src/components/shared/status-shape.tsx`: SVG component rendering circle (ok), triangle (warn), square (err), ring (mute). Props: `kind`, `size`. Uses `var(--ok)`, `var(--warn)`, `var(--err)`, `var(--mute)` colors.
2. Create `frontend/src/components/shared/confirm-dialog.tsx`: modal overlay with title, body, cancel/confirm buttons. Uses Ink tokens for backdrop, border, shadow. Escape key closes. Props: `title`, `body`, `confirmLabel`, `onConfirm`, `onCancel`, `tone` (default or danger).
3. Create `frontend/src/components/layout/time-preset-selector.tsx`: button group with 4 presets ("Since restart", "1h", "24h", "7d"). Reads/writes `state.timePreset` signal. Shows current uptime next to the selector.
4. Rewrite `frontend/src/components/layout/sidebar.tsx`: 240px fixed sidebar with "hassette" wordmark (Newsreader display font), version + connection status (StatusShape for WS state), Cmd-K trigger button (⌘K hint), top-level nav items (Overview, Logs, Config), search input for app filtering, and app list grouped by status (failing > blocked > slow > running > stopped > disabled). Multi-instance apps expand to show instance children with `└` connectors. Parent status = worst-of-children. Active item highlighted with `var(--bg-active)`. Auto-loaded badge on autodiscovered apps. Blocked apps show reduced opacity.
5. Implement the off-canvas drawer for mobile (<768px): sidebar slides in from left on hamburger tap. Backdrop overlay with click-to-close. CSS transition `var(--t-med)`.
6. Verify `bottom-nav.tsx` and its test were deleted in WP04. If not, delete them now.
7. Rewrite `frontend/src/components/layout/status-bar.tsx`: remove session scope toggle. Add time-preset selector. Show connection status and uptime.
8. Rewrite `frontend/src/app.tsx`: update layout to 240px sidebar + 1fr main grid. Update routes — remove `/sessions`, keep `/`, `/apps`, `/apps/:key`, `/apps/:key/:index`, `/logs`. Add `/config` route (placeholder). Wire sidebar navigation. Add hamburger button for mobile. Include confirm-dialog provider and Cmd-K keyboard listener (palette component comes in WP10).
9. Add all component CSS to `frontend/src/global.css`: sidebar classes, nav items, app tree, status shapes, confirm dialog, time-preset selector, status bar, off-canvas drawer, layout grid.

- `components/shared/confirm-dialog.test.tsx`: renders title/body, calls onConfirm on confirm click, calls onCancel on cancel click and escape key, danger tone applies red styling.

- Run `npm run test`.

## Review Guidance

- Sidebar must be exactly 240px fixed, not responsive width — the design spec is explicit about this.
- Status shapes must be SVG (not emoji or CSS shapes) for consistent cross-browser rendering.
- The app list grouping order (failing > blocked > slow > running > stopped > disabled) is a design requirement — verify the sort.
- The off-canvas drawer must use `position: fixed` and `z-index` above the main content, not push the content.
- No `box-shadow` on the sidebar — only `border-right: 1px solid var(--line-1)`.
