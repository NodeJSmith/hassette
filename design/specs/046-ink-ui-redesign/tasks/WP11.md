## Objectives & Success Criteria

Rewrite all Playwright E2E tests to match the new UI structure. Delete session-specific tests. Update mock fixtures for new endpoints and response shapes. After this WP: `uv run nox -s e2e` passes with comprehensive coverage of all user workflows. E2E tests cover: dashboard overview (all state variants), app navigation, handler drill-down, job drill-down, log filtering, config display, theme switching, mobile drawer navigation, and command palette search.

## Subtasks

1. Delete `tests/e2e/test_sessions.py` and `tests/e2e/test_session_toggle.py`.
2. Update `tests/e2e/conftest.py`: remove `_default_scope_all` autouse fixture. Remove `live_server_ws` fixture if it was session-specific. Update viewport constants if needed.
3. Update `tests/e2e/mock_fixtures.py`: add mock responses for `GET /apps/{key}/config` and `GET /apps/{key}/source`. Update `ConnectedPayload` mock to send `uptime_seconds` instead of `session_id`. Add WS mock messages for `invocation_completed` and `execution_completed`. Remove `session_id` from all telemetry mock query params. Add `timed_out` field to listener mock data.
4. Rewrite `tests/e2e/test_dashboard.py`: test all 5 overview state variants (healthy, single failure, multi failure, first install, quiet). Test framework error banner. Test telemetry degradation banner. Test KPI strip rendering.
5. Rewrite `tests/e2e/test_app_detail.py`: test handler list renders with human_description, modifier chips, timed_out count. Test master/detail layout (click handler → detail pane shows invocations). Test action buttons (start/stop/reload with confirm dialog). Test code tab renders source. Test config tab renders with redacted values.
6. Rewrite `tests/e2e/test_apps.py`: test sidebar app list renders with status grouping. Test multi-instance app expands in sidebar. Test clicking app navigates to detail. Test search filters apps.
7. Rewrite `tests/e2e/test_logs.py`: test log table renders with Ink styling. Test level filtering. Test app filter. Test expandable rows.
8. Write `tests/e2e/test_config.py`: test config page renders system config values. Test path fields (app_dir, data_dir, config_dir) are visible.
9. Rewrite `tests/e2e/test_navigation.py`: test sidebar navigation between pages. Test off-canvas drawer on mobile viewport. Test breadcrumb navigation on instance detail.
10. Rewrite `tests/e2e/test_theme.py`: test light/dark toggle. Verify no contrast failures on key elements (status shapes, text on backgrounds).
11. Write `tests/e2e/test_cmd_k.py`: test Cmd-K opens palette. Test search filters items. Test keyboard navigation. Test selecting an app navigates correctly.
12. Rewrite `tests/e2e/test_responsive.py`: test mobile viewport renders off-canvas drawer (not bottom nav). Test handler master/detail collapses to drill-down on narrow viewport.
13. Rewrite `tests/e2e/test_websocket.py`: test WS connection establishes. Test `uptime_seconds` in connected message. Test real-time invocation/execution updates appear.
14. Run full E2E suite: `uv run nox -s e2e`.

## Test Strategy

This WP IS the test strategy — it rewrites all E2E tests. Verify by running `uv run nox -s e2e` and confirming all tests pass.

## Review Guidance

- Every test must use `data-testid` attributes or semantic selectors — not CSS class names, which will change between redesigns.
- Mock fixtures must match the actual backend response models after WP02/WP03 changes (no `session_id` in connected payload, `timed_out` in listener data, `uptime_seconds` in connected, etc.).
- The `_default_scope_all` fixture must be gone — no session scoping in any test.
- Mobile tests must use the defined `MOBILE_VIEWPORT` constant and test the off-canvas drawer, NOT the old bottom-nav.
- The 5 dashboard state variants are acceptance criteria — each needs its own test case, not a parameterized test with shared assertions.
