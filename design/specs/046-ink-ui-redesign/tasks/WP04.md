## Objectives & Success Criteria

Rewrite the frontend plumbing layer: state management, hooks, API client, and WS types. Remove all session references. After this WP: `create-app-state.ts` has no session signals and includes time-preset and WS-event signals, `use-websocket.ts` handles new event types and sends `uptime_seconds`, `use-scoped-api.ts` passes `since` instead of `session_id`, `endpoints.ts` has new endpoint functions and no session params, and `ws-types.ts` includes new event interfaces. Session-related files are deleted. The frontend builds without errors.

## Subtasks

1. Delete `frontend/src/components/layout/session-scope-toggle.tsx` and its test.
2. Delete `frontend/src/utils/session-scope.ts` and its test.
3. Delete `frontend/src/pages/sessions.tsx` and its test.
4. Rewrite `frontend/src/state/create-app-state.ts`: remove `sessionId` and `sessionScope` signals. Add `timePreset: Signal<TimePreset>` (type: `"since-restart" | "1h" | "24h" | "7d"`, default `"since-restart"`, persisted to localStorage). Add `uptimeSeconds: Signal<number | null>` (set from WS connected message). Add `invocationCompleted: Signal<InvocationCompletedPayload | null>` and `executionCompleted: Signal<ExecutionCompletedPayload | null>` for new WS events.
5. Rewrite `frontend/src/hooks/use-websocket.ts`: remove `sessionId` write from connected handler. Read `uptime_seconds` from connected payload and set `state.uptimeSeconds.value`. Add handlers for `invocation_completed` and `execution_completed` message types — write to the corresponding signals.
6. Rewrite `frontend/src/hooks/use-scoped-api.ts`: remove all session scope resolution logic. Replace with time-window computation: resolve `since` from `state.timePreset` and `state.uptimeSeconds` ("since-restart" → `Date.now()/1000 - uptimeSeconds`, "1h" → `Date.now()/1000 - 3600`, etc.). Block fetches until `uptimeSeconds` is available (same loading-gate pattern as the old `sessionId` gate). Pass `since` to the fetcher function.
7. Update `frontend/src/api/endpoints.ts`: remove `sessionId` parameter from all telemetry functions. Add `since: number | null` parameter instead. Remove `getSessionList` and `SessionListEntry` exports. Add `getAppConfig(appKey)` and `getAppSource(appKey)` functions.
8. Update `frontend/src/api/ws-types.ts`: remove `session_id` from `WsConnectedPayload`. Add `uptime_seconds: number`. Add `WsInvocationCompletedPayload` and `WsExecutionCompletedPayload` interfaces. Add both to the `WsServerMessage` union.
9. Update `frontend/src/hooks/use-relative-time.ts` if it needs adjustment for the new tick/uptime pattern.
10. Delete `frontend/src/components/layout/bottom-nav.tsx` and its test (replaced by the off-canvas drawer in WP05). Remove the `BottomNav` import from `frontend/src/app.tsx`.
11. Verify `npm run build` succeeds with all changes.

## Test Strategy

- `state/create-app-state.test.ts`: update to verify new signals exist (`timePreset`, `uptimeSeconds`, `invocationCompleted`, `executionCompleted`) and session signals are absent.
- `hooks/use-websocket.test.ts`: update to verify `uptime_seconds` is read from connected payload, new event types are dispatched to signals, and `sessionId` is not set.
- `hooks/use-scoped-api.test.ts`: rewrite to verify `since` computation for each preset, loading-gate behavior (blocks until `uptimeSeconds` available), and that `session_id` is never passed.
- `api/client.test.ts`: verify existing fetch pattern still works (no session params).
- MSW handlers in `test/handlers.ts`: update to remove session_id from mock responses.
- Test factories in `test/factories.ts`: add `createInvocationCompletedPayload()` and `createExecutionCompletedPayload()` factories.
- Run `npm run test`.

## Review Guidance

- Grep for `session` (case-insensitive) across all frontend `src/` files — zero matches expected outside of generated-types.ts (which is auto-generated and may retain session references from backend models that still exist internally).
- The loading gate in `use-scoped-api.ts` must block on `uptimeSeconds` being non-null before computing `since` — otherwise the first fetch fires with `since=NaN`.
- `TimePreset` type must be exported and used consistently — no string literals for presets outside the type definition.
- The `since` computation must use `Date.now()/1000` (epoch seconds) not `Date.now()` (epoch milliseconds).
