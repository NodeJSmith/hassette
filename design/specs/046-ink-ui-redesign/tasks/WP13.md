## Reference

## Objectives

Enhance the sidebar to match the mockup: add version display, status group headers with counts and collapse, and invocation counts next to app names.

## Subtasks

### 1. Version display

**Mockup ref:** `sidebar.jsx:84–89`

Below the "hassette" wordmark, add a line showing:
```
v{version} · ● connected
```
in `--font-mono`, `--fs-micro`, `--ink-3`.

**Version source:** `version` field on `SystemStatus` (added in WP16), available via `/api/system/status` and the WS `connected` message.

**Connection state variation** (mockup `sidebar.jsx:87–88`): When WS is disconnected/reconnecting, change the StatusShape to warn-tone and the label to "reconnecting…". Use the `connection` signal from app state. Do NOT hard-code "connected" as a static string.

### 2. APPS header with count

**Mockup ref:** `sidebar.jsx:130–132`

Above the app search input, add a section header:
```
APPS  {count}
```
using `SectionHeader`-style: uppercase, `--fs-micro`, `--ink-3`, with count in mono.

When search is active, show `{filteredCount}/{totalCount}` instead of just `{count}` (mockup `sidebar.jsx:131`).

### 3. Status group headers with collapse

**Mockup ref:** `sidebar.jsx:62–69, 154–220`

Replace the flat sorted app list with collapsible status groups:

| Group | Tone | Label | Default state |
|-------|------|-------|--------------|
| err | err | FAILING | expanded |
| blocked | err | BLOCKED | expanded |
| warn | warn | SLOW | expanded |
| ok | ok | RUNNING | collapsed |
| stopped | mute | STOPPED | expanded |
| disabled | mute | DISABLED | collapsed |

Each group header shows: collapse chevron (▼/▶), StatusShape dot, group label in uppercase, count badge.

Empty groups are hidden. Clicking the header toggles collapse. Group headers need `role="button"`, `tabIndex={0}`, `aria-expanded`, and keyboard handlers (Enter/Space) — follow the existing `ht-sidebar__app-expand` button pattern.

**Exception:** When system state is all-healthy, force the RUNNING group open regardless of user's collapse state (mockup `sidebar.jsx:71` — `groupOpenEffective` pattern).

**Group assignment logic** (mockup `sidebar.jsx:40–59`):
- `status === "blocked"` → blocked
- `status === "disabled"` → disabled
- `status === "failed" || status === "crashed"` → err
- Status with warn tone (exhausted_cooling, stopping, shutting_down) → warn
- `status === "stopped" || status === "not_started"` → stopped
- `status === "exhausted_dead"` → err (permanent failure — budget exhausted)
- Everything else → ok
- Multi-instance: use worst-of-children status

**Important:** `exhausted_dead` must map to err (not ok). Without this, an app that has permanently crashed will appear in the RUNNING group.

### 4. Invocation counts

**Mockup ref:** `sidebar.jsx` app entries show a count right-aligned next to each app name.

Add invocation count (right-aligned, mono dim) next to each app name in the sidebar. Data source: `recent_invocations_1h` field on `AppManifestResponse` (added in WP16) — a fixed 1h window count. Do NOT use the app-grid endpoint (it's time-scoped and only fetched on the dashboard — coupling the sidebar to `timePreset` would cause unexpected refetches on every time-window change).

The existing `auto_loaded` badge (mockup `sidebar.jsx:269–276`) should be preserved between the app name and the invocation count. It already exists in the current implementation.

### 5. Exclusions

Do NOT add an "events" nav item. The mockup shows 4 nav items (overview, events, logs, config) but the events page is explicitly deferred. The current 3 nav items (Overview, Logs, Config) are correct.
