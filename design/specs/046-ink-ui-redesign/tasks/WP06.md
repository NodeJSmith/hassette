## Objectives & Success Criteria

Rebuild the dashboard/overview page with all 5 state variants (healthy, single failure, multiple failures, first install, quiet). After this WP, the overview page renders the correct hero card for each state, shows summary cards (apps, activity, system health), displays the recent errors table, shows framework error banners separately from app errors, and shows the telemetry degradation banner when drop counters are non-zero. Real-time updates (via WS invocation/execution signals) trigger KPI and error feed refreshes.

## Subtasks

1. Rewrite `frontend/src/pages/dashboard.tsx`: detect system state from manifests + KPIs + errors data. Render the appropriate hero card for each of the 5 states. Add the greeting text (Newsreader display font). Add metadata line (HA version, app count, run rate).
2. Rewrite `frontend/src/components/dashboard/kpi-strip.tsx`: render KPI cards using Ink tokens. Include error rate, handlers, jobs, uptime. Use `var(--font-mono)` for values. Error rate card gets prominence styling.
3. Rewrite `frontend/src/components/dashboard/app-grid.tsx` and `app-card.tsx`: render app cards with status shapes, display name, run count, error rate. Link to app detail. Inactive apps (stopped/disabled) in a secondary grid with reduced opacity.
4. Rewrite `frontend/src/components/dashboard/error-feed.tsx`: table with columns — time, app, source location (module.function:line), exception type + message, age. Use `var(--font-mono)` for timestamps and locations. Error rows get `var(--err-bg)` tint.
5. Rewrite `frontend/src/components/dashboard/framework-health.tsx`: show framework error count as a chip separate from app errors. Collapsible detail if errors exist. Uses `var(--warn)` or `var(--err)` based on count.
6. Rewrite `frontend/src/components/dashboard/service-status-panel.tsx`: show service health (resource_name, status, role) with status shapes. Subscribe to `serviceStatus` signal for real-time updates.
7. Rewrite the alert banner component in `components/layout/`: telemetry degradation banner (reads `telemetryDegraded` + drop counter signals). Framework error banner for boot-time config issues. Both use `var(--warn-bg)` or `var(--err-bg)` with border accent.
8. Wire real-time refresh: when `invocationCompleted` or `executionCompleted` signals fire, debounce and refetch KPIs + error feed (use existing `useDebouncedEffect` pattern with 500ms delay, 2000ms max-wait).
9. Add component CSS to `global.css` for all overview components.

## Test Strategy

- `pages/dashboard.test.tsx`: test each of the 5 state variants renders the correct hero card. Mock different manifest/KPI/error responses for each state.
- `components/dashboard/kpi-strip.test.tsx`: renders all KPI cards with correct values from mock data.
- `components/dashboard/app-grid.test.tsx`: renders app cards, separates active/inactive, links navigate correctly.
- `components/dashboard/error-feed.test.tsx`: renders error rows with all columns, empty state when no errors.
- `components/dashboard/framework-health.test.tsx`: renders framework error count, collapses/expands.
- `components/dashboard/service-status-panel.test.tsx`: renders service rows with status shapes.
- Run `npm run test`.
