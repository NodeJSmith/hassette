## Step 5: Warnings (non-blocking)

**W1 — T05 bundles FR#12/AC#12 (accessor lazy import fix) with test migration.**
The lazy import fix is a one-line change to a non-test file (`accessors.py`). Bundling it into T05 is pragmatic but slightly obscures it — a reviewer scanning T05's title ("Migrate test field-access paths") might not expect a production code change. The design doc's "Fixes included" section explicitly scopes it here, so this is intentional. No action required; flagged for awareness.

**W2 — FR#8 and AC#7 are claimed by both T02 and T06.**
T02 implements the runtime behavior (Subscription.registration_task wiring); T06 implements the documentation. Dual coverage is appropriate and intentional. Both tasks have Verify items. No conflict.

**W3 — T04 depends on T01, T02, and T03, but T03 only depends on T01.**
T03 (Router extraction) is listed as depending only on T01. T04 depends on T01, T02, and T03. This ordering is correct — T03 can proceed in parallel with T02 since Router extraction does not require the Bus.on() split. No issue.

**W4 — T04 Step 6 note about `_create_cancel_listener` assert timing.**
The Focus section correctly reasons through the assert-on-`_timer` ordering (attach_timer() stores the reference before start() is called). This is sound, but the assert should be verified empirically during implementation — the timing argument is subtle enough that a test covering this path (assert fires correctly) would be valuable. The cancel-listener factory test in T04 Step 10 does cover this path indirectly.
