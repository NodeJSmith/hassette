## Warnings

**W1 — FR#5 split across three tasks with incomplete Verify at T03:**
FR#5 covers Pyright's `reportUnusedCoroutine` for all protected methods including scheduler methods (`add_job`, `run_*`). T02 and T04 each include an FR#5 Verify item for bus and api methods respectively. T03 does **not** include FR#5 in its `implements` list, and its Verify section contains no Pyright-probe item for scheduler methods. The gap is compensated by T06's AC#3/FR#5 probe covering the full canonical list, so this is a warning rather than a gap. The implementer should be aware that Pyright probe coverage of scheduler bare calls is implicit via T06, not explicitly verified in T03.

**W2 — T01 `implements` includes AC#1 and AC#5 but not AC#2:**
AC#2 requires verifying that awaiting a protected method returns the expected type with no warning. T01 builds the handle machinery but no converted public methods exist yet, so a full AC#2 test cannot be written in T01. AC#2 is covered in T02/T03/T04 which is correct sequencing. This is consistent with the task split but worth flagging so the implementer does not expect AC#2 coverage from T01's test suite in isolation.

**W3 — documentation coverage split:**
FR#12's documentation requirement (the shutdown-timing limitation in the troubleshooting page) is verified by T06's Verify item ("the troubleshooting docs document the `self.sub = coro()` shutdown-timing limitation"). T01 also claims FR#12 but its Verify item covers the *runtime observable behavior* test ("a handle held alive does not warn until collected"), not the docs. This split is coherent and correctly sequenced, but the implementer should note that T01's FR#12 claim is about the runtime guarantee and T06's is about the documentation of it.
