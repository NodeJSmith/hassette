## Objectives & Success Criteria

Every Service subclass declares a `restart_spec` class attribute matching its criticality and failure profile per the design. `SchemaVersionError` is extracted as a typed exception from the current bare `RuntimeError` in `DatabaseService._handle_schema_version()`.

- All 8 services declare `restart_spec` with values matching the design doc
- `SchemaVersionError` exists in `src/hassette/exceptions.py` (not a `FatalError` subclass — it's handled by `fatal_error_names`, not the CRASHED path)
- `DatabaseService._handle_schema_version()` raises `SchemaVersionError` instead of `RuntimeError`
- `WebsocketService` does NOT list `InvalidAuthError` in `non_retryable_error_names` (it's a `FatalError` subclass, handled by CRASHED path)
- No `__init_subclass__` warnings fire for any production Service subclass
- Existing tests pass (no behavioral change — watcher isn't using specs yet until WP02 lands)

## Subtasks

1. Add `SchemaVersionError` to `src/hassette/exceptions.py` as a standalone exception (inherits from `Exception`, not `FatalError`)
2. Update `DatabaseService._handle_schema_version()` in `src/hassette/core/database_service.py` to raise `SchemaVersionError` instead of `RuntimeError` for schema version mismatches
3. Add `restart_spec` class attribute to each service:
   - `BusService`: `RestartSpec(restart_type=RestartType.PERMANENT, budget_intensity=2, budget_period_seconds=30)`
   - `SchedulerService`: `RestartSpec(restart_type=RestartType.PERMANENT, budget_intensity=2, budget_period_seconds=30)`
   - `WebsocketService`: `RestartSpec(restart_type=RestartType.TRANSIENT, budget_intensity=5, budget_period_seconds=300, startup_timeout_seconds=60)`
   - `DatabaseService`: `RestartSpec(restart_type=RestartType.TRANSIENT, budget_intensity=3, budget_period_seconds=120, fatal_error_names=("SchemaVersionError",))`
   - `WebApiService`: `RestartSpec(restart_type=RestartType.TRANSIENT, budget_intensity=3, budget_period_seconds=60)`
   - `CommandExecutor`: `RestartSpec(restart_type=RestartType.TRANSIENT, budget_intensity=3, budget_period_seconds=120)`
   - `FileWatcherService`: `RestartSpec(restart_type=RestartType.TEMPORARY, budget_intensity=3, budget_period_seconds=60)`
   - `WebUiWatcherService`: `RestartSpec(restart_type=RestartType.TEMPORARY, budget_intensity=3, budget_period_seconds=60)`
4. Add necessary imports to each service file (`RestartSpec`, `RestartType` from appropriate modules)
5. Write unit test for SchemaVersionError extraction

## Test Strategy

**File:** `tests/unit/test_schema_version_error.py`

- `test_schema_version_error_raised_when_db_ahead` — mock a database schema version ahead of code → `SchemaVersionError` raised (not `RuntimeError`)
- `test_schema_version_error_is_not_fatal_error` — verify `SchemaVersionError` is not a `FatalError` subclass

**File:** `tests/unit/test_service_restart_specs.py`

- `test_all_services_declare_restart_spec` — import all 8 services, verify each has `restart_spec` in `cls.__dict__` (not inherited)
- `test_bus_service_permanent` — verify BusService.restart_spec.restart_type == PERMANENT
- `test_websocket_service_transient` — verify WebsocketService.restart_spec.restart_type == TRANSIENT
- `test_file_watcher_temporary` — verify FileWatcherService.restart_spec.restart_type == TEMPORARY
- `test_database_service_fatal_errors` — verify "SchemaVersionError" in DatabaseService.restart_spec.fatal_error_names

## Review Guidance

- Verify `SchemaVersionError` does NOT inherit from `FatalError` — it must reach the FAILED path (not CRASHED) so that `fatal_error_names` matching works
- Verify `InvalidAuthError` does NOT appear in any service's `non_retryable_error_names` or `fatal_error_names` — it's already handled by the CRASHED path
- Verify each service's `restart_spec` values match the design doc exactly
- Check that `_handle_schema_version()` raises `SchemaVersionError` in all code paths where it currently raises `RuntimeError` for version mismatches (not just one of them)
- Confirm WebsocketService's restart_spec has no entries in `non_retryable_error_names` or `fatal_error_names` that reference internal retry state — validates the design boundary that watcher and internal retries are independent (FR11)
