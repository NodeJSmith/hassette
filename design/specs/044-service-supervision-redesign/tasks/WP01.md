## Objectives & Success Criteria

Introduce the foundational types that all subsequent WPs build on: `RestartType` enum, `RestartSpec` frozen dataclass, `RestartBudget` sliding-window tracker, and the `__init_subclass__` warning on `Service`. After this WP, the types exist, are tested, and the Service base class has a `restart_spec` class attribute — but the ServiceWatcher does not yet use them.

- `RestartType` enum with `PERMANENT`, `TRANSIENT`, `TEMPORARY` values exists in `src/hassette/types/enums.py`
- `RestartSpec` frozen dataclass exists in `src/hassette/resources/base.py` with all fields and defaults per the design
- `Service.restart_spec` class attribute defaults to `RestartSpec()`
- Concrete `Service` subclasses that don't declare `restart_spec` trigger a `warnings.warn()` via `__init_subclass__`
- `RestartBudget` class exists in `src/hassette/core/service_watcher.py` with `record_restart()`, `is_exhausted()`, `reset()`, `_evict_expired()`
- `EXHAUSTED_DEAD` and `EXHAUSTED_COOLING` values exist in `ResourceStatus` enum
- `retry_at: float | None = None` field exists on `ServiceStatusPayload`
- All unit tests pass

## Subtasks

1. Add `PERMANENT`, `TRANSIENT`, `TEMPORARY` to `RestartType` enum in `src/hassette/types/enums.py` (follow existing `StrEnum` pattern)
2. Add `EXHAUSTED_DEAD` and `EXHAUSTED_COOLING` to `ResourceStatus` enum in `src/hassette/types/enums.py`
3. Add `RestartSpec` frozen dataclass to `src/hassette/resources/base.py` with all fields: `restart_type`, `non_retryable_error_names`, `fatal_error_names`, `backoff_base_seconds`, `backoff_multiplier`, `backoff_max_seconds`, `budget_intensity`, `budget_period_seconds`, `startup_timeout_seconds`, `cooldown_seconds`, `max_cooldown_cycles`
4. Add `restart_spec: ClassVar[RestartSpec] = RestartSpec()` class attribute to `Service`
5. Extend `Service.__init_subclass__` to emit `warnings.warn()` when a concrete subclass does not declare its own `restart_spec` (match existing `depends_on` pattern)
6. Add `retry_at: float | None = None` field to `ServiceStatusPayload` in `src/hassette/events/hassette.py`
7. Add `RestartBudget` class to `src/hassette/core/service_watcher.py` with `record_restart()`, `is_exhausted()`, `reset()`, `_evict_expired()` methods
8. Write unit tests for `RestartSpec` and `RestartBudget`

## Test Strategy

**File:** `tests/unit/test_restart_spec.py`

- `test_restart_spec_defaults` — verify all default field values match the design (TRANSIENT, budget_intensity=5, etc.)
- `test_restart_spec_frozen` — verify frozen dataclass raises on attribute assignment
- `test_restart_spec_custom_values` — verify custom field values are preserved

**File:** `tests/unit/test_restart_budget.py`

- `test_budget_empty_not_exhausted` — new budget with no restarts returns `is_exhausted() == False`
- `test_budget_exhausted_at_intensity` — record `intensity` restarts, verify `is_exhausted() == True`
- `test_budget_not_exhausted_below_intensity` — record `intensity - 1` restarts, verify `is_exhausted() == False`
- `test_budget_evicts_expired_timestamps` — record restarts, advance time past window, verify old entries evicted
- `test_budget_reset_clears_all` — record restarts, call `reset()`, verify `is_exhausted() == False`
- `test_budget_boundary_timestamp` — record restart exactly at the window boundary, verify correct eviction behavior
- `test_budget_mixed_expired_and_current` — some timestamps expired, some current, verify only current count

**File:** `tests/unit/test_service_init_subclass.py`

- `test_service_subclass_without_restart_spec_warns` — subclass Service without declaring restart_spec, verify `warnings.warn()` fires
- `test_service_subclass_with_restart_spec_no_warning` — subclass Service with restart_spec declared, verify no warning

## Review Guidance

- Verify `RestartSpec` field names and defaults match the design doc exactly (especially `non_retryable_error_names` not `non_retryable_errors`)
- Verify `RestartBudget` uses `time.monotonic()` not `time.time()` (monotonic clock is essential for sliding window correctness)
- Verify `__init_subclass__` warning does not fire for abstract/intermediate classes — only concrete Service subclasses
- Verify `EXHAUSTED_DEAD` and `EXHAUSTED_COOLING` are added to `ResourceStatus` in `types/enums.py`, not `domain_models.py`
- `RestartBudget` tests that involve time should use `unittest.mock.patch("time.monotonic")` to control the clock
