## Objectives & Success Criteria

Replace the ServiceWatcher's global-config-based restart logic with per-service RestartSpec-driven behavior. After this WP, the watcher reads each service's `restart_spec`, uses `RestartBudget` for sliding-window tracking, implements the three-layer error routing policy, the in-restart guard, shutdown-safe sleeping, budget reset on recovery, and the three exhaustion paths (PERMANENT→shutdown, TRANSIENT→cooldown, TEMPORARY→dead).

- `restart_service()` reads the service's `restart_spec` instead of global config
- Fatal errors (`fatal_error_names`) trigger immediate system shutdown regardless of restart type
- Non-retryable errors (`non_retryable_error_names`) skip restart and go to exhaustion handling
- Budget tracking uses `RestartBudget` per-service sliding window instead of monotonic counter
- `_restarting` set prevents double budget depletion from concurrent/in-restart FAILED events
- All `asyncio.sleep()` calls replaced with `asyncio.wait_for(shutdown_event.wait(), timeout=duration)` pattern
- Long-cooldown task for TRANSIENT exhaustion is spawned in `task_bucket`, keyed per-service, respects `max_cooldown_cycles`
- Budget resets on confirmed recovery (service reaches RUNNING+ready)
- `_on_service_running()` uses per-service `startup_timeout_seconds` instead of global config
- Readiness timeout does not affect the restart budget
- `restart()` exceptions are caught, logged, and service is left in FAILED state without double-counting budget
- Post-BusService-recovery reconciliation scan checks for missed FAILED events
- All integration tests pass

## Subtasks

1. Replace `_attempt_counter: dict[str, int]` with `_budgets: dict[str, RestartBudget]` and `_restarting: set[str]` and `_cooldown_tasks: dict[str, asyncio.Task]` on the ServiceWatcher class
2. Add helper method `_get_service_and_spec(event)` that resolves the service instance and its `restart_spec` from a `HassetteServiceEvent`
3. Rewrite `restart_service()` handler with the six-step flow from the design: check fatal → check non-retryable → check budget → record restart → backoff → restart. Add in-restart guard (check/add to `_restarting` set). Wrap `service.restart()` in try/except to catch and log exceptions without double-counting
4. Implement exhaustion handling: PERMANENT emits CRASHED + shutdown, TRANSIENT emits `EXHAUSTED_COOLING` (with `retry_at` on `ServiceStatusPayload`) + schedules cooldown task, TEMPORARY emits `EXHAUSTED_DEAD`
5. Implement `_cooldown_and_retry()` method: shutdown-safe sleep for `cooldown_seconds`, check `shutdown_event`, reset budget, attempt restart. Track cooldown cycle count; transition to `EXHAUSTED_DEAD` if `max_cooldown_cycles` exceeded. Spawn in `task_bucket`, cancel existing cooldown for same service before starting new one
6. Replace all `asyncio.sleep(backoff)` with `asyncio.wait_for(hassette.shutdown_event.wait(), timeout=backoff)` — early wake = abort
7. Rewrite `_on_service_running()` to use `service.restart_spec.startup_timeout_seconds` instead of global config. On successful readiness, call `budget.reset()` and clear service from `_restarting` set. On timeout, log warning only — no budget impact
8. Add `_reconcile_after_bus_recovery()` method: after BusService reaches RUNNING+ready, iterate `hassette.services`, check each `service.status == ResourceStatus.FAILED`, and look up `_budgets` for a matching entry. Services in FAILED state with no budget entry had their FAILED event dropped during the BusService restart window — treat as if a FAILED event had been received (enter the normal restart flow)
9. Remove references to global config fields (`service_restart_max_attempts`, `service_restart_backoff_seconds`, etc.) from `restart_service()` and `_on_service_running()`
10. Remove the `log_service_event()` handler's reference to `service_restart_max_attempts` in the log message (if present)

## Review Guidance

- Verify `_restarting` set is cleared correctly: on RUNNING event AND on new FAILED event after restart completes
- Verify `_cooldown_tasks` dict is cleaned up: old cooldown cancelled before new one starts for same service

- Exception from `service.restart()` must NOT propagate out of the bus handler — it must be caught and logged
