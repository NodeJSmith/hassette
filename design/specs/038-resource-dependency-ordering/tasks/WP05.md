## Objectives & Success Criteria

`Hassette` shuts down children in reverse topological order — services that depend on others shut down first, services depended upon shut down last. The existing `Hassette.on_shutdown()` pre-sequence for `CommandExecutor` is removed (the graph handles it). An integration test verifies shutdown order matches reverse init order.

## Subtasks

1. Override `_ordered_children_for_shutdown()` on `Hassette` in `src/hassette/core/core.py` to return instances in reverse topological order:
   ```python
   def _ordered_children_for_shutdown(self) -> list[Resource]:
       type_to_instance = {type(c): c for c in self.children}
       return [type_to_instance[t] for t in reversed(self._init_order) if t in type_to_instance]
   ```
   This feeds the existing `_finalize_shutdown()` machinery which handles error isolation via `asyncio.gather(..., return_exceptions=True)`.
2. Remove the `CommandExecutor`-before-`DatabaseService` pre-sequence from `Hassette.on_shutdown()` (`src/hassette/core/core.py`, lines ~407-419). The graph-derived order already places CommandExecutor before DatabaseService in shutdown. Simplify `on_shutdown()` to only contain non-ordering logic (if any remains after the removal; if empty, remove the override entirely).
3. Write an integration test in `tests/integration/test_resource_deps.py`:
   - `test_shutdown_order_is_reverse_init_order` — create a Hassette instance, verify that `_ordered_children_for_shutdown()` returns instances in the reverse of `_init_order` types

## Test Strategy

- `test_shutdown_order_is_reverse_init_order` in `tests/integration/test_resource_deps.py` — verifies the shutdown order is the mathematical reverse of the initialization order by comparing type sequences

## Review Guidance

- `_ordered_children_for_shutdown()` must use `type(c)` dict lookup (exact match), NOT `isinstance` — `_init_order` contains the exact types from `type(c)`, so `isinstance` would be wrong here and could match supertypes
- The `if t in type_to_instance` guard handles the case where `_init_order` includes a type that was removed from `children` (shouldn't happen, but defensive)
- Verify `Hassette.on_shutdown()` is simplified correctly — the `try/except` around CommandExecutor shutdown was defensive; with graph ordering, it's redundant
- The existing `_finalize_shutdown()` gather provides error isolation — do NOT replace it with a sequential loop
