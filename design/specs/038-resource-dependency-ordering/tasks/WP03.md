## Objectives & Success Criteria

All 8 framework services declare `depends_on` and their manual `wait_for_ready()` calls in lifecycle hooks are removed. The `ready_event.wait()` in `WebApiService.before_initialize()` is removed. The existing test suite passes (no behavioral change — the auto-wait from WP02 now handles what the manual calls did).

## Subtasks

1. **CommandExecutor** (`src/hassette/core/command_executor.py`): Add `depends_on: ClassVar[list[type[Resource]]] = [DatabaseService]`. Remove `await self.hassette.wait_for_ready([self.hassette.database_service])` from `on_initialize()` (line ~118). Keep the runtime `wait_for_ready` calls in `register_listener()` (line ~500) and `register_job()` (line ~512) — those are runtime guards, not initialization ordering.
2. **ApiResource** (`src/hassette/core/api_resource.py`): Add `depends_on: ClassVar[list[type[Resource]]] = [WebsocketService]`. Remove `await self.hassette.wait_for_ready(self.hassette._websocket_service)` from `on_initialize()` (line ~67).
3. **StateProxy** (`src/hassette/core/state_proxy.py`): Add `depends_on: ClassVar[list[type[Resource]]] = [WebsocketService, ApiResource, BusService, SchedulerService]`. Remove the 4-element `wait_for_ready` call from `on_initialize()` (lines ~55-62).
4. **AppHandler** (`src/hassette/core/app_handler.py`): Add `depends_on: ClassVar[list[type[Resource]]] = [WebsocketService, ApiResource, BusService, SchedulerService, StateProxy]`. Remove `await self.hassette.wait_for_ready(self.hassette._websocket_service)` from `on_initialize()` (line ~92). Remove the 5-element `wait_for_ready` call and warning guard from `after_initialize()` (lines ~102-111). Simplify `after_initialize()` to unconditionally spawn `self.lifecycle.bootstrap_apps()`.
5. **RuntimeQueryService** (`src/hassette/core/runtime_query_service.py`): Add `depends_on: ClassVar[list[type[Resource]]] = [BusService, StateProxy, AppHandler]`. Remove the 3-element `wait_for_ready` call from `on_initialize()` (lines ~69-75). Preserve the `run_web_api=False` early-return guard.
6. **TelemetryQueryService** (`src/hassette/core/telemetry_query_service.py`): Add `depends_on: ClassVar[list[type[Resource]]] = [DatabaseService]`. Remove `await self.hassette.wait_for_ready([self.hassette.database_service])` from `on_initialize()` (line ~209). Preserve WAL mode validation. Preserve the `run_web_api=False` early-return guard.
7. **WebApiService** (`src/hassette/core/web_api_service.py`): Add `depends_on: ClassVar[list[type[Resource]]] = [RuntimeQueryService]`. Remove `await self.hassette.ready_event.wait()` from `before_initialize()` (line ~36). Remove `await self.hassette.wait_for_ready([self.hassette._runtime_query_service])` from `on_initialize()` (line ~45). Preserve the `run_web_api=False` early-return guard.
8. **ServiceWatcher** (`src/hassette/core/service_watcher.py`): Add `depends_on: ClassVar[list[type[Resource]]] = [BusService]`. No existing `wait_for_ready` to remove — this is a new declaration for a previously-undeclared dependency.
9. Add required imports to each file: `ClassVar` from `typing`, and the dependency types (e.g., `DatabaseService`, `WebsocketService`).

## Test Strategy

No new tests — this WP modifies existing behavior to use the auto-wait mechanism from WP02 instead of manual calls. The existing test suite validates that all services initialize correctly. If any service's test fails, it indicates a migration error (wrong dep type, missing dep, or removed code that had side effects beyond the wait).

## Review Guidance

- Each migration must be 1:1: every removed `wait_for_ready` call must have a corresponding type in `depends_on`
- Runtime `wait_for_ready` calls (in `CommandExecutor.register_listener/register_job`) must NOT be removed — verify they're preserved
- `ready_event.wait()` is removed ONLY from `WebApiService` — verify it's preserved in `BusService`, `SchedulerService`, `FileWatcherService` (those are coordinator-level gates, not service deps)
- The `run_web_api=False` early-return guards in RuntimeQueryService, TelemetryQueryService, and WebApiService must be preserved
- `AppHandler.after_initialize()` must be simplified to unconditionally spawn `bootstrap_apps()` — the old warning log is no longer needed
- Check that imports are added at module level (not lazy imports per project rules)
