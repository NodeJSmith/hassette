## Objectives & Success Criteria

`Hassette.__init__` validates the dependency graph at construction time (missing types raise `ValueError`, cycles raise `ValueError` with cycle path). `_phase_services` dict and `StartupPhase` enum are removed. `run_forever()` startup is simplified to a direct database-first conditional. The computed initialization order is stored as `_init_order` and logged at INFO level. Four broken tests in `test_core.py` are rewritten. `StartupPhase` removal is documented as a breaking change.

## Subtasks

1. In `Hassette.__init__` (`src/hassette/core/core.py`), after all `add_child()` calls, replace the `_phase_services` validation block (lines ~128-137) with graph validation:
   - Compute `all_types = list(dict.fromkeys(type(c) for c in self.children))` (deterministic, deduplicated)
   - Validate every `depends_on` reference exists among children via `issubclass` — raise `ValueError` naming the declaring class and missing type
   - Call `topological_sort(all_types)` and store result as `self._init_order`
   - Log the dependency graph at INFO level (only services with non-empty `depends_on`)
2. Import `topological_sort` from `hassette.utils.service_utils` in `core.py`.
3. Simplify `run_forever()`: replace `self._start_database()` and `self._start_remaining_resources()` with inline logic:
   - `self._database_service.start()` (Phase 1)
   - Wait for database, create session (unchanged)
   - `for child in self.children: if child is not self._database_service: child.start()` (Phase 2)
4. Remove `_start_database()` and `_start_remaining_resources()` methods from `Hassette`.
5. Remove `_phase_services` attribute from `Hassette.__init__`.
6. Remove `StartupPhase` enum from `src/hassette/types/enums.py`.
7. Remove `StartupPhase` from imports and `__all__` in `src/hassette/types/__init__.py`.
8. Remove `StartupPhase` import from `src/hassette/core/core.py`.
9. Rewrite 4 tests in `tests/integration/test_core.py` (lines ~327-364):
   - Replace `test_startup_phase_database_contains_only_database_service` → `test_database_service_starts_first` (verify database_service.start() is called before other services)
   - Replace `test_startup_phase_services_contains_all_non_database_children` → `test_init_order_contains_all_children` (verify `_init_order` includes all child types)
   - Replace `test_start_database_starts_only_phase_members` → `test_init_order_has_no_cycles` (verify topological_sort completes without raising)
   - Replace `test_start_remaining_resources_starts_only_services_phase` → `test_graph_validation_catches_missing_type` (verify ValueError on bad depends_on)
10. Remove the `StartupPhase` import from `tests/integration/test_core.py`.

## Test Strategy

Rewrite 4 tests in `tests/integration/test_core.py`:
- `test_database_service_starts_first` — verify database_service is started before other children in run_forever
- `test_init_order_contains_all_children` — verify `_init_order` contains all child types from `Hassette.children`
- `test_init_order_has_no_cycles` — verify the topological sort completes without raising for the real service graph
- `test_graph_validation_catches_missing_type` — add a dummy `depends_on` referencing a nonexistent type and verify `ValueError`

- `dict.fromkeys` must be used instead of `set()` for deterministic ordering — this was a round 1 finding
