## Objectives & Success Criteria

Integration tests validate startup ordering enforcement via the gate pattern, cycle detection at `__init__` time, and missing-type detection at `__init__` time. A harness consistency test validates that `_DEPENDENCIES` entries correspond to real service `depends_on` declarations, preventing silent drift. All tests pass.

## Subtasks

1. Add `_COMPONENT_CLASS_MAP: dict[str, type[Resource]]` to `src/hassette/test_utils/harness.py`, mapping harness string names to real service classes:
   ```python
   _COMPONENT_CLASS_MAP: dict[str, type[Resource]] = {
       "bus": BusService,
       "scheduler": SchedulerService,
       "app_handler": AppHandler,
       "state_proxy": StateProxy,
       "state_registry": StateRegistry,
       # api_mock and file_watcher are harness-specific; no real class mapping
   }
   ```
   Add required imports for the service classes.
2. Write `tests/unit/test_harness_dependency_consistency.py`:
   - `test_harness_dependencies_match_real_depends_on` ‚Äî for each entry in `_DEPENDENCIES` that has a `_COMPONENT_CLASS_MAP` entry, verify the real service class's `depends_on` contains types matching the harness dep names (via `_COMPONENT_CLASS_MAP` lookup). This is one-directional by design ‚Äî harness `_DEPENDENCIES` is a coarser subset.
3. Write integration tests in `tests/integration/test_resource_deps.py`:
   - `test_service_with_depends_on_waits_for_dep` ‚Äî use `asyncio.Event` gate pattern: create a stub service with `depends_on = [GatedService]`, block GatedService's `mark_ready()` behind an event, verify the dependent service does not proceed to `on_initialize()` until the gate is released
   - `test_service_without_depends_on_proceeds_immediately` ‚Äî create a stub service with empty `depends_on`, verify it proceeds to `on_initialize()` without blocking
   - `test_cycle_detection_at_init_time` ‚Äî create a Hassette with two services that depend on each other, verify `ValueError` at `__init__` time
   - `test_missing_type_detection_at_init_time` ‚Äî create a service with `depends_on` referencing a type not in `Hassette.children`, verify `ValueError` at `__init__` time

## Test Strategy

- `tests/unit/test_harness_dependency_consistency.py` ‚Äî reads `_DEPENDENCIES` and `_COMPONENT_CLASS_MAP` from the harness module and `depends_on` from real service classes; pure metadata comparison, no async
- `tests/integration/test_resource_deps.py` ‚Äî uses the gate pattern documented in CLAUDE.md for timing-sensitive startup ordering tests; uses real or near-real `Hassette` instances for cycle/missing-type detection

## Review Guidance

- The gate pattern must use `asyncio.Event` (not `asyncio.sleep`) ‚Äî per CLAUDE.md regression test patterns
- The harness consistency test must be one-directional ‚Äî it validates harness ‚äÜ real, not real ‚äÜ harness (the harness is intentionally coarser)
- `_COMPONENT_CLASS_MAP` entries for `api_mock` and `file_watcher` may be omitted if they don't map cleanly to real service classes ‚Äî document why
- Integration tests for cycle/missing-type detection should use lightweight stub services, not real framework services ‚Äî keeps tests fast and isolated
