## Objectives & Success Criteria

Add a `topological_sort()` function to `src/hassette/utils/service_utils.py` that accepts a list of `Resource` types, computes a valid initialization order from their `depends_on` ClassVars, and raises `ValueError` with a human-readable cycle path on cycle detection. All unit tests pass covering linear chains, diamonds, cycles, self-deps, empty input, and stable ordering.

## Subtasks

1. Add `topological_sort(types: list[type[Resource]]) -> list[type[Resource]]` to `src/hassette/utils/service_utils.py`. Implement iterative DFS with three-color marking (WHITE/GRAY/BLACK) and an explicit `path` stack for cycle path reconstruction. Return types in initialization order (deps before dependents). Raise `ValueError` with a readable cycle path (e.g., `"Cycle detected: A → B → A"`) when a back edge is found.
2. Export `topological_sort` from `src/hassette/utils/__init__.py`.
3. Write unit tests in `tests/unit/test_topological_sort.py`:
   - `test_linear_chain` — `A → B → C` returns `[C, B, A]`
   - `test_diamond` — `A → {B, C} → D` deduplicates D, returns valid order where D precedes B and C, which precede A
   - `test_empty_list` — returns empty list
   - `test_cycle_raises` — `A → B → A` raises `ValueError` with both node names in the message
   - `test_self_dependency_raises` — `A → A` raises `ValueError`
   - `test_disconnected_components` — nodes with no deps are included in output
   - `test_stable_ordering` — nodes at the same depth appear in input order (deterministic)
   - Use lightweight stub `Resource` subclasses defined in the test module with `depends_on` ClassVars

## Test Strategy

All tests in `tests/unit/test_topological_sort.py`. Uses stub `Resource` subclasses (no real services needed). Tests cover the algorithm's correctness, error reporting, and determinism guarantees.

## Review Guidance

- The implementation must use iterative DFS, not recursive — verify no recursion
- Cycle path must include the full cycle (not just "cycle exists") — check the `ValueError` message format
- Input order must be preserved for nodes at the same depth — verify stable sort behavior
- The function must accept types with empty `depends_on` as root nodes
- `depends_on` is read from `type.depends_on` (ClassVar), not from instances
