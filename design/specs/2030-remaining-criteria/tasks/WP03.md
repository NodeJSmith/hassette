## Objectives & Success Criteria

- A "Sessions" nav item appears in the sidebar with a history/clock icon.
- Clicking it navigates to `/sessions`, which renders a table of sessions with status badges, formatted timestamps, duration, and error info.
- The page fetches data from `GET /telemetry/sessions`.
- E2E tests verify the page renders and the sidebar link works.

## Subtasks

1. Add `IconHistory` SVG icon to `frontend/src/components/shared/icons.tsx` (Lucide clock or history icon).
2. Add `getSessionList(limit?: number)` fetcher to `frontend/src/api/endpoints.ts` calling `GET /telemetry/sessions?limit={limit}`.
3. Create `frontend/src/pages/sessions.tsx` ŌĆö a page component that calls `getSessionList()` via `useApi`, renders a table with columns: Status (badge), Started At, Stopped At, Duration, Error Type, Error Message. Use existing `ht-table` CSS classes and `formatTimestamp`/`formatDuration` utilities.
4. Add `/sessions` route to `frontend/src/app.tsx` in the `<Switch>` block.
5. Add "Sessions" nav item to `frontend/src/components/layout/sidebar.tsx` in the `NAV_ITEMS` array with path `/sessions`, testId `nav-sessions`, and `<IconHistory />` icon.
6. Add E2E test `tests/e2e/test_sessions.py::test_sessions_page_renders` ŌĆö navigate to `/sessions`, verify table is visible with session data.
7. Add E2E test `test_sidebar_sessions_navigation` ŌĆö click the Sessions nav item, verify navigation to `/sessions`.

## Test Strategy

- **E2E tests** in `tests/e2e/test_sessions.py` (new file):
  - `test_sessions_page_renders` ŌĆö navigate to `/sessions`, verify table renders with at least one session row showing status, timestamps
  - `test_sidebar_sessions_navigation` ŌĆö click `[data-testid="nav-sessions"]`, verify URL changes and page renders

## Review Guidance

- Verify the sessions page uses `useApi` (not `useScopedApi` ŌĆö session list is not session-scoped, it shows all sessions by definition).
- Verify timestamps are rendered as formatted dates using existing utilities, not raw floats.
- Verify the sidebar icon follows the existing pattern (SVG component, same size as `IconDashboard`, `IconBoxes`, `IconScrollText`).
