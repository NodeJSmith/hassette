## Objectives & Success Criteria

- `GET /telemetry/sessions?limit=50` returns a typed list of session records with `id`, `started_at`, `stopped_at`, `status`, `error_type`, `error_message`, `duration_seconds`.
- `get_session_list()` in `TelemetryQueryService` returns typed Pydantic models instead of `list[dict]`.
- The endpoint follows the `try/except DB_ERRORS` pattern with graceful degradation (returns empty list on failure).
- The `limit` parameter is validated (1–200, default 50).
- Generated OpenAPI types include `SessionListEntry`.

## Subtasks

1. Add `SessionRecord` Pydantic model to `src/hassette/core/telemetry_models.py` with fields: `id: int`, `started_at: float`, `stopped_at: float | None`, `status: str`, `error_type: str | None`, `error_message: str | None`, `duration_seconds: float | None`. This replaces the `list[dict]` return type.
2. Update `TelemetryQueryService.get_session_list()` in `src/hassette/core/telemetry_query_service.py` to return `list[SessionRecord]` instead of `list[dict]`. Construct model instances from the query rows.
3. Add `SessionListEntry` Pydantic model to `src/hassette/web/models.py` matching the `SessionRecord` fields (or re-export if identical).
4. Add `GET /telemetry/sessions` endpoint to `src/hassette/web/routes/telemetry.py` with `limit: int = Query(default=50, ge=1, le=200)`. Wrap in `try/except DB_ERRORS` with `_reraise_if_not_connection_closed()`, returning empty list on failure.
5. Regenerate `frontend/src/api/generated-types.ts` from the updated OpenAPI schema.
6. Update existing `tests/integration/test_telemetry_query_service.py::TestGetSessionList` to verify the new typed return type.
7. Add integration test for the new endpoint in `tests/integration/test_web_api.py` — verify response shape, limit parameter, and DB error handling.

## Test Strategy

- **Integration tests** in `tests/integration/test_telemetry_query_service.py`:
  - Update `TestGetSessionList` — verify returns `list[SessionRecord]`, field types match, ordering correct
- **Integration tests** in `tests/integration/test_web_api.py`:
  - `test_sessions_endpoint_returns_session_list` — `GET /telemetry/sessions` returns 200 with valid session data
  - `test_sessions_endpoint_limit_parameter` — verify `limit` parameter constrains results
  - `test_sessions_endpoint_db_error_returns_empty` — verify graceful degradation returns empty list

## Review Guidance

- Verify `SessionRecord` uses float timestamps (not ISO strings) — consistent with `last_invoked_at`, `execution_start_ts` elsewhere.
- Verify `duration_seconds` is `float | None` — null when neither `stopped_at` nor `last_heartbeat_at` is set (running session with no heartbeat yet).
- Verify the endpoint imports and uses `DB_ERRORS` and `_reraise_if_not_connection_closed` from the existing pattern.
