## Objectives & Success Criteria

- Expanding a handler row shows `source_location` and `registration_source` in the detail panel, above the invocations table.
- Expanding a job row shows the same fields above the executions table.
- When `source_location` is empty and `registration_source` is null, no source section renders.
- Existing E2E test `test_registration_source_link` is strengthened to assert on actual source content.

## Subtasks

1. In `frontend/src/components/app-detail/handler-row.tsx`, add a source display block inside the `ht-item-detail` div, before `<HandlerInvocations>`. Show `source_location` with `ht-text-muted ht-text-xs` and `registration_source` in a `<code>` block with `ht-text-mono ht-text-xs`. Guard with `{(listener.source_location || listener.registration_source) && ...}`.
2. In `frontend/src/components/app-detail/job-row.tsx`, add the same source display block inside the `ht-item-detail` div, before `<JobExecutions>`. Guard with `{(job.source_location || job.registration_source) && ...}`.
3. Update `tests/e2e/test_app_detail.py::test_registration_source_link` to assert that `source_location` and `registration_source` text content appears in the expanded detail panel (the conftest fixture already provides `source_location="my_app.py:15"` and `registration_source="on_initialize"` at `tests/e2e/conftest.py`).
4. Add E2E test `test_job_registration_source_display` ‚Äî expand a job row, verify source fields appear.
5. Add E2E test `test_source_display_hidden_when_empty` ‚Äî verify no source section renders when both fields are empty/null.

## Test Strategy

- **E2E tests** in `tests/e2e/test_app_detail.py`:
  - `test_registration_source_link` ‚Äî strengthened: expand handler row, assert `source_location` text and `registration_source` text visible
  - `test_job_registration_source_display` ‚Äî expand job row, assert source fields visible
  - `test_source_display_hidden_when_empty` ‚Äî handler/job with empty source fields, verify no source section in DOM

## Review Guidance

- Verify the render guard uses truthy check on `source_location` (non-empty string), not null check ‚Äî `source_location` is `NOT NULL` in the schema with default `""`.
- Verify `registration_source` is rendered as text content via JSX, never via `dangerouslySetInnerHTML`.
- Verify both handler-row and job-row use the same guard and display pattern.

## Visual Verification

| Page | Setup | Verify |
|------|-------|--------|

| App detail page with handler missing source data | Expand a handler where both fields are empty | No source section rendered; invocations table is the first thing in the detail panel |
