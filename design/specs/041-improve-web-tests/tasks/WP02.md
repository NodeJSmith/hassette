## Objectives & Success Criteria

All five functions in `frontend/src/utils/format.ts` have thorough unit tests covering boundary conditions. Tests use `vi.useFakeTimers()` for time-dependent functions and regex matchers for locale-dependent output. `npm test` passes.

## Subtasks

1. Create `frontend/src/utils/format.test.ts`
2. Test `formatTimestamp` — epoch zero, current time, large timestamp. Use regex matchers (`/^\d{2}\/\d{2} \d{1,2}:\d{2}:\d{2} (AM|PM)$/i`) rather than exact strings to avoid ICU version differences. Scope `vi.useFakeTimers()` inside `beforeEach`/`afterEach`
3. Test `formatDuration` — sub-millisecond (<1ms), milliseconds (e.g., 158.0ms), seconds (e.g., 1.5s), boundary at 1000ms, zero
4. Test `pluralize` — count=0 (plural), count=1 (singular), count=2 (default plural with 's'), custom plural
5. Test `formatTriggerDetail` — seconds passthrough (e.g., "30s"), conversion from large seconds ("432000s" → "5d"), mixed units ("3661s" → "1h 1m 1s"), non-seconds passthrough ("30 7 * * 1-5", "07:00")
6. Test `formatRelativeTime` — "just now" (diff < 60s), minutes (e.g., "2m ago"), hours (e.g., "3h ago"), days (e.g., "5d ago"), boundary at 60s, boundary at 3600s, boundary at 86400s. Use `vi.useFakeTimers()` with `vi.setSystemTime()`

## Test Strategy

- File: `frontend/src/utils/format.test.ts`
- Pure function tests — no mocking needed except fake timers for `formatRelativeTime` and `formatTimestamp`
- Test functions: `test_formatTimestamp_epoch_zero`, `test_formatTimestamp_recent`, `test_formatDuration_sub_ms`, `test_formatDuration_ms`, `test_formatDuration_seconds`, `test_pluralize_singular`, `test_pluralize_default_plural`, `test_pluralize_custom_plural`, `test_formatTriggerDetail_seconds`, `test_formatTriggerDetail_days`, `test_formatTriggerDetail_mixed`, `test_formatTriggerDetail_cron_passthrough`, `test_formatRelativeTime_just_now`, `test_formatRelativeTime_minutes`, `test_formatRelativeTime_hours`, `test_formatRelativeTime_days`

## Review Guidance

- Verify no exact string assertions on `formatTimestamp` output — must use regex matchers
- Verify `vi.useFakeTimers()` is scoped per-suite not globally — must be in `beforeEach`/`afterEach`
- Verify boundary values are tested: 59s vs 60s for minutes, 3599s vs 3600s for hours
