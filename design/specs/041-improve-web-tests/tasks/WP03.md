## Objectives & Success Criteria

All untested dashboard components (`app-grid.tsx`, `kpi-strip.tsx`) and shared components (`show-more-button.tsx`, `spinner.tsx`, `status-badge.tsx`, `icons.tsx`) have test files. Components with logic get thorough coverage; trivial wrappers get smoke tests. `npm test` passes.

## Subtasks

1. Create `frontend/src/components/dashboard/app-grid.test.tsx` — renders grid of app cards, handles empty list, passes correct props to AppCard children
2. Create `frontend/src/components/dashboard/kpi-strip.test.tsx` — renders all KPI cards (Apps, Error Rate, Handlers, Jobs, Uptime), handles zero values, error rate color class assignment
3. Create `frontend/src/components/shared/status-badge.test.tsx` — renders correct badge for each status (running, stopped, failed, disabled, blocked), no badge for running apps
4. Create `frontend/src/components/shared/show-more-button.test.tsx` — renders expand/collapse label, fires click handler
5. Create `frontend/src/components/shared/spinner.test.tsx` — smoke test: renders without crashing
6. Create `frontend/src/components/shared/icons.test.tsx` — smoke test: each icon component renders an SVG element

## Test Strategy

- `app-grid.test.tsx` — use factories from `test/factories.ts` for `DashboardAppGridEntry[]`, render with MSW for any API calls. Test: grid renders N cards for N apps, empty list shows placeholder, link navigation to app detail
- `kpi-strip.test.tsx` — mock `useScopedApi` hook (render-with-known-data pattern per mocking layer rule). Test: each KPI card label and value rendered, error rate class applied, zero invocations handled
- `status-badge.test.tsx` — pure prop tests: each status string produces correct badge class/text
- `show-more-button.test.tsx` — render with `expanded=false`, verify "Show more" text; render with `expanded=true`, verify "Show less"; verify `onClick` fires
- `spinner.test.tsx` — `render(<Spinner />)`, expect element exists
- `icons.test.tsx` — render each exported icon, expect SVG element

## Review Guidance

- `kpi-strip` should use `vi.mock(hook)` per the mocking layer rule — it tests rendering given known data, not data fetching
- Verify factories from `test/factories.ts` are used, not inline object literals
