## Objectives & Success Criteria

No E2E test assertion contains a hardcoded seed-data-derived value. All seed-data-dependent assertions use computed constants derived from builder output or regex/structural matchers. Computation-verifying tests import a shared formula helper. Changing a seed data value in `mock_fixtures.py` does not break any E2E test (the acid test). `uv run nox -s e2e` passes.

## Subtasks

1. Extract a `compute_error_rate(total_invocations, total_executions, handler_errors, job_errors)` pure function in `src/hassette/web/telemetry_helpers.py`. Refactor the inline computation in `web/routes/telemetry.py` (dashboard KPIs and app-grid endpoints) to call this helper
2. Add unit tests for `compute_error_rate` in `tests/unit/web/test_telemetry_helpers.py`
3. Extract module-level computed constants in `tests/e2e/mock_fixtures.py` from builder output. Use tier-qualified names: `APP_TIER_TOTAL_INVOCATIONS`, `APP_TIER_TOTAL_ERRORS`, `FRAMEWORK_TIER_TOTAL_ERRORS`, etc. Constants must reference builder objects (e.g., `APP_TIER_TOTAL_INVOCATIONS = default_global_summary.listeners.total_invocations`), not hand-written literals
4. Replace hardcoded seed-data-derived assertions in `tests/e2e/test_dashboard.py`: invocation counts (`"30 inv"`, `"20 exec"`, `"3 inv"`, `"8 exec"`), error rate string (`"9 / 61 invocations"`) — the error rate test (`test_dashboard_error_rate_includes_jobs`) must import `compute_error_rate` from `hassette.web.telemetry_helpers` to derive the expected string, not re-compute it inline — error feed count (`to_have_count(5)`), framework error badge count (`"1"`)
5. Replace hardcoded seed-data-derived assertions in `tests/e2e/test_app_detail.py`: invocation counts (`"10 calls"`, `"20 calls"`), execution counts (`"15 runs"`, `"5 runs"`), source locations (`"my_app.py:15"`, `"my_app.py:30"`)
6. Audit remaining E2E test files (`test_apps.py`, `test_apps_list.py`, `test_logs.py`, `test_sessions.py`, `test_navigation.py`, `test_websocket.py`, `test_responsive.py`, `test_hot_reload.py`) for hardcoded seed-data-derived values and replace as needed
7. Add E2E computed constant convention to `tests/TESTING.md`: all seed-data-derived E2E assertions must use constants computed from builder output with tier-qualified names
8. Verify resilience: modify a seed data value (e.g., change `total_invocations` from 10 to 15 in one listener), run `uv run nox -s e2e`, confirm all tests pass. Revert the change.

## Test Strategy

- `tests/unit/web/test_telemetry_helpers.py` — new tests for `compute_error_rate`: zero total (returns 0.0), all errors (returns 100.0), mixed (returns correct percentage), handler-only errors, job-only errors
- E2E tests: the existing test suite IS the verification — all tests must pass after the assertion migration
- Acid test: temporarily modify `mock_fixtures.py` seed values and verify no assertion breaks

## Review Guidance

- Verify NO hand-written numeric constants in `mock_fixtures.py` — all must be computed from builder output
- Verify the `compute_error_rate` helper is used by both `telemetry.py` routes and E2E tests — no re-derivation
- Verify error-rate test (`test_dashboard_error_rate_includes_jobs`) still semantically verifies that job executions are in the denominator — it should not be weakened to a regex
- Static UI text assertions (page titles, labels, "Status", "Error Rate") must remain unchanged — only seed-data-derived values change
