## Objectives & Success Criteria

All untested layout components (`status-bar.tsx`, `sidebar.tsx`, `alert-banner.tsx`, `error-boundary.tsx`, `session-scope-toggle.tsx`) and pages (`dashboard.tsx`, `apps.tsx`, `logs.tsx`, `sessions.tsx`, `not-found.tsx`) have test files. `dashboard.tsx` gets thorough coverage including debounced refetch and tier filter interaction. `npm test` passes.

## Subtasks

1. Create `frontend/src/components/layout/status-bar.test.tsx` ‚Äî renders connection status indicator, theme toggle, version display. Test each connection state (connecting, connected, disconnected, reconnecting)
2. Create `frontend/src/components/layout/sidebar.test.tsx` ‚Äî renders navigation links, highlights active route, responsive visibility
3. Create `frontend/src/components/layout/alert-banner.test.tsx` ‚Äî renders alert when failed apps exist, hidden when no failures, displays app key and error message
4. Create `frontend/src/components/layout/error-boundary.test.tsx` ‚Äî catches child component errors, displays fallback UI, does not crash parent
5. Create `frontend/src/components/layout/session-scope-toggle.test.tsx` ‚Äî renders scope options (current/all), fires onChange, displays current selection
6. Create `frontend/src/pages/dashboard.test.tsx` ‚Äî thorough coverage: renders KPI strip + app grid + error feed, debounced refetch on reconnect (`statusVersionRef` pattern), tier filter toggle changes error feed scope, conditional rendering paths (with/without framework errors), empty state
7. Create `frontend/src/pages/apps.test.tsx` ‚Äî renders manifest list with status filter, filter changes update displayed apps
8. Create `frontend/src/pages/logs.test.tsx` ‚Äî renders log table with level and app filters, pagination
9. Create `frontend/src/pages/sessions.test.tsx` ‚Äî renders session list with status indicators
10. Create `frontend/src/pages/not-found.test.tsx` ‚Äî smoke test: renders 404 message

## Test Strategy

- `status-bar.test.tsx` ‚Äî mock `useAppState()` to provide different connection states. Tests: `test_connected_indicator`, `test_disconnected_indicator`, `test_reconnecting_indicator`, `test_theme_toggle`
- `sidebar.test.tsx` ‚Äî mock routing. Tests: `test_renders_nav_links`, `test_active_route_highlighted`
- `alert-banner.test.tsx` ‚Äî pure prop test. Tests: `test_shows_alert_for_failed_apps`, `test_hidden_when_no_failures`
- `error-boundary.test.tsx` ‚Äî render child that throws. Tests: `test_catches_error`, `test_shows_fallback`
- `dashboard.test.tsx` ‚Äî mock all hooks (`useScopedApi` for each data section). Tests: `test_renders_kpi_strip`, `test_renders_app_grid`, `test_renders_error_feed`, `test_tier_filter_toggles_scope`, `test_empty_state`, `test_reconnect_refetch_debounce`
- `not-found.test.tsx` ‚Äî `render(<NotFound />)`, expect 404 text

## Review Guidance

- `dashboard.test.tsx` is the most complex ‚Äî verify it tests the debounce logic and tier filter, not just render presence
- Layout components should use `vi.mock(hook)` per the mocking layer rule
- `error-boundary.test.tsx` must silence React/Preact error console output during the throw test
- Verify all connection states are tested for `status-bar`
