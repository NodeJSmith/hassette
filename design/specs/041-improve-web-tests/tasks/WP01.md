## Objectives & Success Criteria

MSW is installed and configured as the default network-level mocking layer for frontend tests. Shared handlers cover all API endpoints with typed responses. Shared factories replace per-file duplication. A `renderWithAppState()` helper exists for components that need `AppStateContext`. `npm test` passes with MSW active and `onUnhandledRequest: 'warn'`.

## Subtasks

1. Add `msw` to `devDependencies` in `frontend/package.json` and run `npm install`
2. Add MSW server lifecycle (`beforeAll`/`afterEach`/`afterAll`) to `frontend/src/test-setup.ts` with `onUnhandledRequest: 'warn'`
3. Verify `client.test.ts` still passes with the global MSW server active — if it conflicts, add `server.close()` in `client.test.ts`'s `beforeAll`/`afterAll`
4. Create `frontend/src/test/handlers.ts` with default `http.get`/`http.post` handlers for every endpoint in `frontend/src/api/endpoints.ts`. Use `HttpResponse.json<T>()` with explicit type parameters from `generated-types.ts`
5. Create `frontend/src/test/factories.ts` with factory functions for all major data objects (`createManifest`, `createAppGridEntry`, `createListener`, `createJob`, `createHealthData`, `createKpis`, `createError`, `createLogEntry`, `createSession`). Each uses `satisfies` against the generated type and accepts `Partial<T>` overrides
6. Create `frontend/src/test/render-helpers.ts` with `renderWithAppState(ui, stateOverrides?)` that wraps the component in `AppStateContext.Provider` with a `createAppState()` instance
7. Migrate existing fetch-mock component tests (`app-card.test.tsx`, `error-feed.test.tsx`) to use MSW handlers and shared factories — remove per-file factory functions
8. Add to `tests/TESTING.md`: the mocking layer rule (MSW for data-fetching behavior, `vi.mock(hook)` for render-with-known-data, direct fetcher injection for hook internals), factory `satisfies` convention, and MSW usage patterns (handler creation, `server.use()` for per-test overrides, `onUnhandledRequest` policy)
