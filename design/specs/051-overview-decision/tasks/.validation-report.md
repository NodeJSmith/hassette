## Issues

### Issue #1 — AC#5 frontend orphan cleanup assigned only to T02 (MEDIUM)

AC#5 covers both backend orphans **and** frontend orphans ("No orphaned backend routes, frontend API functions, or CSS classes"). T02's Verify item for AC#5 reads: "No orphaned backend routes, response models, or query methods remain from the overview." This is backend-only. The frontend orphan verification (API functions in `endpoints.ts`, CSS classes in `global.css`, `app-data.ts` exports) is done in T01, but T01 does not list AC#5 in its `implements` field or its Verify checklist.

**Recommendation:** Either add AC#5 to T01's `implements` list and add a corresponding Verify item ("No orphaned frontend API functions, CSS classes, or utility exports remain"), or rewrite T02's AC#5 Verify item to explicitly cover both frontend and backend orphans and acknowledge the split.

### Issue #2 — AC#4 TelemetryDegradedBanner coverage is incomplete (LOW)

T03's Verify item for AC#4 states: "TelemetryDegradedBanner renders on all pages (apps, handlers, logs, diagnostics, config) when telemetry is degraded." This is concrete, but the Prompt does not direct the implementer to write an automated test for this — it says "write equivalent tests in a layout test file" but as an optional branch ("If `dashboard.test.tsx` had tests for TelemetryDegradedBanner rendering"). The original `dashboard.test.tsx` does have this coverage per the design doc (~380 lines). The conditional phrasing risks the test migration being skipped if the implementer doesn't find the old tests.

**Recommendation:** Change "If `dashboard.test.tsx` had tests..." to an unconditional directive: "Write tests in `app.test.tsx` (or equivalent layout test file) that verify TelemetryDegradedBanner renders when `telemetryDegraded` is true and is hidden when false."

### Issue #4 — No task covers `frontend/src/components/layout/command-palette.test.tsx` update (LOW)

T01 Prompt step 4 says: "Update the command palette test file if it asserts on this entry." T03 step 8 also says: "Update `frontend/src/components/layout/command-palette.test.tsx` if it asserts on an 'Overview' entry." Both tasks reference this file conditionally. The duplication is harmless but could cause the implementer of T01 to leave it to T03, and vice versa. Neither task claims this in `implements`.

Since the command palette change is part of FR#4 (overview no longer routable/accessible) and T01 implements FR#4, the test update should be explicitly owned by T01's Verify section.

**Recommendation:** Add a Verify item to T01: "command-palette.test.tsx has no assertion referencing an 'Overview' entry."
