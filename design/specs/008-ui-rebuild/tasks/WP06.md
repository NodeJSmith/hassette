## Objectives & Success Criteria

Complete the page set. After this WP:
1. Apps list page renders at `/apps` with manifest table, status filter tabs, and links to App Detail
2. Logs page renders at `/logs` with global log viewer, level/app/search filters, and WS streaming
3. Both pages render correctly in light and dark modes

## Subtasks

1. Rewrite Apps page route in `src/hassette/web/ui/router.py`:
   - Fetch `get_all_manifests_snapshot()` for manifest list
   - Pass `manifest_snapshot` and `app_keys` to template

2. Create `src/hassette/web/templates/pages/apps.html`:
   - Status filter tabs (All, Running, Failed, Stopped, Disabled) using Alpine.js state (not HTMX tab switching)
   - Manifest list with expandable instance rows
   - Each app row links to App Detail
   - `data-live-on-app` on manifest list for WS refresh
   - `data-testid` on filter tabs and app rows

3. Rewrite Logs page route in `src/hassette/web/ui/router.py`:
   - Fetch `get_recent_logs(limit=100)` and app keys list

4. Create `src/hassette/web/templates/pages/logs.html`:
   - `log_table` macro with `show_app_column=true` and `app_keys` for filter dropdown
   - Log level filter, app filter, search input
   - WS streaming via `logTable` Alpine.js component

5. Create/update partials: `app_list.html` (manifest rows), `log_entries.html` (log lines)

6. Update `src/hassette/web/static/js/log-table.js` — styling updates for new token system, no functional changes

## Test Strategy

E2E test file: `tests/e2e/test_apps_list.py`
- `test_apps_list_renders_all_apps` — all configured apps visible
- `test_apps_list_status_filter_tabs` — clicking "Running" tab filters to running apps only
- `test_app_row_links_to_detail` — clicking app row navigates to App Detail

E2E test file: `tests/e2e/test_logs.py`
- `test_logs_page_renders_entries` — log entries visible
- `test_logs_level_filter` — selecting ERROR shows only error entries
- `test_logs_search_filter` — typing search text filters entries

## Review Guidance

- Apps page MUST use Alpine.js for tab state, not `hx-on::after-request` inline JS (the old fragile pattern per design doc).
- Logs page must reuse the existing `logTable` Alpine.js component pattern.
- No standalone Bus or Scheduler page — these are explicitly removed.
