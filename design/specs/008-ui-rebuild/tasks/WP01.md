## Objectives & Success Criteria

Backend foundation for the UI rebuild. After this WP:
1. Every predicate class in `predicates.py` has a `summarize() -> str` method that returns a stable, human-readable description
2. The `listeners` table has a `human_description` column populated at registration time
3. `TelemetryQueryService` methods return typed Pydantic models instead of raw dicts
4. All existing tests pass (no regressions)

## Subtasks

1. Add `summarize() -> str` method to each predicate class in `src/hassette/event_handling/predicates.py`:
   - `Guard` → `"custom condition"` (fallback for lambdas)
   - `AllOf` → join child summaries with ` and `
   - `AnyOf` → join child summaries with ` or `
   - `Not` → `"not " + inner.summarize()`
   - `ValueIs` → delegate to condition's repr or summarize if available
   - `StateFrom` → `"from {state}"`
   - `StateTo` → `"→ {state}"`
   - `StateComparison` → `"{from_state} → {to_state}"`
   - `DomainMatches` → `"domain {domain}"`
   - `EntityMatches` → `"entity {entity_id}"`
   - `ServiceMatches` → `"service {domain}.{service}"`
   - `DidChange` → `"changed"`
   - `IsPresent` → `"is present"`
   - `IsMissing` → `"is missing"`
   - `AttrFrom` → `"attr {attr} from {value}"`
   - `AttrTo` → `"attr {attr} → {value}"`
   - `AttrComparison` → `"attr {attr} {from} → {to}"`
   - `StateDidChange` → `"state changed"`
   - `AttrDidChange` → `"attr {attr} changed"`
   - `ServiceDataWhere` → `"service data where {field} = {value}"`

2. Create Alembic migration adding `human_description TEXT` column to `listeners` table (nullable, no default)

3. Update `BusService` listener registration (`src/hassette/core/bus_service.py`) to populate `human_description` from `predicate.summarize()` when recording the listener

4. Add Pydantic models in `src/hassette/core/telemetry_models.py`:
   - `ListenerSummary` — fields matching `get_listener_summary()` columns + `human_description: str | None`
   - `JobSummary` — fields matching `get_job_summary()` columns
   - `HandlerInvocation` — fields matching `get_handler_invocations()` columns
   - `JobExecution` — fields matching `get_job_executions()` columns
   - `GlobalSummary` — fields matching `get_global_summary()` return dict (used by WP05 Dashboard KPIs)
   - `SessionSummary` — fields matching `get_current_session_summary()` return dict (used by WP05 Dashboard session bar)

5. Update `TelemetryQueryService` methods to return typed models: `get_listener_summary()` → `list[ListenerSummary]`, `get_handler_invocations()` → `list[HandlerInvocation]`, `get_job_summary()` → `list[JobSummary]`, `get_job_executions()` → `list[JobExecution]`

## Test Strategy

TDD. Test file: `tests/unit/test_predicate_summarize.py`

- `test_entity_matches_summarize` — `EntityMatches("light.kitchen").summarize() == "entity light.kitchen"`
- `test_state_to_summarize` — `StateTo("on").summarize() == "→ on"`
- `test_allof_summarize` — `AllOf([EntityMatches("light.kitchen"), StateTo("on")]).summarize() == "entity light.kitchen and → on"`
- `test_guard_summarize` — `Guard(lambda e: True).summarize() == "custom condition"`
- `test_not_summarize` — `Not(StateTo("on")).summarize() == "not → on"`

Test file: `tests/unit/test_telemetry_models.py`
- `test_listener_summary_from_dict` — validates `ListenerSummary.model_validate()` with a sample row dict
- `test_handler_invocation_from_dict` — validates `HandlerInvocation.model_validate()`

Integration test: `tests/integration/test_telemetry_query_service.py`
- Add assertions that existing `get_listener_summary()` calls return `ListenerSummary` instances

## Review Guidance

- Every predicate class must have `summarize()`. Grep for `class.*:` in predicates.py and verify each has the method.
- `human_description` must be populated at registration time, not computed at query time.
- Pydantic models must have `model_config = ConfigDict(from_attributes=True)` if needed for ORM compatibility, but since we're using `_row_to_dict()` → `model_validate(dict)`, standard dict validation is sufficient.
- The Alembic migration must be reversible (downgrade drops the column).
