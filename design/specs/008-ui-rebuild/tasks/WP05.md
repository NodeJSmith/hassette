## Objectives & Success Criteria

The secondary entry point. After this WP:
1. Dashboard renders at `/` with HA connection status banner, system KPI strip, app health grid with error rate gradient, recent errors feed, and session info
2. Connection bar shows WebSocket status (connected/disconnected) with pulse dot
3. App health cards link to App Detail, show handler/job counts, error rate percentage, health bar gradient
4. Recent errors feed is session-scoped (uses `session_id` parameter)
5. Dashboard partials refresh on `app_status_changed` WS events

## Subtasks

1. Update Dashboard route in `src/hassette/web/ui/router.py`:
   - Fetch `get_all_manifests_snapshot()` for app grid
   - Fetch `get_system_status()` for connection state
   - Fetch `get_recent_errors(session_id=current_session_id, limit=10)` for error feed
   - Fetch `get_global_summary()` for KPI strip
   - Fetch `get_current_session_summary()` for session bar

2. Create `src/hassette/web/templates/pages/dashboard.html`:
   - Connection bar component (`connection_bar.html`) with WebSocket state
   - KPI strip: apps count, error rate (1h), handlers count, jobs count, uptime
   - App health grid: cards for each app with name, status badge, handler/job counts, error rate, health bar, last activity, link to App Detail
   - Recent errors section: list of recent errors with app name, handler name, error message, timestamp
   - Session info bar: session number, start time, duration, Hassette version

3. Create `src/hassette/web/templates/components/connection_bar.html`:
   - Connected: green background tint, breathing dot, WS URL, latency
   - Disconnected: red background tint, static dot, reconnection message

4. Create partials: `dashboard_app_grid.html`, `dashboard_errors.html`
   - App cards with `data-testid="app-card-{app_key}"`
   - Error items with `data-testid="error-item"`
   - `data-live-on-app` attributes for WS-triggered refresh

5. Add app health gradient CSS:
   - `.ht-health-bar` with `.ht-health-bar__fill` (width based on success rate)
   - `.excellent` (100%, green), `.good` (>95%, green), `.warning` (>90%, yellow), `.critical` (<90%, red)

## Review Guidance

- `get_recent_errors()` MUST use `session_id` parameter (not `since_ts`) for session-scoped error feed.
- App cards must link to `/apps/{app_key}` via `<a>` tag, not JS navigation.
- Connection bar must reflect actual `$store.ws` state (from Alpine.js store).
- Health bar width must be computed from telemetry data, not hardcoded.
