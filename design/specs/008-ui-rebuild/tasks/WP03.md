## Objectives & Success Criteria

Reusable Jinja2 macros and CSS components used across all pages. After this WP:
1. `macros/ui.html` contains macros for: `status_badge`, `health_card`, `handler_row`, `job_row`, `action_buttons`, `log_table`
2. CSS in `style.css` covers all component patterns: badges, health cards, item rows (with expand chevron), invocation detail, error traces, log container, buttons, form controls
3. Components render correctly in both light and dark modes

## Subtasks

1. Add component CSS to `src/hassette/web/static/css/style.css`:
   - `.ht-status-badge` (running/failed/stopped/disabled variants with colored dots)
   - `.ht-health-card` (label, value, detail sub-elements, `.good`/`.warn`/`.bad` value colors)
   - `.ht-health-strip` (4-column grid)
   - `.ht-item-list` (border + shadow container with 1px gap rows)
   - `.ht-item-row` (grid: indicator dot | content | stats, hover, expand chevron)
   - `.ht-item-detail` (recessed background, invocation list, error trace)
   - `.ht-invocation` (grid: dot | time | duration | status | trigger)
   - `.ht-error-trace` (danger-tinted background, mono, pre-wrap)
   - `.ht-btn` (default, danger variants, icon + text)
   - `.ht-log-level-toggle` (segmented button group with active state)
   - `.ht-log-container`, `.ht-log-toolbar`, `.ht-log-entries`, `.ht-log-line`
   - `.ht-section-head` with `.ht-section-count` pill
   - `.ht-breadcrumb`
   - `.ht-meta-item` (mono, dim, strong variant)
   - `.ht-source-link`

2. Rewrite `src/hassette/web/templates/macros/ui.html` with macros:
   - `status_badge(status)` — renders `.ht-status-badge` with correct variant class
   - `health_card(label, value, detail, status_class="")` — renders `.ht-health-card`
   - `handler_row(listener, app_key)` — renders full handler row with Alpine.js expand pattern, htmx.ajax lazy-load, `data-testid="handler-row-{listener.id}"`, stable `id` attribute
   - `job_row(job, app_key)` — renders full job row with expand pattern, `data-testid="job-row-{job.id}"`
   - `action_buttons(app_key, status)` — start/stop/reload buttons with HTMX post
   - `log_table(show_app_column, app_key, app_keys)` — log viewer Alpine.js component

3. Add `format_handler_summary()` and `extract_entity_from_topic()` to `src/hassette/web/ui/context.py`

## Test Strategy

Unit test file: `tests/unit/test_context_helpers.py`
- `test_format_handler_summary_entity_state` — topic `state_changed.binary_sensor.garage_door` + human_description `→ open` → `"Fires when binary_sensor.garage_door → open"`
- `test_format_handler_summary_fallback` — no human_description → uses topic directly
- `test_extract_entity_from_topic` — various topic formats

E2E tests will validate rendering in WP04-WP05 (when pages use these macros).

## Review Guidance

- Handler row macro must use the Alpine.js `@click` + `htmx.ajax()` pattern from the design doc — NOT `hx-trigger="intersect once"`.
- Every `.ht-item-row` must have a stable `id` attribute for idiomorph.
- All `data-testid` attributes must be present on interactive elements.
- All CSS values must reference `--ht-*` tokens.
- Macros must not contain inline `<script>` tags (per `tools/check_template_patterns.py` rule).
