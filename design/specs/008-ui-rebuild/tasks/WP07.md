## Objectives & Success Criteria

Final integration and test hardening. After this WP:
1. E2E conftest seed data includes telemetry records (handler invocations, job executions, errors) for realistic UI assertions
2. Theme toggle tests verify both light and dark mode render correctly
3. WebSocket tests verify live-update partials refresh on `app_status_changed` events
4. Morphing stability test verifies expanded handler row survives a WS-triggered refresh
5. All old E2E test files for removed pages (Bus, Scheduler) are deleted
6. `uv run nox -s e2e` passes with all new tests
7. `uv run nox -s dev` passes (unit + integration tests unbroken)

## Subtasks

1. Update `tests/e2e/conftest.py`:
   - Add telemetry seed data: handler invocations (mix of success/error), job executions, listener summaries with `human_description`
   - Ensure `TelemetryQueryService` returns typed models (from WP01)
   - Add `session_id` to seed data for session-scoped error feed

2. Create `tests/e2e/test_theme.py`:
   - `test_dark_mode_default` — page loads with `data-theme="dark"`
   - `test_light_mode_toggle` — toggle switches to light, all tokens apply
   - `test_theme_persistence` — set light, reload page, still light (localStorage)
   - `test_both_modes_no_raw_colors` — screenshot in each mode, verify no visual breakage

3. Create `tests/e2e/test_websocket.py`:
   - `test_ws_connection_established` — pulse dot shows connected state
   - `test_live_update_refreshes_on_app_status` — trigger `app_status_changed`, verify partial refreshes
   - `test_expanded_row_survives_morph` — expand a handler row, trigger WS refresh, verify row stays expanded (idiomorph + stable id test)

4. Delete old test files:
   - `tests/e2e/test_bus_page.py`
   - `tests/e2e/test_scheduler_page.py`
   - `tests/e2e/test_hot_reload.py` (if hot reload behavior changed — assess first)

5. Run full E2E suite: `uv run nox -s e2e` — all tests pass
6. Run full dev suite: `uv run nox -s dev` — no regressions in unit/integration tests

## Test Strategy

This WP IS the test hardening pass. The tests listed above are the deliverables.

Key regression test: `test_expanded_row_survives_morph` — this directly validates the idiomorph + Alpine.js state preservation that was a CRITICAL finding in the design critique. If this test fails, the core interaction model is broken.

## Review Guidance

- Seed data must be realistic (multiple apps, mix of success/error invocations, varying invocation counts) — not minimal stubs.
- The morph stability test (`test_expanded_row_survives_morph`) is a hard requirement. If it's flaky, investigate the idiomorph matching — don't skip it.
- Verify deleted test files are for pages that no longer exist (Bus, Scheduler). Do not delete tests for functionality that was moved (e.g., listener display moved from Bus page to App Detail — the assertion moves, not disappears).
- `uv run nox -s e2e` is the CI command. Tests must pass with this exact invocation.
