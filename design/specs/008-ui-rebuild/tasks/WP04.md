## Objectives & Success Criteria

The center of gravity page. After this WP:
1. App Detail page renders at `/apps/{app_key}` and `/apps/{app_key}/{index}` with health strip, handler rows (with plain-language summaries, invocation counts, last-fired), job rows (with execution counts, next run), and app-scoped logs
2. Clicking a handler row expands it and lazy-loads invocation history via `htmx.ajax()`
3. Clicking a job row expands it and lazy-loads execution history
4. Error traces display in expanded rows for failed invocations
5. Registration source links display for each handler/job
6. Stats-only partial polls every 5s to keep invocation counts fresh
7. Handler list and job list refresh on `app_status_changed` WS events via live-updates
8. Identity model bug is fixed: jobs filter by `app_key + instance_index`, not `owner_id`
9. Deferred-feature placeholders are present: log level toggle (501), enable/disable toggle (501), run-now buttons (501)
10. All E2E tests pass

## Subtasks

1. Rewrite `src/hassette/web/ui/router.py`:
   - Remove Bus page route and Scheduler page route
   - Rewrite `app_detail_page` and `app_instance_detail_page` to:
     - Fetch listener summaries via `TelemetryQueryService.get_listener_summary()` (returns `ListenerSummary` models)
     - Fetch job summaries via `TelemetryQueryService.get_job_summary()` (returns `JobSummary` models)
     - Fetch scheduler jobs via `SchedulerService.get_all_jobs()` filtered by `app_key + instance_index` (NOT `owner_id`)
     - Compute `format_handler_summary()` for each listener
     - Fetch app-scoped logs via `RuntimeQueryService.get_recent_logs(app_key=...)`

2. Add new partial routes to `src/hassette/web/ui/partials.py`:
   - `/partials/handler-invocations/{app_key}/{listener_id}` — calls `TelemetryQueryService.get_handler_invocations()`, renders `handler_invocations.html`
   - `/partials/job-executions/{app_key}/{job_id}` — calls `TelemetryQueryService.get_job_executions()`, renders `job_executions.html`
   - `/partials/app-handler-stats/{app_key}` — returns just invocation counts and last-fired for each handler (stats-only partial for 5s polling)
   - `/partials/app-health-strip/{app_key}` — health strip cards
   - `/partials/app-handlers/{app_key}` — full handler row list (for WS-triggered refresh)
   - `/partials/app-jobs/{app_key}` — full job row list (for WS-triggered refresh)
   - `/partials/app-logs/{app_key}` — app-scoped log entries
   - Retain instance-scoped variants: `/partials/app-handlers/{app_key}/{index}` etc.

3. Create `src/hassette/web/templates/pages/app_detail.html`:
   - Extends `base.html`, sets `active_page = "apps"`
   - Breadcrumb: Apps / {app_key}
   - App header: title, status badge, instance/PID/uptime meta, controls area
   - Controls: log level toggle (hx-post to 501 endpoint), reload button, stop button
   - Health strip: 4 cards via `health_card` macro (init status, error rate, avg duration, last activity)
   - Event Handlers section: handler rows via `handler_row` macro
   - Scheduled Jobs section: job rows via `job_row` macro
   - Logs section: `log_table` macro (app-scoped)
   - `data-live-on-app` attributes on refreshable sections
   - Stats-only partial with `hx-trigger="every 5s" hx-sync="this:replace"`

4. Create partials: `handler_invocations.html`, `job_executions.html`, `app_health_strip.html`, `app_handlers.html`, `app_jobs.html`, `app_logs.html`
   - `handler_invocations.html` includes the event payload placeholder (`invocation__payload` div, hidden, with future endpoint)
   - `job_executions.html` includes "Run now" button placeholder (501 endpoint)

5. Update `src/hassette/web/static/js/live-updates.js`:
   - Add IntersectionObserver-based visibility tracking
   - Only schedule refreshes for visible elements
   - `hx-sync="this:replace"` on polling partials

## Review Guidance

- The identity model bug fix is a hard blocker: jobs MUST filter by `app_key + instance_index`, never `owner_id`.
- Handler expand MUST use Alpine.js `@click` + `htmx.ajax()` pattern, NOT `hx-trigger="intersect"`.
- Stats-only polling partial (`app-handler-stats`) must be separate from the full handler list partial to avoid morphing expanded rows.
- All `data-testid` attributes from the macro in WP03 must be present.
- Deferred feature buttons must point to endpoints that return 501, not be absent.
