## Objectives & Success Criteria

- Global Logs page (`/logs`) renders the shared log table component with all-app log entries.
- Log table fetches initial entries from `/api/logs/recent` and streams real-time entries via WS `log` subscription.
- Client-side filtering by level (DEBUG/INFO/WARNING/ERROR/CRITICAL), app key, and free-text search.
- Sortable columns (timestamp, level, app, message).
- WS log subscription sends `{ type: "subscribe", data: { logs: true, min_log_level: "INFO" } }` on mount.
- Changing the minimum log level sends an updated subscribe message.
- Ring buffer + version signal pattern ensures efficient rendering at high log volumes.

## Subtasks

1. Complete `frontend/src/pages/logs.tsx` — global log viewer. Fetches `/api/logs/recent` on mount. Composes the shared `<LogTable>` component with `showAppColumn=true`.
2. Enhance `frontend/src/components/shared/log-table.tsx` (started in WP05) — add sortable column headers, level dropdown filter, app key dropdown filter, search input.
3. Implement WS log subscription: send `subscribe` message on mount, update `min_log_level` when level filter changes.
4. Implement efficient log rendering: component reads `logs.version` signal to know when to re-render, then reads `logs.buffer.toArray()` for display. Apply client-side filters (level, app, search) as computed values.
