## Objectives & Success Criteria

- `frontend/` directory exists with a working Vite + Preact project that builds to `src/hassette/web/static/spa/`.
- `npm run build` produces `dist/index.html` + `dist/assets/*.js` + `dist/assets/*.css`.
- `npm run dev` starts Vite dev server with HMR, proxying `/api/*` and `/api/ws` (with `ws: true`) to `localhost:5000`.
- wouter routing is configured with 5 routes: `/` (dashboard), `/apps` (apps list), `/apps/:key` (app detail), `/logs` (global logs), `*` (404).
- Context-based signal injection is wired: `createAppState()` factory, `AppStateContext`, `useAppState()` hook.
- `useWebSocket` hook connects to `/api/ws`, dispatches typed messages via `batch()`, handles reconnection with exponential backoff, refetches page data on reconnect.
- Ring buffer utility class exists for log entries.
- `<ErrorBoundary>` component wraps each page route.
- Layout shell renders sidebar, status bar (with connection state machine), and alert banner.
- `tokens.css` and `global.css` are set up. Fonts self-hosted in `public/fonts/`.
- `openapi-typescript` generates `types.ts` from committed `openapi.json`.
- `app.py` modified: SPA catch-all with guards replaces template routes. `run_web_ui` guard preserved.
- Dockerfile updated with Node.js multi-stage frontend builder.
- `.gitignore` updated for `src/hassette/web/static/spa/` and `frontend/node_modules/`.
- `noxfile.py` updated: `frontend` session builds SPA; `e2e` depends on it; `dev` checks for built SPA.

## Subtasks

1. Initialize `frontend/` with `npm init -y`. Add dependencies: `preact`, `@preact/signals`, `wouter`. Add dev dependencies: `vite`, `@preact/preset-vite`, `typescript`, `openapi-typescript`.
2. Create `frontend/vite.config.ts` with Preact preset, `outDir` pointing to `../src/hassette/web/static/spa`, dev server proxy config (`/api` → `http://localhost:5000`, `/api/ws` with `ws: true`).
3. Create `frontend/tsconfig.json` with Preact JSX pragma, strict mode, path aliases.
4. Move `tokens.css` from `src/hassette/web/static/css/` to `frontend/src/tokens.css`.
5. Create `frontend/src/global.css` — extract shared design primitives (`.ht-card`, `.ht-btn`, `.ht-status-badge`, `.ht-item-row`, etc.) from current `style.css`. Include reset and font-face declarations.
6. Self-host fonts: download Space Grotesk, DM Sans, JetBrains Mono to `frontend/public/fonts/`.
7. Create `frontend/src/state/create-app-state.ts` — factory function returning signals (`appStatus`, `connection`, `logs` with RingBuffer + version signal, `theme`, `sessionId`).
8. Create `frontend/src/state/context.ts` — `AppStateContext` and `useAppState()` hook.
9. Create `frontend/src/utils/ring-buffer.ts` — generic `RingBuffer<T>` class with `push()`, `toArray()`, `length`.
10. Create `frontend/src/api/client.ts` — fetch wrapper with base URL, JSON parsing, error handling.
11. Run `npx openapi-typescript frontend/openapi.json -o frontend/src/api/types.ts` to generate REST types.
12. Create `frontend/src/api/ws-types.ts` — hand-authored TypeScript discriminated union for WS messages (from WP01).
13. Create `frontend/src/api/endpoints.ts` — typed API functions using generated types.
14. Create `frontend/src/hooks/use-websocket.ts` — connects to `/api/ws`, dispatches typed messages via `batch()`, handles reconnection with exponential backoff (carry over from `ws-handler.js`), exposes `onReconnect` callback.
15. Create `frontend/src/hooks/use-api.ts` — data fetching hook with signal-based caching.
16. Create `frontend/src/components/layout/error-boundary.tsx` — catches component errors, shows last good state + retry.
17. Create `frontend/src/components/layout/sidebar.tsx` — 56px icon-rail nav with pulse dot.
18. Create `frontend/src/components/layout/status-bar.tsx` — connection state machine (`connected | reconnecting | disconnected`).
19. Create `frontend/src/components/layout/alert-banner.tsx` — failed apps alert.
20. Create `frontend/src/app.tsx` — root component: `AppStateContext.Provider` → `ErrorBoundary` → Layout → wouter Router with 5 routes. Stub page components initially.
21. Create `frontend/src/main.tsx` — entry point, imports `tokens.css` and `global.css`, renders `<App />`.
22. Create stub page components: `dashboard.tsx`, `apps.tsx`, `app-detail.tsx`, `logs.tsx`, `not-found.tsx` — each renders a placeholder with page title.
23. Update `src/hassette/web/app.py` — replace template routes with SPA catch-all (with extension and `api/` guards), `run_web_ui` guard preserved, mount `/assets` from SPA directory.
24. Update `Dockerfile` — add `FROM node:22-slim AS frontend` stage, `COPY --from=frontend /app/frontend/dist/ /app/src/hassette/web/static/spa/`.
25. Update `.gitignore` — add `src/hassette/web/static/spa/`, `frontend/node_modules/`, `frontend/dist/`.
26. Update `noxfile.py` — add `frontend` session (`npm ci --prefix frontend && npm run build --prefix frontend`), make `e2e` depend on it, add SPA existence check to `dev`.
27. Update `package.json` at repo root — add `build:frontend` script.
28. Verify `npm run build` produces output and `uvicorn` serves the SPA at `/`.

## Review Guidance

- Verify `vite.config.ts` proxy config includes `ws: true` for the WebSocket endpoint and routes `/api/ws` before `/api`.
- Verify `createAppState()` uses signals (not useState) and returns a plain object (not a class).
- Verify `useWebSocket` uses `batch()` from `@preact/signals` for all signal updates in the message handler.
- Verify the SPA catch-all in `app.py` uses the explicit extension list (not "any dot") for the static asset guard.
- Verify `run_web_ui` guard wraps the SPA serving block.
- Verify Dockerfile frontend stage uses `COPY --from=frontend /app/frontend/dist/` (not `/app/src/...`).
- Verify wouter is used (not preact-router or preact-iso).
