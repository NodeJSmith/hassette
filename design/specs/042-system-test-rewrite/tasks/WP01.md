## Objectives & Success Criteria

The system test infrastructure is extended with shared helpers, app fixture files are committed, the HA fixture config is updated, and all three old test files are deleted. After this WP, `uv run nox -s system` should run and find zero tests (the old ones are gone, new ones aren't written yet).

## Subtasks

1. Delete `tests/system/test_startup.py`, `tests/system/test_shutdown.py`, and `tests/system/test_telemetry_lifecycle.py`
2. Add `toggle_and_capture` async helper to `tests/system/conftest.py` ‚Äî registers a `bus.on_state_change` handler for a given entity_id, calls `api.call_service` to toggle it, waits via `wait_for` until at least one event is captured, and returns the captured events list. Accepts: `bus` (Bus), `api` (Api), `entity_id` (str), optional `service_domain`/`service_action` (default `"light"/"toggle"`), optional `timeout` (default 10.0)
3. Add `make_web_system_config` factory to `tests/system/conftest.py` ‚Äî like `make_system_config` but sets `run_web_api=True` and uses a dynamic port (find an available port via `socket.bind(("", 0))`). Returns a tuple of `(HassetteConfig, str)` where the string is the base URL (e.g., `http://localhost:PORT`)
4. Add `system_app_dir` session-scoped fixture to `tests/system/conftest.py` ‚Äî returns `Path(__file__).parent / "apps"`
5. Create `tests/system/apps/__init__.py` (empty)
6. Create `tests/system/apps/trivial_app.py` ‚Äî exports `TrivialApp(App)` with a no-op `on_initialize`
7. Create `tests/system/apps/config_app.py` ‚Äî exports `ConfigAppConfig(AppConfig)` with `env_prefix="config_app_"` and `greeting: str = "hello"`, and `ConfigApp(App[ConfigAppConfig])` that stores `self.app_config.greeting` in `on_initialize`
8. Create `tests/system/apps/bus_handler_app.py` ‚Äî exports `BusHandlerApp(App)` that registers `bus.on_state_change("light.kitchen_lights")` in `on_initialize` and appends events to `self.captured_events: list[RawStateChangeEvent]`
9. Add `input_boolean:` section to `tests/fixtures/ha-config/configuration.yaml` with a `test:` entity named "System Test Toggle"

## Test Strategy

N/A ‚Äî this WP creates test infrastructure and deletes old tests. No testable code changes. Verification is that `uv run nox -s system` runs without import errors (it will collect 0 tests since no new test files exist yet).
