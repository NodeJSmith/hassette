## Objectives & Success Criteria

`tests/system/test_startup.py` contains 3 passing tests and `tests/system/test_shutdown.py` contains 2 passing tests. These validate the core lifecycle against a real HA instance and prove the infrastructure from WP01 works end-to-end.

## Subtasks

1. Create `tests/system/test_startup.py` with `pytestmark = [pytest.mark.system]` and these tests:
   - `test_startup_completes` — `startup_context` yields a Hassette with `session_id > 0`
   - `test_demo_entities_visible` — `api.get_states()` returns a list containing `light.kitchen_lights` and `binary_sensor.movement_backyard`
   - `test_session_persisted_as_running` — query the sessions table via `hassette.database_service.db`, verify a row with `status='running'` exists for the current `session_id`
2. Create `tests/system/test_shutdown.py` with `pytestmark = [pytest.mark.system]` and these tests:
   - `test_clean_shutdown` — after `startup_context` exits: `hassette._shutdown_completed is True`, `hassette.status == ResourceStatus.STOPPED`, `hassette.event_streams_closed is True`, all children are STOPPED, and a fresh `sqlite3.connect` query shows the session row has `status='success'` and non-null `stopped_at`
   - `test_failed_service_cascade_triggers_shutdown` — create a `_AlwaysFailingService(Service)` whose `on_initialize` raises, add it as a child, stub `hassette.shutdown` to just set `shutdown_event`, fire a FAILED event via `hassette.send_event`, `wait_for` until `shutdown_event.is_set()` within 30s. Restore real shutdown in `finally` block.
3. Run `timeout 300 uv run pytest -m system -v -x -n 0 --tb=short` and verify all 5 tests pass

## Test Strategy

This WP IS the tests. The 5 tests verify:
- `test_startup_completes` — basic connectivity and session creation
- `test_demo_entities_visible` — HA API integration returns real entities
- `test_session_persisted_as_running` — telemetry DB write pipeline works
- `test_clean_shutdown` — full shutdown lifecycle with resource cleanup and session finalization
- `test_failed_service_cascade_triggers_shutdown` — ServiceWatcher restart → exhaust → shutdown cascade

## Review Guidance

- All polling must use `wait_for` — no `while loop.time() < deadline` patterns
- No `caplog` imports or log-based assertions
- `test_clean_shutdown` must query the DB via a fresh `sqlite3.connect` (not the hassette connection) since the async connection is closed after shutdown
- `test_failed_service_cascade_triggers_shutdown` must restore the real `shutdown` method in a `finally` block so `startup_context` can clean up
- Verify `filterwarnings("default::DeprecationWarning")` is NOT in pytestmark (was in old tests, not needed)
