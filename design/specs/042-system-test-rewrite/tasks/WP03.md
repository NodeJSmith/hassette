## Objectives & Success Criteria

`tests/system/test_bus.py` contains 8 passing tests covering state change handlers, attribute changes, glob patterns, predicates, debounce, throttle, once-handlers, and multiple handlers on the same entity.

## Subtasks

1. Create `tests/system/test_bus.py` with `pytestmark = [pytest.mark.system]` and these tests:
   - `test_state_change_handler_fires` — register `bus.on_state_change("light.kitchen_lights", handler=capture)`, toggle light, `wait_for` at least 1 event, assert `len(received) >= 1`
   - `test_attribute_change_handler_fires` — register `bus.on_attribute_change("light.kitchen_lights", "brightness", handler=capture)`, toggle light, `wait_for` event. Note: the demo light may not have brightness; if not, use a different attribute or skip this test with `pytest.mark.skip` and a note
   - `test_glob_pattern_matching` — register `bus.on_state_change("light.*", handler=capture)`, toggle `light.kitchen_lights`, `wait_for` event, verify received
   - `test_changed_to_predicate` — register with `changed_to="on"`, ensure light is off first (call `light.turn_off`), then `light.turn_on`, verify handler fires. Then `light.turn_off` — verify handler does NOT fire again (wait briefly, assert still just 1 event)
   - `test_debounce` — register with `debounce=1.0`, toggle 3 times rapidly (no sleep between), wait 2s after last toggle, verify handler fired exactly once
   - `test_throttle` — register with `throttle=2.0`, toggle 3 times with 0.3s gaps, wait 1s, verify handler fired at most once
   - `test_once_handler` — register with `once=True`, toggle twice (with brief pause between), verify handler fired exactly once
   - `test_multiple_handlers_same_entity` — register 2 handlers for `light.kitchen_lights`, toggle once, `wait_for` both received lists to have >= 1 event
2. Run `timeout 300 uv run pytest tests/system/test_bus.py -m system -v -x -n 0 --tb=short` and verify all tests pass

## Test Strategy

The 8 tests verify bus event filtering behaviors that depend on real HA event delivery timing — behaviors that can only be partially tested with mocks:
- `test_state_change_handler_fires`, `test_glob_pattern_matching`, `test_multiple_handlers_same_entity` — basic event routing
- `test_attribute_change_handler_fires` — attribute-level filtering
- `test_changed_to_predicate` — predicate filtering with both positive and negative cases
- `test_debounce`, `test_throttle` — timing-sensitive rate limiting
- `test_once_handler` — one-shot subscription cleanup

## Review Guidance

- Event count assertions must use `>=` for positive cases (HA may emit multiple state_changed events per toggle) and exact counts only where the test explicitly controls the number of toggles and waits for debounce/throttle windows
- `test_changed_to_predicate` must ensure the light starts in a known state (`turn_off` then `turn_on`) rather than assuming initial state
- `test_debounce` and `test_throttle` are the most flake-prone tests in the suite — verify timeouts are generous (at least 2x the debounce/throttle window)
- All tests must use `wait_for` for polling, not `asyncio.sleep` followed by a bare assert
- The `toggle_and_capture` helper from WP01 can be used where appropriate, but some tests need finer control (e.g., debounce tests need multiple toggles without waiting between them)
