## Objectives & Success Criteria

`tests/system/test_api.py` contains 6 passing tests and `tests/system/test_state_proxy.py` contains 4 passing tests. Together they prove the HA API integration and state caching layer work against a real HA instance.

## Subtasks

1. Create `tests/system/test_api.py` with `pytestmark = [pytest.mark.system]` and these tests:
   - `test_get_state_single_entity` ‚Äî `await hassette.api.get_state("light.kitchen_lights")`, verify the result has `.entity_id == "light.kitchen_lights"` and `.state` is a string ("on" or "off")
   - `test_set_state_roundtrip` ‚Äî `await hassette.api.set_state("input_boolean.test", "on", {})`, then `await hassette.api.get_state("input_boolean.test")`, verify `.state == "on"`. Then set to "off" and verify again. (Uses the `input_boolean.test` entity added in WP01)
   - `test_fire_event_received_by_bus` ‚Äî register `bus.on("custom_test_event", handler=capture)`, `await hassette.api.fire_event("custom_test_event", {"key": "value"})`, `wait_for` event received, verify the event data contains `{"key": "value"}`
   - `test_render_template` ‚Äî `await hassette.api.render_template("{{ states('light.kitchen_lights') }}")`, verify result is "on" or "off" (not empty, not an error string)
   - `test_get_config` ‚Äî `await hassette.api.get_config()`, verify the result is a dict containing `"components"` key and `"unit_system"` key
   - `test_get_history` ‚Äî toggle `light.kitchen_lights`, wait 1s, call `await hassette.api.get_history("light.kitchen_lights", start_time=<now minus 2 minutes>)`, verify the result is a non-empty list
2. Create `tests/system/test_state_proxy.py` with `pytestmark = [pytest.mark.system]` and these tests:
   - `test_initial_state_loaded` ‚Äî verify `hassette.state_proxy.states` is non-empty and `"light.kitchen_lights" in hassette.state_proxy`
   - `test_state_change_propagates_to_proxy` ‚Äî read current state of `light.kitchen_lights` from proxy, toggle the light via API, `wait_for` until `hassette.state_proxy.get_state("light.kitchen_lights")["state"]` differs from the original value
   - `test_state_manager_typed_access` ‚Äî `hassette.states.light["kitchen_lights"]`, verify the result has `.state` attribute (string) and `.attributes` (dict-like)
   - `test_state_manager_domain_iteration` ‚Äî `entities = list(hassette.states.light)`, verify `len(entities) > 0` and each entry is a tuple of `(entity_id, state)`
3. Run `timeout 300 uv run pytest tests/system/test_api.py tests/system/test_state_proxy.py -m system -v -x -n 0 --tb=short` and verify all 10 tests pass

## Test Strategy

**test_api.py** (6 tests):
- `test_get_state_single_entity`, `test_set_state_roundtrip` ‚Äî REST API state read/write
- `test_fire_event_received_by_bus` ‚Äî custom event round-trip through HA ‚Üí bus
- `test_render_template` ‚Äî Jinja2 template rendering via HA
- `test_get_config` ‚Äî HA config endpoint
- `test_get_history` ‚Äî HA history/recorder integration

**test_state_proxy.py** (4 tests):
- `test_initial_state_loaded` ‚Äî cache population at startup
- `test_state_change_propagates_to_proxy` ‚Äî real-time cache invalidation
- `test_state_manager_typed_access`, `test_state_manager_domain_iteration` ‚Äî typed state access patterns

## Review Guidance

- `test_set_state_roundtrip` depends on `input_boolean.test` from WP01's HA config change ‚Äî if that entity doesn't exist the test will fail with EntityNotFoundError, not a vague assertion error
- `test_state_change_propagates_to_proxy` must read the initial state BEFORE toggling, then `wait_for` the state to differ ‚Äî do not assume the light starts as "on" or "off"
- `test_get_history` needs a brief `asyncio.sleep(1)` after toggling to give HA's recorder time to persist ‚Äî then use `wait_for` on the API call result
- `test_fire_event_received_by_bus` uses `bus.on()` (generic topic), not `bus.on_state_change()` ‚Äî verify the topic string matches what HA fires
- `test_state_manager_domain_iteration` should verify iteration returns `(entity_id, state)` tuples ‚Äî check the actual return type from `StateManager`
