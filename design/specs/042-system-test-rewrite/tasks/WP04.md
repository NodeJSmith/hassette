## Objectives & Success Criteria

`tests/system/test_scheduler.py` contains 6 passing tests covering `run_in`, `run_every`, `run_once`, job cancellation, group cancellation, and telemetry persistence of job executions.

## Subtasks

1. Create `tests/system/test_scheduler.py` with `pytestmark = [pytest.mark.system]` and these tests:
   - `test_run_in_fires_after_delay` ‚Äî schedule `hassette._scheduler.run_in(callback, 1)`, `wait_for(lambda: len(fired) >= 1, timeout=5.0)`, assert fired
   - `test_run_every_fires_multiple_times` ‚Äî schedule `run_every(callback, seconds=1)`, `wait_for(lambda: len(fired) >= 2, timeout=5.0)`, assert at least 2 firings
   - `test_run_once_at_time` ‚Äî compute a time ~2s in the future, schedule `run_once(callback, at=<time_str>)`, `wait_for(lambda: len(fired) >= 1, timeout=5.0)`
   - `test_job_cancellation` ‚Äî schedule `run_in(callback, 2)`, call `job.cancel()` immediately, `await asyncio.sleep(3)`, assert `len(fired) == 0`
   - `test_group_cancellation` ‚Äî schedule 3 `run_in` jobs with `group="test_group"` (delays 2, 3, 4s), call `hassette._scheduler.cancel_group("test_group")`, `await asyncio.sleep(5)`, assert none fired
   - `test_job_execution_persisted` ‚Äî schedule `run_in(callback, 1)`, `wait_for` until fired, then poll `job_executions` table via `hassette.database_service.db` until a row with correct `session_id` appears (use `wait_for` with an async-friendly predicate or an inline polling loop wrapped in `wait_for`)
2. Run `timeout 300 uv run pytest tests/system/test_scheduler.py -m system -v -x -n 0 --tb=short` and verify all tests pass

## Test Strategy

The 6 tests verify scheduler timing behaviors against the real event loop:
- `test_run_in_fires_after_delay`, `test_run_every_fires_multiple_times`, `test_run_once_at_time` ‚Äî trigger types fire at expected times
- `test_job_cancellation`, `test_group_cancellation` ‚Äî cancellation prevents execution
- `test_job_execution_persisted` ‚Äî telemetry write pipeline records job executions

## Review Guidance

- Cancellation tests use `asyncio.sleep` (not `wait_for`) to wait for the job's scheduled time to pass, then assert the callback was never called ‚Äî this is the correct pattern for "verify something did NOT happen"
- `test_run_once_at_time` needs to compute the target time in the test using hassette's timezone-aware time utilities or `datetime.now()` ‚Äî verify the time string format matches what `run_once(at=...)` expects
- `test_job_execution_persisted` must poll the DB via `wait_for` since the write pipeline is async and batched ‚Äî a bare query immediately after the callback fires may return empty
- All positive-case tests must use `wait_for` with generous timeouts (5s for 1s delays) to account for event loop jitter
- Verify callbacks are simple (append to a list) and don't raise ‚Äî exceptions in callbacks could mask the real test failure
