## Objectives & Success Criteria

Four Docker Deployment pages are rewritten: `docs/pages/getting-started/docker/index.md`, `docs/pages/getting-started/docker/dependencies.md`, `docs/pages/getting-started/docker/image-tags.md`, and `docs/pages/getting-started/docker/troubleshooting.md`. All inline fenced code blocks are converted to snippet files. `mkdocs build --strict` passes after all changes.

The Docker Setup page gives a new user a working `docker-compose.yml` and `hassette.toml` in a single linear flow. The Dependencies page accurately reflects the current dependency installation mechanism. The Image Tags page lists currently valid tags. The Troubleshooting page covers real failure modes with accurate diagnostic steps.

## Subtasks

1. Read the nav audit (`docs/nav-audit.md`) Section 2 for per-page recommendations on Docker pages.

2. Read all four Docker pages and the existing snippet files in `docs/pages/getting-started/docker/snippets/`.

3. Read `docs/pages/getting-started/docker/snippets/docker-compose.yml` and `src/hassette/` to verify the current Docker setup flow is accurate. Cross-check environment variables, volume mounts, and required configuration against the actual codebase.

4. Rewrite `docs/pages/getting-started/docker/index.md`. Convert all inline fenced blocks to snippet files in `docs/pages/getting-started/docker/snippets/`. Existing snippets (`docker-compose.yml`, `hassette.toml`, `my_app.py`) may be reused or updated as needed.

5. Read `docs/pages/getting-started/docker/dependencies.md` (325 lines). Verify accuracy against the current dependency installation mechanism (check `src/hassette/` and the design specs for Docker dep strategy at `design/specs/docker-dep-redesign/design.md` if it exists). Rewrite inaccurate sections. Convert all inline code blocks to snippet files.

6. Read `docs/pages/getting-started/docker/image-tags.md` (155 lines). Verify the listed tags against the actual Docker image tagging scheme in `.github/workflows/build_and_publish_image.yml`. Rewrite stale tag descriptions. Convert all inline code blocks to snippet files.

7. Read `docs/pages/getting-started/docker/troubleshooting.md` (412 lines). Verify each troubleshooting scenario against the current Docker setup. Remove any sections that describe problems that no longer exist or have changed. Convert all inline code blocks to snippet files.

8. Run `python -m py_compile` on every new or modified `.py` snippet file in `docs/pages/getting-started/docker/snippets/`.

9. Run `mkdocs build --strict`. Fix any errors or warnings before committing.

10. Commit all changes.
