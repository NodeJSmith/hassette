## Objectives & Success Criteria

`tools/gen_ref_pages.py` is rewritten to emit `::: module.path` stubs only for modules on the allowlist produced by WP01 (seeded from `hassette.__all__` and `hassette.test_utils.__init__.py` Tier 1, plus curated additions from the nav audit). Docstrings are added or expanded for every class/function on the allowlist that was flagged as thin in WP01's docstring gap list. `mkdocs build --strict` passes. No public API behavior is changed.

After this WP, the API Reference section shows only the public surface — no internal modules (bus.listeners, bus.rate_limiter, core.core, migrations/*, etc.) appear in the generated reference.

## Subtasks

1. Read the nav audit (`design/audits/2026-04-11-docs-nav-audit/audit.md`) Section 6 (allowlist) and Section 7 (docstring gap list). These are the authoritative inputs for this WP.

2. Rewrite `tools/gen_ref_pages.py` to filter generated stubs to the allowlist. The allowlist is a set of dotted module paths (e.g., `hassette.bus.bus`, `hassette.app.app`, `hassette.test_utils`). For each `.py` file found in `SRC_DIR.rglob("*.py")`, check whether its module path is in the allowlist before emitting the `::: module.path` stub. Modules not on the allowlist are silently skipped.

   Implement the allowlist as a frozenset constant at the top of the file (derived from the nav audit's Section 6 output):
   ```python
   PUBLIC_MODULES: frozenset[str] = frozenset({
       # Populate from WP01's allowlist — do not guess
       ...
   })
   ```

3. Verify that the rewritten `gen_ref_pages.py` produces a reference that includes every module on the allowlist and excludes all internal modules. Run `mkdocs build` in debug mode (`GEN_REF_DEBUG=1 mkdocs build`) and inspect the output to confirm.

4. Audit all `[Symbol][hassette.*]` autoref links in the narrative docs. Run:
   ```
   grep -r '\[.*\]\[hassette\.' docs/pages/
   ```
   For each result, verify the target module is on the allowlist. If a link targets an internal module that was just removed from the reference, either (a) update the link to target the allowlisted public symbol that exposes this type, or (b) convert the link to plain text if no public equivalent exists.

5. For each class/function in the docstring gap list (WP01 Section 7), open the relevant source file in `src/hassette/` and add or expand the docstring. Requirements per entry:
   - At least a one-line summary.
   - All parameters with non-obvious types are annotated with a `Args:` section (Google-style docstring).
   - Do NOT change any method signatures, behavior, or non-docstring code.

6. Run `mkdocs build --strict`. Fix any unresolved autoref warnings before committing.

7. Commit all changes.
