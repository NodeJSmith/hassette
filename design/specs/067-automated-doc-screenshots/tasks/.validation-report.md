## Step 1: Requirements Extracted from design.md

### Functional Requirements

| Identifier | Section | Text |
|------------|---------|------|
| FR#1 | Functional Requirements | A wrapper script starts the demo environment, waits for readiness, captures all screenshots defined in a manifest, and tears down the demo |
| FR#2 | Functional Requirements | A YAML manifest defines each screenshot with URL path, output file path, viewport dimensions, and optional element selector |
| FR#3 | Functional Requirements | Full-page screenshots capture at 1400px viewport width to match existing screenshot dimensions |
| FR#4 | Functional Requirements | Element-level screenshots use `data-testid` CSS selectors to crop specific UI components |
| FR#5 | Functional Requirements | CSS animations and transitions are disabled during capture via injected stylesheet |
| FR#6 | Functional Requirements | The wrapper script exits with a non-zero code if any screenshot fails to capture |
| FR#7 | Functional Requirements | The wrapper script can run from any working directory within the repository |

**Total FRs: 7** (FR#1, FR#2, FR#3, FR#4, FR#5, FR#6, FR#7)

### Acceptance Criteria

| Identifier | Section | Text |
|------------|---------|------|
| AC#1 | Acceptance Criteria | Running `uv run python scripts/capture_screenshots.py` from the repo root produces all 16 `docs/_static/web_ui_*.png` files (FR#1, FR#2) |
| AC#2 | Acceptance Criteria | Full-page screenshots are 1400px wide (FR#3) |
| AC#3 | Acceptance Criteria | Detail screenshots (sidebar, status bar, error spotlight, etc.) capture only the target element, not the full page (FR#4) |
| AC#4 | Acceptance Criteria | No CSS animations or transitions are visible in captured screenshots (FR#5) |
| AC#5 | Acceptance Criteria | If the demo environment fails to start, the script exits non-zero with a descriptive error and no orphaned processes (FR#6) |
| AC#6 | Acceptance Criteria | Adding a new screenshot requires only a YAML manifest entry and a Markdown image reference — no script changes (FR#2) |

**Total ACs: 6** (AC#1, AC#2, AC#3, AC#4, AC#5, AC#6)
