## Objectives & Success Criteria

Add 14 new simulate methods to `SimulationMixin` and their supporting event factories. After this WP:

- Every `on_*` Bus subscription method has a corresponding `simulate_*` method on `AppTestHarness`
- All new methods follow the `_require_harness` → construct → send → drain pattern
- Convenience methods delegate with `timeout=timeout` threading
- Lifecycle simulate methods include drain-gap `Note:` in docstrings
- Each new simulate method has at least one integration test

## Subtasks

2. Add `_create_service_registered_event(domain: str, service: str) -> ServiceRegisteredEvent` — uses `_create_hass_event("service_registered", {"domain": domain, "service": service})` with isinstance guard

## Test Strategy

All tests in `tests/integration/test_app_test_harness.py`. Raw handler tests verify handler invocation count and basic payload field. DI tests use typed annotations and assert on specific field values. Convenience delegation tests verify the handler fires (they inherit correctness from the base method tests).
