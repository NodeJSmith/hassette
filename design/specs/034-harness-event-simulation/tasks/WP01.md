## Objectives & Success Criteria

Fix `create_state_change_event` and `create_call_service_event` so they produce real, complete event objects that work with typed DI handlers. Add `_create_hass_event` private helper to reduce boilerplate. After this WP:

- `create_state_change_event` produces `RawStateChangeEvent` with complete `HassStateDict` sub-dicts (entity_id, context, timestamps)
- `create_call_service_event` produces a real `CallServiceEvent` (not SimpleNamespace)
- A handler using `D.StateNew[BinarySensorState]` receives a valid typed model via `simulate_state_change`
- A handler using `D.Domain` receives the correct domain via `simulate_call_service`
- All existing tests pass unchanged

## Subtasks

1. Add `_create_hass_event(event_type: str, data: dict[str, Any]) -> Any` private helper to `src/hassette/test_utils/helpers.py` — builds `HassEventEnvelopeDict` envelope with common fields (`id` with comment explaining it's discarded, `type`, `origin`, `time_fired`, `context`) and delegates to `create_event_from_hass`
2. Refactor `create_state_change_event` to use `make_state_dict` for `old_state`/`new_state` sub-dicts — `make_state_dict(entity_id, str(old_value), attributes=old_attrs) if old_value is not None else None` pattern for both
3. Refactor `create_call_service_event` to use `_create_hass_event("call_service", {"domain": domain, "service": service, "service_data": service_data or {}})` with `assert isinstance(event, CallServiceEvent)` guard — update return type annotation to `CallServiceEvent`
4. Grep for `isinstance(.*CallServiceEvent)` and `"hass.event.call_service"` string comparisons across the test suite — fix any tests that break due to the SimpleNamespace→real-event change
5. Add integration tests in `tests/integration/test_app_test_harness.py`:
   - `test_simulate_state_change_typed_di_state_new` — handler with `D.StateNew[BinarySensorState]` receives valid model
   - `test_simulate_state_change_typed_di_state_old` — handler with `D.StateOld[BinarySensorState]` receives valid model
   - `test_simulate_state_change_none_old_value` — `old_value=None` produces `None` old_state dict, `D.MaybeStateOld` returns `None`
   - `test_simulate_state_change_none_new_value` — `new_value=None` produces `None` new_state dict
   - `test_simulate_call_service_typed_di_domain` — handler with `D.Domain` receives correct domain string
   - `test_simulate_attribute_change_typed_di` — handler with `D.StateNew[SensorState]` works through the delegation path

## Test Strategy

Tests in `tests/integration/test_app_test_harness.py`. Each test creates an `AppTestHarness` with an app whose handler uses typed DI annotations and asserts on the received model's fields. The `None` value tests verify the exact dict-level `None` (not `{"state": None}`).
