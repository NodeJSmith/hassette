## Objectives & Success Criteria

- `CommandExecutor.__init__` creates `asyncio.Queue(maxsize=config.telemetry_write_queue_max)`.
- `put_nowait()` calls catch `QueueFull`, log ERROR, increment `self._dropped_overflow`.
- 75% capacity warning (rate-limited) is logged when queue exceeds threshold.
- `RetryableBatch` dataclass exists. Queue type is `asyncio.Queue[HandlerInvocationRecord | JobExecutionRecord | RetryableBatch]`.
- `_drain_and_persist` handles `RetryableBatch` items (expands into current batch).
- `_persist_batch` error classification: `OperationalError` → retry via `RetryableBatch`; `IntegrityError` → row-by-row FK fallback; `DataError`/`ProgrammingError` → drop + REGRESSION log; other `Exception` → drop + ERROR log.
- FK violation: on `IntegrityError`, roll back, re-issue each record individually, null only the violating row's FK.
- Max retry count = 3. Retry-exhausted records increment `self._dropped_exhausted`.
- Sentinel guard updated: `listener_id == 0` / `job_id == 0` → drop + REGRESSION; `listener_id is None` / `job_id is None` → persist normally.
- `_safe_session_id()` removed. `_build_record()` reads `cmd.source_tier` and `result.is_di_failure`. Session ID from `self.hassette.session_id` directly.
- `_flush_queue` handles `RetryableBatch`, wraps `_persist_batch` in try/except (logs drops, no crash).
- `get_drop_counters()` method on `Hassette` delegates to `CommandExecutor`.
- Shutdown ordering: `CommandExecutor` priority higher than `DatabaseService`.
- `source_tier` included in `persist_batch` INSERT statements.

## Subtasks

1. Define `RetryableBatch` dataclass in `src/hassette/core/command_executor.py`.
2. Change queue initialization to `asyncio.Queue(maxsize=self.hassette.config.telemetry_write_queue_max)`.
3. Update all `put_nowait()` call sites to catch `asyncio.QueueFull`, log ERROR, increment `_dropped_overflow`.
4. Add 75% capacity check with rate-limited WARNING log.
5. Update `_drain_and_persist` to handle `RetryableBatch` items (expand into current batch invocations/job_executions). Add type-exhaustiveness guard.
6. Rewrite `_persist_batch` sentinel guard: `== 0` → REGRESSION drop; `is None` → allow.
7. Add error classification to `_persist_batch`: `OperationalError` → re-enqueue as `RetryableBatch(retry_count+1)`; `IntegrityError` → FK violation handler; `DataError`/`ProgrammingError` → drop + REGRESSION; other → drop + ERROR.
8. Implement FK violation handler: on `IntegrityError`, rollback, loop through records individually, catch per-row, null the violating FK, retry single INSERT, log WARNING.
9. Remove `_safe_session_id()`. Update `_build_record()` to use `self.hassette.session_id` directly, read `cmd.source_tier`, read `result.is_di_failure`.
10. Update `_flush_queue` to handle `RetryableBatch`, wrap `_persist_batch` in try/except (log drop count, increment `_dropped_exhausted`, no raise).
11. Update `TelemetryRepository.persist_batch()` INSERT statements to include `source_tier` column.
12. Set `CommandExecutor` shutdown priority higher than `DatabaseService` in the Resource hierarchy.
13. Add `get_drop_counters()` method on `Hassette` in `src/hassette/core/core.py`.

## Review Guidance

- Verify the sentinel guard distinguishes `== 0` (REGRESSION, drop) from `is None` (orphan, persist). These must NOT be conflated.
- Verify FK violation handler uses row-by-row INSERT (not batch-level null-all).
- Verify `_flush_queue` never raises — shutdown must complete even if DB is closed.
- Verify `RetryableBatch` is handled in both `_drain_and_persist` AND `_flush_queue`.
- Verify `source_tier` in the INSERT is from the record field, not a hardcoded value.
