## Objectives & Success Criteria

- `_make_internal_invoke_fn` is deleted from `bus_service.py`.
- The `db_id is None` branch in `_dispatch` is removed — all listeners go through `_make_tracked_invoke_fn`.
- The inner `db_id is None` guard in `_make_tracked_invoke_fn` (lines 352-358) is removed — `InvokeHandler` is created with `listener_id=None` when `db_id is None`.
- `_make_tracked_invoke_fn` reads `listener.source_tier` and passes it to `InvokeHandler`.
- The `db_id is None` branch in `scheduler_service.py:run_job()` is removed — all jobs go through `ExecuteJob`.
- `run_job()` reads `job.source_tier` and passes it to `ExecuteJob`.
- `BusService` has a `register_framework_listener()` convenience method.
- Framework actors at `core.py:318`, `service_watcher.py:211-216`, `app_handler.py:76-80` are registered via `register_framework_listener()` with `app_key='__hassette__'`, `source_tier='framework'`, and component-prefixed `name`.
- `_register_then_add_route()` reads `listener.source_tier` when building `ListenerRegistration`.
- `reconcile_registrations()` has an explicit guard: `app_key == '__hassette__'` → log WARNING + return (no-op).
- `once=True` deferred cleanup in `SessionManager` runs after session creation, filtered to `source_tier = 'app'`.

## Subtasks

1. Delete `_make_internal_invoke_fn` from `src/hassette/core/bus_service.py`.
2. Remove `db_id is None` branch in `_dispatch` — all listeners use `_make_tracked_invoke_fn`.
3. Remove inner `db_id is None` guard in `_make_tracked_invoke_fn` — create `InvokeHandler` with `listener_id=listener.db_id` (which may be `None`), `source_tier=listener.source_tier`.
4. Add `register_framework_listener()` method on `BusService` that calls `Listener.create(..., source_tier='framework', app_key='__hassette__')` and routes through `_register_then_add_route()`.
5. Update `_register_then_add_route()` to read `listener.source_tier` when building `ListenerRegistration`.
6. Update framework actor registration sites:
   - `core.py:318` → use `register_framework_listener(name='hassette.session_manager.on_service_crashed', ...)`
   - `service_watcher.py:211-216` → use `register_framework_listener(name='hassette.service_watcher.<method>', ...)` for each of the 4 listeners
   - `app_handler.py:76-80` → use `register_framework_listener(name='hassette.app_handler.handle_change_event', ...)`
7. Remove `db_id is None` branch in `scheduler_service.py:run_job()` — all jobs go through `ExecuteJob(source_tier=job.source_tier, ...)`.
8. Update `SchedulerService._enqueue_then_register()` to pass `source_tier` when building `ScheduledJobRegistration`.
9. Add guard in `TelemetryRepository.reconcile_registrations()`: if `app_key == '__hassette__'`, log WARNING and return.
10. Add `once=True` deferred cleanup in `SessionManager` (after `create_session()`): delete `once=True` listener rows where `stopped_at IS NOT NULL` and `source_tier = 'app'`, guarded by `NOT EXISTS` for current-session invocations.

## Test Strategy

- **`tests/integration/test_dispatch_unification.py`**: New test file.
  - `test_all_listeners_produce_telemetry` — register a listener, fire an event, verify `HandlerInvocationRecord` is queued (not silently discarded).
  - `test_pre_registration_listener_produces_orphan_record` — gate registration with `asyncio.Event`, fire handler before gate, verify record with `listener_id=None`.
  - `test_framework_listener_registration` — register via `register_framework_listener()`, verify DB row with `app_key='__hassette__'`, `source_tier='framework'`.
  - `test_framework_listener_produces_telemetry` — register framework listener, fire event, verify execution record with `source_tier='framework'`.
  - `test_reconciliation_guard_rejects_hassette` — call `reconcile_registrations('__hassette__', ...)`, verify it returns without deleting.
  - `test_reconciliation_preserves_framework_actors` — register framework + app actors, run full reconciliation for app, verify framework actors survive.
  - `test_once_true_deferred_cleanup` — create `once=True` app listeners from a stopped session, run cleanup, verify they are deleted. Verify framework `once=True` listeners are NOT deleted.
  - `test_all_jobs_produce_telemetry` — register a job, fire it, verify `JobExecutionRecord` is queued.

## Review Guidance

- Verify `_make_internal_invoke_fn` is completely deleted, not just unused.
- Verify the inner `db_id is None` guard in `_make_tracked_invoke_fn` is also removed (the design specifies removing both the outer and inner guards).
- Verify framework actor names use component prefix pattern (`hassette.<component>.<method>`).
- Verify reconciliation guard is in `TelemetryRepository`, not in `AppLifecycleService`.
- Verify `once=True` cleanup filters by `source_tier = 'app'` to protect framework `once=True` listeners.
