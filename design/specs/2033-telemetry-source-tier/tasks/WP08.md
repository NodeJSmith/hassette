## Objectives & Success Criteria

- `HassetteHarness` supports registering framework actors with `app_key='__hassette__'` and `source_tier='framework'`.
- Integration tests verify the full telemetry pipeline for framework actors: registration → dispatch → execution → persistence → query → endpoint response.
- AC-1 (framework job error → error feed), AC-2 (framework handler error → invocation record), AC-11a (pre-DB queue drain), AC-11b (pre-registration orphan), AC-19 (reconciliation safety) are all covered.
- Shutdown ordering test verifies graceful degradation when DB closes before CommandExecutor.

## Subtasks

1. Extend `HassetteHarness` in `src/hassette/test_utils/harness.py` to support `register_framework_listener()` and `register_framework_job()` methods.
2. Write framework actor end-to-end tests in `tests/integration/test_framework_telemetry.py`:
   - Register a framework listener that raises an exception.
   - Fire the event.
   - Query the dashboard errors endpoint with `?source_tier=framework`.
   - Verify the error record appears with `source_tier='framework'`.
3. Same for a framework scheduled job.
4. Test pre-DB queue drain (AC-11a): mock DB unavailability, enqueue framework execution records, restore DB, verify records persisted.
5. Test pre-registration orphan (AC-11b): gate registration, fire handler, verify `listener_id=None` record in DB.
6. Test reconciliation safety (AC-19): register framework + app actors, run reconciliation for app, verify framework actors survive.
7. Test shutdown ordering: force `DatabaseService` shutdown before `CommandExecutor`, verify graceful degradation.
8. Test tier isolation: register framework actors, call `get_all_app_summaries()`, verify `'__hassette__'` not in returned dict.
9. Test drop counter end-to-end (AC-13): fill the write queue to capacity, enqueue additional records (triggering overflow), call `/telemetry/status`, verify `dropped_overflow > 0` in the response.

## Test Strategy

All tests in this WP are integration tests. They exercise the full pipeline via `HassetteHarness` with a real SQLite database.

- **`tests/integration/test_framework_telemetry.py`**: New test file covering AC-1, AC-2, AC-11a, AC-11b, AC-19, tier isolation, and shutdown ordering.

## Review Guidance

- Verify `HassetteHarness` extension is minimal — just convenience methods, not a parallel registration path.
- Verify AC-11b uses the `asyncio.Event` gate pattern from the spec's verification description.
- Verify reconciliation safety test actually runs `reconcile_registrations` for the app, not just checks DB state.
- Verify shutdown ordering test confirms log output (dropped record count), not just absence of crash.
