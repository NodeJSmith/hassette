## Objectives & Success Criteria

- `SourceTier` field is present on: `Listener`, `ScheduledJob`, `ListenerRegistration`, `ScheduledJobRegistration`, `InvokeHandler`, `ExecuteJob`, `HandlerInvocationRecord`, `JobExecutionRecord`.
- `listener_id` on `HandlerInvocationRecord` is `int | None`. `job_id` on `JobExecutionRecord` is `int | None`.
- `ExecutionResult` gains `is_di_failure: bool` field, set in `track_execution()` via `isinstance(exc, DependencyError)`.
- `Listener.create()` accepts `source_tier: SourceTier = 'app'` parameter.
- All Pydantic response models in `telemetry_models.py` and `web/models.py` have `source_tier: SourceTier` where needed, and nullable FK/registration fields for orphan records.
- `AppConfig` Pydantic field validator rejects `app_key = '__hassette__'` with `ValueError`.
- Pyright passes with no new errors.

## Subtasks

1. Add `source_tier: SourceTier = 'app'` to `Listener` in `src/hassette/bus/listeners.py`. Update `Listener.create()` to accept and propagate it.
2. Add `source_tier: SourceTier` to `ScheduledJob` in `src/hassette/scheduler/classes.py`.
3. Add `source_tier: SourceTier` to `ListenerRegistration` and `ScheduledJobRegistration` in `src/hassette/core/registration.py`.
4. Add `source_tier: SourceTier` to `InvokeHandler` and `ExecuteJob` in `src/hassette/core/commands.py`.
5. Change `HandlerInvocationRecord.listener_id` to `int | None` in `src/hassette/bus/invocation_record.py`. Add `source_tier: SourceTier`.
6. Change `JobExecutionRecord.job_id` to `int | None` in `src/hassette/scheduler/classes.py`. Add `source_tier: SourceTier`.
7. Add `is_di_failure: bool = False` to `ExecutionResult` in `src/hassette/utils/execution.py`. Set it in `track_execution()` using `isinstance(exc, DependencyError)`.
8. Update `telemetry_models.py`: add `source_tier: SourceTier` to `ListenerSummary`, `JobSummary`, `HandlerInvocation`, `JobExecution`, `HandlerErrorRecord`, `JobErrorRecord`, `SlowHandlerRecord`, `SessionRecord`. Make `listener_id`, `app_key`, `handler_method`, `topic` on `HandlerErrorRecord` nullable (`str | None` / `int | None`). Same for `JobErrorRecord`.
9. Update `web/models.py`: make `listener_id`, `app_key`, `handler_method`, `topic` on `HandlerErrorEntry` nullable. Same for `JobErrorEntry`. Add `source_tier: SourceTier` to all response models that include it. Add `dropped_overflow: int = 0`, `dropped_exhausted: int = 0` to `TelemetryStatusResponse`.
10. Add Pydantic field validator on `AppConfig.app_key` (or `HassetteConfig`) that rejects `'__hassette__'`.

## Test Strategy

- **`tests/unit/test_source_tier_models.py`**: New test file.
  - `test_listener_create_default_source_tier` — `Listener.create()` with no `source_tier` produces `'app'`.
  - `test_listener_create_framework_source_tier` — `Listener.create(source_tier='framework')` stores `'framework'`.
  - `test_invoke_handler_carries_source_tier` — `InvokeHandler(source_tier='framework', ...)` accessible.
  - `test_handler_invocation_record_nullable_listener_id` — `HandlerInvocationRecord(listener_id=None, ...)` is valid.
  - `test_handler_error_record_nullable_fields` — `HandlerErrorRecord(listener_id=None, app_key=None, ...)` is valid.
  - `test_execution_result_is_di_failure` — verify `is_di_failure` set correctly for `DependencyError` vs other exceptions.
  - `test_app_config_rejects_hassette_sentinel` — `AppConfig(app_key='__hassette__')` raises `ValueError`.
  - `test_app_config_accepts_normal_app_key` — `AppConfig(app_key='my_app')` succeeds.

## Review Guidance

- Verify `SourceTier` is used everywhere (not bare `str`) — Pyright should catch mismatches.
- Verify nullable fields on error models match what LEFT JOIN queries will produce.
- Verify `is_di_failure` is set from `isinstance(exc, DependencyError)`, not from `error_type` string comparison.
- Verify `AppConfig` validator is a Pydantic field validator (not `__init__` check) so it applies to env-var-sourced values.
