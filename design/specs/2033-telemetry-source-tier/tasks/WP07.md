## Objectives & Success Criteria

- `frontend/src/api/generated-types.ts` is regenerated from the updated OpenAPI schema, reflecting `source_tier` fields and nullable FK fields on all response types.
- The default dashboard view continues to show only app-tier telemetry (no frontend changes needed for the default behavior — the backend defaults to `source_tier='app'`).
- A visible framework health affordance exists on the default dashboard (e.g., a "System Health" collapsible section, a badge/counter, or a toggle filter). The exact UX is the implementer's choice.
- The affordance calls `/telemetry/dashboard/errors?source_tier=framework` and `/telemetry/dashboard/kpis?source_tier=framework` to populate.
- Orphaned error records (null `listener_id` / `job_id`) render as "deleted handler" / "deleted job" in the error feed.
- `TelemetryStatusResponse` TypeScript type includes `dropped_overflow` and `dropped_exhausted`.

## Subtasks

1. Regenerate `frontend/src/api/generated-types.ts` from the updated OpenAPI schema (run the existing codegen script or manually update types).
2. Update error feed components to handle null `listener_id`/`app_key` — render "deleted handler" / "deleted job" placeholder instead of crashing on null.
3. Implement a framework health affordance on the dashboard page:
   - Fetch framework error count and KPIs via `?source_tier=framework`.
   - Display as a visible element (badge, collapsible section, or toggle).
   - Ensure the affordance is present even when framework error count is zero (AC-18).
4. If using a collapsible section: render framework errors with a distinct visual tag ("Framework" badge or similar).
5. Update the telemetry status display (if any) to show `dropped_overflow` / `dropped_exhausted` counts when non-zero.

## Test Strategy

- **E2E tests** (`tests/e2e/test_dashboard.py` — extend existing):
  - `test_dashboard_default_shows_app_errors_only` — verify the default dashboard error feed shows only app-tier errors.
  - `test_framework_affordance_visible` — verify a visible framework health element exists on the default dashboard.
  - `test_framework_affordance_shows_framework_errors` — click/activate the affordance, verify framework errors appear with tier tag.
  - `test_orphan_error_renders_deleted_label` — create an orphan error, verify "deleted handler" label in the error feed.
