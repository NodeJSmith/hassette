## Objectives & Success Criteria

- `get_recent_errors()` uses a single UNION ALL query with LEFT JOINs, single ORDER BY + LIMIT, and `source_tier` filter parameter.
- `get_slow_handlers()` uses LEFT JOIN and `source_tier` filter.
- `get_all_app_summaries()` uses `active_app_listeners`/`active_app_scheduled_jobs` views for registration counts AND filters activity queries by `source_tier = 'app'`.
- `get_global_summary()` filters by `source_tier = 'app'` by default.
- `get_listener_summary()` and `get_job_summary()` include `source_tier` in SELECT.
- All `LIKE 'Dependency%'` patterns replaced with `is_di_failure` flag queries.
- `source_tier` parameter accepted by all query methods (default `'app'`; `'all'` means no filter).
- Orphaned records (null FK from LEFT JOIN) produce valid Pydantic models with nullable fields.

## Subtasks

1. Rewrite `get_recent_errors()` in `src/hassette/core/telemetry_query_service.py`: single UNION ALL with LEFT JOINs, `source_tier` filter, single LIMIT. Map `'all'` to no WHERE clause on source_tier.
2. Update `get_slow_handlers()`: change INNER JOIN to LEFT JOIN, add `source_tier` filter.
3. Update `get_all_app_summaries()`: registration queries use `active_app_listeners`/`active_app_scheduled_jobs` views. Activity queries add `WHERE l.source_tier = 'app'` / `WHERE sj.source_tier = 'app'` in the JOIN condition. Verify `'__hassette__'` never appears in the returned dict.
4. Update `get_global_summary()`: add `source_tier` filter (default `'app'`) to both listener and job subqueries.
5. Update `get_listener_summary()` and `get_job_summary()`: add `l.source_tier` / `sj.source_tier` to SELECT.
6. Replace all `LIKE 'Dependency%'` patterns with `SUM(CASE WHEN hi.is_di_failure = 1 THEN 1 ELSE 0 END)`.
7. Add `source_tier` parameter to all query method signatures (default `'app'`).
8. Update `get_handler_invocations()` and `get_job_executions()` to include `source_tier` in SELECT.
9. Add documentation to the per-app health summary response model specifying that `handler_count` reflects instance 0 only while `total_invocations` aggregates all instances (FR-9b).

## Test Strategy

- **`tests/integration/test_telemetry_query_service.py`** (extend existing file):
  - `test_get_recent_errors_union_all_ordering` — insert mixed handler/job errors with known timestamps, verify correct ordering regardless of type distribution.
  - `test_get_recent_errors_limit_applies_globally` — insert 20 handler errors and 5 job errors, request limit=10, verify the 10 most recent (not 10 handlers + 5 jobs).
  - `test_get_recent_errors_left_join_orphans` — delete a listener registration, verify its error records still appear with null `app_key`.
  - `test_get_recent_errors_source_tier_filter` — insert app and framework errors, filter by `'app'`, verify only app errors returned. Filter by `'framework'`, verify only framework. Filter by `'all'`, verify both.
  - `test_get_all_app_summaries_excludes_hassette` — insert framework actors, call `get_all_app_summaries()`, verify `'__hassette__'` not in returned dict.
  - `test_get_all_app_summaries_activity_filtered` — insert framework invocations, verify they don't inflate app-tier counts.
  - `test_di_failure_flag_query` — insert records with `is_di_failure=1`, verify `di_failures` count uses flag not string match.
  - `test_get_slow_handlers_left_join` — delete a listener, verify its slow invocations still appear.

## Review Guidance

- Verify UNION ALL query has a single `ORDER BY execution_start_ts DESC LIMIT ?` at the end, not per-subquery.
- Verify LEFT JOIN is used (not INNER JOIN) in `get_recent_errors` and `get_slow_handlers`.
- Verify `get_all_app_summaries` filters BOTH registration AND activity queries by `source_tier`.
- Verify no `LIKE 'Dependency%'` patterns remain anywhere in `telemetry_query_service.py`.
- Verify `source_tier = 'all'` omits the WHERE clause entirely (not `WHERE source_tier = 'all'`).
