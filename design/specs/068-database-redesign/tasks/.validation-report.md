## Change from prior version

The `app_key`→`owner_key` rename was dropped after a challenge review verified `owner_key` is a synonym for `app_key` (the column already stores framework keys under `app_key`; `Resource.app_key` is the documented telemetry identity key; `source_tier` already discriminates app vs framework). Consequences:

- **T01 (owner_key codemod) deleted.** T02 and T03 are now dependency roots.
- **FR#12, FR#13, FR#14 withdrawn** (storage / API / frontend rename). **AC#9 withdrawn** (no app_key references). Marked withdrawn in place — not renumbered, to preserve cross-references.
- **AC#5** (static analysis passes) remapped from the withdrawn FRs to the global pyright/build gate; verified by T16.
- `app_key` is retained across all layers. The listener natural key is `(app_key, instance_index, name, topic)`.

## Summary

All 17 active FRs and 13 active ACs trace to at least one task with a Verify item. No functional coverage gaps.
