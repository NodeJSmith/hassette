## Objectives & Success Criteria

- `Resource.initialize()` automatically calls `initialize()` on children in insertion order, after hooks and before `handle_running()`.
- `Service.initialize()` propagates to children after `after_initialize()` hooks (after serve task is spawned).
- Children with status `STARTING` or `RUNNING` are skipped (handles Hassette's explicit `start()` calls).
- Children with status `NOT_STARTED`, `STOPPED`, `FAILED`, or `CRASHED` are eligible for re-initialization.
- Init propagation is sequential (one child at a time, awaited).
- After propagation, Hassette's children (started via `start()`) are not double-initialized.

## Subtasks

1. In `Resource.initialize()` (`src/hassette/resources/base.py:260`), add a child init propagation loop between the `_run_hooks([...])` call and `handle_running()`:
   ```python
   for child in self.children:
       if child.status not in (ResourceStatus.STARTING, ResourceStatus.RUNNING):
           await child.initialize()
   ```
2. In `Service.initialize()` (`src/hassette/resources/base.py:374`), add the same loop after `_run_hooks([self.after_initialize])` and before `_initializing = False`.
3. Write unit tests in `tests/unit/resources/test_lifecycle_propagation.py`:
   - `test_init_propagates_to_children_in_insertion_order` — parent with 3 children, verify init order matches insertion.
   - `test_init_skips_running_children` — pre-start a child, verify propagation skips it.
   - `test_init_skips_starting_children` — child in STARTING status, verify skipped.
   - `test_init_reinitializes_stopped_children` — shutdown a child, then init parent, verify child is re-initialized.
   - `test_init_reinitializes_failed_children` — child in FAILED status, verify re-initialized.
   - `test_init_propagation_runs_before_handle_running` — verify parent status is STARTING during child init, then RUNNING after.
   - `test_service_init_propagation_after_serve_spawn` — Service subclass, verify child init runs after serve task is spawned.

## Review Guidance

- **Design constraint**: Init propagation MUST run before `handle_running()` in `Resource.initialize()` — parent is not RUNNING until children are initialized.
- **Design constraint**: Skip filter is `status not in (STARTING, RUNNING)` — NOT an allowlist of `(NOT_STARTED, STOPPED)`.
- **Design constraint**: Parent hooks run BEFORE child propagation. Bus.on() and Scheduler.run_every() must accept registrations regardless of readiness status.
- **FAIL condition**: A parent reaching RUNNING status while any child is still STOPPED/NOT_STARTED.
