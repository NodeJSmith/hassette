## Objectives & Success Criteria

Remove the manual `child.shutdown()` gather from `Hassette.on_shutdown()`. Add `Hassette._on_children_stopped()` override that calls `close_streams()`. Update `EventStreamService.close_streams()` docstring. After this WP, child shutdown propagation is owned entirely by `_finalize_shutdown()` with timeout enforcement.

After this WP:
- `Hassette.on_shutdown()` is a no-op `pass`
- `Hassette._on_children_stopped()` calls `self._event_stream_service.close_streams()`
- `close_streams()` runs after children emit STOPPED events (via the hook), before Hassette's own STOPPED event
- The double-shutdown pattern is eliminated — timeout enforcement applies to Hassette's direct children

## Subtasks

1. Replace `Hassette.on_shutdown()` body in `src/hassette/core/core.py` with `pass` and a docstring explaining children are shut down by `_finalize_shutdown()` propagation
2. Add `Hassette._on_children_stopped()` override in `src/hassette/core/core.py` that calls `await self._event_stream_service.close_streams()`
3. Update `EventStreamService.close_streams()` docstring in `src/hassette/core/event_stream_service.py` to reference `_on_children_stopped()` as the caller
4. Update `tests/integration/test_lifecycle_propagation.py`: modify `TestHassetteShutdownIdempotent` to verify single-pass shutdown (children shut down once, not twice)
5. Add integration test `test_close_streams_after_children_stopped`: verify that children's STOPPED events are emitted before `close_streams()` is called

## Test Strategy

Tests in `tests/integration/test_lifecycle_propagation.py`:
- `test_hassette_single_pass_shutdown`: verify each child's `shutdown()` is called exactly once (not twice as before)
- `test_close_streams_after_children_stopped`: verify ordering — children's `handle_stop()` fires before `close_streams()`. Use a mock or spy on `EventStreamService.close_streams()` to capture call order relative to children's STOPPED events.

## Review Guidance

- `Hassette.on_shutdown()` MUST be a no-op — any manual child shutdown defeats the timeout enforcement
- `_on_children_stopped()` MUST call `await super()._on_children_stopped()` (currently super is a no-op, but the contract requires it)
- Verify that `close_streams()` ordering is preserved: children STOPPED before streams close
- Check that `before_shutdown()` (bus listener removal, session finalization) is unchanged — it still runs before `_finalize_shutdown()` via the hook chain
