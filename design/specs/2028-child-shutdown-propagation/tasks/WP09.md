## Objectives & Success Criteria

Add `Resource._force_terminal()` method (recursive, cancels tasks), `Service._force_terminal()` override (also cancels `_serve_task`), and `TaskBucket.cancel_all_sync()` (fire-and-forget task cancellation). These are foundational types consumed by the `_finalize_shutdown()` refactor in WP09b.

After this WP:
- `_force_terminal()` recursively walks the subtree and cancels tasks + sets STOPPED on every descendant
- `Service._force_terminal()` also cancels `_serve_task`
- `TaskBucket.cancel_all_sync()` cancels all tracked tasks without awaiting completion

## Subtasks

1. Add `TaskBucket.cancel_all_sync()` to `src/hassette/task_bucket/task_bucket.py` — a non-awaited version that calls `.cancel()` on all tracked tasks without awaiting completion (fire-and-forget). Snapshot the WeakSet before iterating.
2. Add `Resource._force_terminal()` method to `src/hassette/resources/base.py`: check `_shutdown_completed` (return early if True), call `self.cancel()`, call `self.task_bucket.cancel_all_sync()`, set `_shutting_down = False`, `_shutdown_completed = True`, `status = STOPPED`, call `mark_not_ready("shutdown timed out")`, recurse into `self.children`
3. Add `Service._force_terminal()` override in `src/hassette/resources/base.py`: cancel `_serve_task` if running and not done, then call `super()._force_terminal()`
4. Add unit tests in `tests/unit/resources/test_lifecycle_propagation.py`:
   - `test_force_terminal_recurses_to_grandchildren`: create a 3-level tree, call `_force_terminal()` on root, verify all descendants are STOPPED with `_shutdown_completed = True`
   - `test_force_terminal_cancels_task_bucket`: verify `cancel_all_sync()` is called on each resource
   - `test_force_terminal_skips_completed_children`: verify `_shutdown_completed = True` children are skipped
   - `test_service_force_terminal_cancels_serve_task`: create a StubService with a running `_serve_task`, call `_force_terminal()`, verify the task is cancelled

## Test Strategy

Tests in `tests/unit/resources/test_lifecycle_propagation.py`. Use existing `conftest.py` fixtures for `StubResource` and `StubService`. Tests verify:
- Recursive descent of `_force_terminal()` through a 3-level tree
- `cancel_all_sync()` called on each resource in the subtree
- Early return when `_shutdown_completed` is already True
- Service `_serve_task` cancellation

## Review Guidance

- `_force_terminal()` MUST be a method, not a free function — encapsulation of private state
- `_force_terminal()` MUST check `_shutdown_completed` at entry to skip already-completed children
- `cancel_all_sync()` MUST NOT await — fire-and-forget only
- `Service._force_terminal()` MUST call `super()._force_terminal()` after cancelling `_serve_task`
