## Objectives & Success Criteria

Add `Resource._force_terminal()` method (recursive, cancels tasks) and `Service._force_terminal()` override (also cancels `_serve_task`). Add `_on_children_stopped()` hook to `Resource._finalize_shutdown()`. Replace the existing inline force-patch block in the timeout handler with calls to `_force_terminal()`. Add `children_timed_out` flag to gate the hook.

After this WP:
- `_force_terminal()` recursively walks the subtree and cancels tasks + sets STOPPED on every descendant
- `Service._force_terminal()` also cancels `_serve_task`
- `_on_children_stopped()` is a no-op hook called after clean child shutdown, skipped on timeout
- The timeout handler calls `child._force_terminal()` instead of inline flag patching
- Wrap `cleanup()` in `asyncio.timeout` in the base `_finalize_shutdown()`
- `_initializing` warning with `shutdown_event.is_set()` gating (DEBUG vs WARNING)

## Subtasks

1. Add `Resource._force_terminal()` method to `src/hassette/resources/base.py`: check `_shutdown_completed`, call `self.cancel()`, request task cancellation (non-awaited), set flags, recurse into `self.children`
2. Add `TaskBucket.cancel_all_sync()` to `src/hassette/task_bucket/task_bucket.py` — a non-awaited version that calls `.cancel()` on all tracked tasks without awaiting completion (fire-and-forget)
3. Add `Service._force_terminal()` override in `src/hassette/resources/base.py`: cancel `_serve_task` if running, then call `super()._force_terminal()`
4. Add `_on_children_stopped()` no-op hook to `Resource` in `src/hassette/resources/base.py` with docstring specifying success-path-only semantics and super() requirement
5. Refactor `Resource._finalize_shutdown()`: wrap `cleanup()` in `asyncio.timeout`, add `children_timed_out` flag, call `_force_terminal()` in timeout handler, call `_on_children_stopped()` conditionally, add `_initializing` warning with `shutdown_event` gating
6. Add unit tests in `tests/unit/resources/test_lifecycle_propagation.py`:
   - `test_force_terminal_recurses_to_grandchildren`: verify all descendants are STOPPED after `_force_terminal()`
   - `test_force_terminal_cancels_task_bucket`: verify task cancellation is requested
   - `test_service_force_terminal_cancels_serve_task`: verify `_serve_task` is cancelled
   - `test_on_children_stopped_called_on_clean_shutdown`: verify hook fires
   - `test_on_children_stopped_skipped_on_timeout`: verify hook is skipped when children time out
   - `test_cleanup_timeout_fires_on_hung_cleanup`: verify `asyncio.timeout` wraps `cleanup()`
   - `test_initializing_warning_on_shutdown_during_init`: verify warning/debug log level depends on `shutdown_event`

## Test Strategy

All tests in `tests/unit/resources/test_lifecycle_propagation.py`. Use the existing `conftest.py` fixtures for `StubResource` and `StubService`. Tests verify:
- Recursive descent of `_force_terminal()`
- Task cancellation behavior (mock `task_bucket.cancel_all_sync()`)
- Service `_serve_task` cancellation
- Hook call/skip semantics
- Cleanup timeout behavior

## Review Guidance

- `_force_terminal()` MUST be a method, not a free function — encapsulation of private state
- `_force_terminal()` MUST check `_shutdown_completed` at entry to skip already-completed children
- `cancel_all_sync()` MUST NOT await — fire-and-forget only
- `_on_children_stopped()` MUST only run when `children_timed_out is False`
- The `asyncio.timeout` wrapping `cleanup()` replaces the bare `try/except Exception` — verify both TimeoutError and Exception are handled
