## Objectives & Success Criteria

Add `total_shutdown_timeout_seconds` config, add FinalMeta exemption for Hassette, and override `Hassette.shutdown()` to wrap `super().shutdown()` in a total timeout with a `finally` block that ensures `close_streams()`, `handle_stop()`, and terminal status are always set.

After this WP:
- `total_shutdown_timeout_seconds` config exists (default: 30s)
- FinalMeta exempts Hassette from `@final` on `shutdown()`
- `Hassette.shutdown()` wraps `super().shutdown()` in `asyncio.timeout(total_shutdown_timeout_seconds)`
- On total timeout: `_force_terminal()` is called on all children recursively
- `finally` block: sets `_shutdown_completed = True` first, then `handle_stop()`, then `close_streams()`, then terminal status

## Subtasks

1. Add `total_shutdown_timeout_seconds: int = 30` to `src/hassette/config/config.py` with docstring explaining it bounds the entire shutdown (hooks + propagation)
2. Add Hassette exemption to `FinalMeta.__init__` in `src/hassette/resources/base.py`: extend the existing Service check to include `"hassette.core.core.Hassette"`
3. Add `Hassette.shutdown()` override in `src/hassette/core/core.py` with `@final` decorator: wrap `await super().shutdown()` in `asyncio.timeout(self.config.total_shutdown_timeout_seconds)`, with `except TimeoutError` calling `_force_terminal()` on all children, and a `finally` block setting `_shutdown_completed = True` first, then `handle_stop()`, then `close_streams()`, then terminal status
4. Add unit tests in `tests/unit/resources/test_lifecycle_propagation.py`:
   - `test_total_shutdown_timeout_caps_wall_clock`: verify that Hassette shutdown completes within `total_shutdown_timeout_seconds` even when a child hangs
   - `test_total_timeout_force_patches_all_descendants`: verify `_force_terminal()` is called recursively on timeout
   - `test_total_timeout_finally_always_closes_streams`: verify `close_streams()` is called even on timeout
   - `test_total_timeout_sets_shutdown_completed_first`: verify `_shutdown_completed = True` is set before `handle_stop()` and `close_streams()`

## Test Strategy

Tests in `tests/unit/resources/test_lifecycle_propagation.py`. Use a mock Hassette with a child that hangs in `on_shutdown` to trigger the total timeout. Verify:
- Wall-clock time is bounded
- `_force_terminal()` is called on all children
- `close_streams()` always executes
- `_shutdown_completed` is set before any awaits in the `finally` block

## Review Guidance

- FinalMeta exemption MUST use the fully qualified class name `"hassette.core.core.Hassette"` matching the existing Service exemption pattern
- `_shutdown_completed = True` MUST be the first statement in the `finally` block — before any awaits
- `handle_stop()` MUST be called before `close_streams()` so Hassette's own STOPPED event is emitted while streams are open
- `close_streams()` MUST be wrapped in `suppress(Exception)` for error tolerance
- The `@final` on `Hassette.shutdown()` prevents subclasses of Hassette from overriding it
