## Objectives & Success Criteria

- `Bus`, `Scheduler`, `Api`, `ApiSyncFacade`, and `_ScheduledJobQueue` call `mark_ready()` in `on_initialize()` instead of `__init__()`.
- After shutdown + re-initialization (via propagation), readiness is restored for all five classes.
- `StateManager` is verified to already use `after_initialize()` — no change needed.
- All existing tests pass without modification (or with minimal fixture updates).

## Subtasks

1. **Bus** (`src/hassette/bus/bus.py`): Remove `self.mark_ready(reason="Bus initialized")` from `__init__`. Add new `async def on_initialize(self) -> None:` that calls `self.mark_ready(reason="Bus initialized")`.
2. **Scheduler** (`src/hassette/scheduler/scheduler.py`): Remove `self.mark_ready(reason="Scheduler initialized")` from `__init__`. Add new `async def on_initialize(self) -> None:` that calls `self.mark_ready(reason="Scheduler initialized")`.
3. **Api** (`src/hassette/api/api.py`): Remove `self.mark_ready(reason="API initialized")` from `__init__`. Add new `async def on_initialize(self) -> None:` that calls `self.mark_ready(reason="API initialized")`.
4. **ApiSyncFacade** (`src/hassette/api/sync.py`): Remove `self.mark_ready(...)` from `__init__`. Add new `async def on_initialize(self) -> None:` that calls `self.mark_ready(reason="Synchronous API facade initialized")`.
5. **_ScheduledJobQueue** (`src/hassette/core/scheduler_service.py`): Remove `self.mark_ready(reason="Queue ready")` from `__init__`. Add new `async def on_initialize(self) -> None:` that calls `self.mark_ready(reason="Queue ready")`.
6. Write unit tests in `tests/unit/resources/test_lifecycle_propagation.py`:
   - `test_bus_ready_after_initialize_not_after_init` — create Bus, verify not ready, initialize, verify ready.
   - `test_scheduler_ready_after_initialize_not_after_init` — same for Scheduler.
   - `test_leaf_resources_ready_after_restart` — shutdown all leaf resources, re-initialize, verify all are ready again.

## Test Strategy

TDD: write `test_bus_ready_after_initialize_not_after_init` first (RED — currently passes because mark_ready is in __init__). Modify Bus, test goes GREEN. Repeat for each class.

- **Test file**: `tests/unit/resources/test_lifecycle_propagation.py`
- **Tests**: listed above
- **Note**: Some existing integration tests may assume leaf resources are ready immediately after construction. These must be updated to call `initialize()` first or use the HassetteHarness which handles lifecycle automatically.

## Review Guidance

- **Design constraint**: `Bus.on()`, `Scheduler.run_every()`, and similar registration methods must NOT gate on readiness. Verify this — if they do, parent `on_initialize` hooks that register subscriptions before child propagation will break.
- **Design constraint**: `StateManager` already uses `after_initialize()` — do not touch it.
- **FAIL condition**: A leaf Resource that is ready immediately after construction (before `initialize()` is called).
