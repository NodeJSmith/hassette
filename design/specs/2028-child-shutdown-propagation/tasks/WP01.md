## Objectives & Success Criteria

- `Resource.shutdown()` and `Service.shutdown()` are truly idempotent: calling `shutdown()` twice on the same resource executes the shutdown sequence only once.
- `_shutdown_completed` flag is set at end of `_finalize_shutdown()`, checked at top of `shutdown()`.
- `initialize()` resets `_shutdown_completed` and clears `shutdown_event` (single canonical location).
- `start()` resets `_shutdown_completed` before the early-return guard.
- `shutdown_event` replacement is removed from `start()` — only `initialize()` clears it via `.clear()`.
- All existing tests continue to pass.

## Subtasks

1. Add `_shutdown_completed: bool = False` to `LifecycleMixin.__init__()` in `src/hassette/resources/mixins.py`, alongside the existing `_shutting_down` pattern.
2. In `Resource.shutdown()` (`src/hassette/resources/base.py:288`), add `if self._shutdown_completed: return` before the `_shutting_down` guard.
3. In `Service.shutdown()` (`src/hassette/resources/base.py:388`), add the same `_shutdown_completed` check.
4. In `Resource._finalize_shutdown()` (`src/hassette/resources/base.py:244`), set `self._shutdown_completed = True` after `cleanup()` and before `handle_stop()`.
5. In `Resource.initialize()` (`src/hassette/resources/base.py:260`), add `self._shutdown_completed = False` and `self.shutdown_event.clear()` as the first two lines (before the `_initializing` guard).
6. In `Service.initialize()` (`src/hassette/resources/base.py:374`), add the same two resets as first lines.
7. In `LifecycleMixin.start()` (`src/hassette/resources/mixins.py:91`), add `self._shutdown_completed = False` before the `_init_task` early-return guard. Remove the `self.shutdown_event = asyncio.Event()` line.
8. Update `reset_hassette_lifecycle()` in `src/hassette/test_utils/reset.py` to reset `_shutdown_completed = False` on the hassette instance and all children.
9. Write unit tests in `tests/unit/resources/test_lifecycle_propagation.py`:
   - `test_shutdown_completed_prevents_double_shutdown` — call `shutdown()` twice, verify `on_shutdown` hook runs only once.
   - `test_shutdown_completed_reset_by_initialize` — shutdown then initialize, verify `shutdown()` works again.
   - `test_shutdown_event_cleared_by_initialize` — shutdown (sets event), initialize, verify `shutdown_event.is_set()` is False.
   - `test_start_resets_shutdown_completed` — shutdown, then `start()`, verify init task is spawned.

## Review Guidance

- **Design constraint**: `_shutdown_completed` MUST be set in `_finalize_shutdown()`, not in `shutdown()` itself — this ensures the flag is set even if hooks raise exceptions (the finally block calls `_finalize_shutdown`).
- **Design constraint**: `shutdown_event` must use `.clear()` not replacement — preserves references held by serve loops.
- **Design constraint**: `start()` must NOT replace `shutdown_event` — only `initialize()` does that via `.clear()`.
- **FAIL condition**: Any test that calls `shutdown()` then `initialize()` then `shutdown()` and sees `on_shutdown` run only once.
