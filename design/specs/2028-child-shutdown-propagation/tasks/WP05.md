## Objectives & Success Criteria

- `Scheduler` has an `on_shutdown()` hook that awaits `remove_all_jobs()`.
- `App.cleanup()` is simplified to `await super().cleanup(timeout=timeout)` — manual child cleanup removed.
- `StateProxy.on_shutdown()` no longer calls `bus.remove_all_listeners()` or `scheduler.remove_all_jobs()`. Retains `mark_not_ready()`, `states.clear()`, `poll_job = None`, `state_change_sub = None`.
- `ServiceWatcher.on_shutdown()` no longer calls `self.bus.remove_all_listeners()`.
- `AppHandler.on_shutdown()` no longer calls `self.lifecycle.bus.remove_all_listeners()`. Retains `self.lifecycle.shutdown_all()`.
- All existing integration tests pass — particularly `test_state_proxy.py`, `test_app_factory_lifecycle.py`, `test_service_watcher.py`.

## Subtasks

1. **Scheduler.on_shutdown()** (`src/hassette/scheduler/scheduler.py`): Add new method:
   ```python
   async def on_shutdown(self) -> None:
       await self.remove_all_jobs()
   ```
2. **App.cleanup()** (`src/hassette/app/app.py:133-153`): Replace entire body with `await super().cleanup(timeout=timeout)`. Keep the `@final` decorator and the `timeout` parameter override.
3. **StateProxy.on_shutdown()** (`src/hassette/core/state_proxy.py:93-104`): Remove lines calling `await self.bus.remove_all_listeners()` and `self.scheduler.remove_all_jobs()`. Keep `mark_not_ready()`, `states.clear()` under lock, `poll_job = None`, `state_change_sub = None`.
4. **ServiceWatcher.on_shutdown()** (`src/hassette/core/service_watcher.py:39-40`): Remove `await self.bus.remove_all_listeners()`. If the method body is now empty, keep it with just `pass` or remove the override entirely.
5. **AppHandler.on_shutdown()** (`src/hassette/core/app_handler.py:108-113`): Remove `await self.lifecycle.bus.remove_all_listeners()`. Keep `mark_not_ready()` and `await self.lifecycle.shutdown_all()`.
6. Write unit tests in `tests/unit/resources/test_lifecycle_propagation.py`:
   - `test_scheduler_on_shutdown_awaits_remove_all_jobs` — create Scheduler with mocked service, add a job, shutdown, verify `remove_jobs_by_owner` was awaited.
   - `test_app_shutdown_propagates_to_bus_and_scheduler` — create App (using HassetteHarness or stub), shutdown, verify Bus.on_shutdown and Scheduler.on_shutdown were invoked.

## Review Guidance

- **Design constraint**: `Scheduler.on_shutdown()` MUST `await` the Task returned by `remove_all_jobs()`. A fire-and-forget call is a regression.
- **Design constraint**: `App.cleanup()` passes `app_shutdown_timeout_seconds` via the `timeout` param to `super().cleanup()` — this timeout is preserved, not dropped.
- **Design constraint**: `StateProxy.on_shutdown()` MUST keep `poll_job = None` and `state_change_sub = None` — defensive against `on_disconnect()` race.
- **Design constraint**: `AppHandler.on_shutdown()` MUST keep `lifecycle.shutdown_all()` — app instances are in AppRegistry, not the Resource tree.
- **FAIL condition**: `remove_all_jobs()` called but not awaited in `Scheduler.on_shutdown()`.
