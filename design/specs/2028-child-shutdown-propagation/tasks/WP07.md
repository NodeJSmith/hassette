## Objectives & Success Criteria

- Integration tests verify the full restart round-trip: shutdown propagates to children, initialize restores them, readiness is restored.
- Integration tests verify Hassette's on_shutdown + propagation interaction (double-shutdown is idempotent).
- Integration tests verify the simplified parent cleanup paths work end-to-end.
- Full test suite passes: `uv run nox -s dev`.

## Subtasks

1. Write integration test `test_app_restart_preserves_children` in `tests/integration/test_app_factory_lifecycle.py` (or new file `tests/integration/test_lifecycle_propagation.py`):
   - Create an App via HassetteHarness, verify Bus/Scheduler/Api/StateManager children are ready.
   - Shutdown the App, verify children are STOPPED and not ready.
   - Re-initialize the App, verify children are RUNNING and ready again.
2. Write integration test `test_hassette_shutdown_idempotent` in `tests/integration/test_core.py`:
   - Hassette.on_shutdown() manually shuts down children. Then _finalize_shutdown propagation tries again. Verify children's on_shutdown runs only once.
3. Write integration test `test_state_proxy_restart_round_trip` in `tests/integration/test_state_proxy.py`:
   - Shutdown StateProxy (propagates to its Bus/Scheduler children).
   - Re-initialize StateProxy.
   - Verify subscriptions are re-established and state polling resumes.
4. Run full test suite: `uv run nox -s dev` and fix any failures.

## Test Strategy

Integration tests using the existing HassetteHarness and test infrastructure.

- **Test files**: `tests/integration/test_lifecycle_propagation.py` (new), `tests/integration/test_core.py`, `tests/integration/test_state_proxy.py`
- **Tests**: `test_app_restart_preserves_children`, `test_hassette_shutdown_idempotent`, `test_state_proxy_restart_round_trip`
- **Approach**: Use the real Resource lifecycle (not mocks) to verify end-to-end propagation behavior.

## Review Guidance

- **Design constraint**: These tests verify the *composition* of WP01-WP06. Each individual WP has unit tests; these tests verify the WPs work together.
- **FAIL condition**: Any child resource stuck in STOPPED/FAILED state after a parent restart.
- **FAIL condition**: Full test suite (`nox -s dev`) has failures.
