## Objectives & Success Criteria

Validate that replacing `asyncio.wait_for(asyncio.gather(...))` with `async with asyncio.timeout(): await asyncio.gather(...)` does not reproduce the scheduling quirk that caused `test_bus_driven_failed_events_trigger_shutdown_after_max_attempts` to fail. This is a GATE — all subsequent WPs depend on the outcome.

Success: the flaky test passes 50/50 runs with `asyncio.timeout` wrapping the gather in `_finalize_shutdown()`. If it fails, fall back to the existing `asyncio.wait_for` pattern and document the limitation.

## Subtasks

1. In `src/hassette/resources/base.py`, replace the `asyncio.wait_for(asyncio.gather(...), timeout=timeout)` in `Resource._finalize_shutdown()` with `async with asyncio.timeout(timeout): await asyncio.gather(...)`. Keep the same `except TimeoutError` handler.
2. Run `test_bus_driven_failed_events_trigger_shutdown_after_max_attempts` 50x via `uv run pytest tests/integration/test_service_watcher.py::test_bus_driven_failed_events_trigger_shutdown_after_max_attempts --count=50 -x -v -p no:randomly` and record results.
3. If all 50 pass: the prototype validates the design assumption. Commit the change.
4. If any fail: revert the change, document the failure in the design doc's Open Questions, and update all subsequent WPs to use `asyncio.wait_for` instead.
