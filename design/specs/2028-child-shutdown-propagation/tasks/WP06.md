## Objectives & Success Criteria

- `reset_state_proxy()` in `src/hassette/test_utils/reset.py` uses full lifecycle (`shutdown()` + `initialize()`) instead of calling `on_shutdown()` / `on_initialize()` directly.
- `reset_hassette_lifecycle()` resets `_shutdown_completed` on hassette and all children.
- All existing integration tests pass with the updated reset functions.

## Subtasks

1. **`reset_state_proxy()`** (`src/hassette/test_utils/reset.py:19-39`): Replace direct `on_shutdown()` + `on_initialize()` calls with `await proxy.shutdown()` + `await proxy.initialize()`. Verify the states cache is still cleared (it happens in `StateProxy.on_shutdown()` which is called by the full `shutdown()` path).
2. **`reset_hassette_lifecycle()`** (`src/hassette/test_utils/reset.py:76-106`): Add `hassette._shutdown_completed = False` to the reset. Also iterate `hassette.children` and reset `_shutdown_completed = False` on each child recursively.
3. Run `tests/integration/test_state_proxy.py` and `tests/integration/test_core.py` to verify no regressions from updated reset functions.

## Test Strategy

No new tests — this WP updates test infrastructure. Validation is via existing test suite.

- **Validation**: `uv run pytest tests/integration/test_state_proxy.py tests/integration/test_core.py tests/integration/test_app_factory_lifecycle.py -v`

## Review Guidance

- **Design constraint**: `reset_state_proxy` must use full `shutdown()` + `initialize()` to exercise the propagation path. Direct `on_shutdown()` calls bypass child propagation.
- **Design constraint**: `reset_hassette_lifecycle` must reset `_shutdown_completed` on ALL resources in the tree, not just the top-level hassette instance.
- **FAIL condition**: An integration test that fails because a resource is stuck in `_shutdown_completed = True` after a reset.
