# WP01: Fix useWebSocket reconnect loop

## Scope

Stabilize the `options` parameter in `useWebSocket` to prevent infinite reconnect loops when callers pass inline objects.

## Acceptance Criteria

- [ ] Passing `{ onReconnect: () => ... }` to `useWebSocket` creates exactly one WebSocket connection
- [ ] WebSocket connection remains stable across parent component re-renders
- [ ] Existing behavior (message handling, backoff, cleanup) unchanged

## Implementation Steps

1. Add `const optionsRef = useRef(options); optionsRef.current = options;` before the `useEffect`
2. Inside the effect's `onmessage` handler, replace `options?.onReconnect?.()` with `optionsRef.current?.onReconnect?.()`
3. Change `useEffect` dependency array from `[state, options]` to `[state]`
