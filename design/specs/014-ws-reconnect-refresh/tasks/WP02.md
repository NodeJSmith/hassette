# WP02: Auto-refresh useApi data on WebSocket reconnection

## Scope

Add a `reconnectVersion` signal to AppState, make `useApi` read it internally via context, and wire `useWebSocket` to increment it on reconnect (not first connect). No page component changes.

## Acceptance Criteria

- [ ] `reconnectVersion` stays at 0 after first connect
- [ ] `reconnectVersion` increments on reconnect
- [ ] All mounted `useApi` instances auto-refetch when `reconnectVersion` changes
- [ ] No page component files are modified
- [ ] Dashboard's `appStatus`-driven `appGrid` refetch remains intact
- [ ] Reading `reconnectVersion` does not cause component re-renders (uses `useSignalEffect`)
- [ ] A `useApi` instance mounted when `reconnectVersion > 0` fetches exactly once (initial fetch only)

## Implementation Steps

### Step 1: Add reconnectVersion to AppState
- File: `frontend/src/state/create-app-state.ts`
- Add `reconnectVersion: signal(0)` to the returned object

### Step 2: Make useApi reconnect-aware
- File: `frontend/src/hooks/use-api.ts`
- Import `useAppState` from `../state/context`
- Import `useSignalEffect` from `@preact/signals`
- Add a `useSignalEffect` that reads `reconnectVersion.value` (auto-tracked by the signal system, no component re-render) and calls `refetch()` when the value is > 0
- Track the initial value at mount time via a ref to avoid double-fetching when mounted after a reconnect has already occurred

### Step 3: Wire useWebSocket to increment reconnectVersion
- File: `frontend/src/hooks/use-websocket.ts`
- Add `hasConnectedRef = useRef(false)` inside the hook
- On `"connected"` message: if `hasConnectedRef.current === true`, increment `state.reconnectVersion.value`; else set `hasConnectedRef.current = true`
- Remove `UseWebSocketOptions` interface and `options` parameter
- Remove `optionsRef` (from WP01) since the callback approach is superseded

### Step 4: Update WebSocketProvider call site
- File: `frontend/src/app.tsx`
- Confirm `useWebSocket(state)` call has no second argument (already the case)
- Remove any `UseWebSocketOptions` import if present
