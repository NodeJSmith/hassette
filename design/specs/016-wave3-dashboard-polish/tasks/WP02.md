## Objectives & Success Criteria

All 4 string-mapping sites use `status.ts` functions instead of inline maps. No inline `Record<string, string>` status maps remain in component files. CSS classes are applied via the mapped `StatusVariant`, not raw backend strings. Existing E2E tests continue to pass (no visual regression).

## Subtasks

1. **`frontend/src/components/shared/status-badge.tsx`** ‚Äî remove `VARIANT_MAP` and `BADGE_VARIANT_MAP`. Import `statusToVariant` from `status.ts`. Both badge sizes use `statusToVariant(status)` to derive the CSS class.
2. **`frontend/src/components/shared/health-bar.tsx`** ‚Äî import `healthGradeToVariant`. Replace `${healthStatus}` with the mapped variant in the CSS class: `ht-health-bar__fill--${healthGradeToVariant(healthStatus)}`.
3. **`frontend/src/components/dashboard/kpi-strip.tsx`** ‚Äî import `errorRateToVariant`. Replace `${data.error_rate_class}` with `errorRateToVariant(data.error_rate_class)`.
4. **`frontend/src/components/app-detail/health-strip.tsx`** ‚Äî remove `STATUS_COLOR_MAP` (lines 13-18). Import `errorRateToVariant` for the error rate class (line 31) and `statusToVariant` for the status card (line 27). The status card uses a double-mapping chain because health-card CSS uses good/warn/bad vocabulary:
   - `status "running"` ‚Üí `statusToVariant()` ‚Üí `"success"` ‚Üí `"good"` CSS class
   - `status "failed"` ‚Üí `statusToVariant()` ‚Üí `"danger"` ‚Üí `"bad"` CSS class
   - `status "stopped"` ‚Üí `statusToVariant()` ‚Üí `"warning"` ‚Üí `"warn"` CSS class
   - `status "disabled"` ‚Üí `statusToVariant()` ‚Üí `"neutral"` ‚Üí `""` CSS class
   Create a small local helper (e.g., `variantToHealthCardClass`) for this second mapping step. Add an inline comment explaining the chain.
5. **Update CSS class names** in `frontend/src/global.css` ‚Äî the health-bar fill classes currently use raw grade names as bare selectors (`.ht-health-bar__fill.excellent` at line 1207, `.good` at 1208, `.warning` at 1209, `.critical` at 1210). Replace these with variant-based selectors: `.ht-health-bar__fill--success`, `.ht-health-bar__fill--warning`, `.ht-health-bar__fill--danger`. Remove the old `.excellent`, `.good`, `.warning`, `.critical` selectors ‚Äî health-bar.tsx is the only consumer (confirmed by grep), so the switch is atomic within this WP.
6. **Do NOT touch** `handler-row.tsx:31-32` or `job-row.tsx:31-32` ‚Äî the dot color ternaries are threshold logic and stay inline per design.
