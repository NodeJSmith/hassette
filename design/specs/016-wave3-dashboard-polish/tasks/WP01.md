## Objectives & Success Criteria

A single `frontend/src/utils/status.ts` module provides all status-to-variant mappings for the frontend. All mapping functions accept `string`, validate against a known set, return a typed `StatusVariant`, and `console.warn` for unknown values. Unit tests pass for every known status and for unknown/unexpected inputs.

## Subtasks

1. Create `frontend/src/utils/status.ts` with:
   - `type StatusVariant = "success" | "danger" | "warning" | "neutral"`
   - `const APP_STATUSES` set and `statusToVariant(status: string): StatusVariant` ‚Äî maps running‚Üísuccess, failed‚Üídanger, stopped‚Üíwarning, disabled‚Üíneutral, blocked‚Üíwarning, starting‚Üíneutral, shutting_down‚Üíneutral. Unknown ‚Üí neutral + console.warn.
   - `const HEALTH_GRADES` set and `healthGradeToVariant(grade: string): StatusVariant` ‚Äî maps excellent‚Üísuccess, good‚Üísuccess, warning‚Üíwarning, critical‚Üídanger. Unknown ‚Üí neutral + console.warn.
   - `const ERROR_RATE_CLASSES` set and `errorRateToVariant(cls: string): StatusVariant` ‚Äî maps good‚Üísuccess, warn‚Üíwarning, bad‚Üídanger. Unknown ‚Üí neutral + console.warn.
2. Write `frontend/src/utils/status.test.ts` ‚Äî unit tests for all three functions.

## Test Strategy

TDD ‚Äî write tests first in `frontend/src/utils/status.test.ts` using vitest:
- `test_statusToVariant_known_values` ‚Äî each known status maps correctly
- `test_statusToVariant_unknown_warns` ‚Äî unknown string returns "neutral" and calls console.warn
- `test_healthGradeToVariant_known_values`
- `test_healthGradeToVariant_unknown_warns`
- `test_errorRateToVariant_known_values`
- `test_errorRateToVariant_unknown_warns`

## Review Guidance

- All three functions must accept `string` (not union types) ‚Äî the functions ARE the validation boundary
- `console.warn` must fire for unknown values ‚Äî this is the runtime detection mechanism for backend/frontend version skew
- Default return must be `"neutral"` for all three functions ‚Äî never throw
- The union types (`StatusVariant`, etc.) are exported for downstream consumers but NOT used as function parameter types
