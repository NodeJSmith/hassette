## Objectives & Success Criteria

The test suite catches broken migration chains (missing revisions, dangling references, model/migration divergence) before they reach production. Four new tests pass. A broken chain like the `003` crash-loop incident would be caught in CI.

## Subtasks

1. **Revision chain integrity** ‚Äî add `test_migration_chain_has_no_gaps` to `tests/integration/test_database_service.py`. Load `ScriptDirectory` from the migrations path, walk from base to heads via `walk_revisions()`, assert no gaps or dangling `down_revision` references. This is ~10 lines and catches the exact failure mode from the crash-loop.
2. **Fresh apply to head** ‚Äî add `test_fresh_db_migrates_to_head`. Create an empty SQLite DB in `tmp_path`, run `alembic upgrade head`, verify it completes without error. Verify all 5 expected tables exist (`sessions`, `listeners`, `scheduled_jobs`, `handler_invocations`, `job_executions`).
3. **Sequential upgrade** ‚Äî add `test_sequential_upgrade_from_each_revision`. For each revision in the chain, stamp a fresh DB at that revision, then upgrade to head. Catches intermediate migrations that break the chain.
4. **Schema drift detection** ‚Äî add `test_migration_schema_matches_models`. After running all migrations on a fresh DB, compare the resulting table/column set against `Base.metadata` (from the SQLAlchemy models). Flag any tables or columns present in models but missing from migrations, or vice versa.

All tests use the same Alembic config pattern as `DatabaseService._run_migrations()` (`database_service.py:206-211`): programmatic `Config()` with `script_location` and `sqlalchemy.url` set dynamically. Use `tmp_path` for fresh DBs.

## Test Strategy

## Review Guidance

- Tests must use programmatic Alembic config (not `alembic.ini`) to match production behavior
- Each test must use a fresh `tmp_path` DB ‚Äî no shared state between tests
- Schema drift test should compare table names and column names/types, not full DDL (SQLite type affinity makes exact DDL comparison brittle)
- Tests must not import from migration version files directly ‚Äî use `ScriptDirectory` API
