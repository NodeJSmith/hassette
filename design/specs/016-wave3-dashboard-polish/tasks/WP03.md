## Objectives & Success Criteria

A `local-storage.ts` utility owns all localStorage access with a consistent `hassette:` prefix. The existing `ht-theme` key is migrated. Multi-instance app expand state in the app list survives page navigation. Unit tests pass for localStorage helpers including corrupt data and stale key pruning. E2E test verifies expand state persists.

## Subtasks

1. Create `frontend/src/utils/local-storage.ts` with:
   - `const STORAGE_PREFIX = "hassette:"` — single source of truth for key prefix
   - `getStoredSet(key: string): Set<string>` — reads `STORAGE_PREFIX + key` from localStorage, parses JSON array, returns Set. try/catch with fallback to empty Set on corrupt/missing data.
   - `setStoredSet(key: string, value: Set<string>): void` — writes JSON array to `STORAGE_PREFIX + key`. try/catch (silently fails if localStorage unavailable).
   - `getStoredValue<T>(key: string, fallback: T): T` — generic get with fallback. try/catch.
   - `setStoredValue<T>(key: string, value: T): void` — generic set. try/catch.
   - `migrateKey(oldKey: string, newKey: string): void` — reads old key, writes to `STORAGE_PREFIX + newKey`, deletes old key. No-op if old key doesn't exist.
2. Write `frontend/src/utils/local-storage.test.ts` — unit tests using vitest with mocked localStorage.
3. **Migrate `ht-theme`** in `frontend/src/state/create-app-state.ts` — on module load, call `migrateKey("ht-theme", "theme")`. Update the theme signal initialization to read from `hassette:theme` via `getStoredValue("theme", "dark")`. Update the theme toggle to write via `setStoredValue("theme", newTheme)`.
4. **Wire expand persistence** in `frontend/src/components/apps/manifest-row.tsx`:
   - On mount: read expanded set via `getStoredSet("expanded-apps")`, initialize `expanded` signal from whether this app_key is in the set.
   - On toggle: update the stored set (add or remove the app_key).
   - Note: pruning stale keys (deleted apps) can be done by the parent `manifest-list.tsx` — after fetching manifests, intersect stored keys with manifest app_keys and write back the pruned set.
5. **Add stale key pruning** in `frontend/src/components/apps/manifest-list.tsx` — after manifests load, prune expanded-apps keys that don't match any manifest app_key.

## Review Guidance

- All localStorage access must go through `local-storage.ts` — no direct `localStorage.getItem`/`setItem` calls elsewhere
- try/catch on every localStorage read (corrupt data, private browsing mode)
- `ht-theme` migration must be backward-compatible — read old key, write new key, delete old
- Stale key pruning must not run on every render — only after manifest fetch
