## Objectives & Success Criteria

A `useDebouncedEffect` hook provides generic trailing debounce for signal changes. The dashboard page debounces all three data fetches (KPIs, app grid, errors) on `appStatus` changes with 500ms delay. Reconnection refetches remain immediate (via `reconnectVersion`). Unit tests pass for the debounce hook. E2E test verifies debounced behavior.

## Subtasks

1. Create `frontend/src/hooks/use-debounced-effect.ts` with:
   - `useDebouncedEffect(getValue: () => unknown, delayMs: number, callback: () => void): void` ŌĆö watches the return value of `getValue()`, and when it changes, starts a 500ms trailing debounce timer. On timeout, calls `callback()`. Cleans up timer on unmount and on new change (restarts timer).
   - Uses `useEffect` internally with the value as a dependency, plus `useRef` for the timer.
2. Write `frontend/src/hooks/use-debounced-effect.test.ts` ŌĆö unit tests using vitest with fake timers.
3. **Update dashboard** in `frontend/src/pages/dashboard.tsx`:
   - Remove the current `useEffect` block (lines 22ŌĆō31) that watches `lastStatus` and calls `appGrid.refetch()`.
   - Import `useDebouncedEffect`.
   - Add: `useDebouncedEffect(() => appStatus.value, 500, () => { void kpis.refetch(); void appGrid.refetch(); void errors.refetch(); })`.
   - Remove the `lastStatus` const and `refetchAppGrid` callback (no longer needed).
4. **Document invariant** in `frontend/src/state/create-app-state.ts` ŌĆö add a comment on `reconnectVersion` and `appStatus` signals explaining that reconnection triggers immediate refetch via `reconnectVersion` (in `useApi`), while `appStatus` changes trigger debounced refetch. These must remain independent code paths.

## Test Strategy

TDD ŌĆö write tests first in `frontend/src/hooks/use-debounced-effect.test.ts`:
- `test_debounce_coalesces_rapid_calls` ŌĆö trigger 5 value changes in quick succession, advance timers by 500ms, verify callback called once
- `test_debounce_resets_on_new_change` ŌĆö trigger change, advance 300ms, trigger another change, advance 500ms, verify callback called once (not twice)
- `test_debounce_fires_after_delay` ŌĆö trigger one change, advance 500ms, verify callback called
