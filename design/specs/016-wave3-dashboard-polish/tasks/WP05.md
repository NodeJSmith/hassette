## Objectives & Success Criteria

Dashboard app cards display invocation and execution counts alongside the existing handler/job counts. The data is already in `DashboardAppGridEntry` — this is a frontend-only display change. E2E test verifies counts render on app cards.

## Subtasks

1. **Update `frontend/src/components/dashboard/app-card.tsx`** — add a stats row below the existing handler/job count row:
   - Show `{total_invocations} inv · {total_executions} exec` using `--ht-text-dim` color, `--ht-text-xs` size, mono font (`ht-text-mono`).
   - Use the existing `pluralize` utility or a simpler format since "inv" and "exec" are abbreviations.
   - Only show the row if `total_invocations > 0 || total_executions > 0` — avoid visual noise on apps that haven't fired yet.
2. **Add E2E test** in `tests/e2e/test_dashboard.py`.

## Test Strategy

E2E test in `tests/e2e/test_dashboard.py`:
- `test_app_card_shows_invocation_and_execution_counts` — load dashboard, find app card for `my_app` (which has seed data with invocations), verify the card contains invocation and execution count text.

## Review Guidance

- Must use design tokens: `--ht-text-dim` for color, `--ht-text-xs` for size, mono font
- Row should not appear for apps with zero invocations AND zero executions
- Verify the counts come from `total_invocations` and `total_executions` (handler invocations + job executions), not from `total_errors`
