## Objectives & Success Criteria

- `ScheduledJob.cancel()` delegates to `self._scheduler.cancel_job(self)` ‚Äî raises `RuntimeError` when `_scheduler` is None
- `Scheduler.cancel_job()` performs the full cancel: owner guard, idempotency via `_dequeued`, DB write, `_dequeue_job`, set `_dequeued = True`
- `Scheduler.cancel_group()` delegates to `cancel_job` per-member ‚Äî no inline cancel logic
- `Scheduler._dequeue_job()` renamed from `remove_job` ‚Äî calls `scheduler_service.dequeue_job(job)` only (no inline dict cleanup)
- `Scheduler._remove_all_jobs()` renamed from `remove_all_jobs`
- `ScheduledJob.cancelled` boolean flag removed
- Three dispatch guards removed from `SchedulerService` (`_dispatch_and_log`, `run_job`, `reschedule_job`); `_dispatch_and_log` guard replaced with `if job._dequeued: return`
- Stale inline comment at `scheduler.py:192-194` removed
- All existing tests updated and passing
