## Objectives & Success Criteria

- `ScheduledJob` has `_scheduler` and `_dequeued` fields (both `repr=False, compare=False`)
- `_ScheduledJobQueue` has a `remove_item_sync(job)` method that performs synchronous heap removal without the async lock
- `SchedulerService` has a `dequeue_job(job)` method that calls `remove_item_sync`, fires `_fire_removal_callbacks` unconditionally, and calls `kick()` when the job was in the heap
- The old `SchedulerService.remove_job` (async task-spawning wrapper at line 493) is removed
- `Scheduler.add_job()` sets `job._scheduler = self` before calling `scheduler_service.add_job`
- All existing tests pass (with temporary adapter if needed — this WP does not yet rewire cancel_job)

## Subtasks

1. Add `_scheduler: "Scheduler | None" = field(default=None, repr=False, compare=False)` to `ScheduledJob` in `src/hassette/scheduler/classes.py`
2. Add `_dequeued: bool = field(default=False, repr=False, compare=False)` to `ScheduledJob` in `src/hassette/scheduler/classes.py`
3. Update `matches()` docstring in `classes.py` to remove `cancelled` and add `_scheduler`/`_dequeued` to the non-compared fields list
4. Add `remove_item_sync(job) -> bool` method to `_ScheduledJobQueue` in `src/hassette/core/scheduler_service.py` — calls `self._queue.remove_item(job)` directly, no lock
5. Add `dequeue_job(job) -> bool` method to `SchedulerService` — calls `self._job_queue.remove_item_sync(job)`, fires `self._fire_removal_callbacks([job])` unconditionally, calls `self.kick()` when removed
6. Remove the public `SchedulerService.remove_job` method (line 493–499) — zero callers after this refactor
7. Set `job._scheduler = self` in `Scheduler.add_job()` before `self.scheduler_service.add_job(job)`
8. Write unit tests for the new sync dequeue path

## Test Strategy

- `tests/unit/core/test_scheduler_service_dequeue.py` (new file):
  - `test_dequeue_job_removes_from_heap` — verify job is absent from queue after sync dequeue
  - `test_dequeue_job_returns_false_when_not_in_heap` — verify idempotent no-op
  - `test_dequeue_job_fires_removal_callbacks_when_removed` — verify callback fires when job was in heap
  - `test_dequeue_job_fires_removal_callbacks_when_not_removed` — verify callback fires even when job was already popped
  - `test_dequeue_job_calls_kick_only_when_removed` — verify kick is conditional
  - `test_dequeue_job_is_synchronous` — verify inline return without a yield point
- `tests/unit/test_scheduler_resource.py`:
  - `test_add_job_sets_scheduler_back_reference` — verify `job._scheduler is scheduler` after `add_job`

## Review Guidance

- Verify `_scheduler` and `_dequeued` are excluded from `__repr__`, `__eq__`, `__hash__` via `repr=False, compare=False`
- Verify `remove_item_sync` does NOT acquire the `FairAsyncRLock` — it must be pure synchronous
- Verify `_fire_removal_callbacks` is called unconditionally (not gated on `removed`) — this prevents dict leaks when the serve loop already popped the job
- Verify the old `SchedulerService.remove_job` (task-spawning wrapper) is fully removed, not just commented out
- Verify `job._scheduler = self` is set BEFORE `scheduler_service.add_job(job)`, not after
